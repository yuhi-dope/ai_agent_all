# a01. 電力・データセンター戦略

## 1. データセンターは必要か

### 結論：現段階では不要。ただし3,000社超で検討フェーズに入る。

シャチョツーはAnthropicのAPIを**利用する側**であり、推論基盤はAnthropic/AWSが担う。

| レイヤー | 担当 | 自前DC必要？ |
|---------|------|-------------|
| LLM推論基盤 | Anthropic / AWS | Anthropicの問題（AWS提携済み） |
| アプリケーション | シャチョツー | クラウド（Supabase等）で十分 |
| ユーザーデータ | Supabase/PostgreSQL | 数TB程度ならマネージドDBで余裕 |

### 参考：Anthropic vs OpenAI のインフラ戦略

- **Anthropic**：AWSから数十億ドル規模の投資を受け、AWSインフラを活用
- **OpenAI**：MicrosoftのAzureを基盤としつつ、Stargate計画で自前DC建設も推進

### 自前DC検討が必要になる3つの閾値

1. 月間インフラ費が **1.5億円** を超える（後述：約3,000社以上で到達）
2. 独自モデルのファインチューニング/トレーニングを行う場合
3. データ主権の法規制で国内保存が必須になった場合

## 2. 月間API費用1億円 = 何社で到達するか

### 1社あたりのAPI消費量の推定

| 企業規模 | 日次API呼出 | 平均トークン/回 | 月額API費用（概算） |
|---------|-----------|---------------|-------------------|
| 小規模（社員10名） | 50-100回 | 3,000 tok | 約1-3万円 |
| 中規模（社員50名） | 300-500回 | 3,000 tok | 約5-10万円 |
| 大規模（社員200名） | 1,000-3,000回 | 3,000 tok | 約15-50万円 |

※ Claude Sonnet主体、一部Opus利用のミックス（平均約 $0.02/call）
※ 注意: 本試算はClaude（Anthropic）ベースの単価。01_事業計画.md ではGemini 2.5 Flash
  （Claudeの1/3〜1/5コスト）を主LLMとして採用しており、実際のコストはこの試算の
  1/3〜1/5に圧縮される。DC投資判断時は01の最新コスト前提を使用すること。

### 1億円到達に必要な社数（API単価ベース）

| シナリオ | 平均月額/社 | 必要社数 |
|---------|-----------|---------|
| 中小企業中心 | 5万円 | 約2,000社 |
| 中堅企業中心 | 15万円 | 約700社 |
| 大企業中心 | 40万円 | 約250社 |
| **現実的ミックス** | **8万円** | **約1,200社** |

エージェントが高度化しマルチステップ推論（1タスクで10-30回API呼出）が増えた場合、1社あたりの消費が跳ね上がり **500社前後** で到達する可能性もある。

ただし、後述するDC建設前の最適化施策（セマンティックキャッシュ、OSSモデル併用等）により、**実質的なDC検討ラインは3,000社以上に後退する。**

## 3. DC建設の前にやるべき最適化施策

**「API外注 vs 自前DC」は二択ではない。** 中間に複数の選択肢があり、これらを段階的に実施することでDC建設判断を大幅に後ろ倒しできる。

### 3-1. セマンティックキャッシュの導入

B2B SaaSでは類似クエリが繰り返し発生する。

```
例：「売上レポートを出して」「先月の売上を集計して」
→ 類似クエリのキャッシュヒット率：推定30-50%
→ API費用を30-50%削減可能（DC不要）
```

| 項目 | 内容 |
|------|------|
| 導入コスト | 数百万円（ベクトルDB + キャッシュロジック開発） |
| 期待削減率 | API費用 30-50% |
| 導入タイミング | 月間API費 500万円超 |

### 3-2. Anthropicボリュームディスカウント交渉

大口顧客向けにAnthropicはボリューム契約を提供している。

| 項目 | 内容 |
|------|------|
| 導入コスト | 0円（契約交渉のみ） |
| 期待削減率 | 20-40% |
| 導入タイミング | 月間API費 2,000万円超 |

### 3-3. OSSモデルへのタスク分離

全タスクにClaude Opus/Sonnetを使う必要はない。簡易タスク（分類、要約、定型生成等）はオープンソースモデル（Llama、Mistral等）をGPUクラウド（CoreWeave、Lambda Labs等）で動かすことで大幅にコスト削減できる。

| 項目 | 内容 |
|------|------|
| 導入コスト | 数百万円（GPUクラウド契約 + ルーティング開発） |
| 期待削減率 | Anthropic API依存度を50%削減 |
| 導入タイミング | 月間API費 5,000万円超 |

### 3-4. 最適化施策の累積効果

```
元のAPI費用：月間1億円（1,200社時点）
  - セマンティックキャッシュ：▲30%  → 7,000万円
  - ボリューム契約：▲25%          → 5,250万円
  - OSSモデル分離：▲30%           → 3,675万円

実質月間コスト：約3,700万円（63%削減）
```

**月間API費1億円（1,200社）でも、最適化後は3,700万円で済む。**
DC建設（年間運用5.5億円）を正当化するには、最適化済みでも月間1.5億円を超える必要がある。
→ つまり **約3,000社以上** がDC検討の実質的なライン。

## 4. イーロン方式による北海道DC建設の費用試算

### 4-1. イーロン・マスク方式とは（xAI Memphis DC：約122日で建設）

- **既存建物の転用**（新築しない）
- **モジュラー型GPU冷却コンテナ**を並べる
- **規制交渉を並行処理**（建設と許認可を同時進行）
- **液冷方式**で空調設備を最小化

### 4-2. 必要なGPU規模の逆算

```
月間1億円 ÷ $0.02/call ÷ 150円/$
  ≒ 月間3,300万回のAPI呼出
  → 秒間約12.7リクエスト（ピーク時3-5倍 ≒ 秒間50リクエスト）
  → 必要GPU：NVIDIA H100 約200-500基（推論専用）
```

### 4-3. 初期建設費用の内訳

| 項目 | 費用（億円） | 備考 |
|------|------------|------|
| GPU調達（H100 x 300基） | 15-20 | 1基 $25,000-35,000（2026年時点） |
| サーバー・ネットワーク機器 | 3-5 | ラック、スイッチ、ストレージ等 |
| 建物・コンテナ型DC | 3-5 | プレファブ/液冷コンテナ方式 |
| 電力インフラ | 2-4 | 受変電設備、自家発電（LNG） |
| 液浸冷却設備 | 1-2 | GRC、LiquidCool等 |
| ネットワーク回線 | 0.5-1 | 東京IX直結の専用線 |
| 設計・施工・許認可 | 1-2 | 建設会社、コンサル費用 |
| 予備費 | 2-3 | 想定外コスト |
| **合計** | **27-42億円** | **中央値 約35億円** |

### 4-4. 年間運用コスト

| 項目 | 年間費用（億円） | 備考 |
|------|----------------|------|
| 電力費 | 3-5 | 300基 x 700W ≒ 210kW + 冷却・付帯 → 約500kW |
| 人件費 | 1-2 | DC運用エンジニア 5-10名 |
| 保守・部品交換 | 1-2 | GPU故障率 年3-5% |
| 推論スタック運用 | 0.3-0.5 | vLLM/TGI等の運用・モデル更新対応 |
| 回線費 | 0.3-0.5 | 専用線維持 |
| **合計** | **5-10億円** | **中央値 約8億円** |

### 4-5. 隠れコスト（見落としやすいリスク）

| リスク | 内容 | 影響 |
|--------|------|------|
| **GPU減価償却** | H100は2-3年で性能的に陳腐化（毎世代2倍の性能/電力効率で進化） | 3.5年回収計算の前提が崩れる。途中でGPU入替費用が発生 |
| **モデル更新** | Anthropic APIなら自動更新。自前推論ではモデル差し替え工数が発生 | 運用負荷増 |
| **冗長構成** | 本番運用にはGPUの1.5-2倍の予備機が必要 | GPU調達費が1.5倍に膨らむ可能性 |
| **北海道の採用難** | DCエンジニアの採用は都市部でも困難、北海道はさらに厳しい | リモート運用 or 高額報酬で対応 |

### 4-6. 建設スケジュール（日本での最速ルート）

| ステップ | 期間 | 内容 |
|---------|------|------|
| Step 1: 用地確保 | 2-4週間 | 工場跡地・倉庫を買収/リース、特区活用 |
| Step 2: 電力確保 | 並行実施 | 自家発電を先行設置（系統電力は後接続）or 大口電力契約付き工場を買収 |
| Step 3: 建設 | 3-6ヶ月 | コンテナ型DC設置 + 液浸冷却導入 |
| **合計** | **6-9ヶ月** | 日本の規制環境を考慮 |

※ 電力引込みは通常1-2年かかるため、**自家発電の先行設置**が最短化の鍵。

## 5. 北海道提携による投資回収シミュレーション

### 5-1. 北海道を選ぶ物理的根拠

熱力学的原理：

```
データセンターの消費電力 ≒ 全てが最終的に熱になる
冷却コスト ∝ 1 / (許容温度 - 外気温)
→ 気温が低いほど冷却の電力コストが指数的に下がる
```

Google・Facebookが北欧にDCを建てるのと同じ原理。

| 項目 | 北海道（石狩・苫小牧） | 評価 |
|------|---------------------|------|
| 冷却 | 年間平均気温8.9°C、外気冷却でPUE 1.1台 | ◎ |
| 電力 | 北海道電力 + 再エネ（風力・太陽光）豊富 | ◎ |
| 地震 | 本州より低リスク | ○ |
| 遅延 | 東京まで光ファイバーで約15ms | △ |
| 土地 | 広大・安価・拡張性抜群 | ◎ |
| 水 | 冷却用水が豊富 | ◎ |
| 先行事例 | さくらインターネット石狩DC運営済み、ラピダス千歳工場建設中 | ◎ |

### 5-2. 提携なしの場合

```
API外注を続けた場合：年間12億円（成長で増加）
自前DC（提携なし）：初期投資35億円 + 年間運用8億円

年間削減額 = 12億 - 8億 = 4億円/年
投資回収 = 35億 ÷ 4億 ≒ 約9年 ← 割に合わない
```

### 5-3. 北海道提携ありの場合

| 提携メリット | 効果 |
|------------|------|
| 補助金・助成金 | 経産省AIインフラ補助（初期投資の最大1/3）→ ▲10-15億円 |
| 電力優遇 | 北電との大口契約 + 再エネ直接購入 → 電力費30-50%削減 |
| 土地無償/格安貸与 | 自治体の企業誘致枠 → 土地費ほぼゼロ |
| 固定資産税減免 | 過疎地域指定で最大3年免除 |
| 雇用助成 | 地元雇用で人件費補助 |

```
初期投資：35億 - 補助金12億 = 23億円
年間運用：8億 - 電力削減2億 - 各種減免0.5億 = 5.5億円
年間削減額 = 12億 - 5.5億 = 6.5億円/年

投資回収 = 23億 ÷ 6.5億 ≒ 約3.5年
```

### 5-4. GPU減価償却を考慮した補正

```
H100の実質寿命：2-3年（性能的陳腐化）
3年目にGPU世代交代が必要 → 追加投資10-15億円

補正後の回収計算：
  初期23億 + 3年目追加12億 = 総投資35億円
  5年間の累積削減 = 6.5億 x 5年 = 32.5億円
  → 実質回収期間：約5.5年

ただしGPU入替で性能2倍 → 処理能力に余裕が生まれ、
さらなるスケールを追加投資なしで吸収可能。
```

### 5-5. 比較まとめ

| | API外注継続 | 自前DC（提携なし） | 自前DC（北海道提携） |
|--|-----------|-----------------|-------------------|
| 初期投資 | 0円 | 35億円 | 23億円 |
| 年間コスト | 12億円（増加） | 8億円 | 5.5億円 |
| 投資回収（単純） | - | 約9年 | 約3.5年 |
| 投資回収（GPU入替込） | - | 10年超 | **約5.5年** |
| 拡張性 | API上限に依存 | 自由 | 自由 |
| データ主権 | 海外依存 | 国内完結 | 国内完結 |

## 7. インフラ段階的スケール計画（改訂版）

DC建設前に最適化施策を段階的に実施し、投資判断を後ろ倒しにする戦略。
その間にGPU世代が進み、同じ予算でより高性能なDCが建てられるメリットもある。

| Phase | 条件 | 施策 | 効果 |
|-------|------|------|------|
| 1 | 現在〜初期 | Supabase + Anthropic API | - |
| 2 | 月間API費 500万円超 | **セマンティックキャッシュ導入** | API費 30-50%削減 |
| 3 | 月間API費 2,000万円超 | **Anthropicボリューム契約交渉** | さらに 20-40%削減 |
| 4 | 月間API費 5,000万円超 | **簡易タスクをOSSモデルに分離**（GPUクラウド） | Anthropic依存度を50%削減 |
| 5 | ユーザー3,000社超 | 専用クラウド（AWS/GCP東京リージョン） | 安定性確保 |
| 6 | 月間インフラ費 1.5億円超 | **北海道DC建設検討開始**（北海道提携活用） | 年間コスト大幅削減 |
| 7 | 独自モデル訓練・フルスケール | 北海道メインDC + 印西エッジDC | 完全自立運用 |

**原則：DC建設は最後の手段。その前にソフトウェアで解決できることを全てやり切る。プロダクトで勝つことに集中し、インフラは段階的にスケールさせる。**
