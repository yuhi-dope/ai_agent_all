## 1. 歯科医院の基本構造

### 1.1 市場規模

```
歯科医院数: 約67,000施設（コンビニ約56,000より多い）
歯科医師数: 約10.7万人
DX意欲あり層（5名以上）: 約1.2万施設
```

### 1.2 歯科医院の組織

```
院長 ─┬─ 歯科医師（勤務医。0-3名）
      ├─ 歯科衛生士（DH: 2-10名）← 超人手不足（有効求人倍率22.6倍）
      ├─ 歯科助手（DA: 1-5名）  ← 無資格でもできる補助業務
      └─ 受付・事務（1-3名）     ← ★レセプト担当はここ

特徴:
  - 院長 = 経営者 = メインの治療者 = 最終決裁者
  - 院長が診療以外に1日2-3時間の事務作業をしている
  - 事務長を置いている医院は少ない（大きい医院のみ）
```

### 1.3 収入構造

```
■ 保険診療（全体の70-90%）
  患者が3割負担 + 保険者（社保/国保）が7割負担
  → 7割分を毎月レセプトで請求 ← ★ここがBPO対象

■ 自費診療（全体の10-30%）
  インプラント、矯正、セラミック、ホワイトニング等
  → レセプト不要。患者が全額支払い

■ 収入例（1日患者30名の医院）:
  保険: 1日約15-20万円 × 22日 = 月330-440万円
  自費: 月50-150万円
  合計: 月400-600万円
  → このうち保険の7割分（月230-310万円）をレセプトで請求
```

### 1.4 歯科医院の1日

```
AM 8:30  受付開始。予約患者の確認、カルテ準備
AM 9:00  診療開始。院長+勤務医で3-4チェア同時進行
         衛生士がスケーリング（歯石除去）等を実施
AM 12:00 午前の部終了
PM 2:00  午後の部開始
PM 6:00  診療終了

翌月10日 レセプト提出期限（厳守）
```

## 2. レセプトとは何か

### 2.1 定義

```
レセプト（診療報酬明細書）
  = 保険診療の内容を保険者に請求するための書類
  = 「この患者にこの治療をしたので、この金額を払ってください」という請求書

  1患者 × 1保険者 × 1ヶ月 = 1レセプト
```

### 2.2 レセプト請求の流れ

```
歯科医院 → 支払基金 or 国保連 → 審査 → 保険者 → 支払い

  ① 診療（毎日）: 治療内容をレセコンに入力
  ② 月末: レセコンからレセプトデータを出力
  ③ 点検: エラー・不整合をチェック（★AI化対象）
  ④ 修正: 問題のあるレセプトを修正
  ⑤ 翌月10日: オンライン or 紙で提出
  ⑥ 審査: 支払基金/国保連が内容を審査
  ⑦ 翌々月: 審査通過分が入金（約2ヶ月後）

  → 査定（減額）or 返戻（差戻し）が発生する可能性あり
```

### 2.3 査定と返戻

```
■ 査定（さてい）:
  「この請求はルール違反だから減額します」
  → お金が減る。取り戻せない（再審査請求は可能だが手間）
  → 年間の査定率0.5%改善 = 年間30-100万円の収益改善

■ 返戻（へんれい）:
  「記載不備があるので差し戻します」
  → 修正して再提出すればお金は戻る
  → ただし再提出の手間 + 入金が1ヶ月遅れる

■ 査定率の目安:
  優秀な医院: 0.1%以下
  平均的な医院: 0.3-0.5%
  チェック不十分な医院: 1-2%

■ 査定の原因TOP5:
  1. 病名と処置の不整合（40%）
  2. 算定回数の超過（20%）
  3. 同月算定不可の組み合わせ（15%）
  4. 部位の不整合（10%）
  5. 記載漏れ（10%）
  → ほとんどがルールベースで検出可能 = AI化しやすい
```

## 3. レセプト点検の完全業務フロー（★最重要セクション）

### 3.0 全体フロー図

```
Step 1: レセコンからデータ出力
  │
Step 2: レセプトの内容確認
  │
Step 3: チェック（病名、算定ルール、部位、回数等）← ★最も時間がかかる
  │
Step 4: エラー修正
  │
Step 5: 提出
  │
Step 6: 返戻・査定への対応
```

### Step 1: レセコンからデータ出力

```
■ レセコン（レセプトコンピュータ）:
  歯科医院の業務システム。診療内容の記録、会計、レセプト作成を行う

■ 主要なレセコン:
  - Dentis（デンティス）: クラウド型。最近急成長
  - ORCA（オルカ）: オープンソース。日医標準レセプトソフト
  - パノラマ: 老舗。シェア大
  - With: クラウド型
  - Receive（レシーブ）: 中堅

■ 出力形式:
  - 電子レセプト（XML/CSV）: オンライン請求用
  - 紙レセプト: 一部の医院（減少中）
  - 点検リスト: レセコン内蔵の簡易チェック結果

■ 出力タイミング:
  月末（最終診療日の翌日）にレセコンの「レセプト作成」を実行
  → 1患者×1ヶ月分のレセプトが自動生成される
```

### Step 2: レセプトの構造

```
■ 1枚のレセプトに含まれる情報:

  ┌─ 患者情報
  │    保険者番号、被保険者番号、氏名、生年月日、性別
  │    保険種別（社保/国保/後期高齢者/公費等）
  │
  ├─ 傷病名（病名）
  │    病名コード、病名、開始日、転帰（継続/治癒/中止）
  │    ★ 処置に対応する病名がないと査定される
  │
  ├─ 診療行為（処置・検査等）
  │    診療行為コード、名称、点数、回数、部位
  │    実施日（日付ごと）
  │
  ├─ 医薬品（投薬）
  │    薬剤コード、名称、用量、日数
  │
  └─ 特記事項
       算定の根拠となる補足情報

■ 歯式表記（部位の表現方法）:
  右上8 7 6 5 4 3 2 1 | 1 2 3 4 5 6 7 8 左上
  右下8 7 6 5 4 3 2 1 | 1 2 3 4 5 6 7 8 左下

  1=中切歯、2=側切歯、3=犬歯、4=第一小臼歯、5=第二小臼歯
  6=第一大臼歯、7=第二大臼歯、8=第三大臼歯（親知らず）

  例: 「右下6番」= 右下の第一大臼歯
```

### Step 3: チェック項目（★AI化の中核）

```
■ チェック1: 病名と処置の整合性（最重要）

  ルール: 処置に対応する傷病名（保険病名）が必要

  よくある査定パターン:
  ┌────────────────────┬──────────────────────┬──────────────┐
  │ 処置               │ 必要な病名            │ ない場合      │
  ├────────────────────┼──────────────────────┼──────────────┤
  │ 抜髄（根管治療）   │ 歯髄炎、歯髄壊死      │ 査定          │
  │ 感染根管治療       │ 根尖性歯周炎          │ 査定          │
  │ SRP（歯石除去深部）│ 慢性歯周炎            │ 査定          │
  │ P基検（歯周検査）  │ 歯周炎 or 歯肉炎      │ 査定          │
  │ CR（コンポジットレジン）│ う蝕（C）         │ 査定          │
  │ In（インレー）     │ う蝕（C）             │ 査定          │
  │ FMC（全部鋳造冠）  │ う蝕（C）or 歯冠崩壊  │ 査定          │
  │ 抜歯               │ 残根、歯根破折等      │ 査定          │
  │ 義歯               │ 歯の欠損（部分/全部）  │ 査定          │
  └────────────────────┴──────────────────────┴──────────────┘

■ チェック2: 算定ルール

  同月に算定できない組み合わせ:
  - SRP と P基検 の同月算定（P基検→SC→再検査→SRP の順序が必要）
  - 抜髄と感染根管治療の同月同歯算定
  - 初診料と再診料の同日算定

  回数上限:
  - 歯周基本検査（P基検）: 月2回まで
  - スケーリング（SC）: 同一部位は月1回
  - フッ化物歯面塗布: 月1回

  算定間隔:
  - SPT（歯周病安定期治療）: 前回から1ヶ月以上
  - 補管（補綴物管理料）: 装着月のみ

■ チェック3: 部位の整合性

  - 上顎の病名なのに下顎の処置 → 査定
  - 右側の病名なのに左側の処置 → 査定
  - 既に抜歯済みの歯に処置 → 査定

■ チェック4: 保険適用の確認

  保険適用外の処置:
  - インプラント（原則自費）
  - 矯正（一部を除き自費）
  - ホワイトニング（自費）
  - セラミック冠（CAD/CAM冠は一部保険適用）

  混合診療の禁止:
  - 同一の歯に保険と自費を同時に適用できない
  - 例: 根管治療（保険）→ セラミック冠（自費）= OK（工程が別）
  - 例: 保険の検査と自費の処置を同日 = グレーゾーン

■ チェック5: 記載漏れ・誤り

  - 患者の保険証情報の誤り（記号・番号）
  - 転帰の未記載（治療が終わった病名の「治癒」記載漏れ）
  - 古い病名の放置（何年も前の病名が「継続」のまま）
  - 摘要欄の記載不足
```

### Step 4: エラー修正

```
■ 修正の流れ:
  チェックでエラー検出 → 院長に確認 → レセコンで修正 → 再チェック

■ よくある修正パターン:
  1. 病名追加: 処置に対応する病名を追加
  2. 病名削除: 不要なレセプト病名を削除
  3. 転帰の修正: 「治癒」「中止」を記載
  4. 回数修正: 算定回数の訂正
  5. 部位修正: 歯式の誤りを修正
```

### Step 5: 提出

```
■ 提出方法:
  オンライン請求（主流）: レセコン → 電子レセプト → 送信
  紙請求（減少中）: 印刷 → 郵送 or 持参

■ 提出先:
  社保（協会けんぽ・組合健保）→ 社会保険診療報酬支払基金
  国保（国民健康保険）→ 国民健康保険団体連合会（国保連）

■ 提出期限: 翌月10日（厳守。遅れると入金が1ヶ月遅れる）
```

### Step 6: 返戻・査定への対応

```
■ 返戻の場合:
  返戻レセプト受領 → 修正 → 再提出 → 審査 → 入金
  ※ 再提出期限: 返戻から6ヶ月以内

■ 査定の場合:
  査定通知受領 → 内容確認 → 再審査請求（納得できない場合）
  ※ 再審査請求期限: 査定通知から6ヶ月以内
  ※ 再審査で覆る確率: 30-50%程度
```

## 4. 歯科特有の算定ルール（★AI化に必須）

### 4.1 主要な処置と点数（2024年度改定時点の参考値）

```
■ 初診・再診:
  初診料: 264点（¥2,640）
  再診料: 56点（¥560）
  ※ 1点 = 10円

■ う蝕（むし歯）治療:
  CR（コンポジットレジン充填）: 100-200点
  In（インレー）: 200-400点
  FMC（全部鋳造冠）: 600-800点
  CAD/CAM冠: 1,200点前後

■ 歯周治療（歯周病）:
  P基検（歯周基本検査）: 200点
  SC（スケーリング）: 72-86点/3分の1顎
  SRP（歯根面清掃）: 60-72点/歯
  SPT（安定期治療）: 120-380点

■ 根管治療:
  抜髄（しんけいを取る）: 230-430点
  感根治（感染根管治療）: 150-380点

■ 抜歯:
  普通抜歯: 150-260点
  難抜歯: 470点
  埋伏歯: 1,050点

■ 義歯:
  部分義歯: 700-1,500点（歯数による）
  総義歯: 2,300点前後
```

### 4.2 歯周治療の順序ルール（★よく査定される）

```
正しい順序:
  P基検（検査）→ SC（スケーリング）→ 再検査 → SRP → 再検査 → SPT

  P基検: 歯周ポケットの深さを測る
    ↓ 4mm以上のポケットあり
  SC: 歯肉縁上の歯石除去（全顎。通常2-3回に分けて実施）
    ↓ SC完了後1ヶ月以上
  P基検（再評価）: 改善度を確認
    ↓ まだ4mm以上のポケットあり
  SRP: 歯肉縁下の歯石除去（ポケットの深い歯のみ）
    ↓ SRP完了後1ヶ月以上
  P基検（再評価）
    ↓ 安定
  SPT: 定期メンテナンス（3ヶ月ごと等）

  ※ この順序を飛ばすと査定される
  ※ 例: P基検なしでいきなりSRP → 査定
```

### 4.3 補管（補綴物管理料）と6ヶ月ルール

```
補管 = 冠やブリッジを装着した月に算定
  → その後6ヶ月以内に同じ歯を再治療すると問題

6ヶ月ルール:
  冠やブリッジを装着 → 6ヶ月以内に脱離・再装着 → 保険請求できない
  ※ やむを得ない事由（外傷等）は例外
```

## 6. AI化マッピング

| チェック項目 | AI化可否 | 自動化度 | 根拠 |
|---|---|---|---|
| 病名と処置の整合性 | ◎ | 90% | ルールベースで検出可能 |
