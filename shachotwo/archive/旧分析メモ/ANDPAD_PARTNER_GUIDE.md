# ANDPAD パートナープログラム申請手順書

> 作成日: 2026-03-21
> 根拠: ADR-011（ANDPADパートナー戦略）
> 目的: 23 万社・68 万ユーザーへのチャネル確保

---

## 1. ANDPAD 連携の目的（ADR-011 要約）

### なぜ ANDPAD か

| 指標 | 数値 |
|---|---|
| 導入企業数 | 23 万社 |
| ユーザー数 | 68 万人 |
| 主要ユーザー | 建設・工務店・設備工事の施工管理担当者 |

ANDPAD は施工プロセスの記録・管理ツールであり、シャチョツーとは競合しない。

**役割分担**:
- ANDPAD = 現場の施工記録・工程管理・安全書類の収集
- シャチョツー = 会社の頭脳（見積自動作成・原価学習・経営判断支援）

この補完関係を前面に出してパートナー提携を進める。

### 連携によるユーザー価値

> 「ANDPAD に蓄積した原価実績データをシャチョツーが読み込み、次の見積がより正確になる。」

具体的には:
- 過去工事の原価実績 → 工種別単価の自動学習
- 工程表・施工計画 → 見積根拠の自動補完
- 安全書類テンプレート → シャチョツーの安全書類生成パイプラインに連携

## 2. 申請前の準備

以下を事前に用意する。申請フォームで全て求められる。

### 必須書類・情報

| 項目 | 内容 | 準備状況 |
|---|---|---|
| 会社登記情報 | 登記簿謄本または登録番号 | 要取得 |
| 事業概要資料 | 1 ページのプロダクト紹介（PDF） | 要作成（Week 7 Day 7） |
| API 連携ユースケース説明 | 何のデータをどのように使うかを明記した文書 | 下記参照 |
| プライバシーポリシー | 公開 URL が必要 | 要確認 |
| 利用規約 | 公開 URL が必要 | 要確認 |

### API 連携ユースケース説明（記載例）

```
連携するデータ:
- 工事情報 API: 案件名・工事種別・契約金額・工期
- 原価実績 API: 工種別の実績単価・使用材料・外注費
- 安全書類 API: 作業員名簿・危険有害業務リスト

使用目的:
- 原価実績データをシャチョツーの見積エンジンに学習させ、過去実績に基づく見積精度向上
- 工事情報をトリガーとした見積ドラフト自動生成
- 安全書類の自動転記（ANDPAD → シャチョツー見積書への添付）

データ保管:
- 取得データは自社 Supabase（GCP 東京リージョン）に保管
- テナント分離（RLS）により他社データと混在しない設計
- ANDPAD から取得したデータを第三者に提供しない
```

## 3. 申請手順

### Step 1: ANDPAD 開発者ポータルにアクセス

URL: `https://partners.andpad.jp`（または `https://developer.andpad.jp`）

- トップページの「パートナー申請」または「API 利用申請」リンクを確認
- アカウント作成が必要な場合は会社メールアドレスで登録

### Step 2: 「パートナー申請」フォームに記入

記入項目（想定）:
- 会社名・代表者名・事業内容
- プロダクト名・概要・対象ユーザー
- 連携の目的・ユースケース（上記の説明文をそのまま転記）
- 月間想定 API コール数（初期: 1,000 回/月 程度）

### Step 3: API アクセス申請（対象 API）

優先度順に以下のスコープを申請する:

| 優先度 | API スコープ | 用途 |
|---|---|---|
| 高 | 工事情報 API | 案件情報の読み込み・見積トリガー |
| 高 | 原価実績 API | 工種別単価の学習データとして使用 |
| 中 | 安全書類 API | 安全書類生成パイプラインとの連携 |
| 低（Phase 3） | ユーザー管理 API | SSO・シングルサインオン連携 |

### Step 4: ANDPAD 担当者との打ち合わせ

- 申請後、通常 2〜4 週間以内に ANDPAD の担当者から連絡が来る
- 打ち合わせでは以下を準備する:
  1. 1 ページのプロダクト紹介資料
  2. 連携のシステム構成図（簡易版で可）
  3. パイロット企業の見込み社数（「建設業 3〜5 社でパイロット検討中」と伝える）

### Step 5: サンドボックス環境での API 検証

- ANDPAD からサンドボックスアクセスが付与される
- `workers/connector/andpad.py` のスケルトンを用意して接続テストを実施
- 検証期間: 2〜4 週間（パイロット開始前に完了させる）

## 4. 連携実装計画

### Phase 1（今すぐ — Week 7）

- ANDPAD パートナー申請フォームの送付のみ
- コードは書かない
- 目的: 申請受理・担当者コンタクトの確立

### Phase 2（パイロット後 — Week 10 以降）

`workers/connector/andpad.py` を実装する。

```python
# 実装予定のインターフェース（スケルトン）
class ANDPADConnector:
    async def get_project_list(self, company_id: str) -> list[Project]: ...
    async def get_cost_actuals(self, project_id: str) -> list[CostActual]: ...
    async def get_safety_docs(self, project_id: str) -> list[SafetyDoc]: ...
```

実装時の注意:
- 認証トークンは `tool_connections` テーブルに暗号化して保管（平文禁止）
- API レスポンスは `company_id` に紐づけてキャッシュ（Redis、TTL: 1 時間）
- レート制限: ANDPAD の制限値に合わせた throttle 設定

### Phase 3（マーケットプレイス掲載 — PMF 後）

- ANDPAD アプリマーケットへの掲載申請
- 掲載条件（想定）: セキュリティ審査・利用規約審査・サポート体制確認
- 掲載後は ANDPAD のユーザー画面から直接シャチョツーを発見できる

## 5. 営業での使い方

パイロット企業へのデモ・商談で以下のフレーズを使う:

> 「ANDPAD の原価データを読み込んで、次の見積がより正確になります。」

具体的なデモシナリオ:
1. ANDPAD の原価実績画面を見せる（担当者の PC で）
2. 「このデータをシャチョツーにインポートすると...」と説明
3. 類似工事の見積ドラフトが自動生成される様子を実演

ANDPAD 未導入企業に対しては:
> 「ANDPAD と一緒に導入すれば、現場記録と見積が連携して、経営全体が見えるようになります。」

## 6. Week 7 アクション

| 日程 | アクション | 完了基準 |
|---|---|---|
| Day 1（月） | ANDPAD 開発者ポータル（partners.andpad.jp）にアクセスし、パートナー申請フォームの全項目を確認・記入開始 | フォーム記入 80% 完了 |
| Day 3（水） | 申請フォーム送付・申請番号またはメール受信を確認 | 送付完了・控え保管 |
| Day 7（日） | 自社プロダクト紹介資料 1 ページ版を作成（Canva 等を使用） | PDF 形式で保存完了 |

### プロダクト紹介資料 1 ページ版の構成（Canva テンプレート推奨）

```
[上段] シャチョツーとは？（3 行以内）
  建設業の見積作成を AI が自動化。過去実績から学習し、精度が上がり続ける。

[中段左] 課題                    [中段右] 解決
  見積作成 8 時間/件              → 30 分/件（AI ドラフト）
  単価が属人的で引き継げない      → 過去見積から自動学習
  ANDPAD データが活用できていない → 原価実績を見積に自動反映

[下段] パイロット案内
  最初の 3 ヶ月無料。見積時間 50% 削減保証。
  問い合わせ先: [メールアドレス] / [電話番号]
```
