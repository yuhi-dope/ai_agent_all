## 0. エグゼクティブサマリー

### シャチョツー製造業版とは何か

「社長の見積の頭を再現するAI」を入口に導入し、データが溜まるにつれて経営参謀へ進化するBPOサービス。

```
見積AI（入口の薬）→ データ蓄積 → 経営提案（効く薬）
                                    ↑
                              ここが本当の価値
```

### なぜ製造業か

- 見積作業が**最も属人化**している産業（社長の頭の中にしかない）
- 見込み生産ではなく**個別受注**の中小企業が多い（月50-150件の見積）
- 建設業の積算AIと**見積構造が類似**（図面→工程推定→単価×工数）
- 市場規模: 対象セグメント約10万社。DX意欲層15%で約15,000社

### 料金体系

| プラン | 月額 | 対象 | 内容 |
|---|---|---|---|
| スターター | ¥15万 | ¥5-10億 | ブレイン + 見積AI（月50件） |
| スタンダード | ¥25万 | ¥10-50億 | ブレイン + 見積AI（月150件）+ 2ndモジュール |
| プレミアム | ¥40万 | ¥50-100億 | ブレイン + 見積AI（無制限）+ 3モジュール + バックオフィス |

### 1.2 データフライホイール

```
見積AIを使う
  → 見積データが溜まる（工程/工数/材料/単価）
    → 精度が上がる（類似部品検索、チャージレート学習）
      → もっと使う（精度が上がるから任せる範囲が広がる）
        → さらにデータが溜まる
          → 受注/失注分析ができるようになる
            → 経営提案ができるようになる
              → 解約できなくなる（データ=自社の経営知）
                → ARPU上昇（追加モジュール契約）
                  → N社データでネットワーク効果発動
```

**フライホイールの起点は「見積AIの日常利用」。** だから最初の導入体験（Stage 0）で「おお」と言わせることが全ての起点。

### 1.3 各Stageで溜まるデータと提案の関係

| Stage | 溜まるデータ | 可能になる提案 | 必要な期間 |
|---|---|---|---|
| 0 | 自社設備・チャージレート・材料マスタ | - （まだ提案できない） | Day 1 |
| 1 | 見積修正履歴（工程変更/工数変更/単価変更） | 見積精度の自動改善 | Month 1-3 |
| 2 | 受注/失注結果、実加工時間、不良率 | 受注率分析、利益率可視化、設備稼働率推定 | Month 3-6 |
| 3 | N社の匿名化データ（チャージレート分布、受注率分布） | 業界ベンチマーク、Tier昇格提案、産業横展開提案 | Month 6+（10社以上のデータ必要） |

### 1.4 競争優位の源泉

| 優位性 | 説明 | 模倣困難性 |
|---|---|---|
| **個社ゲノム** | 社長の暗黙知（得意先ルール/加工ノウハウ/判断基準）がデータ化される | 極高。数ヶ月の利用で溜まるもので、他社が一朝一夕に複製できない |
| **実績データ** | 見積→受注→実加工の一気通貫データ。見積ソフトには実績が溜まらない | 高。見積だけのSaaSでは取れないデータ |
| **ネットワーク効果** | N社の匿名化データで市場単価・ベンチマークを提供 | 極高。後発がN社分のデータを集めるのに数年かかる |
| **業種テンプレート** | 建設/製造/歯科…のゲノムテンプレが蓄積。新業種参入時の初期体験が良い | 中。設計・データが必要だが模倣は可能 |

## 2. 見積AIの本質: 3つの役割を代行する

### 2.1 見積1件の作業分解

```
見積1件のフロー:

  ① 引合い受付・管理          → 営業事務の役割
     「図面来たよ」「納期いつ？」「相見積もりらしい」

  ② 図面→工程→工数を決める   → ★技術見積の役割（本丸）
     「MCで3面加工→研磨→メッキ外注」
     「段取り30分、サイクル12分、SUS304だから被削性0.6」

  ③ 材料費・外注費を調べる     → 調達の役割
     「SUS304丸棒φ50、今いくら？メッキ外注はA社 or B社」

  ④ 利益率を決めて見積書作成   → 営業事務 + 経営判断
     「A社向けだから15%で」「急ぎだから1.2倍」→ PDF送付
```

### 2.2 各役割のAI代替度

| 役割 | 今誰がやっているか | AI代替度 | シャチョツーの対応 |
|---|---|---|---|
| 営業事務 | 事務員 or 社長 | ◎ ほぼ自動化 | 引合い管理 + 見積書自動生成 + メール送付 |
| **技術見積** | **社長/ベテラン** | **○ 8割AI→2割人が修正** | **図面OCR + LLM工程推定 + エンジン計算** |
| 調達情報 | 社長 or 購買 | ○ マスタ管理+自動更新 | ゲノムに材料・外注単価を蓄積 |
| 経営判断 | 社長 | △ 提案はするが決めるのは人 | 受注率分析から最適利益率を提案 |

### 2.3 売り文句の核

```
刺さらない言い方:
  × 「見積の事務作業を効率化します」（営業事務の話→安い価値に聞こえる）
  × 「材料費を自動で調べます」（調達の話→それだけなら要らない）

刺さる言い方:
  ◎ 「社長、あなたの見積の"頭"を再現します」
  ◎ 「社長がいなくても8割の精度で見積が出せるようになります」
  ◎ 「後継者が見積できるようになるまでの橋渡しをします」
  ◎ 「使うほど賢くなって、半年後には経営判断を助けます」
```

## 3. 「Excel見積で十分」層の動かし方

### 3.1 Excelの限界を自覚させるアプローチ

| # | アプローチ | 刺さる層 | 具体的なアクション |
|---|---|---|---|
| 1 | **速度訴求** | 回答遅延で失注している会社 | 「見積回答を翌日→2時間に。先に回答した会社が受注する」 |

| 3 | **実績の不可視** | データ経営に関心ある2代目 | 「過去3年の受注率、失注理由、分かりますか？Excelでは分析できません」 |
| 4 | **★無料見積再現テスト** | **全層に有効。最強のリードジェン** | 下記参照 |

### 3.2 無料見積再現テスト（リードジェネレーション武器）

```
営業フロー:
  1. 「過去の見積書10件と図面をください」（NDA締結）
  2. AIで見積を再現し、実見積との比較レポートを無料作成
  3. 誤差率と「AI化した場合の月間時間削減」を提示
  4. 「おお」体験 → スターター契約

  コスト: LLM費用 ¥30-60/10件 + 人件費2時間 ≒ 実質¥0
  転換率目標: 30%以上（「おお」と言った人はほぼ契約する）
```

## 4. 技術戦略

### 4.1 図面OCR精度

| 項目 | 方針 |
|---|---|
| エンジン | Document AI（構造化抽出）+ Gemini 2.5 Flash マルチモーダル（工程推定） |
| 精度目標 | 寸法読み取り95%以上、工程推定80%以上（人間確認前提） |
| 対応形式 | CAD出力のPDF。手書き図面は対象外と明示 |
| 製造図面の特殊記号 | 幾何公差(GD&T)・表面粗さ・溶接記号はGeminiマルチモーダルで認識 |
| 検証方法 | パイロット1社から過去見積20件を借り、AI見積vs実見積を比較。誤差率±20%以内が合格 |

### 4.2 3D CADデータ対応

| 対象 | 時期 | 理由 |
|---|---|---|
| 2D PDF図面 | Phase 1（今） | 金属切削は2D図面が主流。α型のメイン |
| DXF/DWG | Phase 2 | 板金はDXFが標準。β型で必須 |
| STEP/IGES | Phase 2+ | プラ成型は3Dが必須。Open Cascade (pythonOCC)でジオメトリ解析 |
| フルCAM連携 | Phase 3+ | 工程推定を3Dモデルから直接行う |

### 4.3 チャージレート初期設定

```
3段階オンボーディング:

  Day 0（自動）:
    YAMLデフォルト値で即座に使える状態にする
    「業界平均」として提示
    → 地域別×設備年式別にYAMLを拡張（東大阪と諏訪で¥1,000-2,000違う）

  Day 1（30分ヒアリング）:
    「御社のMCのチャージレートは？」を15設備分聞く
    → ゲノムに反映。これだけで精度が大幅改善

  Month 1（自動学習）:
    見積修正データから実態チャージレートを逆算して提案
    「御社のMCは平均¥9,200/hで使われています。
     デフォルトの¥8,000を更新しますか？」
```

### 4.4 見積精度の合格ライン

| 段階 | 精度目標 | 説明 |
|---|---|---|
| Level 0（デフォルト値のみ） | ±30-40% | 業界平均値。「桁は合っている」 |
| Level 1（ヒアリング後） | **±15-20%** | 自社チャージレート反映済み。**合格ライン** |
| Level 2（3ヶ月運用後） | ±5-10% | 実績学習済み。「ほぼそのまま出せる」 |

熟練者の見積でも±10-15%のブレがある。Level 1で±20%以内なら「AIの下書きとして十分」。

## 6. 競合ポジショニング

| 競合 | ポジション | シャチョツーとの違い |
|---|---|---|
| meviy（ミスミ） | 即時見積 → ミスミから購入 | 受発注プラットフォーム。自社の見積力は育たない |
| CADDi | 図面データ管理 + 発注先マッチング | 発注者向け。加工業者の見積業務は解決しない |
| Terminal Q | 金属加工の自動見積 | 見積特化SaaS。ゲノム/データ蓄積/経営提案なし |
| 見積RICH/見積革命 | 見積書作成ソフト | 工程推定AIなし。Excelの延長 |
| TECHS/AMMIC | 生産管理SaaS | 見積機能はあるが工程推定AIなし。管理寄り |
| **シャチョツー** | **BPO型。見積AI→データ蓄積→経営参謀** | **唯一「使うほど賢くなり、経営提案までする」** |

差別化の核心: ベースがテンプレートであり、そこから自社独自のブレインを入れていくことで個別のオーダーメイドBPOができる。他社はSaaS（ツール提供）、シャチョツーはBPO（業務代行+知能蓄積）。
