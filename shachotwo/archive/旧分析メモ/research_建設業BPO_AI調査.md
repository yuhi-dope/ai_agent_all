## 1. 積算（Cost Estimation）

### 1.1 公共工事の積算の仕組み

公共工事の予定価格は、国土交通省が定める「公共建築工事標準単価積算基準」に基づいて算出される。単価は以下の優先順位で採用される:

1. **市場単価・単位施工単価** (最優先)
2. **複合単価**
3. **刊行物の掲載価格**
4. **見積単価**

#### 設計単価・市場単価・見積単価の違い

| 単価種別 | 定義 | 用途 | データソース |
|---|---|---|---|
| **設計単価** | 発注者が予定価格算定時に使用する公定単価 | 予定価格の積算 | 国交省・各自治体が毎年度公表 |
| **市場単価** | 市場における実際の取引価格を調査・反映した単価 | 市販の建築施工単価誌に掲載。材料+施工を含む「施工単位」あたりの価格 | 建設物価調査会、経済調査会が月次調査 |
| **複合単価** | 標準的な材料費・労務費・機械費等の内訳を積み上げた単価 | 市場単価がない工種で使用 | 積算基準書の歩掛 x 単価で算出 |
| **見積単価** | 製造業者・専門工事業者から取得する見積価格 | 特殊工事、刊行物に掲載がない場合 | 複数業者から見積取得、最安値ベースで決定 |

#### 積算の構造

```
工事費 = 直接工事費 + 共通仮設費 + 現場管理費 + 一般管理費等

直接工事費 = Σ(数量 x 単価)
  - 材料費: 建設物価・積算資料の掲載単価
  - 労務費: 公共工事設計労務単価
  - 機械費: 機械経費積算基準
```

## 2. 安全書類（グリーンファイル）

### 2.1 グリーンサイトとは

- **提供元**: MCデータプラス(エムシーディースリー株式会社)
- **概要**: 安全書類(グリーンファイル)をインターネット上で作成・提出・確認できるクラウドサービス
- **登録企業数**: **100万社超**(2026年2月末時点)、元請契約企業数949社
- **登録作業員数**: 160万人以上(2021年時点)
- **市場シェア**: 文書作成・管理ソリューション分野で **4年連続No.1、約50%のシェア**
- **効果**: 1年以上利用したユーザーの80%が月50%以上の業務削減を実感
- **CCUS連携**: 1現場あたり月間10時間を削減

### 2.2 全建統一様式の種類一覧

全建統一様式は一般社団法人全国建設業協会が発行する標準書式で、以下の種類がある:

| 様式番号 | 書類名 | 用途 |
|---|---|---|
| 第1号 | 再下請負通知書 | 下請の再下請を元請に通知 |
| 第1号-甲 | 下請負業者編成表 | 下請の組織体制を一覧化 |
| 第2号 | 施工体制台帳 | 工事に関わる全業者の体制記録 |
| 第3号 | 施工体系図 | 施工体制の図式化 |
| 第4号 | 工事作業所災害防止協議会兼施工体系図 | 安全協議会の運営記録 |
| **第5号** | **作業員名簿** | **現場入場する全作業員の一覧(最重要)** |
| 第5号-別紙 | 社会保険加入状況 | 社会保険の加入確認 |
| 第6号 | 工事安全衛生計画書 | 安全衛生の計画 |
| 第7号 | 新規入場時等教育実施報告書 | 安全教育の記録 |
| 第8号 | 安全ミーティング報告書 | 日々のKY活動記録 |
| 第9号 | 有資格者一覧表 | 資格保有者の管理 |
| 第10号 | 持込機械等(移動式クレーン等)使用届 | 重機の使用届出 |
| 第11号 | 持込機械等(電動工具等)使用届 | 電動工具等の使用届出 |
| 第12号 | 火気使用願 | 火気使用の許可申請 |

その他、以下のような関連書類も存在する:
- 通勤用車両届
- 工事用車両届
- 危険物・有害物持込使用届
- 安全衛生日誌

### 2.3 最も手間がかかる安全書類

**作業員名簿(第5号)が圧倒的に手間がかかる**。理由:

1. **人数が多い**: 現場に入場する全作業員分の作成が必要(大規模現場で数百人)
2. **記載項目が多い**: 氏名、生年月日、住所、血液型、社会保険番号、資格、健康診断日、入場年月日、雇入年月日、経験年数、特別教育、技能講習、免許等
3. **更新頻度が高い**: 作業員の入退場のたびに更新が必要
4. **社会保険加入状況の確認**: 2024年以降、社会保険加入が厳格化。健康保険・年金・雇用保険の番号確認が必要
5. **二重入力**: 元請ごとに書式が異なる場合、同じ情報を何度も入力する必要がある
6. **有資格者の管理**: 資格証の有効期限管理、写し添付が必要

**AI自動化の機会**: 作業員マスタデータから自動生成、資格期限アラート、入退場管理との連携が大きな効率化ポイント。

## 3. 出来高・請求

### 3.1 出来高請求の仕組み

#### 基本概念

- **出来高払い**: 工事の仕上がり状況(出来形)に応じて工事代金を分割支払いする方式
- **出来形**: 工事の目的物のできあがった部分
- **出来高**: 出来形に相応する請負代金額

#### 出来高請求のフロー

```
1. 契約時に出来高払いの条件を合意(支払回数、検査時期等)
2. 月次(または任意の期間)で出来高調査書を作成
   - 工事区分、工種、種別、細別、規格を記載
   - 設計数量に対する進捗率を算定
   - 進捗率 x 契約金額 = 出来高金額を算出
3. 出来高検査: 施工箇所が設計図通りか確認
4. 検査合格後、出来高に基づく請求書を発行
5. 元請は請求から1か月以内に支払い義務あり
```

#### 請求書の構造

```
当月請求額 = 当月出来高累計額 - 前月までの出来高累計額

記載項目:
- 注文番号
- 契約金額
- 前月迄出来高累計額
- 当月出来高明細(工種別)
- 当月請求額
```

### 3.2 電子帳簿保存法対応の要件

2024年1月より電子取引のデータ保存が完全義務化された。

#### 3つの保存区分

| 区分 | 内容 | 義務 |
|---|---|---|
| 電子帳簿等保存 | 会計ソフト等で作成した帳簿・書類の電子保存 | 任意 |
| スキャナ保存 | 紙の書類をスキャンして電子保存 | 任意 |
| **電子取引** | **メール・Web等で授受した電子データの保存** | **義務** |

#### 建設業で対象となる主な書類

- 注文書・発注書
- 見積書
- 請求書(出来高請求書含む)
- 契約書
- 領収書

#### 保存要件

1. **真実性の確保**: タイムスタンプ付与、または履歴が残るクラウドシステムの利用
2. **検索機能**: 取引年月日・取引金額・取引先で検索可能なこと
3. **保存期間**: 確定申告書の提出期限の翌日から**7年間**
4. **改ざん防止**: 訂正・削除の履歴が残ること

**備考**: 履歴が残るクラウドシステムを利用する場合、タイムスタンプは不要。電子契約に切り替えれば**印紙税が不要**になるメリットあり。

**API公開が不明/限定的なサービス:**
- グリーンサイト、蔵衛門、SPIDERPLUS、Photoruction、KENTEM等は公開APIの情報が見つからなかった。個別連携やCSVエクスポートが主な連携手段と思われる。

**シャチョツーへの示唆**: 建設業SaaSのAPI公開は極めて限定的。ANDPAD APIへの接続を最優先とし、それ以外はCSV/Excel取り込みやRPA的なアプローチが現実的。

## 6. シャチョツーへの示唆(まとめ)

### 6.1 BPO自動化の優先領域

| 優先度 | 領域 | 理由 | 難易度 |
|---|---|---|---|
| **1** | 安全書類(作業員名簿)自動生成 | 最も手間がかかる。マスタデータ管理で大幅効率化可能 | 中 |
| **2** | 出来高管理・請求書自動生成 | Excel依存が多い。電帳法対応ニーズ高い | 中 |
| **3** | 積算支援(拾い出しAI) | 市場が大きいが、単価データのライセンスがボトルネック | 高 |
| **4** | 経営ダッシュボード(赤字工事予測) | データ統合が価値。他SaaSとの連携が課題 | 中 |
| **5** | 行政手続き(経審・許可更新) | 定型業務でAI化しやすい。競合少ない | 低 |

### 6.2 データ連携戦略

```
Phase 1: ANDPAD API連携(デファクト施工管理)
Phase 2: グリーンサイトCSV取り込み(安全書類データ)
Phase 3: どっと原価WEB-API連携(原価データ)
Phase 4: 建設物価データファイル契約(単価マスタ)
```

### 6.3 注意点

- **単価データは有償ライセンス**: 建設物価調査会・経済調査会のデータは無償公開されていない。ビジネスモデルに組み込む場合、データライセンス費用を考慮
- **公共工事設計労務単価はPDFのみ**: スクレイピング or 手動データ化が必要
- **グリーンサイトのシェアが圧倒的**: 安全書類領域で戦うならグリーンサイト連携が必須
- **中小企業向けは未開拓**: 大手向けSaaSは充実しているが、中小建設業(社員10-50名)向けの簡単・低価格なソリューションが不足
