## 1. 不動産管理業の基本構造

### 1.1 市場規模

```
賃貸住宅管理業者: 登録業者約8,900社（国交省）
管理戸数: 全国約1,600万戸（民間賃貸住宅の約80%が管理委託）
シャチョツーのターゲット: 管理戸数500-5,000戸の中小管理会社
```

### 1.2 不動産管理会社の役割

```
オーナー（物件所有者）が管理会社に委託する業務:
  ① 入居者募集（空室を埋める）
  ② 家賃の集金・送金
  ③ 入居者対応（クレーム、修繕要望）
  ④ 建物の維持管理（修繕、清掃、設備点検）
  ⑤ 契約管理（更新、退去手続き）
  ⑥ オーナーへの報告（月次収支報告）

→ オーナーの代わりに「大家業」を代行するのが管理会社
```

### 1.3 関係者構造

```
オーナー（物件所有者）
  │
  ├─ 管理会社 ← ★シャチョツーのクライアント
  │    ├─ 入居者（テナント）
  │    ├─ 仲介会社（入居者を見つけてくる）
  │    ├─ 保証会社（滞納時に立替払い）
  │    ├─ 修繕業者（設備故障・原状回復）
  │    └─ 保険会社（火災保険等）
  │
  └─ 銀行（ローン返済。オーナー側）
```

### 1.4 収入構造

```
管理会社の収入源:
  ① 管理手数料: 家賃の3-5%（最も安定した収入）
     例: 1,000戸 × 平均家賃8万 × 5% = 月400万円
  ② 仲介手数料: 家賃1ヶ月分（入居時のみ）
  ③ 更新手数料: 更新料の一部（更新時のみ）
  ④ 修繕マージン: 修繕工事の10-30%
  ⑤ その他: 鍵交換、ハウスクリーニング等

→ ①が生命線。入居者が家賃を払わない = 管理会社の売上も止まる
→ だから家賃管理・督促が最重要業務
```

### 1.5 管理会社の組織（10-50名）

```
社長 ─┬─ PM部門（プロパティマネジメント）← ★家賃管理はここ
      │    ├─ PM担当A（200-500戸を担当）
      │    ├─ PM担当B
      │    └─ PM担当C
      │
      ├─ リーシング部門（入居者募集）
      │    ├─ 営業担当
      │    └─ 内見案内
      │
      ├─ 修繕・BM部門（ビルマネジメント）
      │    ├─ 修繕手配
      │    └─ 定期巡回
      │
      └─ 経理・事務
           ├─ 経理（家賃消込・送金）← ★AI化の主ターゲット
           └─ 事務
```

### 1.6 管理の種類

| 種類 | 対象 | 主な業務 |
|---|---|---|
| 賃貸管理（PM） | アパート・マンション | 家賃管理、入居者対応、修繕 |
| ビルメンテナンス（BM） | オフィスビル・商業施設 | 設備管理、清掃、警備 |
| 分譲マンション管理 | 分譲マンション | 管理組合の運営支援 |

※ シャチョツーの初期ターゲットは賃貸管理（PM）

## 2. 家賃管理・督促とは何か

### 2.1 定義

```
家賃管理 = 入居者からの家賃入金を確認し、オーナーに送金する業務
督促 = 家賃を支払わない入居者に対して支払いを求める業務

  入力: 請求データ（誰がいくら払うべきか）+ 入金データ（実際の入金）
  処理: 消込（突き合わせ）→ 未入金の検出 → 督促 → オーナー送金
  出力: 消込結果、督促記録、送金明細、収支報告書
```

### 2.2 消込（けしこみ）とは

```
消込 = 「請求」と「入金」を突き合わせて、誰が払ったかを確定する作業

  請求一覧:                    入金一覧（銀行明細）:
  田中太郎  ¥80,000           ¥80,000 タナカタロウ
  鈴木花子  ¥65,000           ¥65,000 スズキハナコ
  山田次郎  ¥72,000           ¥72,000 ヤマダ
  佐藤一郎  ¥90,000           （入金なし）

  → 田中: OK、鈴木: OK、山田: OK（名義が微妙に違うが金額一致）
  → 佐藤: 未入金 → 督促対象

問題点:
  - 手作業で月40-60時間かかる（1,000戸の場合）
  - 名義不一致（家族が振り込む、会社名義等）→ 手動で判断
  - 部分入金（金額が違う）→ 何の支払いか特定が難しい
  - 複数物件のオーナーが違う → 送金先の仕分けが複雑

→ ★AI化の最有力ターゲット
```

## 3. 家賃管理・督促の完全業務フロー（★最重要セクション）

### 3.0 全体フロー図（月次サイクル）

```
月初:
  Step 1: 請求データの確認・生成
    │
  Step 2: 入金確認・消込 ← 最も時間がかかる（月の1-5日）
    │
  Step 3: 滞納検出・督促開始
    │
月中:
  Step 4: 督促の段階的エスカレーション
    │
月末:
  Step 5: オーナーへの送金
    │
  Step 6: 月次報告書の作成
```

### Step 1: 請求データの確認・生成

```
■ 家賃の構成要素:
  月額家賃（賃料）: 物件の基本家賃
  共益費（管理費）: 共用部の維持費
  駐車場代: 駐車場を借りている場合
  水道代: 管理会社が一括徴収する場合
  その他: 町内会費、インターネット代等

  例: 田中太郎さんの月額請求
    賃料: ¥65,000
    共益費: ¥5,000
    駐車場: ¥8,000
    水道代: ¥2,000
    合計: ¥80,000

■ 請求データの管理:
  管理ソフト（いい生活、賃貸革命、ESいい物件One等）で管理
  or Excelで管理（中小はこちらが多い）

■ 月初の確認事項:
  - 新規入居者の追加（日割り計算が必要な場合あり）
  - 退去者の削除（日割り計算）
  - 家賃改定の反映
  - 前月の滞納繰越額の加算
```

### Step 2: 入金確認・消込

```
■ 入金経路:
  ① 口座振替（最も多い。60-70%）
     管理会社 → 収納代行会社（SMBCファイナンスサービス等）
     → 入居者の口座から自動引落
     → 振替結果データが管理会社に返却

  ② 銀行振込（20-30%）
     入居者が自分で管理会社の口座に振り込む
     → 銀行明細のCSVをダウンロード → 手動で消込

  ③ クレジットカード（増加中。5-10%）
     カード決済サービス経由
     → 決済データが管理会社に連携

  ④ 現金持参（減少中。1-5%）
     入居者が管理会社の窓口に持ってくる
     → 領収書発行 → 手入力で消込

■ 消込の手作業フロー（銀行振込の場合）:
  1. 銀行の管理画面にログイン
  2. 入出金明細をCSVでダウンロード
  3. Excelで請求一覧と突き合わせ
  4. 金額一致 + 名義一致 → 消込OK
  5. 金額一致 + 名義不一致 → 手動で確認
     （例: 「タナカ」「田中太郎」「タナカタロウ」「(株)田中商事」）
  6. 金額不一致 → 部分入金? 複数月分? → 手動で確認
  7. 入金なし → 滞納リスト入り
  8. 不明入金 → 誰の入金か特定（最も時間がかかる）

■ 口座振替の場合:
  振替結果データ（CSV）に「振替OK/振替NG（残高不足等）」が記載
  → NGの入居者 = 滞納候補
  → OKの入居者 = 自動消込
  → 口座振替は消込が楽（★AI化以前に口座振替への移行を推奨）

■ 消込の所要時間:
  1,000戸の場合: 月40-60時間（経理1-2名がかかりきり）
  500戸の場合: 月20-30時間
  → 管理戸数に比例して増加
```

### Step 3: 滞納検出・督促開始

```
■ 滞納の検出:
  消込完了 → 未消込 = 滞納
  → 滞納リスト作成（名前、物件、金額、滞納月数）

■ 滞納率の実態:
  全国平均: 5-8%
  都市部: 3-5%（保証会社加入率が高い）
  地方: 8-12%
  → 1,000戸管理で常時50-80戸が何らかの滞納状態

■ 督促開始の判断:
  口座振替NGの場合: 翌日から督促開始
  振込未入金の場合: 支払期日の翌日から
  ※ 「うっかり忘れ」が8割。最初の連絡で7割が解決
```

### Step 4: 督促の段階的エスカレーション

```
■ 督促フロー（業界標準）:

  [Day 1-3] 電話連絡
    「お支払いの確認です。行き違いでしたら申し訳ございません」
    → 70%がこの段階で解決

  [Day 4-7] SMS/メール
    「お家賃の入金が確認できておりません。〇月〇日までにお支払いください」
    → さらに15%が解決

  [Day 8-14] 督促状（書面）
    郵送。「期日までにお支払いがない場合は保証会社に連絡します」
    → さらに5%が解決

  [Day 15-30] 内容証明郵便
    法的な証拠力のある書面。「〇日以内に支払わなければ法的措置を検討」
    → さらに3%が解決

  [Month 2] 保証会社への代位弁済請求
    保証会社加入の場合: 保証会社が管理会社に立替払い
    → 保証会社が入居者に直接請求（管理会社の手を離れる）

  [Month 3+] 法的手続き
    保証会社未加入の場合:
    明渡請求訴訟 → 強制執行
    ※ 弁護士費用: 30-50万円
    ※ 訴訟から退去まで3-6ヶ月

■ 保証会社の仕組み:
  入居時: 入居者が保証会社と契約（保証料: 家賃の0.5-1ヶ月分）
  滞納時: 管理会社が保証会社に「代位弁済請求」
    → 保証会社が管理会社に家賃を立替払い
    → 保証会社が入居者に求償（取り立て）
  メリット: 管理会社は滞納リスクから解放される
  現在: 新規契約の80-90%が保証会社加入

AI化ポイント:
  - 滞納検出の自動化（消込と連動）
  - 督促テンプレートの自動生成（SMS、メール、書面）
  - 督促進捗の一元管理
  - 保証会社への代位弁済請求の自動化
```

### Step 5: オーナーへの送金

```
■ 送金額の計算:
  オーナーへの送金額 = 入金額 - 管理手数料 - 立替経費

  例: 10戸のアパート（オーナーAさん）
    家賃合計: ¥800,000
    入金額: ¥720,000（1戸滞納）
    管理手数料(5%): ¥36,000
    修繕費立替: ¥30,000（先月の水漏れ修理）
    送金額: ¥720,000 - ¥36,000 - ¥30,000 = ¥654,000

■ 送金タイミング:
  月末 or 翌月10日（管理委託契約による）

■ 送金明細:
  物件ごとの入金状況、管理手数料、立替経費、送金額を記載
  → Excelで手作業作成が多い（★AI化対象）

■ 複数オーナーの場合:
  管理会社は数十〜数百人のオーナーに送金
  → オーナーごとに物件を仕分けて送金額を計算
  → 送金ミス = 信頼失墜 → 管理委託の解約リスク
```

### Step 6: 月次報告書の作成

```
■ オーナーへの月次報告:
  ① 収支報告書（入金・支出・送金額の明細）
  ② 滞納状況報告（誰がいくら滞納しているか）
  ③ 修繕報告（実施した修繕の内容・費用）
  ④ 空室状況（空室の募集状況・反響数）

■ 年次報告:
  確定申告用の年間収支明細
  → 不動産所得の申告に必要（オーナーの税理士に提出）

AI化ポイント: 消込データから月次報告書を自動生成
```

## 4. 判断分岐・例外パターン（★AI化に必須）

### 4.1 退去月の日割り計算

```
月途中の退去 → 日割り家賃の計算が必要
  日割り家賃 = 月額家賃 × 居住日数 ÷ 月の日数
  例: 3/15退去、月額¥80,000 → ¥80,000 × 15 ÷ 31 = ¥38,710

敷金の精算:
  敷金 - 原状回復費用 = 返金額
  ※ 国交省「原状回復をめぐるトラブルとガイドライン」に基準あり
```

### 4.2 法人契約 vs 個人契約

```
法人契約: 会社が社員の社宅として契約
  - 家賃は会社が支払い（滞納リスク低い）
  - 入居者（社員）が変わっても契約は継続
  - 解約は会社の判断（転勤等）

個人契約: 入居者本人が契約
  - 滞納リスクは入居者個人の信用力次第
  - 保証会社が必須
```

### 4.3 更新料の徴収

```
更新時期（通常2年ごと）:
  更新料: 家賃1-2ヶ月分（地域による）
  更新事務手数料: ¥10,000-30,000
  火災保険更新: ¥15,000-20,000（2年分）
  保証会社更新: ¥10,000-20,000

フロー:
  更新6ヶ月前 → 更新案内送付 → 入居者の意思確認 → 更新 or 退去

注意:
  更新案内を送り忘れ → 法定更新（自動更新）→ 更新料を取り損ねる
  → ★期限管理のAI化が重要
```

### 4.4 保証会社審査落ち

```
新規入居申込 → 保証会社審査 → 不承認の場合:
  ① 別の保証会社で再審査
  ② 連帯保証人を追加して再審査
  ③ 敷金を増額して契約（オーナー了承が必要）
  ④ 入居不可
```

### 4.5 生活保護受給者の家賃代理納付

```
自治体から管理会社に直接家賃が振り込まれる制度
  → 入金名義が「〇〇市」になる
  → 消込で特殊対応が必要
  → 金額が住宅扶助基準内であること
```

### 4.6 オーナーチェンジ

```
物件が売買された場合:
  旧オーナー → 新オーナーに管理が移行
  - 敷金の引継ぎ
  - 管理委託契約の再締結
  - 入居者への通知
  - 送金先の変更

→ データの一斉更新が必要（★移行ツールとしてのAI価値）
```

### 4.7 家賃値上げ交渉中の入金処理

```
値上げ通知済み → 入居者が旧家賃で入金 → 差額をどう扱うか
  ① 旧家賃で消込して差額は後日請求
  ② 一部入金として処理して差額を滞納扱い
  → 法的にはグレーゾーン。管理会社の方針による
```

## 6. AI化マッピング

| 業務ステップ | AI化可否 | 自動化度 | 人間の確認 |
|---|---|---|---|
| 請求データ生成 | ◎ | 90% | 変更チェック |
| 口座振替の消込 | ◎ | 95% | 不要（データ一致） |
| 銀行振込の消込（名義一致） | ◎ | 90% | 不要 |
| 銀行振込の消込（名義不一致） | ○ | 70% | 確認必要 |
| 不明入金の特定 | △ | 50% | 判断必要 |
| 滞納検出 | ◎ | 95% | 不要 |
| 督促テンプレート生成 | ◎ | 90% | 最終確認 |
| 督促進捗管理 | ◎ | 85% | 判断のみ |
| 保証会社への代位弁済請求 | ○ | 70% | 書類確認 |
| 送金額計算 | ◎ | 95% | 最終確認 |
| 月次報告書生成 | ◎ | 90% | 最終確認 |
| 日割り計算 | ◎ | 95% | 不要 |

### AI化の優先順位

```
Phase 1（即座に価値）:
  ① 銀行振込の自動消込（名義マッチング+金額マッチング）
  ② 滞納の自動検出+督促リスト生成
  ③ 送金額の自動計算+送金明細生成
  → 消込作業の60-70%を自動化

Phase 2（3ヶ月後）:
  ④ 督促テンプレートの自動送信（SMS/メール）
  ⑤ 月次報告書の自動生成
  ⑥ 更新期限の自動管理+案内送付

Phase 3（6ヶ月後）:
  ⑦ 不明入金のAI特定（過去パターン学習）
  ⑧ 滞納予測（入金パターンから遅延リスクを予測）
```
