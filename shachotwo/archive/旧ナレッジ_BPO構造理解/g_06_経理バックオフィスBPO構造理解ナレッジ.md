# 経理・バックオフィスBPO 構造理解ナレッジ

> **背景**: 自社が子会社を設立し、外部コンサル会社と連携して建設企業等の経理BPOを実施。
> BPO業務を通じて既存顧問士業の品質を可視化し、士業事務所変更の営業リストを獲得するビジネスモデル。

---

## 1. ビジネスモデル全体像

```
自社（親会社）
  └── 子会社（経理BPO実行主体）
        ├── 外部コンサル会社（実務指導・ノウハウ提供）
        └── 建設クライアント群の経理を受託
              │
              ▼
        経理データを扱う過程で
        既存顧問税理士・会計士の仕事ぶりが丸見えになる
              │
              ▼
        「対応が遅い / 高い / ミスが多い」が定量的に可視化
              │
              ▼
        士業変更の提案 → 提携士業事務所へ紹介（営業リスト化）
```

### 収益ポイント
| 収益源 | 形態 | 備考 |
|---|---|---|
| 経理BPO月額フィー | 固定月額 | クライアント企業から |
| 士業紹介フィー | 一時金 or 顧問料の継続キックバック | 提携士業事務所から |
| AI自動化によるマージン拡大 | 内部効率化 | BPO原価を下げて利益率向上 |

### フライホイール構造
```
BPO受託数↑ → 経理データ蓄積↑ → 士業品質の可視化精度↑
  ↑                                        ↓
  └──── 士業紹介実績↑ ← 営業リスト精度↑ ←┘
```

---

## 2. 経理BPOの業務範囲

| レイヤー | 業務例 | 自動化余地 | 難易度 |
|---|---|---|---|
| 記帳代行 | 仕訳入力・領収書整理・通帳突合 | 高 | 低 |
| 請求管理 | 請求書発行・入金消込・督促 | 高 | 低〜中 |
| 支払管理 | 買掛管理・振込データ作成 | 中〜高 | 中 |
| 月次決算 | 試算表作成・部門別PL・経費分析 | 中 | 中〜高 |
| 給与計算 | 勤怠集計・社保計算・明細発行 | 中 | 中（法改正追従が肝） |
| 届出・法定調書 | 年末調整・償却資産・法定調書 | 低〜中 | 高（判断が必要） |

## 3. 経理BPO業務フロー

### 3-1. 記帳代行フロー

```
┌─────────────────────────────────────────────────────────┐
│ クライアント（建設会社）                                    │
│  現場・事務所から証憑を回収                                  │
│  - 領収書・レシート                                        │
│  - 請求書（受領分）                                        │
│  - 通帳コピー or ネットバンクCSV                             │
│  - クレジットカード明細                                     │
│  - 出金伝票・経費精算書                                     │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ① 証憑回収・デジタル化                                      │
│  - スキャン / スマホ撮影 → クラウドストレージにアップ           │
│  - OCRでテキスト抽出（日付・金額・取引先・摘要）               │
│  - 原本は電帳法要件に従いタイムスタンプ付与して保存             │
│                                                            │
│  ★ ボトルネック: 建設業は現場が散在し証憑が集まらない          │
│  ★ 対策: LINE/Slackで写真送付 → 自動OCR → 仕訳候補生成      │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ② 仕訳起票                                                 │
│  - 取引先×勘定科目のパターンマッチで自動仕訳候補を生成         │
│  - 過去仕訳の学習で80%は自動分類可能                         │
│  - 建設業特有の勘定科目に注意:                               │
│    - 完成工事原価 / 未成工事支出金                            │
│    - 工事未払金 / 完成工事未収入金                            │
│    - 外注費（一人親方・下請）                                 │
│  - インボイス番号の確認・適格/非適格の判定                     │
│                                                            │
│  ★ 判断が必要なもの: 工事別の原価按分、勘定科目の判断          │
│  ★ ここで税務判断に踏み込むと税理士法に抵触するので線引き必須   │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ③ 通帳突合・残高確認                                        │
│  - ネットバンクCSV or 通帳コピーと仕訳を突合                   │
│  - 不一致があれば原因調査（未記帳・二重計上・計上漏れ）         │
│  - 現金出納帳との照合                                        │
│  - クレジットカード明細との照合                               │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ④ 月次チェック・修正                                        │
│  - 勘定科目の異常値チェック（前月比で大きく変動した科目）       │
│  - 未処理取引の洗い出し                                      │
│  - 仮勘定・仮払金の精算確認                                  │
│  - 消費税区分の確認（課税/非課税/不課税/免税）                 │
└──────────────┬──────────────────────────────────────────┘
               ▼
       会計ソフトに仕訳データ反映
```

### 3-2. 請求管理フロー（手形・ファクタリング・出来高混在版）

```
┌──────────────────────────────────────────────────────────┐
│ ① 請求データ作成                                            │
│  - 工事台帳 / 出来高データから請求額を算出                     │
│  - 契約条件に応じた請求パターン:                              │
│    A) 出来高払い: 月末に当月出来高を査定→翌月請求              │
│    B) 完成払い: 引渡し完了後に一括請求                        │
│    C) 分割払い: 着手金30%→中間金40%→完成金30% 等              │
│    D) 前受金+出来高: 着手時に前受金→以降月次出来高払い         │
│  - インボイス要件（登録番号・税率区分）を満たす形式で発行       │
│                                                            │
│  ★ 実務: 出来高の査定は元請⇔下請で認識がズレることが多い      │
│    → 「出来高確認書」を元請に承認してもらってから請求が鉄則     │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ② 請求書送付・管理                                          │
│  - PDF生成 → メール or 郵送                                  │
│  - 送付日・支払期日をDB管理                                   │
│  - 得意先別の与信管理（支払いサイト: 建設は60-120日が一般的）   │
│  - 回収条件の管理（得意先ごとに支払手段が異なる）:             │
│    - 現金振込のみ                                           │
│    - 手形併用（例: 50%現金 + 50%手形）                       │
│    - 電子記録債権（でんさい）                                │
│    - ファクタリング利用                                      │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ③ 入金消込（決済手段別の処理分岐）                            │
│                                                            │
│  ■ パターンA: 現金振込                                      │
│    - ネットバンクCSVと請求データを自動マッチング               │
│    - 振込手数料が差引かれている場合の差額処理                  │
│    仕訳: 普通預金 ××× / 完成工事未収入金 ×××               │
│          支払手数料 ×× /                                    │
│                                                            │
│  ■ パターンB: 約束手形（受取手形）                           │
│    - 手形受領時: 受取手形 / 完成工事未収入金                  │
│    - 満期日に入金: 普通預金 / 受取手形                       │
│    - 手形割引（期日前に現金化）:                              │
│      普通預金 ××× / 受取手形 ×××                           │
│      手形売却損 ×× /（割引料）                              │
│    - 手形期日: 通常90-120日。資金繰り圧迫の主因               │
│    - 不渡りリスク: 手形が不渡りになった場合の回収不能処理       │
│                                                            │
│  ■ パターンC: 電子記録債権（でんさい）                       │
│    - 手形の電子版。受領・譲渡・割引がオンラインで完結          │
│    - 受領時: 電子記録債権 / 完成工事未収入金                  │
│    - 期日入金: 普通預金 / 電子記録債権                       │
│    - 分割譲渡が可能（手形にはない利点）                       │
│                                                            │
│  ■ パターンD: ファクタリング（売掛金の早期現金化）            │
│    - 2社間: 自社 ⇔ ファクタリング会社（手数料10-20%）         │
│    - 3社間: 自社 ⇔ 元請 ⇔ ファクタリング会社（手数料1-9%）   │
│    仕訳:                                                    │
│      普通預金 ××× / 完成工事未収入金 ×××                   │
│      売上債権売却損 ×× /（手数料）                          │
│    ★ ファクタリングは借入ではないのでBSに負債が載らない        │
│      → 経審スコアに影響しない（=建設業で好まれる理由）         │
│                                                            │
│  ■ パターンE: 出来高部分払い+相殺                            │
│    - 当月出来高から前月の不良品手直し費用を相殺されるケース     │
│    - 値引き・返品・前受金精算との複合消込                     │
│    仕訳: 普通預金 ×××     / 完成工事未収入金 ×××           │
│          完成工事原価 ×× /（相殺分）                        │
│                                                            │
│  ★ 1つの工事で複数パターンが混在するのが建設業の常態           │
│    例: 着手金=現金、中間金=手形50%+現金50%、完成金=でんさい    │
│  ★ 消込の自動化は「請求ID×決済手段×分割回」の3軸で管理       │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ④ 督促・回収                                                │
│  - 支払期日超過 → 自動リマインドメール                        │
│  - 一定期間超過 → 担当者にエスカレーション                     │
│  - 回収不能リスク → 貸倒引当金の検討（税理士と連携）          │
│  - 手形不渡り時の対応:                                       │
│    ① 不渡手形勘定に振替: 不渡手形 / 受取手形                 │
│    ② 回収交渉（分割払い等）                                 │
│    ③ 回収不能確定 → 貸倒損失計上（税理士の判断必要）         │
└──────────────────────────────────────────────────────────┘
```

### 3-3. 支払管理フロー（支払手段混在版）

```
┌──────────────────────────────────────────────────────────┐
│ ① 請求書受領・登録                                          │
│  - 下請・外注先・資材業者からの請求書を受領                    │
│  - OCRで読み取り → 金額・支払期日・振込先を自動抽出            │
│  - インボイス登録番号の確認（国税庁DBと照合）                  │
│  - 工事別に原価を紐付け                                      │
│  - 支払条件の確認:                                           │
│    - 末締め翌月末払い / 末締め翌々月10日払い 等               │
│    - 現金100% / 現金+手形 / でんさい 等                      │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ② 支払承認                                                  │
│  - 支払一覧表を作成 → 社長 or 経理責任者の承認                │
│  - 予算対比チェック（工事予算をオーバーしていないか）           │
│  - 資金繰り表と照合（支払可能か）                             │
│  - 支払優先度の判定:                                         │
│    ① 法定（税金・社保）→ ② 手形決済 → ③ 外注費              │
│    → ④ 材料費 → ⑤ 経費                                     │
│  ★ 手形の不渡りは2回で銀行取引停止→倒産なので最優先           │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ③ 決済手段別の実行                                           │
│                                                            │
│  ■ 現金振込                                                │
│    - 全銀フォーマットでFBデータ作成                           │
│    - ネットバンクにアップロード → 振込実行                     │
│    - 振込手数料の管理（先方負担 / 当方負担）                   │
│    仕訳: 工事未払金 ××× / 普通預金 ×××                    │
│                                                            │
│  ■ 支払手形の振出                                           │
│    - 手形帳から振出 → 相手先に郵送 or 手渡し                  │
│    - 手形期日管理台帳に登録（期日忘れ=不渡り=致命的）          │
│    - 振出時: 工事未払金 / 支払手形                            │
│    - 期日決済時: 支払手形 / 当座預金                          │
│    ★ 当座預金の残高管理が命。期日に残高不足=不渡り             │
│                                                            │
│  ■ 電子記録債務（でんさい）                                  │
│    - 発生記録 → 相手先のでんさい口座に通知                    │
│    - 発生時: 工事未払金 / 電子記録債務                        │
│    - 期日決済時: 電子記録債務 / 当座預金                      │
│    ★ 手形と違い紛失リスクがない。中小建設業で普及拡大中        │
│                                                            │
│  ■ 相殺処理                                                │
│    - 元請⇔下請で双方に債権債務がある場合に相殺               │
│    - 相殺通知書を作成・双方合意                              │
│    仕訳: 工事未払金 / 完成工事未収入金（相殺分）              │
└──────────────┬──────────────────────────────────────────┘
               ▼
┌──────────────────────────────────────────────────────────┐
│ ④ 支払記録・消込                                             │
│  - 支払完了データと買掛金（工事未払金）を消込                  │
│  - 源泉徴収が必要な支払の処理:                               │
│    - 個人の一人親方 → 源泉徴収不要（報酬ではなく請負代金）    │
│    - 個人の設計士・士業 → 源泉徴収必要（10.21%）             │
│    - 法人 → 源泉徴収不要                                    │
│  ★ 一人親方が「請負」か「雇用」かの判断を間違えると            │
│    源泉徴収漏れ+社保未加入のダブルリスク                      │
└──────────────────────────────────────────────────────────┘
```

### 3-6. 年次業務フロー（詳細版）

#### 月次サイクル（毎月の繰り返し）
```
月初 ──────────────────────────────────────────── 月末
  │                                                │
  │ 1日-5日: 前月の証憑回収締切・督促               │
  │ 5日-10日: 前月分の記帳完了・通帳突合             │
  │ 10日: 源泉所得税の納付期限（前月分）             │
  │ 10日-15日: 請求書発行（当月分）                  │
  │ 15日-20日: 月次決算処理・試算表作成              │
  │ 20日-25日: 給与計算・明細作成                    │
  │ 25日: 給与振込（会社による。末日の会社も多い）     │
  │ 末日: 社会保険料の納付・住民税の納付             │
```

## 4. 建設業特有の経理論点（実務処理の具体例付き）

### 4-1. 工事別原価管理

**概要**: 1工事=1プロジェクトとして原価を集計。工事台帳と会計の紐付けが命。

**原価4要素と実際の処理**:

| 原価要素 | 内容 | 実務での処理 |
|---|---|---|
| 材料費 | 資材・部材の購入費 | 納品書の工事番号と照合して工事別に計上。現場で共用する資材（釘・テープ等）は按分 |
| 労務費 | 自社社員の人件費 | 出面表（現場ごとの作業日数）から工事別に配賦。1日に複数現場を回る職人は時間按分 |
| 外注費 | 下請・一人親方への支払 | 注文書→請求書の工事番号で紐付け。最も金額が大きい（売上の50-70%） |
| 経費 | 重機リース・交通費・仮設費等 | 現場ごとのリース契約書で紐付け。共通経費は完工高比で按分 |

**実際の仕訳例（マンション新築工事の場合）**:
```
【着工時】仕訳なし（受注時に工事台帳を起票するのみ）

【材料仕入時】
  未成工事支出金（材料費） 5,000,000 / 工事未払金 5,000,000
  ※ 工事番号: A-2024-001 を摘要に記載

【外注費発生時】
  未成工事支出金（外注費） 8,000,000 / 工事未払金 8,000,000

【自社労務費（月末に配賦）】
  未成工事支出金（労務費） 2,000,000 / 労務費配賦 2,000,000
  ※ 出面表: 職人Aが当月20日のうち15日を本工事に従事 → 75%配賦

【工事完成・引渡し時】
  完成工事未収入金 30,000,000 / 完成工事高 30,000,000  ← 売上計上
  完成工事原価   15,000,000 / 未成工事支出金 15,000,000  ← 原価振替
```

**工事完成基準 vs 工事進行基準の実務判断**:
```
■ 工事完成基準（中小建設業の90%以上がこちら）
  - 引渡し完了まで売上を計上しない
  - 施工中の原価は全て「未成工事支出金」（BS=資産）
  - メリット: 処理がシンプル
  - デメリット: 期をまたぐ大型工事で利益が偏る

■ 工事進行基準（収益認識基準）
  - 進捗度に応じて売上を段階計上
  - 進捗度 = 発生原価 ÷ 見積総原価（原価比例法が一般的）
  - 適用要件: 工事収益総額・原価総額・進捗度を信頼性をもって見積もれること
  - 実務: 見積精度が低い中小では適用困難。大手ゼネコンや年商10億超で採用

■ BPOでの判断ポイント
  - クライアントがどちらを採用しているか初回で確認
  - 経審では完工高（完成工事高）が点数に影響するため、
    計上タイミングで戦略的に選ぶケースもある（税理士と要相談）
```

### 4-2. 出来高請求と資金繰り

**出来高請求の実際のフロー**:
```
工事期間: 6ヶ月、契約金額: 3,000万円の場合

月次出来高の流れ:
  1月（着工）: 出来高 500万 → 2月初に出来高確認書提出 → 2月末請求
               → 元請の支払サイト60日 → 4月末に入金
  2月: 出来高 600万 → 同上のサイクル → 5月末に入金
  3月: 出来高 700万 → ... → 6月末に入金
  ...

★ 問題: 1月に材料費・外注費を払うのに、入金は4月末
  → 3ヶ月分の運転資金が必要
  → 年商1億の会社なら常時2,500-3,000万円の運転資金が必要
```

**資金繰りの実務処理**:
```
┌──────────────────────────────────────────────────────────┐
│ 資金繰り表（キャッシュフロー予測）の作り方                     │
│                                                            │
│ ① 入金予定:                                                │
│   - 工事別の出来高予定 × 元請ごとの支払サイト                │
│   - 手形の満期日                                           │
│   - ファクタリング入金予定                                   │
│                                                            │
│ ② 出金予定:                                                │
│   - 外注費（月末締め翌月末払いが多い）                       │
│   - 材料費（都度払い or 月末締め）                           │
│   - 給与・社保（毎月25日 or 末日）                           │
│   - 税金（法人税・消費税・源泉税等の期日）                   │
│   - 手形の決済日（絶対に遅れてはいけない）                   │
│   - リース料・保険料                                        │
│                                                            │
│ ③ 過不足の判定:                                            │
│   - 不足が見込まれる月 → 手形割引 or ファクタリング or       │
│     短期借入を検討                                          │
│   - 余裕がある月 → 手形割引を控えて割引料を節約              │
│                                                            │
│ ★ 月次で社長に報告する最重要帳票。                           │
│   「来月300万足りません」を早期に把握できるかが経理BPOの価値   │
└──────────────────────────────────────────────────────────┘
```

### 4-3. 外注費・一人親方

**実務での判断フローチャート**:
```
支払先は法人？個人？
  │
  ├─ 法人 → インボイス登録番号確認 → あり: 仕入税額控除OK
  │                                → なし: 経過措置（80%→50%→0%控除）
  │
  └─ 個人（一人親方）
       │
       ├─ 契約形態は？
       │   ├─ 請負契約（自分の道具・自分の判断で施工）
       │   │   → 外注費で処理。源泉徴収不要
       │   │   → インボイス登録番号確認
       │   │
       │   └─ 実態が雇用（時間拘束・指揮命令あり・道具は会社支給）
       │       → ★偽装請負リスク
       │       → 本来は給与処理+社保加入が必要
       │       → 労基署の調査で発覚すると遡及で社保料+延滞金
       │
       └─ 設計士・測量士・コンサル（個人）
           → 報酬として処理。源泉徴収10.21%必要
           → 支払調書の作成（年間5万円超）
```

**インボイス制度の実務インパクト（2023年10月〜）**:
```
下請A社（適格請求書発行事業者）から100万円+消費税10万円の請求
  → 仕入税額控除: 10万円全額OK

一人親方B（免税事業者=インボイス未登録）から100万円+消費税10万円の請求
  → 経過措置期間中の仕入税額控除:
    2023/10-2026/9: 80%（8万円）控除可、2万円は控除不可
    2026/10-2029/9: 50%（5万円）控除可、5万円は控除不可
    2029/10〜: 0% 控除不可

★ 実務上の対応パターン:
  ① 一人親方にインボイス登録を促す（消費税の申告義務が発生するため嫌がる人も多い）
  ② 登録しない場合、控除できない消費税分を値下げ交渉する
  ③ 消費税分を自社が負担する（実質的な原価アップ）
  → BPOとしては、下請台帳にインボイス登録状況を管理するのが第一歩
```

### 4-4. 経営事項審査（経審）

**経審スコアの構成と経理の影響**:
```
総合評定値(P) = 0.25×X1 + 0.15×X2 + 0.20×Y + 0.25×Z + 0.15×W

X1: 完成工事高（完工高）
  → 売上の計上タイミングで変わる
  → 期末に完成見込みの工事を前倒しで引渡し完了にして完工高を上げる、
    という調整が（合法的に）行われることがある
  → BPOで月次の完工高を正確に追跡する意味がここにある

X2: 自己資本額・平均利益額
  → 利益が出ている＝スコアが高い。赤字工事は早めに損切り判断が必要
  → 自己資本を厚くするため、役員報酬を下げて内部留保を増やす等の戦略

Y: 経営状況（財務諸表から算出）
  → 8つの経営指標:
    ① 純支払利息比率（借入コストの低さ）
    ② 負債回転期間（負債の少なさ）
    ③ 総資本売上総利益率（稼ぐ力）
    ④ 売上高経常利益率（利益率）
    ⑤ 自己資本対固定資産比率（財務安定性）
    ⑥ 自己資本比率（自己資本の厚さ）
    ⑦ 営業キャッシュフロー（キャッシュ創出力）
    ⑧ 利益剰余金（蓄積された利益）
  ★ 全て経理処理の正確性に直結。仕訳ミスがスコアに影響する
  ★ ファクタリングは借入ではないので①②に悪影響しない → 好まれる理由

Z: 技術職員数・元請完工高
  → 経理というより人事・施工管理の領域

W: 社会性等
  → 社保加入状況（一人親方の偽装請負はここにも影響）
  → 退職金制度の有無
  → 建設業退職金共済（建退共）の加入
  → 法定外労災保険の加入
```

**経審を意識した経理処理の実務テクニック（税理士と連携）**:
```
1. 完工高の最大化
   - 期末月に引渡し可能な工事は極力当期に完成させる
   - 継続的な小工事（メンテナンス等）も完工高にカウント

2. 借入金の圧縮
   - 決算期末に一時的に借入金を返済（→期首に再借入）
   - ファクタリングで売掛金を現金化して借入返済に充てる

3. 未成工事支出金の適正管理
   - 着手していない工事の前渡金を未成工事支出金に含めない
   - 赤字確定工事は工事損失引当金を計上（PLで損失認識）

4. 減価償却の実行
   - 赤字でも減価償却は実施する（経審では未償却は減点対象）

5. 役員報酬の設計
   - 利益を内部留保に回すため、適正な役員報酬水準を設計
   → 税理士の判断領域だが、BPO側は月次数値を正確に提供する役割
```

### 4-5. 建設業許可の財務要件

**財務要件と実務での管理方法**:
```
■ 一般建設業の場合
  - 自己資本額 ≥ 500万円
  - OR 500万円以上の残高証明書
  → 中小でも通常クリア可能。ただし赤字が続くと危険

■ 特定建設業の場合（元請として4,000万円以上の下請契約を締結する場合）
  - 欠損の額 ≤ 資本金の20%
  - 流動比率 ≥ 75%
  - 資本金 ≥ 2,000万円
  - 自己資本 ≥ 4,000万円
  → 特定建設業は要件が厳しい。月次で流動比率を監視する必要あり

■ BPOでの管理ポイント
  - 毎月の月次決算で上記指標をダッシュボード化
  - 閾値に近づいたらアラート → 社長に報告
  - 許可更新の5年ごとの期日管理（期限切れ=建設工事ができなくなる）
  - 変更届の30日ルール（役員変更・営業所変更等を放置すると過料）
```

### 4-6. 建退共・中退共（退職金共済）

**実務での処理**:
```
■ 建設業退職金共済（建退共）
  - 公共工事では加入がほぼ必須（経審の加点対象）
  - 証紙方式: 1日あたり320円の証紙を手帳に貼付
  - 経理処理:
    証紙購入時: 退職金共済掛金 ××× / 現金預金 ×××
  - ★ 現場ごとに証紙の交付・受払を管理する必要がある
  - ★ 公共工事では「建退共の証紙受払簿」の提出を求められる

■ 中小企業退職金共済（中退共）
  - 事務職員など建退共の対象外の従業員向け
  - 月額掛金（5,000-30,000円）を会社が負担
  - 経理処理:
    毎月: 退職金共済掛金 ××× / 普通預金 ×××
```

### 4-7. JV（共同企業体）の経理

**実務での処理（大型工事で発生）**:
```
■ JVとは
  - 複数の建設会社が共同で1つの工事を受注する形態
  - 公共工事の大型案件で多い
  - 幹事会社（スポンサー）が経理を一括管理

■ 出資比率に応じた処理
  例: A社60%、B社40%のJVで、工事原価1億円の場合
  A社の仕訳: 未成工事支出金 60,000,000 / JV出資金 60,000,000
  B社の仕訳: 未成工事支出金 40,000,000 / JV出資金 40,000,000

■ BPOでの注意点
  - JV会計は幹事会社が作成するJV精算書に基づいて処理
  - 自社がJV構成員の場合、精算書の内容を検証する必要がある
  - JV工事の完工高は出資比率分のみ自社の完工高に計上
  → 経審スコアに影響するので計上漏れは致命的
```

## 7. 注意すべきリスク

### 税理士法リスク
- **記帳代行**: 無資格でOK（税理士法2条に該当しない）
- **税務相談・税務申告**: 税理士の独占業務（違反は2年以下の懲役 or 100万円以下の罰金）
- **線引き**: 「この仕訳はこうすべき」は税務判断に近いのでグレー。「過去の仕訳パターンに基づく候補提示」なら問題ない
- **対策**: 判断が必要なものは顧問税理士に確認するフローを明確にする

### 情報管理リスク
- 経理データ = 売上・利益・取引先・給与 = 最高機密
- 子会社⇔コンサル会社間のデータアクセス権限を明確化
- NDA + 情報管理規程は最低限必要
- クラウド会計のアカウント権限管理（閲覧/編集/管理者）

### 士業変更に伴うリスク
- 既存顧問が決算期途中で辞任すると、申告に支障が出る
- 引き継ぎ期間（最低2-3ヶ月）を確保する必要がある
- 顧問変更は決算終了後のタイミングがベスト
