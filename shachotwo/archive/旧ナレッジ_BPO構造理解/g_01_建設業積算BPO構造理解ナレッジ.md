## 1. 建設業の基本構造

### 1.1 市場規模と特徴

```
建設投資総額: 約80兆円/年（2026年度予測）
建設業者数: 約47万社（許可業者）
就業者数: 約480万人
平均年齢: 約44歳（他産業より高い）
中小企業比率: 99%以上
```

### 1.2 公共工事 vs 民間工事

| 項目 | 公共工事 | 民間工事 |
|---|---|---|
| 発注者 | 国・自治体・公団 | 企業・個人 |
| 入札 | 必須（競争入札） | 不要（指名・随意契約） |
| 積算基準 | 国交省基準（公開） | 各社独自 |
| 単価 | 設計労務単価（公表） | 市場実勢・交渉 |
| 内訳書 | 厳密な様式 | 比較的自由 |
| 予定価格 | あり（事前公表/事後公表） | なし |
| 利益率 | 低め（5-10%） | 交渉次第（10-20%+） |
| 透明性 | 高い（情報公開） | 低い |
| **AI化しやすさ** | **◎（ルールが標準化）** | **△（各社バラバラ）** |

### 1.3 元請・下請構造

```
発注者（国/自治体/企業）
  │
  ├─ 元請（ゼネコン/地場の総合建設会社）
  │    ├─ 一次下請（専門工事会社: 鉄筋、型枠、電気、配管等）
  │    │    ├─ 二次下請（さらに専門的な業者）
  │    │    └─ 二次下請
  │    ├─ 一次下請
  │    └─ 一次下請
  │
  └─ 設計事務所（図面を描く人たち。施工はしない）

※ シャチョツーのターゲット = 元請の中小建設会社（10-300名）
```

### 1.4 入札の仕組み

```
■ 一般競争入札
  条件を満たす業者は誰でも参加可能
  → 価格競争が激しい

■ 指名競争入札
  発注者が「この会社に入札させる」と指名
  → 実績のある会社が有利

■ 総合評価方式
  価格だけでなく技術力・実績も評価
  → 「安かろう」では勝てない

■ 随意契約
  入札なしで直接契約（小規模工事や緊急時）

入札のキーワード:
  予定価格: 発注者が設定する上限価格
  最低制限価格: これ以下では失格（ダンピング防止）
  落札率: 落札額 ÷ 予定価格 × 100（通常80-95%）
```

### 1.5 工事の流れ

```
受注 → 着工準備 → 施工 → 検査 → 引渡し → 瑕疵担保
│        │          │       │        │          │
│        │          │       │        │          └─ 引渡し後の不具合対応（2年/10年）
│        │          │       │        └─ 竣工（完成）
│        │          │       └─ 中間検査、完成検査
│        │          └─ 毎日: 朝礼→作業→安全確認→日報
│        └─ 施工計画書作成、安全書類作成、近隣挨拶
└─ 契約、着手届

※ 積算は「受注前」のフェーズ。入札に勝つために行う
```

### 1.6 建設会社の組織（10-300名規模）

```
社長 ─┬─ 工事部長 ─┬─ 現場監督A（現場常駐）
      │             ├─ 現場監督B
      │             └─ 現場監督C
      │
      ├─ 積算担当（1-3名。兼任の場合も多い）← ★ここがAI化対象
      │
      ├─ 安全管理（兼任が多い）
      │
      ├─ 営業（社長自身が兼任も多い）
      │
      └─ 事務・経理（2-5名）
```

## 2. 積算とは何か

### 2.1 定義

```
積算 = 工事にいくらかかるかを事前に計算すること

  入力: 設計図面、仕様書、数量計算書
  処理: 数量拾い出し → 単価選定 → 歩掛適用 → 諸経費計算
  出力: 工事費内訳書（見積書）

  → この金額で入札 or 見積提出 → 受注を目指す
```

### 2.2 なぜ重要か

```
1. 赤字受注の防止
   積算が甘い → 実際のコストが上回る → 赤字（数百万〜数千万の損失）

2. 入札での勝率
   積算が高すぎる → 競合に負ける
   積算が安すぎる → 赤字 or 最低制限価格割れで失格

3. 実行予算の基準
   積算結果 = 現場の予算目標。原価管理の出発点

4. 後継者問題
   積算できる人が55歳以上。10年後に半減
   → AI化の最大のモチベーション
```

### 2.3 積算の種類

| 種類 | タイミング | 精度 | 用途 |
|---|---|---|---|
| 概算積算 | 入札前の初期段階 | ±20-30% | 入札参加の判断 |
| 詳細積算 | 入札書類作成時 | ±5-10% | 入札金額の決定 |
| 実行予算 | 受注後 | ±3-5% | 現場の原価管理 |

### 2.4 積算ソフト

```
■ ATLUS（アトラス）: 最大手。公共工事向け。年間利用料50-100万円
■ Gaia（ガイア）: 中堅。使いやすさに定評
■ 武蔵: 内田洋行。中小向け
■ RIBC: 公益財団法人が提供。低価格

※ 中小建設会社ではExcel積算も多い（ソフト代が高いため）
※ シャチョツーが狙うのはこの「Excelで頑張っている層」
```

## 3. 積算の完全業務フロー（★最重要セクション）

### 3.0 全体フロー図

```
Step 1: 設計図書の受領・読み込み
  │
Step 2: 数量拾い出し ← 最も時間がかかる（全体の50-60%）
  │
Step 3: 単価の選定
  │
Step 4: 歩掛の適用
  │
Step 5: 諸経費の計算
  │
Step 6: 内訳書の作成・提出
```

### Step 1: 設計図書の受領・読み込み

```
■ 入手する書類:
  ① 図面（設計図）: 平面図、断面図、構造図、詳細図
  ② 仕様書: 工事全体の施工条件・基準
  ③ 特記仕様書: その工事固有の条件
  ④ 数量計算書: 発注者が計算した数量（公共工事の場合あり）
  ⑤ 設計書: 予定価格の根拠となる数量・単価一覧

■ 読み込みのポイント:
  - 図面と仕様書の矛盾がないか確認
  - 特記仕様書に「標準と異なる条件」がないか確認
  - 数量計算書がある場合 → そのまま使えることが多い（★AI化の起点）
  - 数量計算書がない場合 → 図面から自分で拾い出す（時間かかる）
```

### Step 2: 数量拾い出し（★最も時間がかかる）

```
■ 分類体系（工種 → 種別 → 細別 → 規格）

  大分類: 工種（こうしゅ）
    中分類: 種別（しゅべつ）
      小分類: 細別（さいべつ）
        仕様: 規格（きかく）

■ 具体例: 道路改良工事

  工種          種別          細別              規格              数量    単位
  ─────────────────────────────────────────────────────────────────────
  土工          掘削工        バックホウ掘削    0.8m3級           1,500   m3
  土工          掘削工        人力掘削          ─                 30      m3
  土工          埋戻工        埋戻し            ─                 800     m3
  土工          残土処理工    残土運搬          DID10km以内       700     m3
  コンクリート工 型枠工        普通型枠          ─                 250     m2
  コンクリート工 鉄筋工        鉄筋加工組立      SD345 D16         15,000  kg
  コンクリート工 コンクリート工 生コンクリート    21-8-40 N/mm2     300     m3
  舗装工        下層路盤工    砕石              RC-40             200     m3
  舗装工        上層路盤工    粒調砕石          M-30              150     m3
  舗装工        アスファルト舗装工 表層          密粒度(20) t=5cm  2,000   m2

■ 拾い出しの方法:
  ① 数量計算書がある場合: Excelから転記（最も早い）
  ② 設計書PDFがある場合: PDFから数量を読み取る
  ③ 図面から拾う場合: 長さ・面積・体積を手計算

  → ①②がAI化の最有力ターゲット
```

### Step 3: 単価の選定

```
■ 単価の種類と優先順位:

  1. 設計労務単価（公共工事の労務費）
     - 国交省が毎年2月に公表
     - 都道府県別 × 職種別の日額
     - 例: 東京都の普通作業員 = ¥24,000/日（2025年度）
     - 例: 東京都のとび工 = ¥28,000/日
     - ★全国共通の公開データ → AI化しやすい

  2. 市場単価（材料・機械の実勢価格）
     - 「建設物価」「積算資料」（月刊誌）に掲載
     - 例: 生コンクリート 21-8-40 = ¥15,500/m3（東京）
     - 年4回程度改定

  3. 見積単価（専門業者からの見積）
     - 大きな工種（鉄骨、杭等）は業者見積が必須
     - 2-3社から取って比較

  4. 自社実績単価
     - 過去の工事で実際にかかった単価
     - 最も信頼性が高いが、蓄積が必要
     - ★シャチョツーの「育つ単価DB」のコンセプト

■ 単価選定の判断フロー:
  公共工事設計労務単価にある? → YES → そのまま使用
    ↓ NO
  建設物価/積算資料にある? → YES → 掲載単価を使用
    ↓ NO
  業者見積が取れる? → YES → 2-3社比較
    ↓ NO
  過去実績がある? → YES → 実績単価を使用
    ↓ NO
  → 類似工種から推定（ここが最も属人的。AIの出番）
```

### Step 4: 歩掛（ぶがかり）の適用

```
■ 歩掛とは:
  ある作業を完了するのに必要な「人・機械・材料」の標準的な量

■ 具体例: バックホウ掘削 100m3 の歩掛

  必要資源          数量      単位    単価        金額
  ───────────────────────────────────────────────
  バックホウ運転    0.8日     日      ¥55,000     ¥44,000
  普通作業員        0.5人     人      ¥24,000     ¥12,000
  軽油              80L       L       ¥130        ¥10,400
  ───────────────────────────────────────────────
  合計（100m3あたり）                              ¥66,400
  → 単価 = ¥664/m3

■ 歩掛の入手先:
  ① 国交省「土木工事標準積算基準書」（★最重要。公開文書）
  ② 各地方整備局の補正基準
  ③ 自社の実績歩掛

■ 補正係数:
  標準歩掛 × 補正係数 = 実際の歩掛

  補正の例:
  - 夜間施工: ×1.3〜1.5
  - 狭隘な場所: ×1.2〜1.3
  - 水中施工: ×1.5〜2.0
  - 寒冷地: ×1.1〜1.2
  - 市街地（交通規制あり）: ×1.1〜1.2
```

### Step 5: 諸経費の計算

```
■ 工事費の積み上げ構造（★これを暗記する）

  ┌─ 直接工事費（Step 2-4で計算した金額の合計）
  │    ├─ 材料費
  │    ├─ 労務費
  │    └─ 機械経費
  │
  ├─ 共通仮設費（率 or 積み上げ）
  │    ├─ 運搬費
  │    ├─ 準備費
  │    ├─ 事業損失防止施設費
  │    ├─ 安全費
  │    ├─ 役務費
  │    ├─ 技術管理費
  │    └─ 営繕費
  │
  ├─ 現場管理費（率計算）
  │    ├─ 現場従業員給料
  │    ├─ 法定福利費
  │    ├─ 現場事務所費
  │    ├─ 通信交通費
  │    └─ その他
  │
  ├─ 一般管理費等（率計算）
  │    ├─ 本社経費
  │    ├─ 利益
  │    └─ 付帯費
  │
  └─ 消費税

■ 計算の流れ:
  直接工事費 = Σ(各工種の金額)
  純工事費 = 直接工事費 + 共通仮設費
  現場管理費 = 純工事費 × 現場管理費率
  工事原価 = 純工事費 + 現場管理費
  一般管理費等 = 工事原価 × 一般管理費率
  工事価格 = 工事原価 + 一般管理費等
  工事費 = 工事価格 + 消費税

■ 諸経費率の目安（公共土木）:
  共通仮設費率: 直接工事費の5-15%（工事規模による）
  現場管理費率: 純工事費の20-40%（工事規模による）
  一般管理費率: 工事原価の10-20%（工事規模による）

  ※ 率は工事規模が大きいほど下がる（スケールメリット）
  ※ 国交省が率表を公開している → AI化しやすい
```

### Step 6: 内訳書の作成

```
■ 工事費内訳書の構成:

  ① 鏡（かがみ）: 表紙。工事名、金額、会社名
  ② 内訳書: 工種別の金額一覧
  ③ 明細書: 種別・細別の金額
  ④ 単価表: 各項目の単価根拠

■ 出力形式:
  - 公共工事: 電子入札システムの指定フォーマット
  - 民間工事: Excel or PDF（比較的自由）

■ 提出まで:
  内訳書作成 → 上長チェック → 入札金額決定（経営判断）→ 提出
  ※ 最終的な入札金額は積算結果に「経営判断」を加味して決める
```

## 4. 判断分岐・例外パターン（★AI化に必須）

### 4.1 設計変更時の対応

```
工事中に設計変更が発生（よくある）:
  → 変更部分の数量を再計算
  → 新しい単価が必要な場合は協議
  → 変更契約（増額 or 減額）

AI化ポイント: 変更前後の数量差分を自動計算
```

### 4.2 特殊な施工条件

```
夜間施工 → 労務費×1.3〜1.5、機械経費割増
狭隘施工 → 大型機械が入れない→小型機械→歩掛変更
水中施工 → 仮締切・排水費用が追加
高所作業 → 足場費用追加
既設構造物近接 → 振動・騒音対策費用追加

AI化ポイント: 条件フラグから自動で補正係数を適用
```

### 4.3 新単価の作成

```
歩掛表にない工種の場合:
  ① 類似工種の歩掛を参考に作成
  ② 業者見積から逆算
  ③ 実績データから作成

→ 最も属人的な判断。AI化の難易度が高い部分
→ ただし「過去の新単価作成事例」を学習させれば改善可能
```

### 4.4 公共工事 vs 民間工事の積算の違い

| 項目 | 公共工事 | 民間工事 |
|---|---|---|
| 数量計算書 | 発注者が提供（多い） | 自分で拾い出し |
| 単価基準 | 国交省基準 | 自社基準 |
| 諸経費率 | 率表で計算 | 交渉・実績ベース |
| 提出様式 | 厳密（電子入札） | Excel/PDF |
| 価格交渉 | なし（入札一発） | あり（値引き交渉） |
| 利益率 | 固定的 | 案件による |

### 4.5 低入札価格調査制度

```
予定価格の70-85%以下で入札 → 「本当にこの金額でできるのか」調査される
  → 根拠を説明できないと失格
  → AI積算で根拠を明確化できる = 低価格でも説得力のある入札が可能
```

## 6. AI化マッピング

| 積算ステップ | AI化可否 | 自動化度 | 人間の確認 | 必要な入力 | 出力 |
|---|---|---|---|---|---|
| 設計図書の読み込み | ◎ | 80% | 矛盾チェック | PDF/Excel | 構造化データ |
| 数量拾い出し（数量計算書あり） | ◎ | 90% | 最終確認 | Excel/PDF | 工種別数量一覧 |
| 数量拾い出し（図面から） | △ | 40% | 全項目確認 | 図面PDF | 工種別数量一覧 |
| 設計労務単価の適用 | ◎ | 95% | 不要 | 地域+職種 | 日額単価 |
| 市場単価の適用 | ○ | 70% | 異常値チェック | 材料名+地域 | 単価候補 |
| 歩掛の適用 | ○ | 75% | 補正係数の確認 | 工種+条件 | 複合単価 |
| 諸経費計算 | ◎ | 95% | 不要 | 直接工事費+工事規模 | 諸経費金額 |
| 内訳書作成 | ◎ | 95% | 最終確認 | 全計算結果 | Excel/PDF |
| 新単価作成 | △ | 30% | 必須 | 類似事例 | 推定単価 |
| 入札金額決定 | × | 0% | 経営判断 | 積算結果 | （人間が決める） |

### AI化の優先順位

```
Phase 1（即座に価値が出る）:
  ① 数量計算書Excel/PDFの構造化取り込み → 数量一覧の自動生成
  ② 設計労務単価・諸経費率の自動適用
  ③ 内訳書テンプレートへの自動流し込み

Phase 2（3ヶ月後）:
  ④ 自社実績DBによる単価候補表示
  ⑤ 歩掛の自動適用（標準歩掛表のDB化）
  ⑥ 補正係数の自動提案

Phase 3（6ヶ月後）:
  ⑦ 図面OCRからの数量推定
  ⑧ N社データによる市場単価推定
  ⑨ 落札率予測
```

## 7. 学習ロードマップ（1-2週間）

### Day 1-3: 業界の全体像を掴む

```
□ このドキュメントのSection 1-2を通読（2時間）
□ YouTubeで「建設業 積算 入門」を検索して3-5本視聴（2時間）
□ 「よくわかる建設業の積算」等の入門書を1冊購入して通読（3時間）
□ 国交省の設計労務単価表をダウンロードして眺める（30分）
□ 積算ソフトATLUSのデモ動画を見る（1時間）
```

### Day 4-7: 業務フローの詳細理解

```
□ このドキュメントのSection 3を精読（3時間）
□ 「土木工事標準積算基準書」の目次と主要項目を確認（2時間）
□ 実際の数量計算書（公開されている入札結果から入手）を読む（2時間）
□ 実際の内訳書（同上）を読む（2時間）
□ 建設物価or積算資料のサンプルを確認（1時間）
□ Section 4の例外パターンを読む（2時間）
□ Section 5の用語を暗記（隙間時間に繰り返し）
```

### Day 8-10: 実地体験

```
□ パイロット企業の積算担当者に「横で1日見せてください」と依頼
□ 実際の積算作業を横で見ながらメモ
  - どの書類をどの順番で見ているか
  - どこで判断に迷っているか
  - どのツール/ソフトを使っているか
  - 1案件にどれくらい時間がかかっているか
□ 終了後に「このステップはAIで代替できそうか」を整理
```

### Day 11-14: AI設計に落とし込む

```
□ 積算フローの各ステップを「入力→処理→出力」で再定義
□ 各ステップのAI化可否を判断（Section 6を参考に）
□ Level 1プロトタイプの機能要件を書く
□ パイロット企業に「こういうツールを作ります」とデモ資料を見せる
□ フィードバックを反映して要件を修正
```

```

4. 数量計算書のExcelを見て、工種・種別・細別・数量・単位を特定できるか
```

## 付録: 積算AI Level 1 の入出力仕様（設計用メモ）

```
■ 入力:
  - 数量計算書（Excel or PDF）
  - 設計書PDF
  - 工事種別（土木/建築）
  - 地域（都道府県）
  - 年度

■ 処理:
  1. Document AI / LLM でドキュメントを構造化
  2. 工種/種別/細別/規格/数量/単位を抽出
  3. 設計労務単価DB、市場単価DBから単価候補を表示
  4. ユーザーが単価を確認・修正
  5. 諸経費を自動計算
  6. 内訳書を生成

■ 出力:
  - 数量一覧（構造化JSON/Excel）
  - 単価候補付き内訳書（Excel/PDF）
  - 諸経費計算書

■ 精度目標:
  - 数量抽出: 70-80%（数量計算書Excelの場合は90%+）
  - 単価推定: 60-70%（設計労務単価は95%+）
  - 人間の確認: 全項目必須（Level 1では）
```
