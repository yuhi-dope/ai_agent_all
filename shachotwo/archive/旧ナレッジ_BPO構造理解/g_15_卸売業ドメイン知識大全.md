# 卸売業ドメイン知識大全

> 中小卸売業（従業員10-300名）の業務・文化・課題を網羅的に理解するためのドキュメント。
> 製品紹介ではなく、業界そのものの深い理解を目的とする。
> **対象読者**: 卸売業に提案・支援を行う全ての人。営業・PM・エンジニア・経営者。
> **関連**: `e_15a_卸売業_全貌.md`（シャチョツーの卸売BPO設計に特化）

## 1. 卸売業とは何か

### 1.1 業種分類

卸売業は日本標準産業分類（JSIC）の「I 卸売業、小売業」に属し、中分類で6つに分かれる。ただし実態はもっと細かく、同じ「食品卸」でも酒類専門と冷凍食品専門では全く別の商売。

#### 食品卸（約5.5万社）

卸売業の最大ボリュームゾーン。市場規模約48兆円。

| 区分 | 概要 | 得意先 | 特殊要件 |
|---|---|---|---|
| 総合食品卸 | 常温+冷蔵+冷凍。数千SKU | スーパー、飲食店、給食 | 3温度帯管理、賞味期限、HACCP |
| 酒類卸 | 日本酒、ビール、ワイン、焼酎 | 居酒屋、料飲店、小売酒屋 | 酒類販売業免許、蔵元との関係 |
| 菓子卸 | 菓子、スナック、珍味 | コンビニ、スーパー、駄菓子屋 | 季節商品の在庫リスク |
| 水産卸 | 鮮魚、冷凍水産 | 寿司屋、鮮魚店、スーパー | 鮮度管理が最重要。朝3時のセリ |
| 青果卸 | 野菜、果物 | スーパー、八百屋、飲食店 | 市場経由 or 産直。天候で価格変動 |
| 業務用食品卸 | 飲食店向け業務用サイズ | 飲食チェーン、ホテル、病院 | 大容量パック、レシピ提案力 |

食品卸はFAX依存率が最も高い（80%超）。理由は得意先が個人飲食店で、FAX以外の発注手段を持っていないから。オーナーの平均年齢は55-60歳。「FAXを辞めてWeb発注にしてください」は通用しない。

#### 建材・住宅設備卸（約3.5万社）

市場規模約25兆円。建設業と密接。

| 区分 | 概要 | 得意先 | 特殊要件 |
|---|---|---|---|
| 木材卸 | 構造材、内装材、合板 | 工務店、建設会社 | 産地（国産/輸入）、等級管理 |
| 建材卸 | サッシ、屋根材、外壁材 | 工務店、リフォーム会社 | メーカー多数（LIXIL/YKK/ニチハ） |
| 住設卸 | キッチン、バス、トイレ | 工務店、設備屋 | 品番が膨大。色・サイズの組合せ |
| 管材卸 | 配管、継手、バルブ | 設備屋、管工事会社 | 規格（JIS/ANSI）、材質（鉄/銅/SUS/塩ビ） |
| 電材卸 | 電線、配線器具、照明 | 電気工事会社 | パナソニック/東芝/日東工業等 |

建材卸の特徴は「現場直送」。倉庫から建設現場にトラックで直接納品する。現場には搬入時間の指定があり（「朝8時にクレーン待ち」等）、配送の調整が複雑。品番数が膨大（5万点以上）で、品番の表記揺れが日常的に発生する。

#### 機械・金属卸（約3.0万社）

市場規模約40兆円。いわゆる「技術商社」「機械商社」。

```
特徴:
  ・単なる「モノを流す」のではなく、技術提案が付加価値
  ・「この用途ならこのメーカーのこの型番が最適」と提案できる営業力
  ・ミスミ、MonotaRO、Amazonに置き換えられない「相談機能」
  ・見積対応が重要な業務（仕入先への照会→マージン上乗せ→得意先に提出）
  ・リードタイムが長い商品が多い（特注品: 2-6ヶ月）
  ・粗利率15-25%。卸の中では利幅が大きい
```

#### 繊維・衣料卸（約2.0万社）

市場規模約8兆円。ファストファッションとECに押されて縮小傾向。

```
特徴:
  ・シーズン商品（春夏/秋冬）。売れ残り=即赤字
  ・アパレルメーカーから小売への中間流通
  ・返品率が高い（シーズン落ちで20-40%が返品）
  ・粗利率20-35%だが、返品・値引きを考慮すると実質10-15%
  ・展示会商談で半年先の発注を取る。先読みが命
```

#### 医薬品卸（約1.0万社）

市場規模約10兆円。寡占化が進行。

```
4大卸:
  メディパルホールディングス（アルフレッサ）
  スズケン
  東邦薬品
  バイタルKSK
  → この4社で市場の80%を占める

中小の医薬品卸は少数。規制が厳しく参入障壁が高い。
  ・薬機法に基づく医薬品卸売販売業の許可
  ・管理薬剤師の設置義務
  ・品質管理基準（GDP: Good Distribution Practice）
  ・温度管理の記録義務（2-8℃保管品あり）
  ・トレーサビリティ（ロット管理）
  ・粗利率5-8%。最も薄利な卸売業
```

#### 化学品卸（約1.5万社）

市場規模約15兆円。法規制対応が重い。

```
特徴:
  ・危険物取扱いが多い（消防法、毒劇法、化管法）
  ・SDS（安全データシート）の管理・提供義務
  ・保管場所の制約（危険物倉庫、温度管理）
  ・得意先の業種が多岐にわたる（製造業、食品、農業、建設）
  ・少量多品種の小分け（ドラム缶→一斗缶→小瓶）が付加価値
```

### 1.2 卸売業の存在意義（なぜメーカー直販にならないのか）

「中抜き」が叫ばれて20年以上。それでも卸が存在する理由。

```
■ 品揃え機能（ワンストップ）
  飲食店が毎日使う食材: 調味料、酒類、冷凍食品、乾物、缶詰…
  → メーカーは自社商品しか出せない
  → 卸は100社以上のメーカーの商品を1回の注文で届ける
  → 個人飲食店が50社のメーカーと個別取引するのは現実的に不可能

■ 小口対応
  メーカーの最小出荷単位: ケース（24本入り等）
  飲食店の注文: 「バラで3本だけ」
  → 卸がケースを開けてバラ出荷する。メーカーはやらない

■ 与信・代金回収
  零細飲食店が味噌メーカーと直接取引 → メーカーは与信判断できない
  卸が間に入ることで、メーカーは卸に1本で請求。回収リスクは卸が負う
  → 卸の「金融機能」。貸倒れリスクを引き受けている

■ 配送頻度
  飲食店: 毎日配送が必要（生鮮品）
  メーカー: 週1回のルート配送が精一杯
  → 卸の倉庫に在庫して、毎日配送する「即納力」が価値

■ 情報提供
  新商品情報、市場動向、棚割提案、販促企画
  → 営業マンが足で稼ぐ情報が、小売・飲食店にとっての価値
  → ここがAIに置き換わると卸の付加価値が問われる
```

### 1.3 中小卸売業の財務構造

#### 売上規模

```
従業員数と売上の目安:
  10名:  年商3-10億円
  30名:  年商10-30億円
  50名:  年商20-50億円
  100名: 年商50-150億円
  → 製造業と比べて1人あたり売上が大きい（モノを作らず流すだけなので）
```

#### 利益構造

```
典型的なP/L構成（食品卸、年商20億円の場合）:

  売上高:           2,000,000千円（100%）
  ──────────────────────────────
  売上原価:
    仕入原価:       1,760,000千円（88%）★売上のほとんどが仕入
  ──────────────────────────────
  粗利益:             240,000千円（12%）
  ──────────────────────────────
  販管費:
    人件費:           120,000千円（ 6%）
    物流費（配送）:     50,000千円（ 2.5%）
    倉庫費:            20,000千円（ 1%）
    販促費:            10,000千円（ 0.5%）
    その他:            30,000千円（ 1.5%）
  ──────────────────────────────
  販管費計:           230,000千円（11.5%）
  ──────────────────────────────
  営業利益:            10,000千円（ 0.5%）
```

#### 業種別の粗利率の違い

| 業種 | 粗利率 | 営業利益率 | 特徴 |
|---|---|---|---|
| 食品卸 | 8-12% | 0.5-2% | 薄利多売。配送コストが利益を食う |
| 建材卸 | 10-15% | 1-3% | 品番が膨大。在庫リスクあり |
| 機械・金属卸 | 15-25% | 3-5% | 技術提案の付加価値。利幅が取れる |
| 繊維・衣料卸 | 20-35% | 1-3% | 表面上は高粗利だが返品で目減り |
| 医薬品卸 | 5-8% | 0.3-1% | 最薄利。規模の経済が必須 |
| 化学品卸 | 10-20% | 2-4% | 専門性が参入障壁 |

#### なぜ利益率が低いのか

- **仕入原価が高い**: 売上の80-90%が仕入。粗利は残りカスから全経費を捻出する
- **配送コストの重さ**: 自社配送の場合、ドライバー+車両+燃料で粗利の20-30%が消える
- **メーカーの値上げ転嫁が遅い**: メーカーが値上げしても、得意先への転嫁に3-6ヶ月かかる。その間は逆ざやに近い状態
- **得意先の価格交渉**: 「もう1%安くしてくれないと、他の卸に切り替える」→ 1%粗利が減ると利益が半分消える
- **貸倒れリスク**: 年商20億の卸で年1回500万円の焦げ付きがあると、営業利益の50%が吹き飛ぶ

### 1.4 組織構造

```
■ 従業員50名の食品卸（典型例）:

  社長（1名）
  ├── 大口得意先への営業（チェーン店本部商談は社長が出る）
  ├── メーカーとの仕入条件交渉
  └── 与信判断の最終権限（「あの店、もう掛売りは止めよう」は社長判断）

  営業部（15名）
  ├── エリア担当制。1人で得意先30-80軒
  ├── 毎朝ルート配送+営業活動（配送しながら御用聞き）
  ├── 得意先の「いつもの」を全部知っている
  ├── 新商品の提案、棚割提案、販促企画
  └── 平均年齢45歳。ベテラン営業が得意先との関係を握っている

  受注部（5名）★会社のボトルネック
  ├── FAX注文の読取→販売管理システムへの入力
  ├── 電話注文の受付（「いつもの頼む」→ 何を何個か聞き出す）
  ├── 在庫確認+欠品時の仕入先への緊急発注
  ├── パートタイマー3名+正社員2名
  └── ここが止まると出荷が止まる。全社が止まる

  物流部（10名）
  ├── ピッキング（注文に合わせて商品を集める）
  ├── 検品（商品名・数量・賞味期限の確認）
  ├── 出荷（ルート別に仕分け→トラックに積込み）
  ├── 配送（自社配送 or 運送会社委託）
  └── 早朝4-5時出社。飲食店への朝納品は6-8時

  管理部（5名）
  ├── 経理: 請求書発行、入金確認、消込、仕入先への支払い
  ├── 経理: 仕入先からの請求書と自社の発注・納品データの照合
  ├── 在庫管理: 棚卸し、入出庫管理、在庫データの補正
  ├── 総務: 給与計算、社会保険、労務管理
  └── ここも1人=1部署の世界

  商品部（2名）
  ├── 商品マスタの管理（新商品登録、廃番処理、価格更新）
  ├── メーカーとの商談（仕入条件、販促企画）
  └── 商品マスタのメンテナンスが追いつかない（月数十件の変更）
```

### 1.5 卸売業の文化

#### FAXは「文化」である

FAXが残る理由は技術的な問題ではなく、文化的な問題。

```
なぜFAXが残るのか:
  1. 得意先が変わらない
     個人飲食店のオーナーは60代。「FAXで注文する」が30年来の習慣
     「アプリで注文して」→ 「スマホ持ってないよ」「老眼で見えない」
     → 得意先にFAXを辞めさせることは不可能

  2. FAXの方が「間違いない」という信念
     手書きで商品名と数量を書く → 確実
     Webフォームでプルダウン選択 → 「間違えて選んでしまう」恐怖
     → 高齢の得意先にとってFAXの方が安心

  3. 「いつもの」が電話で済む
     電話1本「いつもの頼むわ」→ 30秒で発注完了
     Webで10品目を1つずつ選択 → 5分かかる
     → 得意先にとってはFAXや電話が「最速のUI」

  4. 卸の営業が御用聞きで回る
     営業マンが毎日得意先を訪問 → その場で注文を聞く
     → 営業マンがメモして会社に戻ってFAXまたは入力
     → 得意先はシステムを使う必要がない

結論: FAXを廃止するのではなく、FAXのまま受け取ってAIが処理する
      → これがシャチョツーのキラーメッセージ
```

#### 「口座」の重さ

卸売業で「口座を持つ」とは、取引先として登録されていること。銀行口座ではない。

```
口座開設のプロセス:
  1. 取引申込書の提出（会社情報、決算書、代表者の経歴）
  2. 信用調査（帝国データバンク or 東京商工リサーチで評点確認）
  3. 与信限度額の設定（月間取引額の1.5-2倍が目安）
  4. 取引条件の決定（掛率、支払いサイト、配送条件）
  5. 営業担当の割当て

口座を閉じる = 取引停止 = 「あの会社とは縁を切る」
  → 30年取引があっても、貸倒れが発生すると即口座閉鎖
  → 逆に、一度口座を開くとなかなか閉じない（情で続く）
```

#### 「建値」と「仕切値」

卸売業の商談で毎日使われる用語。

```
建値（たてね）:
  メーカーが設定する「この価格で売ってほしい」という希望小売価格
  → 実際にこの価格で売れることは少ない

仕切値（しきりね）:
  メーカーが卸に対して設定する卸売価格。仕入原価
  → 建値の50-70%が一般的

卸の利益 = 得意先への販売価格 - 仕切値 - 配送コスト等

例:
  建値（希望小売）: ¥1,000
  仕切値（卸への卸売）: ¥550（55掛）
  卸の販売価格（A社向け）: ¥700（70掛）
  粗利: ¥150（粗利率21.4%）
  配送コスト: ¥30
  実質利益: ¥120（実質利益率17.1%）
```

#### 義理人情の取引

```
・「うちは30年この卸から仕入れてる」→ 少し高くても切り替えない
・営業マンが冠婚葬祭に出る。得意先の子供の入学を知っている
・年末年始の挨拶、お中元、お歳暮は欠かさない
・「あの営業マンが辞めたから、卸も変える」→ 人についてくる取引
・新規開拓は「紹介」が最強。飛び込みでは口座が開かない
```

## 2. 受発注の実務フロー

### 2.1 1日の流れ（食品卸の場合）

```
■ AM 4:00-5:00: 出荷準備
  前日夜～早朝に届いたFAX注文を回収
  昨日の受注入力分のピッキングリスト出力
  物流チームが出荷準備開始（冷凍→冷蔵→常温の順）

■ AM 5:00-6:00: 出荷・配送開始
  ルート別にトラックへ積込み
  配送ドライバー出発（飲食店は朝6-8時着が多い）

■ AM 7:00-8:00: FAX注文の入力開始
  今朝のFAXを1枚ずつ読取・入力
  電話注文の受付開始（「いつもの」が飛んでくる）

  FAX注文の処理フロー:
    1. FAX受信（複合機のトレイに紙が溜まる）
    2. 得意先の特定（FAXの送信元番号 or 店名を確認）
    3. 商品の特定（★最難関。略称・手書き・独自呼称の解読）
    4. 数量・単位の確認（ケース？バラ？本？kg？）
    5. 納品日の確認（記載なし→翌日と推定。「急ぎ」→当日？）
    6. 販売管理システムに入力（商品コード+数量+得意先+納品日）
    7. 在庫確認（在庫あり→引当/在庫なし→仕入先に発注 or 欠品連絡）
    8. 次のFAXへ

  1枚あたり: 10-15分。30枚で5-7時間。パート3名体制

■ AM 10:00-12:00: 仕入先への発注
  在庫が切れそうな商品の仕入発注
  メーカーへの定期発注（週2回のルート発注が多い）
  緊急発注（当日欠品しそうな商品）

■ PM 1:00-3:00: 午後便の出荷
  午前中に入力した注文のピッキング→出荷
  午後便は当日受注・当日出荷のケースが多い

■ PM 3:00-5:00: 翌日分の準備
  翌日配送分の確定→ピッキングリスト出力
  在庫の確認（明日の出荷分が足りるか）
  欠品アラート→得意先への連絡

■ PM 5:00以降: 事務処理
  入力ミスの修正
  仕入先からの出荷通知の確認
  翌日のFAXが届き始める（閉店後に翌日分を送る得意先）
```

### 2.2 FAX注文の実態

FAXの「読み」が受注業務の最大の難所。

```
■ 典型的なFAX注文書（居酒屋からの手書き）:

  ┌─────────────────────────────────┐
  │ ○○居酒屋 注文書           │
  │ 3/29 納品分                │
  │                             │
  │ いつものビール  2ケ         │
  │ サバ水         3つ          │
  │ ポン酢         5           │
  │ 焼肉タレ甘口   2           │
  │ からあげ粉      1           │
  │ あと日本酒いつもの          │
  │                             │
  │ よろしくお願いします        │
  └─────────────────────────────────┘

  受注担当が解読:
  「いつものビール 2ケ」
    → ○○居酒屋のいつものビール = アサヒスーパードライ350ml 24本入り
    → 「2ケ」= 2ケース
    → 商品コード: ASH-SD350-24、数量: 2ケース

  「サバ水 3つ」
    → サバの水煮缶。この居酒屋がいつも頼むのはマルハニチロの190g缶
    → 「3つ」= 3ケース？3缶？ → この店は「つ」=ケース
    → 商品コード: MN-SW190-24、数量: 3ケース

  「ポン酢 5」
    → ミツカン味ぽん？ヤマサ昆布ぽん酢？エバラ？
    → この店の過去注文を確認 → ミツカン味ぽん1Lが定番
    → 「5」= 5本
    → 商品コード: MT-APN1L、数量: 5本

  「あと日本酒いつもの」
    → この店のいつもの日本酒 = 八海山清酒 1.8L
    → 数量の記載なし → 前回と同じ3本か？ → 電話確認

このように、1枚のFAXに5-10分かけて「翻訳」する作業が
月500-2,000回発生する。
```

### 2.3 「いつもの」注文パターン

```
中小卸の得意先の注文パターン:
  80%: リピート注文（前回と同じ or ほぼ同じ）
  15%: 定番+追加（いつもの+季節商品 or 新商品）
  5%:  完全新規（新メニュー導入等）

「いつもの」の実態:
  ・個人飲食店: 週2-3回、ほぼ同じ商品を同じ数量
  ・スーパー: 週1回の定番発注+売れ行きに応じた追加
  ・給食: 献立に合わせて週単位で変動するが、月単位では安定

「いつもの」がAIにとって最大のチャンス:
  ・得意先×曜日×商品の組み合わせを学習
  ・「火曜日の○○居酒屋はビール2ケース+焼酎1本」とパターン化
  ・FAXに「いつもの」と書いてあるだけで、自動的に前回と同じ内容を入力
  ・異常値検知: 「いつもの倍の数量」→ 確認フラグを立てる
```

## 3. 掛率の仕組み

### 3.1 掛率の基本

掛率（かけりつ）は卸売業の利益構造の根幹。全ての価格交渉はこの「掛率」で行われる。

```
掛率 = 販売価格 ÷ 定価（メーカー希望小売価格） × 100

例: 定価¥1,000の商品を¥700で販売 → 70掛（ななじゅうがけ）

業界の言い方:
  「A社には70で出してる」= A社への掛率は70%
  「この商品は55で仕入れてる」= 仕入原価は定価の55%
  「65掛でいけますか」= 定価の65%で売ってくれませんか
```

### 3.2 得意先ランク別の掛率体系

```
ランクの決め方:
  S: 年間取引額 5,000万円以上。大手チェーン、大口法人
  A: 年間取引額 1,000-5,000万円
  B: 年間取引額 300-1,000万円
  C: 年間取引額 300万円未満。個人飲食店の大半

掛率テーブルの例（食品卸の場合）:

  カテゴリ       仕入掛率   S掛率   A掛率   B掛率   C掛率
  ─────────────────────────────────────────────────────
  調味料         55%       65%    68%    72%    78%
  酒類           60%       70%    73%    76%    80%
  冷凍食品       50%       62%    65%    70%    75%
  缶詰・乾物     52%       64%    67%    71%    76%
  業務用食材     55%       66%    69%    73%    78%

  → S掛率65%で仕入掛率55%なら、粗利は10pt（=定価の10%分の利益）
  → C掛率78%で仕入掛率55%なら、粗利は23pt
  → 大口ほど薄利、小口ほど厚利。だが大口を失うと売上激減
```

### 3.3 掛率の複雑さ（地獄の条件分岐）

```
基本掛率（得意先ランクで決定）
  ↓
+ カテゴリ別掛率（商品カテゴリで異なる場合あり）
  ↓
+ 数量割引（10ケース以上→-2%、100ケース以上→-5%）
  ↓
+ 特売掛率（メーカー販促期間中の特別価格。期間限定）
  ↓
+ 新商品導入掛率（初回のみ特別価格で出す）
  ↓
+ 期間限定プロモーション（「今月だけ全品2%OFF」等）
  ↓
+ 年間リベート（年間購入額に応じた事後値引き）

→ 1つの商品の「正しい売価」を出すのに、5-6個の条件を確認する必要がある
→ ベテラン営業は頭の中で計算できる。新人は間違える
→ 間違えると赤字販売（仕入原価を下回る価格で出してしまう）
→ 得意先に「前回と値段が違う」と指摘される → 信用を失う
```

### 3.4 掛率管理の実態

```
管理方法:
  ・Excelの掛率マスタ → 更新が追いつかない。バージョン管理なし
  ・販売管理システムの価格マスタ → 設定が複雑で正しく登録されていない
  ・営業の頭の中 → 「A社はこの商品だけ特別に67掛で出している」
  ・紙の掛率表 → 3年前に作ったきりで更新されていない

問題:
  ・営業が独自に掛率を決めてしまう（「特別に安くしておくよ」）
  ・赤字販売に気づかない（仕入原価の変更が反映されていない）
  ・同じ得意先で営業担当が変わると掛率がわからなくなる
  ・メーカーの仕入原価改定（年1-2回）への追随が遅れる
```

## 4. リベートの種類と計算

### 4.1 リベートとは

メーカーから卸への「事後的な値引き」。年間の取引実績に応じて後から支払われる。卸の利益の重要な構成要素。

```
■ メーカー→卸のリベート（仕入リベート）

  数量リベート:
    年間仕入額が○○万円以上 → 仕入額の1-3%を返金
    例: 年間仕入5,000万円 × 2% = 100万円

  達成リベート:
    メーカーが設定した目標額を達成 → ボーナス的に返金
    例: 年間目標3,000万円達成 → ¥50万円の達成金

  新商品導入リベート:
    新商品を得意先に導入した実績に対して支払われる
    例: 新商品○○を20店舗に導入 → 1店舗あたり¥5,000

  販促リベート:
    メーカーの販促企画に協力した対価
    例: チラシ掲載、試食販売の実施 → 実費+マージン

■ 卸→得意先のリベート（売上リベート）

  年間達成リベート:
    得意先の年間購入額に応じて後から値引き
    例: 年間購入3,000万円超 → 購入額の0.5%を返金
    → ¥15万円が翌年の3-4月に支払われる

  早期支払リベート:
    支払いサイトを短縮した場合のインセンティブ
    例: 通常60日サイト → 30日で支払い → 0.3%値引き
```

### 4.2 リベート管理の実態

```
問題:
  ・リベートの計算が年に1回。年間実績を集計して計算する作業が膨大
  ・メーカーごとにリベートの条件が違う。100社のメーカー × 異なる条件 = 地獄
  ・「あと100万円仕入れれば次のリベートランクに上がる」という判断ができていない
  ・得意先へのリベート支払いも同様に複雑
  ・リベートの未収金が決算時に問題になる（計上タイミングの会計処理）
```

## 5. 食品卸の特殊性

### 5.1 賞味期限管理（1/3ルール）

日本独自の商慣習。食品ロスの大きな原因でもある。

```
1/3ルール（食品業界の暗黙のルール、法律ではない）:
  製造日から賞味期限までの期間を3等分し:
    最初の1/3以内: メーカー→卸への納品期限
    次の1/3以内:    卸→小売への販売期限
    残り1/3:        小売での販売可能期間

例: 賞味期限6ヶ月の商品（製造日1/1、賞味期限6/30）
  ・メーカー→卸: 2/28までに納品しないとNG
  ・卸→小売: 4/30までに納品しないとNG
  ・小売: 6/30まで販売可能

卸にとっての影響:
  ・仕入れた商品を長期間倉庫に置けない
  ・販売期限を過ぎた在庫は返品 or 廃棄 → 直接損失
  ・先入先出（FIFO）の徹底が必須
  ・「日付逆転」（古い在庫より新しい在庫を先に出す）は厳禁
  ・賞味期限の短い商品（乳製品: 2週間、豆腐: 5日等）は毎日の管理が命
```

### 5.2 温度帯管理

```
3温度帯の管理:
  常温（ドライ）:      15-25℃。調味料、缶詰、乾物、菓子
  冷蔵（チルド）:      0-10℃。乳製品、生鮮、惣菜、練り物
  冷凍（フローズン）:  -18℃以下。冷凍食品、アイスクリーム

倉庫の温度帯:
  ・3温度帯対応の倉庫を持つ卸は中堅以上
  ・中小卸は常温倉庫のみ。冷蔵・冷凍は外部委託 or 冷蔵庫のみ
  ・冷凍倉庫のコスト: 冷蔵の2-3倍（電気代が支配的）

配送の温度帯:
  ・同じ得意先への納品でも温度帯ごとに別配送になることがある
  ・常温便、チルド便、フローズン便の3便で同じ店に届けるのは非効率だが、
    混載すると温度管理が崩れる
  ・2温度帯対応車両（常温+冷蔵の仕切り車）で効率化する卸が増加

HACCP対応（2021年6月～義務化）:
  ・倉庫の温度記録の保管義務
  ・配送車両の温度記録
  ・衛生管理計画の策定・実施
  ・中小卸の対応状況: 形式的に書類を作成しているが、
    温度記録が手書き or 記録していないケースが多い
```

### 5.3 アレルゲン管理

```
食品表示法で表示義務のある特定原材料8品目:
  えび、かに、くるみ、小麦、そば、卵、乳、落花生

表示推奨21品目:
  アーモンド、あわび、いか、いくら、オレンジ、カシューナッツ、
  キウイフルーツ、牛肉、ごま、さけ、さば、大豆、鶏肉、バナナ、
  豚肉、まつたけ、もも、やまいも、りんご、ゼラチン、マカダミアナッツ

卸にとっての影響:
  ・得意先（飲食店、給食）からのアレルゲン照会に対応する義務
  ・商品マスタにアレルゲン情報を登録・管理
  ・メーカーのアレルゲン情報変更への追随
  ・「この商品に○○は入っていますか」→ 即答できないとクレーム
```

## 7. 与信管理の実務

### 7.1 与信管理とは

「掛売り」（後払い）が卸売業の基本取引形態。代金回収のリスクを卸が負う。

```
掛売りの流れ:
  月初-月末: 商品を納品（納品書を渡す）
  月末: 締め日。1ヶ月分の納品をまとめて請求書を発行
  翌月末 or 翌々月末: 入金（支払いサイト30日 or 60日）

→ 商品を渡してから代金をもらうまで30-90日のタイムラグ
→ その間に得意先が倒産したら、全額が焦げ付く（貸倒れ）
```

### 7.2 与信限度額の設定

```
与信限度額の決め方:
  ① 信用調査（帝国データバンク、東京商工リサーチ）
     評点50点以上: 比較的安全
     評点40-50:   要注意
     評点40未満:  現金取引 or 取引見送り

  ② 月間取引額の1.5-2倍を目安に設定
     月間取引額100万円 → 与信限度150-200万円

  ③ 自社の体力との比較
     「この取引先が飛んだら、うちも危ない」金額は設定しない
     → 卸の月間利益が500万円なら、1社への与信限度は500万円以下が理想

  ④ 定期的な見直し（年1回以上）
     決算書の入手、支払い実績の確認、風評情報のチェック
```

### 7.3 危険サインの見分け方

```
得意先の危険サイン（ベテランが見ているポイント）:

  支払い遅延:
    ・初めての遅延: 「忘れてました」で済む
    ・2回目: 偶然ではない。資金繰りが苦しい可能性
    ・3回目以上: 赤信号。与信限度の引き下げ or 現金取引への切替

  注文パターンの変化:
    ・急に注文量が減る → 売上が落ちている
    ・急に注文量が増える → 他の卸から切られて集中している可能性
    ・いつもと違う商品を大量に注文 → 転売?

  その他:
    ・電話に出なくなる
    ・営業担当への態度が変わる
    ・従業員が辞めたという情報
    ・近隣で競合店がオープン
    ・オーナーが体調を崩した

  対応の段階:
    ① 社内アラート共有（営業+経理）
    ② 訪問して直接確認
    ③ 支払いサイトの短縮（60日→30日、30日→現金）
    ④ 与信限度の引き下げ
    ⑤ 取引停止（口座閉鎖）
```

### 7.4 貸倒れの実態

```
中小卸の貸倒れ統計:
  ・年1回以上の貸倒れ経験: 全体の3-5%
  ・平均損失額: 500-3,000万円
  ・食品卸は飲食店の廃業率が高い（年間8%前後）ため、貸倒れリスクが特に高い

貸倒れの影響:
  ・粗利率10%の卸で500万円の貸倒れ → 5,000万円分の売上利益が消える
  ・資金繰りの悪化（入るはずのお金が入らない）
  ・精神的ダメージ（「あの社長を信用していたのに」）
```

## 8. 在庫管理の課題

### 8.1 在庫の基本指標

| 指標 | 計算方法 | 食品卸の目安 | 建材卸の目安 |
|---|---|---|---|
| 在庫回転率 | 年間売上原価 ÷ 平均在庫 | 12-20回/年 | 4-8回/年 |
| 在庫日数 | 365 ÷ 在庫回転率 | 18-30日 | 45-90日 |
| 欠品率 | 欠品件数 ÷ 受注件数 | 1-3% | 2-5% |
| 死蔵在庫率 | 6ヶ月以上動かない在庫 ÷ 総在庫 | 3-8% | 5-15% |

### 8.2 ABC分析の実務

```
A品（売上の上位20%の商品 → 全売上の80%）:
  ・欠品は絶対NG。安全在庫を厚く持つ
  ・発注頻度: 週2-3回
  ・在庫管理: 毎日チェック

B品（売上の中位30%の商品 → 全売上の15%）:
  ・適正在庫を維持。多少の欠品は許容
  ・発注頻度: 週1回
  ・在庫管理: 週次チェック

C品（売上の下位50%の商品 → 全売上の5%）:
  ・在庫は最小限。受注発注（取り寄せ）も検討
  ・発注頻度: 月1-2回
  ・在庫管理: 月次チェック
  ・死蔵在庫候補。定期的に見直して廃番検討

問題:
  ・ABC分析をやっている中小卸は少ない
  ・「昔から置いている」だけで、売れていない商品が倉庫の場所を占有
  ・「なくなったら困る」という恐怖で在庫を抱えすぎる
  ・逆に、売れ筋商品の在庫切れが起きて機会損失
```

### 8.3 死蔵在庫

```
死蔵在庫の発生原因:
  ・得意先の閉店（在庫を引き取ってもらえない）
  ・メーカーの新商品切替（旧品が残る）
  ・需要予測の外れ（多すぎた発注）
  ・季節商品の売れ残り
  ・返品在庫（賞味期限が近い、パッケージ汚損）

処理方法:
  ・値下げ販売（掛率を下げて特売）
  ・従業員販売（社員に安く売る）
  ・フードバンクへの寄付（食品の場合。税制上のメリットあり）
  ・廃棄（最終手段。廃棄費用がかかる。食品は産廃扱い）

中小卸の死蔵在庫:
  年商20億の食品卸で、平均在庫1.5億円のうち5-8%（750-1,200万円）が死蔵
  → これが倉庫スペースを圧迫し、新商品の在庫場所がなくなる
```

## 9. IT導入の現状

### 9.1 主なシステムとツール

| ツール | 概要 | 普及率（中小） | 価格帯 |
|---|---|---|---|
| **販売管理システム** | 受注→売上→請求→入金を管理 | 80-90% | 月額1-10万円。初期50-500万円 |
| **在庫管理** | 入出庫、在庫数、ロケーション管理 | 50-60% | 販売管理に含む or 月額1-5万円 |
| **EDI（電子データ交換）** | 取引先とのデータ連携 | 大手: 80%、中小: 10-20% | 月額5-30万円 |
| **流通BMS** | 日本の標準EDI規格 | 大手: 50%、中小: 5-10% | 月額5-20万円 |
| **WMS（倉庫管理）** | ピッキング、棚管理、バーコード管理 | 10-20% | 月額5-30万円 |
| **BtoB EC** | 得意先がWeb上で発注 | 5-10% | 月額3-30万円 |
| **FAX-OCR** | FAXを画像認識で電子化 | 3-5% | 月額3-10万円 |
| **BI（経営分析）** | 売上・粗利・在庫の分析 | 5-10% | 月額1-10万円 |

**販売管理システムの実態**:

```
主要なパッケージ:
  ・弥生販売（弥生）: 年商3億以下の小規模向け。安い
  ・PCA商魂/商管: 中小向け。食品卸に強い
  ・奉行シリーズ（OBC）: 中堅向け。カスタマイズ性あり
  ・SMILE V（OSK）: 業種特化のカスタマイズ対応
  ・楽商（ジャスト）: 食品卸に特化
  ・大塚商会系: 中堅向け。SIerのサポート込み
  ・SAP Business One: 大手向け。高額だが多機能

問題:
  ・導入から10年以上経過。サポート切れ or バージョンアップ未対応
  ・「入れたけどカスタマイズしすぎて、誰もメンテできない」
  ・マスタ管理が崩壊（登録ミス、重複登録、廃番放置）
  ・APIが公開されていない古いシステムが多い → 外部連携が困難
```

### 9.2 EDIの実態

```
EDI（Electronic Data Interchange）= 電子データ交換
  受発注データを、人手を介さずにシステム間で自動連携する仕組み

日本の卸売業のEDI:
  ① 流通BMS（Business Message Standards）
     2007年策定の日本標準EDI。XML or CSV
     主にスーパー←→食品卸の受発注で使われている
     大手スーパー（イオン、セブン、ライフ等）は流通BMS対応を要求
     中小卸: 「BMS対応しないと取引切られる」が、導入コストが高い

  ② Web-EDI
     WebブラウザでEDIデータを送受信。流通BMSより簡易
     中小の取引先に対応しやすい

  ③ 従来型EDI（JCA手順/全銀手順）
     電話回線ベースの旧式。2024年のISDN終了で使えなくなる問題
     → IP-VPN or インターネット手順への移行が必要だが、対応が遅れている

EDI普及率:
  大手卸（売上500億以上）: 80%以上がEDI対応
  中堅卸（売上50-500億）: 30-50%
  中小卸（売上50億未満）: 10-20%
  → 中小卸の得意先の大半はFAX or 電話。EDIは大手得意先との取引のみ
```

### 9.3 なぜIT導入が進まないのか

```
理由1: 得意先が変わらない
  「うちの得意先は個人飲食店100軒。FAXを辞めてくれるはずがない」
  → 受発注のデジタル化は、自社だけでは完結しない。得意先の協力が必要
  → 得意先にWeb発注を強制 → 「面倒だから他の卸に変える」と言われるリスク

理由2: 販売管理システムとの連携
  → 古い販売管理システムにAPI連携機能がない
  → 「新しいシステムに入れ替える」→ 数百万円の投資+データ移行の手間
  → 移行期間中（3-6ヶ月）の業務混乱を恐れる

理由3: 商品マスタの整備コスト
  → 10,000 SKUのマスタを正しく整備するのに数ヶ月かかる
  → 整備してもすぐに古くなる（月50件の変更）
  → 「いつまで経ってもマスタが完璧にならない」→ 導入を先送り

理由4: 費用対効果の見えにくさ
  → 受注入力を自動化しても「パートさんの仕事がなくなるだけ」に見える
  → 「その分パートさんを減らすのか？」→ 減らしたくない。長年の付き合いだし
  → 省力化よりも「正確性の向上」「欠品の防止」で価値を訴求すべき

理由5: 経営者の年齢
  → 中小卸の経営者の平均年齢は60歳前後
  → 「あと5年で引退する。今さらシステムを変えるのは…」
  → 後継者がいれば後継者主導で導入が進むことがある
```

## 10. 卸売業の用語集

営業・提案の場面で「この人はわかっている」と思ってもらうための必須用語。

| 用語 | 読み | 意味 |
|---|---|---|
| 掛率 | かけりつ | 定価に対する販売価格の比率。「70掛」=定価の70% |
| 建値 | たてね | メーカーの希望小売価格 |
| 仕切値 | しきりね | メーカーが卸に出す卸売価格。仕入原価 |
| リベート | リベート | 取引実績に応じた事後的な値引き（キャッシュバック） |
| 口座 | こうざ | 取引先として登録されていること。銀行口座ではない |
| 掛売り | かけうり | 後払いの取引。納品後に月末締め翌月払い等 |
| 消込 | けしこみ | 入金と請求の突合処理。どの請求に対する入金かを特定 |
| サイト | サイト | 支払いまでの期間。「30日サイト」=締日から30日後に支払い |
| 手形 | てがた | 一定期日に支払いを約束する有価証券。サイト60-120日 |
| 先日付 | さきひづけ | 将来の日付を指定した取引。「来月10日納品で」 |
| バラ | バラ | ケース単位ではなく、1個単位での出荷 |
| 入数 | いりすう | 1ケースに入っている商品の数。「入数24」=24個入り |
| SKU | エスケーユー | Stock Keeping Unit。在庫管理の最小単位 |
| 棚割 | たなわり | 小売店の棚に何をどう並べるかの計画。卸の提案力の見せ所 |
| 返品 | へんぴん | 得意先から商品が戻ってくること。食品は賞味期限切れが主因 |
| 欠品 | けっぴん | 注文に対して在庫がない状態。飲食店には致命的 |
| 先入先出 | さきいれさきだし | FIFO。古い在庫から先に出す。食品卸の鉄則 |
| 日付逆転 | ひづけぎゃくてん | 古い賞味期限の商品より新しい賞味期限の商品を先に出してしまうこと。厳禁 |
| ピッキング | ピッキング | 倉庫で注文に合わせて商品を集める作業 |
| 検品 | けんぴん | 商品の数量・品質・賞味期限を確認する作業 |
| ロット | ロット | 製造単位。同じ条件で作られた製品のグループ |
| 特売 | とくばい | 期間限定の値引き販売。メーカーの販促と連動 |
| 押し込み | おしこみ | メーカーが卸に対して、目標達成のために大量購入を迫ること |
| 棚落ち | たなおち | 小売店の棚から商品が撤去されること。卸にとっては売上減 |
| 配荷率 | はいかりつ | 取扱店舗数 ÷ 全店舗数。メーカーが重視する指標 |
| 流通BMS | りゅうつうビーエムエス | 日本の標準EDI規格。Business Message Standards |
| JAN | ジャン | Japanese Article Number。バーコードの規格。13桁 |

## 12. 卸売業の経営者が使う指標

| 指標 | 計算方法 | 目安 |
|---|---|---|
| 粗利率 | 粗利 ÷ 売上 | 食品卸8-12%、建材卸10-15% |
| 営業1人あたり売上 | 売上 ÷ 営業人数 | 食品卸1-3億/人、建材卸2-5億/人 |
| 在庫回転率 | 売上原価 ÷ 平均在庫 | 食品卸12-20回、建材卸4-8回 |
| 売掛金回転日数 | 売掛金 ÷ 売上 × 365 | 30-60日。長いほどキャッシュフローが苦しい |
| 買掛金回転日数 | 買掛金 ÷ 仕入 × 365 | 30-45日。売掛金回転日数より短いと資金が先に出る |
| 欠品率 | 欠品件数 ÷ 受注件数 | 2%以下が目標 |
| 誤出荷率 | 誤出荷件数 ÷ 出荷件数 | 0.1%以下が目標 |
| 受注処理単価 | 受注処理コスト ÷ 受注件数 | ¥500-1,500/件。ここを下げるのがBPOの価値 |
| 得意先あたり粗利 | 粗利 ÷ 得意先数 | 赤字得意先の特定に使う |
| 死蔵在庫率 | 6ヶ月以上不動在庫 ÷ 総在庫 | 5%以下が目標 |
