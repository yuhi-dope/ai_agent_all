## Step 0: 基盤構築 + Day 1 Value（2-3週間）

**目的**: 全Stepの土台 + 初日から価値を提供できる3機能。ここが腐ると全部やり直しになるので最初にしっかり作る。

### やること

```
■ プロジェクト初期化
  - [ ] 既存コード整理・新規プロジェクト構成
  - [ ] FastAPI サーバー + Supabase Auth + JWT認証基盤
  - [ ] DB スキーマ作成（companies, users, knowledge_items, genome_templates）
  - [ ] RLS（Row Level Security）全テーブル適用
  - [ ] LLM抽象化レイヤー（model_tier → モデル自動選択）
  - [ ] LangGraph 基盤セットアップ（エージェント状態管理）
  - [ ] フロントエンド基盤（Next.js + Tailwind + shadcn/ui）
  - [ ] CI/CD パイプライン（lint + type check + test + security scan）
  - [ ] ステージング環境（Cloud Run）

■ チャネル基盤（チャネルファースト・アーキテクチャ）
  - [ ] LINE Bot 受信基盤（Messaging API Webhook）
  - [ ] メール受信基盤（IMAP ポーリング / Webhook）
  - [ ] ファイルアップロード基盤（Web UI + API）
  - [ ] チャネル統一メッセージ形式の定義（全チャネル → 共通フォーマット変換）
  ※ SaaS API連携はオプション。中小企業の実態はLINE/メール/Excel/紙/FAX

■ 統合ファイル処理エンジン（基本版）
  - [ ] Excel 構造推論エンジン（ヘッダー検出・シート間関係・数式解析）
  - [ ] PDF → テキスト抽出（Google Document AI）
  - [ ] Word/PowerPoint → テキスト抽出
  - [ ] CSV/TSV パーサー
  - [ ] 画像 → OCR（Google Document AI 日本語手書き対応）
  - [ ] ファイル種別自動判定 + ルーティング

■ Day 1 Value（ナレッジゼロでも初日から使える3機能）
  - [ ] ファイルチェック機能（Excel/PDF等をアップロード → 整合性チェック・異常検知）
  - [ ] 議事録自動作成（音声入力 → Whisper書き起こし → 構造化議事録）
  - [ ] 文書下書き機能（用途指定 → LLMが下書き生成）
  ※ これらはナレッジ蓄積ゼロの初日から即価値を提供する

■ セキュリティ基盤（全プラン共通・最初から入れる）
  - [ ] TLS 1.3 必須
  - [ ] RLS全テーブル適用（company_idベースのテナント分離）
  - [ ] 監査ログ基盤（audit_logs テーブル）
  - [ ] 基本的なRBAC（CEO / Employee の2ロールから開始）

■ 課金基盤（従量課金対応）
  - [ ] 利用量カウンター（ファイル処理件数・Q&A回数の記録）
  - [ ] 月次利用量集計API
  - [ ] Stripe連携基盤（¥30K基本料 + 従量課金メーター）
```

### 完了基準
- `docker compose up` で開発環境が立ち上がる
- ユーザー登録 → ログイン → テナント分離が動作する
- CI/CDでlint・test・security scanが回る
- LINE Bot / メール / ファイルアップロードで入力を受け付けられる
- Excel/PDF/画像をアップロードしてチェック結果が返る（Day 1 Value）
- 音声から議事録が生成される（Day 1 Value）
- 利用量がカウント・集計される

### 自動化レベル対応
> Step 0 では **Level 0（リマインダー・通知のみ）** の基盤を構築。
> 全企業はLevel 0から開始し、信頼度が上がるにつれてLevel 1以降へ進む。

### なぜ先にやるか
> 認証・RLS・LLM抽象化は後付けが極めて困難。LangGraphも同様。
> ここをスキップすると Step 1 以降で全部書き直しになる。
> チャネル基盤を最初に入れるのは、中小企業の現実的な入力手段（LINE/メール/Excel）を
> SaaS APIより優先するため。Day 1 Valueを先に作ることで、パイロット初日から「使える」状態にする。

## Step 1: MVP（6-7週間）

**目的**: パイロット3社に「これ使いたい」と思ってもらえる最小限のプロダクト。

**ビジネスフェーズ**: Phase A — パイロット3社 × ¥30,000/月（基本料）+ 従量課金

### MVPの定義: 「社長が話すだけで会社のナレッジが貯まり、社員が質問できる」

```
MVP = チャネル入力 + ファイル処理 + 構造化 + Q&A + テンプレート + 能動提案
    = 社長にとって「これは便利」と実感できる最小セット
    = Day 1 Valueで即価値 → ナレッジ蓄積で価値増大
```

### やること

```
■ ナレッジ取り込み（チャネルファースト）
  - [ ] LINE Bot → テキスト/画像/音声受信 → LLM構造化 → 保存パイプライン
  - [ ] メール転送 → 自動取り込み → 構造化
  - [ ] 対話型引き出し（LLMが追加質問を生成して深掘り — LINE/Web両対応）
  - [ ] 音声入力 → Whisper書き起こし → 構造化（Step 0の議事録機能を拡張）
  - [ ] FAX受信（eFAX連携 → PDF → OCR → 構造化）

■ 統合ファイル処理エンジン（高度版）
  - [ ] Excel理解エンジン拡張（テンプレート学習・異常検知・マクロ対応）
  - [ ] 音声ファイル処理（MP3/WAV/M4A → Whisper → 構造化）
  - [ ] 動画ファイル処理（音声トラック抽出 → Whisper → 構造化）
  - [ ] CADファイル基本対応（メタデータ抽出・サムネイル生成）
  - [ ] ファイル処理従量カウント連携（¥50/件）

■ Q&A エンジン
  - [ ] ベクトル検索（pgvector + Voyage AI Embedding）
  - [ ] LLM応答生成 + 引用（根拠ナレッジ）表示
  - [ ] セマンティックキャッシュ（類似質問の高速応答）
  - [ ] プロンプトキャッシュ適用（入力コスト-90%）
  - [ ] LINE Bot経由のQ&A応答
  - [ ] Q&A従量カウント連携（¥30/回）

■ 3つのAIペルソナ（マルチユーザー基盤）
  - [ ] 参謀ペルソナ（経営層向け — 戦略的助言・リスク分析・意思決定支援）
  - [ ] アシスタントペルソナ（管理職向け — 業務管理・進捗確認・レポート作成）
  - [ ] ヘルパーペルソナ（一般社員向け — Q&A・手順案内・書類作成支援）
  - [ ] 階層×部門のマトリクスアクセス制御
  - [ ] ペルソナ別プロンプト・応答トーンの最適化
  - [ ] 追加ユーザー課金連携（マネージャー¥5,000/月、一般¥2,000/月）

■ Company Genome（業界テンプレート）
  - [ ] テンプレート基盤（業種選択 → 80%完成状態でスタート）
  - [ ] 建設業テンプレート（標準部署・フロー・ルール・遺伝子）
  - [ ] 製造業テンプレート（受注生産/見込生産フロー・品質管理・BOM）
  - [ ] 歯科テンプレート基本版（一般歯科/予防歯科/自費診療の基本フロー）
  - [ ] 建設業固有遺伝子（積算・施工管理・協力会社・現場・入札）
  - [ ] 製造業固有遺伝子（生産管理・品質・在庫・BOM・設備保全）
  - [ ] 歯科固有遺伝子基本版（歯科診療・レセプト・リコール・ユニット管理）

■ 可視化 + デジタルツイン
  - [ ] 業務フロー抽出 + Mermaid図自動生成
  - [ ] 意思決定ルール抽出 + ツリー表示
  - [ ] ナレッジ充足度マップ（ヒートマップ）
  - [ ] 会社モデル（9次元）の構築
  - [ ] ギャップ検出（部署×カテゴリの充足度）

■ 能動提案（Level 0: リマインダー・通知）
  - [ ] リスク早期検出エンジン
  - [ ] 改善提案エンジン
  - [ ] What-if シミュレーション
  - [ ] 能動提案の通知・レビューUI
  - [ ] LINE Bot / メールへの提案通知配信
  - [ ] 建設業固有パターン（工程遅延・原価超過・協力会社逼迫）
  - [ ] 製造業固有パターン（在庫切れ・設備故障予兆・品質不良傾向）
  - [ ] バッチAPI適用（提案生成を非同期化）

■ フロントエンド
  - [ ] ナレッジ入力画面（テキスト + 音声 + ファイル + LINE連携状態表示）
  - [ ] Q&A 画面（チャット形式 + 引用表示）
  - [ ] ダッシュボード（充足度マップ + フロー図 + 提案一覧）
  - [ ] 利用量・課金ダッシュボード（従量課金の透明性）
  - [ ] PWA設定（Service Worker + オフライン対応）
```

### 完了基準

```
□ LINE Bot / メール / ファイルアップロードでナレッジが取り込める
□ Excel/PDF/Word/画像/音声ファイルが自動処理される
□ 建設・製造・歯科テンプレートで80%完成状態からスタートできる
□ テキスト/音声/ドキュメントからナレッジが構造化される
□ Q&A回答に引用（根拠ナレッジ）が表示される（LINE Bot経由含む）
□ 3つのAIペルソナが階層・部門に応じて動作する
□ 業務フローがMermaid図で自動可視化される
□ AIが能動的にリスクと改善を提案する（Level 0: 通知のみ）
□ What-if分析でパラメータ変更の影響をシミュレーションできる
□ 従量課金が正確にカウント・請求される
```

### KPI

| 指標 | 目標 |
|------|------|
| ナレッジ入力セッション数/月 | 20+ |
| ナレッジ充足率 | 30%+ |
| Q&A応答精度 | 70%+ |
| 能動提案の採用率 | 30%+ |
| 社長の「便利」実感 | 3社中3社 |
| Day 1 Value利用率（初日） | 3社中3社 |

### MVP提供の運用

```
期間:     0-3ヶ月（Phase A）
対象:     建設1社 + 製造1社 + 歯科1社
単価:     ¥30,000/月（基本料）+ 従量課金（初期は少額）
入力:     LINE Bot / メール転送 / ファイルアップロード（SaaS APIは不要）
フォロー:  週1回の対面フォロー
ゴール:    成功事例 + 推薦コメント
撤退基準:  3社中2社以上が「価値を感じない」→ ピボット検討
```

### なぜこの範囲がMVPか
> Q&Aだけだと「ChatGPTでよくない？」と言われる。
> テンプレート（80%完成スタート）+ 能動提案（AIが先に気づく）があって初めて
> 「これは自社専用のAI参謀だ」と感じてもらえる。
> チャネルファースト（LINE Bot/メール）により、SaaSを使っていない中小企業でも
> 普段使いのツールからそのまま入力できる。Day 1 Valueで初日から価値を体感。
> BPO（自動実行）は価値の証明後でよい。まず「知の整理と活用」で刺す。
