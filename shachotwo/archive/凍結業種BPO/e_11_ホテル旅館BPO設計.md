## 1. レベニューマネジメントAI（★実装対象）

### 問題の本質

```
- RevPAR（Revenue Per Available Room）= 稼働率 × ADR（平均客室単価）を最大化するのがホテル経営の本質
- 大手チェーンは専用システム（IDeaS、Duetto等）を導入済み
- 中小ホテル・旅館は支配人の勘で料金設定。以下が起きる：
  ・繁忙期に安すぎる料金→機会損失
  ・閑散期に高すぎる料金→空室
  ・競合の料金変更に気づかない
  ・OTAごとの在庫配分が非最適
```

### 支払いロジック

```
■ 50室のホテル、ADR ¥10,000、稼働率70%:
  月間売上 = 50室 × ¥10,000 × 70% × 30日 = ¥1,050万

■ ADR 5%改善（¥10,500）: 月+¥52.5万 = 年+¥630万
■ 稼働率5%改善（75%）: 月+¥75万 = 年+¥900万
■ 両方改善: 年+¥1,530万
■ → BPO月10万円（年120万円）で ROI 5〜12倍
```

### 不動産管理との構造比較

| 項目 | 不動産（家賃管理） | ホテル（レベニュー管理） |
|---|---|---|
| 収益の単位 | 月額家賃（固定） | 1泊料金（変動） |
| 稼働率管理 | 空室率の最小化 | 稼働率×単価の最大化 |
| チャネル管理 | SUUMO/HOME'S | じゃらん/楽天/Booking.com |
| 価格戦略 | 相場ベース（年1回見直し） | 日次で変動（需給連動） |
| 共通パターン | 「空室=損失。チャネル管理+価格最適化」 |

### 課題テーブル

| 課題 | 影響額（年間） |
|---|---|
| 料金設定が勘と経験 | ADR最適化で年数百万円の機会損失 |
| OTAの在庫配分が非最適 | 手数料率の高いOTAに偏る |
| 競合の料金変更に気づかない | 価格競争力の喪失 |
| 繁忙期の料金設定が遅い | 売り切り→追加値上げの機会逸失 |
| 閑散期の対策不足 | 空室の放置 |

## 2〜7. 拡張ロードマップ

- **② OTA管理**: じゃらん/楽天/Booking.com/Expediaの在庫・料金同期。EC「商品登録」横展開
- **③ 口コミ管理**: 5サイト×月50件の返信。多言語対応。歯科「集患マーケ」横展開
- **④ シフト・労務**: フロント・調理・清掃の複合シフト。介護「シフト管理」横展開
- **⑤ 食材管理**: 宿泊人数連動の発注。飲食業「仕入・発注」横展開
- **⑥ 設備管理**: 客室設備の予防保全。不動産「修繕・設備管理」横展開
- **⑦ インバウンド対応**: 多言語FAQ・チャットボット。LLMで生成
