## 1. 調剤報酬請求AI（★実装対象）

### 1.1 問題の本質

```
調剤報酬請求の現状:
  ・処方箋受付→調剤→服薬指導→薬歴記載→会計→請求
  ・調剤報酬の加算体系が複雑（服薬管理指導料、かかりつけ薬剤師指導料等）
  ・後発医薬品調剤体系加算（後発率80%以上等）の管理
  ・返戻率2-4%。主な原因は保険情報の不一致、算定要件不備
  ・2年ごとの報酬改定で算定ルールが変わる
```

### 1.2 支払いロジック

```
■ 直接効果①：加算の取り漏れ防止
  1処方箋あたりの平均点数: 約900点
  月間処方箋枚数1,500枚の薬局:
  ・服薬管理指導料の算定漏れ: 月5件 × 59点 × 10円 = 月2,950円
  ・地域支援体制加算の取得: 月1,500枚 × 39点 × 10円 = 月58.5万円（取れるなら巨大）
  ・後発医薬品調剤体系加算: 体制整備で月数万円増

■ 直接効果②：返戻率の改善
  月間請求額400万円 × 返戻率1%改善 = 月4万円 = 年48万円

■ 直接効果③：事務員の請求業務時間削減
  月20-30時間 → 50%削減 = 月1.5-2.3万円

■ 合計効果：年80-200万円
  → BPO月5万円（年60万円）で ROI 1.3〜3.3倍
```

### 1.3 歯科レセプトとの差分（ゲノム差分）

| 項目 | 歯科 | 調剤薬局 |
|---|---|---|
| 請求先 | 社保・国保 | 社保・国保（同一） |
| 法定期限 | 毎月10日 | 毎月10日（同一） |
| 主な審査 | 病名×処置の整合性 | 処方内容×調剤内容の整合性 |
| 加算体系 | か強診加算等 | 服薬管理指導料、かかりつけ薬剤師等 |
| 固有の複雑さ | 部位×病名 | 薬剤の相互作用チェック、後発品変更可否 |
| 共通エンジン | ルールベースチェック→AI予測→レポート生成（同一） |

### 1.4 主な算定ポイント（ゲノムに詰める内容）

**調剤基本料**
- 調剤基本料1〜3（処方箋集中率、グループ薬局の受付回数で区分）
- 地域支援体制加算1〜4（要件: 夜間対応、在宅実績、かかりつけ実績等）

**主な加算**
- 服薬管理指導料（対面/オンライン）
- かかりつけ薬剤師指導料・包括管理料
- 服用薬剤調整支援料（ポリファーマシー対策）
- 在宅患者訪問薬剤管理指導料
- 後発医薬品調剤体系加算（後発率80%以上等）

**返戻の主な原因**
1. 保険情報の不一致（被保険者番号、公費負担者番号）
2. 処方箋の記載不備（用法、日数）
3. 算定要件不備（指導内容の記録不足）
4. 薬剤の適応外使用

### 1.5 課題テーブル

| 課題 | 具体的な問題 | 影響額（年間） | 発生頻度 |
|---|---|---|---|
| 加算の取り漏れ | 服薬管理指導料等を算定し忘れ | 年数万〜数十万円 | 毎月 |
| 地域支援体制加算の要件管理 | 実績要件のカウントが手作業 | 加算取得/喪失で年数十万円 | 常時 |
| 返戻対応の事務負荷 | 月20件の返戻×再請求30分 = 月10時間 | 人件費15万円 | 毎月 |
| 報酬改定への対応 | 2年ごとの改定で算定ルール変更 | 誤請求リスク | 2年ごと |
| 後発品比率の管理 | 80%を下回ると加算が取れない | 年数十万円の差 | 常時 |

## 2〜6. 拡張ロードマップ

> 以下は#1モジュール（調剤報酬請求AI）のPMF確認後に実装。

### ② 在庫管理・医薬品（Tier1 / ¥2-5万）
- 期限切れ廃棄=直接損失。デッドストック月数万〜数十万
- 発注点管理+需要予測
- 製造業の「在庫最適化」パターンの横展開

### ③ 薬歴作成支援（Tier1 / ¥2-3万）
- 薬剤師が閉店後に薬歴を書く。1件5-10分×100件/日
- LLMで薬歴ドラフト生成
- 士業の「書類作成支援」パターンの横展開

### ④ 地域支援体制加算管理（Tier2 / ¥1-2万）
- 実績要件の自動カウント+充足チェック
- 歯科の「施設基準管理（か強診）」パターンの横展開

### ⑤ シフト管理（Tier2 / ¥1-2万）
- 管理薬剤師の常駐義務
- 歯科の「シフト管理」パターンの横展開

### ⑥ 届出・許認可（Tier3 / ¥1万）
- 薬局開設許可更新、保険薬局指定更新
- 歯科の「届出・許認可管理」パターンの横展開
