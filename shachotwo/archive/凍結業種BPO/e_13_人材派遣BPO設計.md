## 1. 契約管理・抵触日管理AI（★実装対象）

### 問題の本質

```
- 労働者派遣法: 同一の派遣先×同一の組織単位で3年が上限（個人単位）
- 事業所単位の抵触日: 派遣先事業所で最初に派遣を受け入れた日から3年
- 抵触日の管理をExcelでやっている→漏れが発生
- 抵触日超過=違法派遣→行政処分（業務停止命令）+みなし雇用
- 派遣契約書・個別契約書・通知書の作成が手作業で膨大
```

### 支払いロジック

```
■ 抵触日超過1件の損害: 行政処分+違約金+信用失墜 = 数百万〜数千万円
■ 契約書作成工数: 月30-50時間（スタッフ100名規模） × 時給1,500円 = 月4.5-7.5万円
■ → BPO月7万円で抵触日超過リスク回避（ROI = 実質∞）+ 契約書作成50%削減
```

### 士業「期限管理」との構造比較

| 項目 | 士業（期限管理） | 人材派遣（抵触日管理） |
|---|---|---|
| 管理対象 | 申告期限、控訴期限 | 抵触日、契約期間 |
| 違反の結果 | 懲戒+賠償 | 行政処分+みなし雇用 |
| 管理単位 | 顧問先×期限 | 派遣先×スタッフ×抵触日 |
| 共通パターン | 期限DB管理→アラート→書類自動生成 |

### 課題テーブル

| 課題 | 影響額（年間） |
|---|---|
| 抵触日の管理漏れ | 行政処分リスク（事業停止） |
| 契約書の作成が手作業 | 人件費54-90万円 |
| 同一労働同一賃金の比較表 | 作成工数+法令違反リスク |
| 事業報告書（毎年6/30） | 作成に丸2-3日 |

## 2〜7. 拡張ロードマップ

- **② マッチング**: スキル×資格×勤務地×時給の最適マッチング。AI推薦
- **③ 勤怠・給与**: 派遣先ごとの勤務体系。交通費・深夜・残業。歯科「経理・給与」横展開
- **④ 請求書生成**: 派遣先×スタッフ×月の請求書。建設業「出来高・請求」横展開
- **⑤ 法令対応**: 事業報告書。同一労働同一賃金。士業「法改正対応」横展開
- **⑥ スタッフフォロー**: 定期面談・満足度調査。離職予兆検知
- **⑦ 教育管理**: 年8h以上のキャリアアップ教育義務。士業「研修管理」横展開
