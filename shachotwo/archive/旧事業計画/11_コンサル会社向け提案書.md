## 2. 御社が今、直面している3つの課題

### 課題①: コンサルタントの「頭の中」が資産にならない

```
現状:
  ・プロジェクト知見がコンサルタント個人のPC・メモに散在
  ・「前に似た案件やったよね」→ 検索できない
  ・提案書を毎回ゼロから作成（過去の知見が再利用できない）
  ・ベテランに聞かないと分からないことが多い

結果:
  ・ベテラン退職 = 知見ごと消失（年間退職率15-25%）
  ・ジュニアの立ち上がりに6ヶ月〜1年
  ・同じ失敗を繰り返す（学習が組織に蓄積しない）
```

### 課題②: コンサルタントの稼働率に天井がある

```
現状:
  ・コンサルタント1人 = 同時に3〜5社が限界
  ・定型的な分析・調査・資料作成に時間を取られる
  ・クライアントからの質問対応で1日が終わる

結果:
  ・売上 = コンサルタント数 × 担当社数 × 単価
  ・担当社数に天井がある限り、売上は人数に比例 → スケールしない
  ・採用→教育→戦力化のサイクルが遅い
```

### 課題③: コンサル終了後、クライアントが元に戻る

```
現状:
  ・3-6ヶ月のプロジェクトでノウハウを移転
  ・でもコンサル終了後、半年で効果が薄れる
  ・「知識移転」と言いつつ、報告書を渡して終わり
  ・継続契約の提案理由が「定例ミーティング」しかない

結果:
  ・クライアント満足度は高いが、LTVが伸びない
  ・「いい仕事だったけど、もう大丈夫です」と言われる
  ・リカーリング収入がない → 毎期ゼロから営業
```

## 3. シャチョツーで何が変わるか

### 3.1 御社のコンサルタントがパワーアップする（社内利用）

```
Before:
  コンサルタント1人 → 同時3-5社 → 売上 ¥150-250万/月/人

After:
  コンサルタント1人 + シャチョツー → 同時8-12社 → 売上 ¥400-600万/月/人

シャチョツーがやること:
  ・過去プロジェクトの知見を即座に検索・引用
    → 「前にやった製造業の原価改善、どうやったっけ？」→ 3秒で回答
  ・提案書のドラフトを自動生成
    → 過去の成功提案書 + クライアント固有情報 → たたき台が30分で完成
  ・クライアントの定型質問にAIが回答
    → 「前回のミーティングで決まったことは？」→ AIが引用付きで回答
  ・業界ベンチマークをリアルタイム提供
    → 「この会社の原価率は同業他社と比べてどうか？」→ 即座に比較
```

### 3.2 ジュニアが即戦力になる（育成加速）

```
Before:
  新人コンサルタント → ベテランに付いて6ヶ月OJT → ようやく独り立ち
  → 年間採用5人のうち、半年以内に2人離職（業界平均）

After:
  新人 + シャチョツー → 2ヶ月で独り立ち
  → AIに聞きながらクライアント対応ができる
  → ベテランの時間を新人教育に取られない

  新人が聞けること:
  「この業界の標準的な利益率は？」→ AIが業界データで回答
  「前任者はこのクライアントにどう対応してた？」→ 過去の対応履歴から回答
  「この提案で抜けてる観点は？」→ AIが過去の類似提案と比較してチェック
```

### 3.3 コンサル終了後もクライアントにリカーリング収入（OEM提供）

```

  クライアントにとっての価値:
  「コンサルが終わっても、コンサルの知見がAIに残っている」
  「新しい社員が入っても、AIに聞けばコンサルが教えてくれた基準がわかる」
  → 解約率が極めて低い（ナレッジが蓄積するほど手放せない）

  御社にとっての価値:
```

## 7. セキュリティ — クライアントのデータを守る仕組み

```

  ✅ RLSポリシーの自動監査・穴検出
  ✅ 異常アクセスパターンのリアルタイム検知
  ✅ APIキー・パスワードの自動ローテーション
  ✅ 依存パッケージの脆弱性自動チェック
  ✅ 週次セキュリティレポート自動生成

  第三者認証（取得予定）:
  ・ISMS（ISO 27001）: 準備中
```

## 8. なぜ自社開発ではなくシャチョツーか

```
「AIツールなら自社で作れるのでは？」という疑問にお答えします。

■ 作れるもの（≒ コモディティ）
  ✅ LLMを使ったQ&Aチャット
  ✅ ドキュメントのRAG検索
  ✅ 簡易的な業務自動化

■ 作れないもの（≒ シャチョツーの本当の価値）
  ❌ 500社分の業種別暗黙知データベース（遺伝子プール）
     → N社のパターンから学んだ精度は、自社30社分では出せない
  ❌ 50業種対応のテンプレート基盤
     → 各業種の法令・業界標準・専門用語を網羅
  ❌ 業界ベンチマーク（「同業他社比で原価率+3%」の分析）
     → 統計的に意味のあるベンチマークには数百社のデータが必要
  ❌ 20以上のSaaSコネクタ（kintone, freee, Slack, ANDPAD...）
     → 1コネクタ=2-4週間の開発。20コネクタ=1年分の工数
  ❌ AI駆動セキュリティ（RLS自動監査・異常検知・脆弱性自動修正）
     → セキュリティ専門知識 + LLM統合 + 運用ノウハウが必要
  ❌ 24/365のSaaS運用・保守・セキュリティアップデート
     → コンサルが本業の会社がここにリソースを割くのは非効率

■ 開発 vs 導入のコスト比較
  ┌──────────────────┬──────────────┬──────────────┐
  │                  │ 自社開発      │ シャチョツー   │
  ├──────────────────┼──────────────┼──────────────┤
  │ 初期費用         │ ¥5,000万〜    │ ¥0           │
  │ 開発期間         │ 12-18ヶ月     │ 即日利用開始   │
  │ 月額運用コスト    │ ¥200万〜      │ ¥3万+従量    │
  │ エンジニア採用    │ 2-3名必要     │ 不要          │
  │ セキュリティ保守  │ 自社責任      │ 当社が負担    │
  │ 機能アップデート  │ 自社で開発    │ 自動反映      │
  │ 業種テンプレート  │ ゼロから構築   │ 50業種対応    │
  │ ベンチマーク精度  │ 自社データのみ │ 500社+のデータ│
```

## 10. よくある質問

### Q: うちのコンサルタントが使いこなせるか？

```
・UIは「社長に聞く感覚」で質問するだけ。ITスキル不要
・導入初期にオンボーディング支援（対面 or オンライン）を提供
・使うほど回答精度が上がるので、成功体験が早い
・まず1チームで試して、効果を確認してから全社展開を推奨
```

### Q: クライアントのデータが混ざらないか？

```
・DB層で完全テナント分離（RLS: 行レベルセキュリティ）
・A社のコンサルデータがB社に漏れることは構造的に不可能
・アプリにバグがあってもDB層でブロックされる
・AI駆動でRLSポリシーの穴を常時監査
```

### Q: AIの回答が間違っていたら？

```
・AIの回答には必ず「引用元」が表示される（根拠が確認できる）
・回答が間違っていた場合、フィードバックで精度が向上
・重要な判断はAIの回答を参考にしつつ、コンサルタントが最終判断
・シャチョツーは「参謀」であり「意思決定者」ではない
```

### Q: 競合のコンサル会社も使えるのか？

```
・データは完全にテナント分離。他社のデータは一切見えない
・御社がOEMで提供する場合、御社ブランドで提供可能
・業界ベンチマークは匿名化データのみ使用（個社特定不可能）
・先行導入することで、競合より早くリカーリング収入を確保できる
```

### Q: シャチョツーが事業撤退したらどうなる？

```
・データはJSON/CSVで一括エクスポート可能（ベンダーロックインなし）
・解約後30日間のエクスポート猶予 → 削除証明書発行
・SLA（サービス品質保証）を契約に明記
・サイバー保険加入済み
```

## [参考] Salesforceモデルとの構造比較

```
■ Salesforce × Accenture の成功モデル

  Step 1: Salesforce を「CRMツール」としてコンサル会社に提供
  Step 2: コンサル会社が「Salesforce導入支援」を新サービス化
  Step 3: クライアントの業務がSalesforce前提で設計される
  Step 4: コンサル会社がSalesforceなしでは提案できなくなる
  → Salesforceの年商 ¥5兆円の40%以上がパートナー経由

■ Salesforceのマージン構造（実際の数値）
  OEMパートナー: Salesforceが25%、パートナーが75%
  ISVパートナー: Salesforceが15%、パートナーが85%
  年商$20M超:    Salesforceが15%、パートナーが85%（OEM）
  → パートナーに圧倒的に有利な条件 → だからパートナーが本気で売る
  → 結果: 世界最大のSaaSエコシステム（$5兆+）を構築

■ シャチョツー × 御社 で同じ構造を作る

  マージン構造: Salesforce準拠（御社75% / 当社25%）
  ボリューム割引: 売上増加に応じて御社80%→85%に拡大

  Step 1: シャチョツーを「コンサル強化ツール」として御社に提供
  Step 2: 御社が「ナレッジDXパッケージ」を新サービス化（OEM）
  Step 3: クライアントのナレッジがシャチョツーに蓄積される
  Step 4: 御社のコンサル提案に「シャチョツー」が標準組み込み
  → 御社の売上UP + 高マージンリカーリング収入 + クライアント満足度UP

■ なぜ当社は25%で成立するのか
  当社が75%をパートナーに渡しても成立する理由:
  ① 変動コスト（LLM+インフラ）が売価の2%以下 → 25%でも十分な粗利
  ② パートナー経由の導入社数増 → 学習フライホイールが加速
     → テンプレ精度・Q&A精度・ベンチマーク精度が全モジュールで向上
     → 直販の成約率・単価が上がる
  ③ データネットワーク効果 → 競合が追いつけない壁が厚くなる
  → 25%は「データ獲得への投資」。本当の利益は学習効果で回収

■ 先行者優位
  この構造は「最初にやった会社」が圧倒的に有利。
  500社分のナレッジが蓄積されると、後発は精度で追いつけない。
  御社が今始めれば、競合コンサル会社より先にこの優位を獲得できます。
```
