## 0. フリーミアムの目的を間違えない

```
■ フリーミアムの目的は「無料で使わせること」ではない

  目的は3つ:
  ① 導入の意思決定コストをゼロにする（試す前の離脱を消す）
  ② バイラルループを起動する（使うこと自体が宣伝になる）
  ③ パートナーに温まったリードを供給する（CACゼロの見込み客）

  つまり: 無料ユーザーは「コスト」ではなく「マーケティング装置」

■ 無料ユーザー1人あたりの価値

  A. 有料転換の期待値:
     転換率10% × LTV ¥1,200,000（ARPU ¥100K × 12ヶ月） = ¥120,000/人

  B. データ貢献の価値:
     匿名化データが業界テンプレートの精度を向上
     → 全ユーザーの体験品質が上がる → 転換率がさらに上がる

  C. バイラル価値:
     1無料ユーザーが平均0.3人を追加招待（k-factor）
     → 招待された人の10%が有料転換 → 追加 ¥36,000/人

  → 無料ユーザー1人の期待値: 約¥156,000
  → 無料ユーザー1人の運用コスト: 約¥200-500/月（後述）
  → 6ヶ月で回収可能。投資として極めて優秀。
```

```
■ なぜ2段階（Free → ¥30K）ではなく3段階か

  問題: ¥0 → ¥30,000 の心理的ジャンプが大きすぎる

  Starter ¥9,800の役割:
  ・「月1万円なら試してみよう」← 部長どころか課長でも即決
  ・IT導入補助金適用後: 実質¥2,450-4,900/月 ← 缶コーヒー数本/日
  ・「無料では物足りないけど、¥30Kはまだ早い」層を確実に捕捉
  ・1人で使い倒す社長がここに滞留 → 他部署展開でGrowthに上がる

  参考: HubSpotの成功パターン
  ・Free CRM → Starter $20/月 → Professional $800/月 → Enterprise
  ・Starter→Professional の転換率はFree→Professionalの3倍

■ 重要: Starter は「パートナー不要で自走できる」設計
  → パートナーのCSキャパに依存しない成長レイヤー
  → パートナーはGrowth以上の大型案件に集中できる
```

## 3. Free プランの設計思想（最重要セクション）

### 3.1 「Aha Moment」の特定と無料での到達保証

```
■ シャチョツーの Aha Moment とは何か

  ユーザーが「これは手放せない」と感じる瞬間。
  Slackでは「2,000メッセージ送信」、Dropboxでは「1ファイル保存」。

  シャチョツーの Aha Moment:
  ━━━━━━━━━━━━━━━━━━━━━━
  「自分の会社のことを質問したら、的確な答えが返ってきた」
  ━━━━━━━━━━━━━━━━━━━━━━

  具体例:
  ・建設業の社長が「杭工事の坪単価は？」と聞いたら、
    業界テンプレート + 自社のアップロード済み見積書から即答
  ・製造業の社長が「SUS316Lの切削条件は？」と聞いたら、
    テンプレートの標準条件 + 自社の過去実績を組み合わせて回答
  ・歯科の院長が「矯正の説明で患者に刺さる言い方は？」と聞いたら、
    テンプレートのカウンセリングパターンから複数案を提示

■ なぜ Free で Aha Moment に到達できるか

  秘密兵器: 業界テンプレート（Genome）

  テンプレートがない場合:
    → 空のナレッジベース → Q&Aしても「情報がありません」→ 離脱

  テンプレートがある場合（シャチョツーの設計）:
    → 登録した瞬間に業界知識80%がプリロード
    → 初回Q&Aからそこそこ的確な回答が返る
    → 自社データを足すほど精度が上がる → 「もっと入れたい」→ ハマる

  Free プラン（50件ナレッジ + テンプレート）でも:
    テンプレート由来の知識でQ&Aの70%に回答可能
    + 自社データ50件で残り30%のうち半分をカバー
    → 全体で約85%のQ&Aに何らかの回答が返る
    → Aha Moment の到達率: 推定 80%以上

■ Aha Moment 到達までの設計フロー

  Step 1: LINE で友達追加 or メールで登録（30秒）
  Step 2: 業種・規模を選択（1分）→ テンプレート自動適用
  Step 3: 「何か質問してみてください」のプロンプト
  Step 4: 質問する → テンプレートベースで回答が返る ← Aha Moment
  Step 5: 「もっと正確にするには、御社の資料をアップロード」
  Step 6: ファイル1つアップロード → 回答精度が上がる ← 2nd Aha Moment

  → 登録から Aha Moment まで 5分以内
```

### 3.2 Free の制限設計：「制限を感じるタイミング」のコントロール

```
■ 制限設計の原則

  ❌ ダメな制限: 初日に制限に当たる
     → 価値を感じる前にストレスを感じる → 離脱

  ❌ ダメな制限: 永遠に制限に当たらない
     → 無料で十分 → 有料に上がる理由がない

  ✅ 良い制限: 価値を実感した直後（2-4週間後）に制限に当たる
     → 「もっと使いたいのに」→ 自然にアップグレード

■ 各制限が効くタイミングの設計

  Q&A 30回/月:
    Day 1-7:   1-2回/日 × 7日 = 7-14回 → まだ余裕
    Day 8-14:  2-3回/日 × 7日 = 14-21回 → 「よく使うようになった」
    Day 15-21: 3-4回/日 → 30回に到達 ← ここで制限
    → 「毎日使うようになった頃に制限」= 完璧なタイミング

  ファイル 10件/月:
    Week 1: 見積書2-3件 + マニュアル1件 = 3-4件
    Week 2: 追加で3-4件
    Week 3: 10件に到達 ← ここで制限
    → Q&Aの制限と同時期に到達（相乗効果で転換を後押し）

  ナレッジ 50件:
    テンプレート由来: カウントしない（これが重要）
    自社登録分のみカウント
    → 50件 = 約1ヶ月の投入量（1日2-3件ペース）
    → 1ヶ月で「自社データが入るほど精度が上がる」を体験済み
    → 「もっと入れたいけど上限」→ 転換

  能動提案 月1件プレビュー:
    → 「こんなリスクを検知しました」のタイトルだけ見える
    → 詳細を見るにはアップグレード
    → 好奇心ドリブンの転換（「何が書いてあるか気になる」）

■ 制限到達時のUX（超重要）

  ❌ ダメなUX:
    「Q&A上限に達しました。有料プランにアップグレードしてください」
    → 冷たい。離脱する。

  ✅ 良いUX:
    「今月のQ&A 30回を使い切りました！

     📊 今月の利用状況:
     ・Q&Aで解決した質問: 28件
     ・ファイルから抽出したルール: 8件
     ・推定削減時間: 約4.5時間

     💡 Starterプラン（月¥9,800）なら:
     ・Q&A 200回/月（今の6.7倍）
     ・ファイル処理 50件/月
     ・音声文字起こし対応
     ・What-if分析

     📝 来月1日にFreeプランのQ&Aはリセットされます

     [Starterにアップグレード ¥9,800/月]
     [来月まで待つ]
     [パートナーに相談する → 導入支援付きで始める]」

  → 3つの選択肢を必ず提示:
    A. セルフでアップグレード（即決層）
    B. 待つ（まだ温まっていない層 → 来月また使う → いずれ転換）
    C. パートナー経由（本格導入したい層 → パートナーにリード供給）
```

### 3.3 Free プランの「やらないこと」リスト

```
■ Free で絶対にやらないこと

  ① BPO（自動実行）を提供しない
     → BPOは最もコストが高い機能（外部API呼び出し・SaaS操作）
     → 無料で提供するとコストが爆発する
     → かつ、BPOはGrowth（¥30K）の最大の差別化要因

  ② SaaSコネクタを提供しない
     → kintone/freee/ANDPAD連携はGrowthの売り
     → 連携すると離脱しにくくなる = 有料の特権

  ③ マルチユーザーを提供しない
     → 「他の社員にも使わせたい」= 最強の転換トリガー
     → 社長が1人で使って満足 → 「部下にも使わせたい」→ Growth

  ④ エクスポートを提供しない
     → データの持ち出し = 無料のまま使い続ける口実
     → 「レポートをPDFで出したい」→ Starter以上

  ⑤ カスタマーサポート（人的）を提供しない
     → セルフサービスのみ（ヘルプ記事 + AIチャットボット）
     → 人的サポートコストをゼロに抑える
```

```
     → 想定: 5-10%がこのトリガー

  ③ What-if分析:
     「もし原価率を3%下げたら？」のシミュレーション

  ① マルチユーザー需要（最大のトリガー）:
     「経理にも使わせたい」「現場監督にも」

  ・ChatGPT Plus: $20/月（約¥3,000）← 汎用AIで業界知識なし

  ・NotePM: ¥4,800/月〜 ← 社内wiki。AIなし
  → ¥9,800 は「業界特化AIアシスタント」として妥当。安すぎず高すぎない
```

### 5.4 バイラル装置③ 紹介プログラム

```
■ 仕組み（Free/Starter/Growth 共通）

  紹介する側:
  ・紹介リンクを共有
  ・紹介先が Free 登録 → 紹介者に Q&A +20回/月（Freeなら30→50回）
  ・紹介先が Starter/Growth に転換 → 紹介者に1ヶ月分の利用料を割引

  紹介される側:
  ・紹介リンクから登録 → Q&A +10回のボーナス（初月のみ）
  ・Starter 初月50% OFF（¥4,900）

■ なぜ金銭ではなく利用枠で報酬するか

  ① 中小企業の社長に現金キャッシュバックは刺さらない
     → 月¥3,000のキャッシュバックは端金。手続きが面倒
  ② 利用枠の増加は「もっと使える」= プロダクト体験の向上
     → 使えば使うほど価値を感じる → 有料転換率が上がる
  ③ 管理が簡単
     → 銀行振込等の事務コストゼロ

■ バイラル係数の試算

  紹介プログラムのk-factor:
  ・月間アクティブユーザーのうち紹介を実施する割合: 5%
  ・紹介1回あたりの登録率: 40%（知人からの紹介は信頼度が高い）
  ・1人あたり平均紹介数: 2人/年

  → k-factor（紹介のみ）: 0.05 × 0.4 × (2/12) = 0.003/月
  → 小さいが、レポートフッター(0.075) + ベンチマーク(0.05) と合算で
     月間バイラル係数: 約0.13
  → 1,000人 → 月130人の新規登録が自動的に発生
```

### 8.2 最重要チャネル: パートナー経由 Free 登録


## 9. 反論への回答（フリーミアム導入の懸念）

```
■ 懸念1:「無料ユーザーのサポートコストが膨大になるのでは？」

  回答: Freeプランは人的サポートゼロ設計。
  ・ヘルプ記事 + AIチャットボット（シャチョツー自身が回答）のみ
  ・人的CS は Starter 以上に限定
  ・Free ユーザーからの問い合わせ先は「パートナーに相談」ボタン
    → サポートコストをパートナーに転嫁（パートナーは見込客獲得として歓迎）

■ 懸念2:「無料で十分だから有料に上がらないのでは？」

  回答: 制限設計が「ちょうど足りない」ようにチューニングされている。
  ・Q&A 30回/月 = 毎日1回。ビジネスで毎日1回しか質問できないのは明らかに不足
  ・ナレッジ50件 = 1部門の一部だけ。全社のナレッジには到底足りない
  ・1ユーザー = 社長だけ。社員に使わせたくなるのは時間の問題
  ・BPOなし = 手動作業は減らない。自動化の「味」すら見せない

  さらに: 能動提案のプレビュー（タイトルだけ表示）が好奇心を刺激
  → 「リスクを検知しました: ○○○○の可能性」← 詳細はアップグレードで
  → 経営者が最も見たい情報がゲートの向こう側にある

■ 懸念3:「月¥9,800のStarterが¥30,000のGrowthを食うのでは？」

  回答: StarterとGrowthは明確に異なるペルソナ。
  ・Starter = 社長1人で完結する企業（10人以下の小規模）
    → もともとGrowthの¥30Kは高すぎる層。Starterがなければ逃す
  ・Growth = チーム利用が必要な企業（10人以上）
    → 社員にも使わせたい → Starterでは不可能 → 自然にGrowthへ

  Starterがなかった場合:
    Free→Growth の転換率: 5-7%（ジャンプが大きすぎて離脱）
  Starterがある場合:
    Free→Starter→Growth: 10% × 40% = 4% + Free→Growth直行3% = 7%
    → ほぼ同じだが、Starter滞留中の売上（¥9,800/月）が追加収益

■ 懸念4:「パートナーのOEMマージンがStarterだと小さすぎるのでは？」

  回答: パートナーにとってStarterは「育成中の案件」。
  ・Starter ¥9,800 × 75% = パートナー取り分 ¥7,350/月/社
  ・パートナーは10社のStarterを持てば ¥73,500/月
  ・そのうち40%がGrowthに移行 = 4社 × ¥30K × 75% = ¥90,000/月
  → 時間とともにGrowth比率が上がり、パートナー収益も自然成長

  さらに: パートナーの本丸の稼ぎは導入支援パッケージ（¥50-300万）
  → Starter期間中に関係構築 → Growthに上がる際にパッケージ提案

■ 懸念5:「競合にFreeを真似されたら終わりでは？」

  回答: Freeは「入口」であって「堀」ではない。堀は3つ:
  ① 業界テンプレート（Genome）: 蓄積データに基づく精度は模倣困難
  ② ネットワーク効果: 参加企業が増えるほどベンチマークの価値が上がる
  ③ スイッチングコスト: 投入したナレッジ・ルール・フローは他社に持ち出せない

  競合がFreeを出しても:
  → テンプレートなし = Day 1 Value なし = 離脱
  → ベンチマークなし = ネットワーク効果なし
  → 後発は常にデータ量で負ける（Winner Takes Most）
```

## 11. KPI ダッシュボード

```
■ 週次で追跡する指標

  ┌─────────────────────────────┬──────────────┬──────────────┐
  │ 指標                         │ 目標（Phase C）│ 危険ライン    │
  ├─────────────────────────────┼──────────────┼──────────────┤
  │ Free 新規登録数（週）         │ 300+          │ < 100        │
  │ Free → Starter 転換率（月）   │ 10%+          │ < 5%         │
  │ Free → Growth 直行率（月）    │ 3%+           │ < 1%         │
  │ Starter → Growth 転換率（月） │ 5%+（月次）    │ < 2%         │
  │ Free Aha Moment 到達率        │ 80%+          │ < 60%        │
  │ バイラル係数（k-factor）       │ 0.1+          │ < 0.05       │
  │ Free 月間コスト/ユーザー      │ < ¥200        │ > ¥500       │
  │ Free 7日リテンション          │ 60%+          │ < 40%        │
  │ Free 30日リテンション         │ 40%+          │ < 25%        │
  │ 制限到達率（月内に制限に当たる）│ 30%+          │ < 15%        │
  └─────────────────────────────┴──────────────┴──────────────┘

  ■ Aha Moment 到達率の計測方法:
    「登録後7日以内に Q&A を3回以上実行」= Aha Moment 到達と定義
    → この定義はデータが溜まったら要見直し（相関分析で最適化）

  ■ 制限到達率が低い場合:
    → 制限が緩すぎる。Q&A 30回→20回に引き下げを検討
    → または利用自体が少ない → オンボーディングに問題

  ■ 制限到達率が高すぎる場合（>60%）:
    → 制限がきつすぎる。Free での価値体験が不十分な可能性
    → Q&A 30回→40回に引き上げを検討
```
