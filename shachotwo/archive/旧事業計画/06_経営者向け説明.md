## 2. 何が変わるのか — Before / After

### 2.1 Before（現状の典型的な課題）

```
■ 社長の頭の中問題
  ・「この案件どうする？」→ 毎回社長に聞くしかない
  ・社長が出張中 → 判断が止まる → 現場が遅れる
  ・社長に聞く → 社長の時間が奪われる → 経営に集中できない

■ 属人化問題
  ・ベテランの退職 → ノウハウが丸ごと消える
  ・引継ぎは「OJT」という名の放置
  ・新人が独り立ちするまで6ヶ月〜1年

■ 情報散在問題
  ・Excel、紙、社長の記憶、LINEのやり取り…
  ・「あの情報どこだっけ」で毎日30分消える
  ・同じ質問に何度も答えている
```

### 2.2 After（シャチョツー導入後）

```
■ デジタル事務員が24時間対応
  ・社員が「社長に聞く代わりに」AIに質問
  ・AIが社長の判断基準に基づいて回答（引用付き）
  ・社長は「本当に自分しかできないこと」に集中

■ ナレッジの永続化
  ・LINEで送る、メールを転送する、Excelをアップロードする、それだけ
  ・退職者のナレッジもAIの中に残り続ける
  ・新人は「AIに聞きながら」独り立ち → 育成期間半減

■ AIが先に教えてくれる
  ・「来月の工程が危ないです」（リスク検出）
  ・「この原価率は業界平均より高いです」（改善提案）
  ・「もし〇〇したら売上が15%伸びる可能性があります」（What-if分析 ※AI推定値）
```

## 6. セキュリティ — 御社のナレッジを守る仕組み

### 6.1 社長向けの要約（1分で読める）

```
御社のナレッジは、以下の仕組みで守られています:

  ✅ データ保管は日本国内サーバー
     → Supabase（AWS東京リージョン）+ GCP東京リージョン
     → AI処理はGoogle Gemini API（東京）をメイン利用（保存なし・学習不使用）
     → Claude APIはフォールバック利用（米国・保存なし・学習不使用・DPA締結済み）
     → PII（個人情報）は自動検出・マスキング後にAI処理。原文は送信しない設計

  ✅ 銀行レベルの暗号化
     → AES-256-GCM（世界標準・軍用レベルの暗号化）

  ✅ 他社のデータとは完全に壁で分離
     → データベースの仕組み（RLS）で強制。アプリのバグでも漏れない

  ✅ AIの学習にデータは一切使われない
     → Google / Anthropic社の商用APIポリシーで契約上保証

  ✅ 当社スタッフも御社データに無断アクセスできない
     → アクセスした場合は全て監査ログに記録

  ✅ 万一の漏洩時はサイバー保険で補償
     → 72時間以内に御社に通知。対応フロー策定済み

  ✅ 解約時はデータを完全削除し、証明書を発行
     → 30日間のエクスポート猶予 → 削除 → 削除証明書PDF送付
```

## 9. よくある質問（意思決定者向け）

### Q: ChatGPTとの違いは？

```
ChatGPT = 汎用的な「物知り」。誰に聞いても同じ答え。
シャチョツー = 御社専用の「参謀」。御社のナレッジに基づいて回答。

例: 「この案件の原価率はどのくらいが妥当？」
  ChatGPT: 「一般的に建設業の原価率は70-80%です」
  シャチョツー: 「御社の過去3年の類似案件の原価率は73%です。
              今回の案件は△△の特殊工法があるので、76-78%を見込むべきです。
              根拠: [2024年○○工事の実績データ]」
```

### Q: 導入に失敗したらどうなる？

```
・基本料¥30,000/月 + 少額の従量。最大リスクは月¥40,000程度。
・合わなければ違約金なしで終了。
・データはJSON/CSVでエクスポート可能。ロックインなし。
・削除証明書を発行するので、データが残る心配もなし。
```

### Q: 社員がちゃんと使ってくれるか心配

```
・LINEで送る、メールを転送する、それだけ。研修不要。
・「AIに聞いてから社長に聞く」というルールを1つ決めるだけ。
・使うほど回答精度が上がるので、成功体験が早い。
・まず1部門の2-3名で始めて、効果を実感してから全社展開。
・充足度マップで「まだAIが知らない分野」が見える
  → 社員が自主的にナレッジを追加し始める。
```
