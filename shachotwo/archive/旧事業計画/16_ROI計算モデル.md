## 1. ROI計算フレームワーク

### 1.1 基本構造

```
ROI(%) = (年間効果合計 − 年間投資額) / 年間投資額 × 100

年間効果合計 = A + B + C + D + E
  A: 社長の時間価値回復
  B: 意思決定の質向上
  C: 属人化リスク削減
  D: 業務自動化効果（BPO）
  E: ネットワーク効果（ベンチマーク）

年間投資額 = (基本料¥30K + 従量課金 + ユーザー追加費) × 12
```

## 2. 効果カテゴリ別 定量化モデル

### A. 社長の時間価値回復

#### A-1. 社長の時間単価算出

```
■ 売上100億円企業の社長報酬（業種別推定）

  建設業:  年収 2,500〜4,000万円（中央値 3,000万円）
  製造業:  年収 2,000〜3,500万円（中央値 2,800万円）
  歯科:   年収 3,000〜5,000万円（中央値 4,000万円）※院長兼経営者

■ 時間単価計算式

  時間単価 = 年収 / (年間稼働日数 × 日間労働時間)

  一般的な社長: 年間280日 × 10h = 2,800h
  → 年収3,000万の場合: ¥30,000,000 / 2,800h = ¥10,714/h

  ただし、社長の時間価値 ≠ 社長の報酬。
  社長の1時間の意思決定が動かす金額で評価すべき。

■ 意思決定ベースの時間価値（推奨計算式）

  実効時間価値 = 年間売上 × 社長関与率 / 年間労働時間

  売上100億 × 社長関与率20% = 20億円分の判断
  20億 / 2,800h = ¥714,286/h（≒ 約70万円/h）

  → 控えめに見ても「社長の判断1時間 = 10〜70万円の価値」
  → ROI計算では保守的に報酬ベース（¥10,000/h）を使用
  → 提案書では意思決定ベースの数字も併記（インパクト最大化）
```

### 5.2 計算ロジック（Excel/Webツール共通）

```
■ Step 1: 月額コスト自動推定
  基本料: ¥30,000
  ユーザー追加: (マネージャー数 × ¥5,000) + (一般社員数 × ¥2,000)
  従量課金推定:
    ファイル処理: 従業員数 × 15件/月 × ¥50
    Q&A: 従業員数 × 20回/月 × ¥30
    BPO: 従業員数 × 5件/月 × ¥200
  コネクタ: SaaS数 × ¥5,000

■ Step 2: 社長の時間単価算出
  入力あり → 年収 / (280日 × 週間労働時間/5)
  入力なし → 業種別デフォルト年収 / 2,800h

■ Step 3: 各効果カテゴリの計算

  A_時間効果 = 削減時間(業種別) × 時間単価 × 12
  B_判断質   = 年間判断件数(売上連動) × ミス率 × 平均損失 × 削減率
  B_判断速   = 月間機会損失件数 × 平均損失額 × 12 × 改善率
  C_属人化   = 退職リスク効果 + 社長不在リスク効果
  D_BPO      = 自動化工数 × 人件費単価 × 12
  E_NW効果   = コンサル代替価値

■ Step 4: 課題チェックボックスによる係数補正

  チェック数に応じて効果を加算:
  ・Q&A過負荷 → A効果を1.3倍
  ・属人化リスク → C効果を1.5倍
  ・情報散在 → B判断質を1.2倍
  ・教育コスト → C効果に教育コスト削減分を加算
  ・ミス繰返し → B判断質を1.3倍
  ・見積ばらつき → 業種固有効果を加算
  ・SaaS未連携 → D効果を1.3倍
  ・トレンド不足 → E効果を1.2倍
  ・事業承継 → C効果を2.0倍
  ・拠点ばらつき → 拠点効果を加算（事業数×年間効果）

■ Step 5: Level別タイムラインROI算出

  Level 0 効果 = Day 1 Value（ファイルチェック + 議事録 + 文書下書き）
  Level 1 効果 = Level 0 + データ収集自動化
  Level 2 効果 = Level 1 + 下書き作成 + Q&A
  Level 3 効果 = Level 2 + 承認付き実行（BPO）
  Level 4 効果 = Level 3 + 自律実行

  各Levelの到達月（標準）:
    Level 0: 月1（Day 1）
    Level 1: 月2-3
    Level 2: 月3-5
    Level 3: 月6-8
    Level 4: 月12以降

■ Step 6: 出力計算

  月次ROI      = (月間効果合計 − 月額費用) / 月額費用 × 100
  年間ROI      = (年間効果合計 − 年間投資額) / 年間投資額 × 100
  投資回収月数  = 年間投資額 / 月間効果合計 （切り上げ）
  3年累計効果  = Σ(月間効果 − 月額コスト) over 36ヶ月
  Day 1 ROI    = Day 1 Value月額効果 / 初月コスト × 100
```

## 8. 補足: ROI計算の前提と注意事項

```
■ 保守的計算の原則
  ・効果はすべて「保守的」シナリオを標準採用
  ・機会獲得効果（B-2）は保守値でも大きいため、営業では控えめに提示し
    顧客自身に「ありそうだ」と納得してもらう
  ・定性効果（安心感、社長のストレス軽減 等）はROI計算に含めない

■ 効果発現のタイムライン（Level進行と連動）
  ・月1（Day 1）: Level 0。ファイルチェック・議事録・文書下書きで即時効果
  ・月2-3: Level 1。データ収集自動化が加わり、Q&A精度も向上
  ・月3-5: Level 2。下書き作成自動化 + Q&A本格活用。効果が本格化
  ・月6-8: Level 3。承認付きBPO実行。効果が大幅に拡大
  ・月12以降: Level 4。自律実行（高信頼タスク限定）。最大効果

■ 従量課金と効果の連動性
  ・従量課金 = 利用量 = 効果の代理変数
  ・利用が少ない（＝効果が小さい）→ コストも低い → ROIは維持される
  ・利用が多い（＝効果が大きい）→ コストも増加するが効果の方が大きい
  ・この構造により「払い損」が発生しにくい設計

■ 効果測定の方法（導入後の振り返り用）
  ・社長の週間労働時間の変化（導入前後で比較）
  ・社内Q&A件数の変化（シャチョツー経由 vs 直接質問）
  ・判断所要時間の変化（主要判断の所要日数をトラッキング）
  ・キーマン退職時の引継ぎ期間（Before/Afterで比較）
  ・Levelの進行速度（導入企業平均との比較）

■ この計算モデルが使えない場合
  ・従業員5名未満の企業 → 従量課金が最低限でも基本料¥30K/月のROI確保が課題
  ・社長が経営に関与していない企業 → 時間価値モデルが適用できない
  ・情報セキュリティ要件が極めて高い企業 → 追加対応コストが必要
```
