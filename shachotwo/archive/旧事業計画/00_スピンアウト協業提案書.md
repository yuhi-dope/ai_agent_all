## 1. 提案の趣旨

```
■ 結論
  シャチョツーを私個人で独立・法人化し、
  御社のコンサル・人材事業の「強力なリード獲得チャネル兼フック商材」として
  最大限活用していただくためのご提案です。

■ 本提案のゴール
  ・御社の売上（トップライン）を落とさない
  ・御社にとって新たな収益源（リード獲得チャネル）を生む
  ・私が開発に専念することで、プロダクトのスケールを加速する
  ・双方にとって Win-Win の長期的パートナーシップを構築する
```

## 2. スピンアウトの背景

```
■ これまでの経緯
  ・業務外（個人開発枠）で本システムを開発
  ・会社から「個人開発」の許可を得た上で進めてきた
  ・一定の顧客評価と売上を獲得できた
  ・御社には営業チャネル・顧客基盤の提供で多大なるご支援をいただいた

  ・独立により御社との関係を「上下」ではなく「対等なパートナー」に昇華させたい

■ 御社への感謝と姿勢
  ・初期顧客を作ってくれた恩は十分に認識している
  ・だからこそ、独立後も御社に利益が還元される仕組みを提案する

## 3. 協業スキーム（提案のコア）

### 3.1 [確定] 既存顧客・御社経由の新規販売（OEM/代理店モデル）

```
■ 商流（お金の流れ）

  [顧客] →（月額利用料を支払）→ [御社] →（システム利用料を支払）→ [新会社（私）]

  ・御社は引き続き「自社商材」として本システムを販売
  ・御社が顧客へ請求し、販売手数料（マージン）を差し引いた額を当方に支払う
  ・顧客との契約主体は御社のまま → 顧客から見れば何も変わらない

■ ブランド使用権 [確定]
  ・御社に対し、OEM（ホワイトラベル）または販売代理店としての権利を付与
  ・御社が自社のフック商材として既存顧客に売りやすいよう、
    御社ブランドでの再販を許可する

■ 販売手数料（御社の取り分）[確定]
  ・初回提示: 顧客請求額の 20% を御社の販売手数料とする
  ・交渉上限: 最大 30% まで譲歩可能（30%が絶対上限）
  ・理由: 営業利益率を高い水準で維持するため

  例: 基本料 ¥30,000/月 + 従量課金 ¥70,000/月 = 月額合計 ¥100,000 の場合
    20%時: 御社 ¥20,000 / 当方 ¥80,000
    30%時: 御社 ¥30,000 / 当方 ¥70,000

  ※ 一般的なSaaS代理店マージンは 20-30%。20%は業界標準の下限で妥当な提示
  ※ 手数料は基本料+従量課金の合計額に対して計算
  ※ 新価格モデル: 基本料¥30,000/月 + 従量課金（ファイル処理¥50/件、Q&A¥30/回、BPO実行¥200/件、追加ユーザー¥2,000-5,000/月、コネクタ¥5,000/個/月）
  ※ 従量課金が¥500,000/月を超えた月は固定プランへの移行を提案
  ※ プラン別の料率差は設けず、全プラン一律

■ 御社のメリット
  ・顧客との契約主体は御社のまま → 売上（トップライン）が落ちない
  ・開発・保守・CS工数が手離れ → 利益率が向上（売るだけの体制）
  ・プロダクトの品質向上は当方が全額負担で行う

■ 当方のメリット
  ・独立初期の不安定な時期に確実な固定収益（ベースロード）を確保
  ・既存顧客の信用と実績を活用できる
```

### 3.2 [確定] 新規リードの相互送客（シナジー効果）

```
■ 棲み分けの構造

  [当方] AIツール（フック商材・月3万〜＋従量・全業界）
    ↓ アップセルニーズ発生時
  [御社] コンサルティング（高単価）/ 人材紹介（高単価）

  → 商材のレイヤーが異なるため、バッティング（競合）しない
  → 当方は御社の「無料のリード獲得チャネル」になる

■ 具体的な送客フロー

  ① 当方が独自マーケティングで全業界にAIツールを拡販
  ② ツール導入企業から以下のニーズが発生:
     ・「業務プロセス自体を見直したい」→ コンサルニーズ → 御社へ送客
     ・「AIを使える人材がほしい」→ 人材紹介ニーズ → 御社へ送客
     ・「DX推進を本格的にやりたい」→ コンサルニーズ → 御社へ送客
  ③ 御社は送客された顧客にコンサル・人材紹介を提供

■ 逆方向の送客
  ・御社がコンサルに入った企業 → 業務効率化ツールとしてシャチョツーを提案
  ・御社の人材紹介先企業 → 採用後のオンボーディング効率化としてシャチョツーを提案

■ 横断型サービス業への展開による送客モデルの強化（新規追加）

  当方のターゲットを商社・人材紹介・コンサル等の「横断型サービス業」に拡大。
  これにより、送客パイプラインが二重・三重に拡大する:

  パターンA: 人材紹介会社にシャチョツーを導入
    → その人材紹介会社の顧客企業にもシャチョツーを紹介（二次展開）
    → 御社の人材事業と「紹介会社同士の連携」が可能に

  パターンB: コンサル会社にシャチョツーを導入
    → コンサル会社がクライアントにシャチョツーを推奨（OEM的）
    → 御社のコンサル事業と棲み分け or 協業

  パターンC: 商社にシャチョツーを導入
    → 商社の取引先（製造業等）にシャチョツーを紹介
    → 製造業の既存ターゲットへの新チャネルが生まれる

  → 「横断型サービス業」は顧客でありながら販売チャネルにもなる
  → 御社にとっても送客元が増え、コンサル・人材事業のリード数が拡大

■ 送客時の紹介料 [確定]
  ・当方→御社への送客: 紹介料なし（恩返し+パートナーシップの一環）
    → 「無料で送客します」は交渉上の強力なカード
  ・御社→当方への送客: 3.1 のOEMモデルに乗せる（御社が代理店として販売）
    → 別途紹介料は設けず、商流をシンプルに保つ
```

## 9. [確定] SLA・品質保証・CS責任分界

```
■ サーバー稼働率の保証: 段階的引き上げ [確定]
  ・初年度: 99.0%（月間ダウンタイム 7.3時間以内）
  ・2年目〜: 99.5%（月間ダウンタイム 3.6時間以内）

  理由:
  ・1人体制では深夜障害対応に限界がある。99.5%は1回のインシデントで未達リスク
  ・運用実績がない段階で高いSLAを約束して未達になると信頼を失う
  ・低めに約束して超過達成する方が信頼構築に有利
  ・1年間で運用実績・ノウハウが溜まり、Phase Cで人員追加後に引き上げる
  ・「実績として99.5%を超えていたので正式にSLAに反映」と言える

■ 障害発生時の対応スピード [確定]

  重大度の定義:
    重大障害: サービス全停止、またはデータ損失の恐れがある障害
    軽微障害: 一部機能の制限・性能低下など、業務継続は可能な障害

  対応スピード（「対応開始」と「復旧完了」を分離して定義）:
  ┌──────────┬──────────────┬──────────────┐
  │ 重大度    │ 対応開始      │ 復旧完了      │
  ├──────────┼──────────────┼──────────────┤
  │ 重大障害  │ 2時間以内     │ 8時間以内     │
  │ 軽微障害  │ 翌営業日以内  │ 3営業日以内   │
  └──────────┴──────────────┴──────────────┘

  補足:
  ・「対応開始」= 障害を認知し、調査・復旧作業に着手した時点
  ・「復旧完了」= サービスが正常稼働に復帰した時点
  ・深夜帯（22:00-8:00）の重大障害は、翌8:00を起点に2時間以内
    （1人体制のため深夜即時対応は現実的でない。Phase Cで人員追加後に24h対応へ移行）
  ・障害発生時は御社担当者へメール+チャットで即時通知

■ カスタマーサポート（CS）の責任分界点 [確定]

  ハイブリッド型を採用。ただし適用範囲は以下に限定:

  対象: 御社経由の既存顧客 + 御社が新規獲得した顧客のみ
  ───────────────────────────────────────────────
  御社:  契約・請求・一般的な操作質問（一次対応）
  当方:  バグ・障害・技術的な問い合わせ（二次対応・技術）
  窓口:  問い合わせ窓口は御社。技術系は当方に転送
  ───────────────────────────────────────────────

  当方が独自に直販した顧客:
  → 当方が100%全CS対応（御社は一切関与しない）

  メリット:
  ・御社が顧客接点を保持 → 顧客との関係性を維持
  ・当方は開発に集中できる
  ・御社の工数は最小限（技術系は全て当方が巻き取る）

■ SLA未達時のペナルティ [確定]
  ・未達月のシステム利用料を10%減額
  ・厳しすぎず、かつ誠意を示せる水準
```

  │                                                          │

  │                                                          │

  │     → メリット: 関係性を安定させるのに十分な期間          │

  │                                                          │
  │   案3: 3年間                                             │

  │     → デメリット: 当方の自由度が下がる                    │
  │                                                          │
  ├──────────────────────────────────────────────────────────┤

## 17. 交渉時の想定Q&A

```
Q: 「辞めて競合になるんじゃないの？」
A: 商材レイヤーが違います。当方はAIツール（月3万+従量課金のフック商材）、
   御社はコンサル（月50-200万の本命商材）と人材紹介。
   むしろ、当方の顧客基盤が御社への送客パイプラインになります。
   しかも送客は紹介料なし。御社にとっては無料のリード獲得チャネルです。

Q: 「うちの売上が減るのでは？」
A: 既存顧客の契約主体は御社のままです。トップラインは変わりません。
   さらに、開発・保守コストが手離れするので利益率は上がります。

Q: 「システムの権利はうちにあるのでは？」
A: 業務時間外に、御社が許可した「個人開発枠」で開発したものです。
   ただし、御社の営業チャネルのおかげで顧客を獲得できた恩があるため、
   販売手数料を業界標準水準の20%でご提案しています。

Q: 「もっと高いマージンが欲しい」
A: 当方のコスト構造（インフラ・API・人件費）を考慮すると、30%が上限です。
   ただし、送客による御社コンサル・人材事業へのリード提供価値
  （実質的な追加収益・紹介料なし）も合わせてご評価ください。

Q: 「出資させてほしい」
A: まずは業務提携で実績を積み、双方にとって最適な関係性を検証させてください。
   事業が軌道に乗った段階で、資本提携の可能性は排除しません。

Q: 「独立したら品質が下がるのでは？」
A: 専業になることで開発リソースが2倍以上になり、品質は向上します。
   SLAを契約に明記し、品質保証をコミットします。

Q: 「今の顧客を持っていかれるのでは？」
A: 御社経由の顧客に対して当方が直接営業することは契約で禁止します。
   御社のチャネルは御社のものとして保護されます。

Q: 「うちのブランドで売り続けられるの？」
A: はい。OEM/ホワイトラベルの権利を御社に付与します。
   御社の自社商材として、御社のブランドで販売を継続できます。

Q: 「CSは誰がやるの？」
A: 御社経由の顧客は、契約・請求・一般操作は御社が窓口、
   バグ・障害・技術系は当方が対応するハイブリッド型です。
   当方が直販した顧客のCSは当方が100%対応しますのでご負担はありません。
```

```
■ 絶対に譲れない線

  ・送客の紹介料: 当方→御社は無料にする（強力なカード）
  ・並行稼働期間: 退職後1-3ヶ月間の優先対応

  ・手数料率は20%で提示。「業界標準」と説明
  ・会社が反発したら25%を出す。「御社の恩を考慮し上乗せ」と説明
  ・それでも不十分なら30%。「これが上限」と明確に線引き

  2. 退職時期・引き継ぎ（スケジュールに直結）
