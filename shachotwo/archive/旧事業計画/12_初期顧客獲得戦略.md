# シャチョツー 初期顧客獲得戦略

> **目標: 3業種 × 15社 + α = 50社を12ヶ月以内に獲得し、データ蓄積のフライホイールを起動する**
> ターゲット: 売上10億〜100億円の中小〜中堅企業（従業員10-300名）
> チャネル: LINE・メール・ファイルを入口に「月3万でデジタル事務員を雇う」
> セールスピッチ: 「月3万円からスタート。使った分だけ払う。事務員1人分以下で会社のAI参謀が手に入る」

## 1. なぜ「売上10億〜100億円」に拡大するのか

```
■ 事業計画（01_事業計画.md）との関係
  メインターゲットは「10-300名の中小企業」。
  旧戦略では100億円前後の中堅企業のみに絞っていたが、
  月額¥30,000の基本料+従量課金への価格改定により、
  10億円規模の企業でも十分にROIが出るビジネスモデルに進化した。

  ① 月3万円なら「試してみよう」のハードルが圧倒的に低い
    ・売上10億企業: 月3万円 = 年間売上の0.004%。稟議不要レベル
    ・売上100億企業: 月3万円は端金。即決可能
    ・旧モデルの¥200K-500K → 稟議必要 → PoC要求 → 3-6ヶ月の意思決定
    ・新モデルの¥30K → 「とりあえず入れてみよう」→ 使った分だけ自然増加

  ② チャネルファースト設計でDay 1 Valueを実現
    ・LINE/メール/ファイルが主要入口 → 社長が普段使いのツールで完結
    ・初日からファイルを投げれば構造化が始まる = 初日から価値が出る
    ・「SaaSにログインして使う」ではなく「LINEで聞けばAIが答える」
    ・PoCが不要。月3万円で「PoC」と「本番」の境目がない

  ③ ARPUは自然成長する
    ・基本料¥30K → ファイル処理50円/件 → Q&A 30円/回 → BPO 200円/件
    ・追加ユーザー¥2,000/人 → マルチユーザー（参謀・アシスタント・ヘルパー）
    ・使えば使うほど料金が上がる = 使えば使うほど価値を感じている証拠
    ・small ARPU目標: ¥5-8万 / medium: ¥8-15万 / large: ¥15-25万

  ④ 10億〜30億企業はコンサル経由でなくても直販で取れる
    ・社長が直接LINEで問い合わせてくる規模感
    ・「月3万でデジタル事務員を雇う」はSNS・口コミで拡散しやすい
    ・100億企業はコンサル経由が効果的（従来の戦略も並行）

  ⑤ データ量は「社数」でカバーする
    ・旧: 15社（100億規模）= 1社から大量データ
    ・新: 50社（10-100億の幅広い規模）= 多様なパターンを収集
    ・遺伝子プールの精度向上は「社数の多様性」が鍵
    ・50社のデータ ≒ 業種別ベンチマークの統計的意味が出るライン

■ 注意: ターゲットの「重心」は売上30-50億規模
  ・30-50億企業がARPU¥8-15万で最もバランスが良い
  ・10-30億企業は入口（月3万スタート→ARPUグロース期待）
  ・50-100億企業はARPU¥15-25万の高単価ゾーン
  ・3層すべてをカバーすることでフライホイールが加速する
```

```

    ・「月3万でデジタル事務員を雇う」のメッセージでSNS拡散
    ・LP → LINE登録 → 即日利用開始
    ・10-30億企業はここで取る（社長が自分で決められる金額）

    ・SIer向け紹介フィー: 15%永続（SIerの保守費を失っても補填される設計）

## 5. ARPU成長モデル（旧「段階移行」に代わる新モデル）

### 5.1 旧モデルとの比較

```
■ 旧モデル: 段階移行（PoC → 移行価格 → 正規料金）
  無料PoC 3ヶ月 → ¥30K×3ヶ月 → ¥100-250K×3ヶ月 → ¥200-500K
  問題点:
    ・無料PoCでの離脱が多い（「タダだから使う」→有償化で離脱）
    ・価格移行のたびに解約リスク
    ・12ヶ月かけてやっと正規料金
    ・PoC対応の人的コストが膨大

■ 新モデル: Day 1有償 + 自然ARPU成長
  月1: 基本料¥30K + 従量（少額）= ¥35-40K
  月3: 使い慣れて従量増加 = ¥50-60K
  月6: マルチユーザー追加 + BPO利用開始 = ¥80-100K
  月12: フル活用 = ¥100-200K

  メリット:
    ・Day 1から有償 = 「お金を払う覚悟のある顧客」だけが入る
    ・月3万スタートなのでPoC不要。「使ってみる」= 本番利用
    ・ARPU成長 = 顧客が価値を感じている証拠 = 解約リスクが低い
    ・「料金が上がるタイミング」がない → 価格移行チャーンがゼロ
```

### 5.2 ARPU成長の4段階

```
■ Stage 1: 入門（月1-2）ARPU ¥3-5万
  ・社長1人がLINEでQ&Aを使い始める

  ・Q&A頻度が週10回→週30回に増加
  ・追加ユーザー1-2名（経理、幹部）
  ・「これ、部下にも使わせたい」

  ・マルチユーザー3-5名（参謀+アシスタント+ヘルパー）

## 6. データ蓄積戦略

### 6.1 50社のデータが生むフライホイール

```
■ 50社 × 多様な規模 = 強力な遺伝子プール

  旧戦略: 15社（100億規模のみ）→ 同質的なデータ
  新戦略: 50社（10-100億の幅）→ 多様なパターン

  ・10-30億企業のデータ → 「小規模ならではの課題パターン」を収集
  ・30-50億企業のデータ → 「成長期特有の組織課題」を収集
  ・50-100億企業のデータ → 「大規模化の暗黙知」を収集
  ・3業種×3規模帯 = 9セグメントのデータ

■ チャネルファースト設計がデータ蓄積を加速

  旧: 社長が「対話セッション」で入力 → 月10-20件のペース
  新: LINE/メール/ファイルで日常的に投入 → 月50-100件のペース

  ・LINEでの質問 = Q&Aログ = ナレッジの種
  ・メール転送されたファイル = 構造化の素材
  ・音声メモ = 暗黙知の直接キャプチャ
  → データ蓄積速度が旧モデルの3-5倍

■ 12ヶ月マイルストーン

  M3:  9社稼働 → 合計900件のナレッジ
  M6:  19社稼働 → 合計4,000件 → 業種別パターン初期版
  M9:  39社稼働 → 合計12,000件 → ベンチマーク初期版提供開始
  M12: 50社稼働 → 合計20,000件 → 遺伝子精度80% → オンボーディング高速化
```

## 7. パートナーインセンティブ設計（新価格モデル対応）

### 7.1 コンサル・SIer 共通インセンティブ

```
■ 紹介フィー: 基本料+従量課金の15%を永続支払い

  例: 紹介した企業のARPUが¥100,000/月の場合
    → 毎月¥15,000がパートナーに永続で入る
    → 10社紹介 × ¥15,000 = ¥150,000/月の不労所得

  ※ 旧モデル: 紹介フィーは月額の20-30%だが固定料金ベース
  ※ 新モデル: 15%だが「ARPUが成長する」ので収入も自然増加
    → 紹介時ARPU ¥30K × 15% = ¥4,500/月
    → 12ヶ月後ARPU ¥120K × 15% = ¥18,000/月
    → パートナー収入が4倍に自然成長

■ OEM提供（大手パートナー向け）
  基本料¥30K × 75% = ¥22,500/月/社（パートナー取り分）
  従量課金の75%もパートナーに
  → パートナーの顧客が使えば使うほど、パートナーの収入も増える

■ SIer向け「共存→侵食→置換」インセンティブ
  共存フェーズ: SIerの保守契約に上乗せでシャチョツーを提案
    → SIerは紹介フィー15%を得ながら、保守契約も維持
  侵食フェーズ: シャチョツーがSIerの一部機能を代替
    → SIerの保守契約が縮小しても、紹介フィーで補填
  置換フェーズ: シャチョツーがメインに
    → SIerは紹介フィー（永続）で安定収入を確保
  → SIerにとって「敵ではなく、収入源の転換」として受け入れやすい設計
```

### 7.2 パートナー報酬シミュレーション（新モデル）

```
■ 建設業コンサルが10社紹介した場合（12ヶ月後）

  紹介時: 10社 × 平均ARPU ¥40K × 15% = ¥60,000/月
  6ヶ月後: 10社 × 平均ARPU ¥80K × 15% = ¥120,000/月
  12ヶ月後: 10社 × 平均ARPU ¥120K × 15% = ¥180,000/月

  → 年間収入: 約¥150万（しかも顧客が使い続ける限り永続）

  OEM型の場合:
  12ヶ月後: 10社 × 平均ARPU ¥120K × 75% = ¥900,000/月
  → 年間¥1,080万のリカーリング収入

■ SIerが20社紹介した場合（12ヶ月後）

  紹介フィー: 20社 × 平均ARPU ¥100K × 15% = ¥300,000/月
  → 年間¥360万。保守契約の一部が減っても、新たな収入源として十分
```
