# BPO タスクカタログ — 6業種×全ドメイン（チャネルファースト版）

> **目的**: 各業種×責務ドメインごとに「何を自動化するか」を網羅的に定義する。
> - 開発者: 何のコネクタ・ロジックを作るか判断する根拠
> - 営業: 「御社ではこれが自動化できます」の引き出し
> - 09_導入ジャーニー: Day 31-90 の BPO 提案タスクの元ネタ
>
> **新設計方針（02_プロダクト設計.md 準拠）**:
> - **チャネルファースト**: SaaS API前提 → LINE Bot / メール(IMAP) / ファイルアップロード / 音声(Whisper) / FAX(eFAX) が主要入力。SaaS APIはオプション
> - **Level 0-4 段階的自動化**: 全企業Level 0から開始。タスク単位で昇格管理
> - **従量課金**: BPO実行 ¥200/件（基本料 ¥30,000/月 に加算）
> - **Day 1 タスク**: 初日から使える機能を明示（🟢マーク）
>
> **凡例**:
> - Level: 0=リマインダー / 1=データ収集 / 2=下書き作成 / 3=承認付き実行 / 4=自律実行
> - チャネル: 主要な入出力チャネル（SaaS APIはオプション）
> - ROI: 1回あたりの削減時間
> - 優先: ★★★=Day31-60で提案 / ★★=Day61-90 / ★=90日以降
> - Day 1: 🟢=初日から使える / —=段階的に有効化
> - 業種実現性: 各業種セクション末尾に✅可能/🟡条件付き/❌不可を記載

### 7.2 従量課金との対応

```
■ BPO実行 ¥200/件 の対象
  Level 2以上のタスク実行が従量課金の対象。
  Level 0-1（通知・データ収集のみ）は基本料 ¥30,000/月 に含む。

  例: 製造業（月40件のBPO実行の場合）
    基本料: ¥30,000/月
    BPO実行: 40件 × ¥200 = ¥8,000/月
    ファイル処理: 100件 × ¥50 = ¥5,000/月
    合計: ¥43,000/月
    → 手動での工数換算: 約40h/月 → 時給¥1,075相当

■ Day 1 タスクの課金
  Day 1タスク（🟢）は初月無料トライアルに含む。
  ファイルチェック、文書ドラフト、リマインダーが中心。
  → 初日から価値を体感 → 翌月の有料継続へ。
```

### 7.4 Level別の提案戦略

```
02_プロダクト設計.md の Level 0-4 段階的自動化モデルとの対応:

■ Day 1（導入即日）: Level 0 タスクを全有効化
  → リマインダー、チェックリスト、アラート通知
  → リスクゼロ。「通知するだけ」で価値を体感
  → 🟢Day 1 マークの51タスクが対象

■ Day 7-14（初期設定後）: Level 0→1 の自動昇格
  → ファイルアップロード導線の設定完了後
  → Excel/PDF/CSVの読み取り・分析が可能に

■ Day 31-45（信頼構築期）: Level 1→2 を提案
  → ★★★ タスクから優先的に下書き作成を提案
  → 例: 請求書ドラフト、見積書ドラフト、レポート自動生成
  → 初回ROI計測開始

■ Day 45-60（実行移行期）: Level 2→3 を提案
  → 承認付き実行（下書き→承認→確定→送信）
  → 金額・外部送信を含むタスクは必ず承認チェックポイント

■ Day 90+（信頼蓄積後）: Level 3→4 を検討
  → 6回以上連続無修正承認の実績が必要
  → 自律実行中も結果レポートは必ず送信
  → いつでもLevel 3に戻せる（安全弁）
```
