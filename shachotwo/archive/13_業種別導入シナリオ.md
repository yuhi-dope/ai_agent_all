# シャチョツー 業種別導入事例シナリオ

> **用途**: 提案書・LP・営業トーク・稟議資料の素材として再利用可能。
> 各シナリオは売上100億円前後の中小〜中堅企業を想定。
> 架空だがリアルな企業プロファイルで、経営者が「これは自分だ」と感じる解像度で描写する。

> **価格モデル**: 基本料 ¥30,000/月 + 従量課金（02_プロダクト設計.md Section 15.1 準拠）
> **チャネル**: LINE Bot / メール(IMAP) / ファイルアップロード / 音声(Whisper) / FAX(eFAX)
> **自動化レベル**: Level 0（リマインダー）→ Level 4（自律実行）の段階的進行

### 1.2 Before — 導入前の社長の1日

```
06:30  起床。LINEを開くと未読47件。
       現場A所長「昨日の検査でひび割れ指摘されました。対応方針ご判断ください」
       現場B主任「鉄筋屋が明日来れないと。代わりの業者心当たりありますか」
       → 朝食を食べながら返信。妻に「またスマホばかり」と言われる。

07:30  出社。駐車場で営業部長に捕まる。
       「社長、○○不動産の案件、概算いくらで出しますか？」
       → 先代なら即答できた。自分は図面を見ないと答えられない。
       ▶ 【非効率ポイント】積算の「相場感」が先代の頭の中にしかなかった。
         概算見積りに毎回2-3時間かかる。

08:00  朝礼。安全注意事項を伝える。
       ▶ 【属人化】ヒヤリハット情報が現場ごとに閉じている。
         「去年の現場Cで同じ事象があったはず」と思うが、記録が見つからない。

08:30  積算部に行き、○○不動産案件の概算を一緒に検討。
       「この地域の杭工事、いくらが相場？」→ ベテラン積算担当（62歳）に聞く。
       ▶ 【属人化】彼が辞めたら、この「相場感データベース」が消える。

10:00  協力会社の社長が来社。新規取引の相談。
       「うちの腕は確かです」と言うが、評価基準がない。
       先代は「あの会社は○○の件で△△だったからダメ」と即判断できた。
       → 自分は工事部長に「前に使ったことある？」と聞くしかない。
       ▶ 【判断ミスリスク】協力会社評価が口伝え。
         実績・品質・安全の記録が散在。

10:30  東京支店長から電話。
       「入札案件、参加するかどうか今日中に判断を」
       → 手持ち工事の状況、技術者の空き、利益率の見通し…全部自分の頭の中で計算。
       ▶ 【ボトルネック】社長1人に判断が集中。
         支店長は「社長に聞かないと決められない」状態。

12:00  昼食を取りながら銀行担当者と面談。
       融資条件の確認。「御社の粗利率は業界平均と比べてどうですか？」
       → 答えられない。そもそも「業界平均」のデータを持っていない。
       ▶ 【機会損失】ベンチマークなしでの経営判断。

13:30  現場巡回（2現場）。片道40分。
       現場A: 問題なし。所長と5分話して終了。
       現場B: 工程が2日遅れ。「なぜ遅れた？」→ 資材納入遅延。
       → 「前にも同じ業者で遅延あったよね？」→ 誰も記録を持っていない。
       ▶ 【再発】同じ失敗の繰り返し。ナレッジが蓄積されない。

16:00  帰社。机の上に決裁書類7件。
       外注費承認、出張申請、備品購入…1件1件中身を確認。
       ▶ 【非効率】月に50件超の決裁。うち30件は定型的で、
         判断基準さえ明文化されていれば部長決裁で済む。

17:30  工事会議（週1回）。5現場の所長がそれぞれ報告。
       → 全部口頭。議事録はあるが、後から検索できない。
       「3ヶ月前の現場Dの鉄骨の件、どうなった？」→ 誰も即答できない。

19:00  退社するつもりが、千葉営業所から電話。
       「近隣からクレーム。社長どうしましょう」
       → 先代ならこの手の対応は山ほどやってきた。
         自分は「前例はどうだった？」と聞くが、前例の記録がない。

20:30  帰宅。夕食後、翌日の入札書類を確認。
       ▶ 【恒常的長時間労働】1日14時間。判断の質が落ちる悪循環。
```

**Before まとめ — 1日で失われている価値**

| 問題 | 時間ロス | リスク |
|---|---|---|
| 積算の相場感が属人的 | 概算見積り1件2-3h（月8件=16-24h） | 受注判断の遅延・失注 |
| 協力会社評価が口伝え | 評価に毎回1-2h | 品質・安全事故リスク |
| 社長に判断が集中 | 1日15件の確認依頼 × 平均15分 = 3.75h | 意思決定ボトルネック |
| ヒヤリハット・失敗の未記録 | — | 同じ事故・遅延の再発 |
| 決裁の形骸化 | 月50件 × 20分 = 16.7h | 本質的判断に時間を使えない |

### 1.3 After — 導入6ヶ月後の社長の1日

```
06:30  起床。シャチョツーの朝のダイジェストを確認（LINE通知1件のみ）。
       「本日の注意: 現場Bの鉄筋工事について、過去3年の類似工程で
        平均2日の遅延が発生しています。予備日の確保を推奨します」
       → 【能動提案（Level 1）】過去の全現場データから自動でリスク検出。
         所長に「予備日を確保しておけ」とLINEで1本送るだけ。

07:30  出社。営業部長が来る前に、シャチョツーで概算を確認済み。
       「○○不動産の案件、RC造8F・延床4,200㎡なら概算12.8億円。
        過去の類似案件3件の平均単価から算出。
        杭工事は○○地域の実績から坪単価△△円を推奨」
       → 【Q&A + ナレッジ】先代＋ベテラン積算担当の相場感がデジタル化。
         営業部長に「12.8億ベースで提案しろ」と即答。所要時間2分。
       ▶ 効果: 概算見積り 2-3h → 5分（-97%）

08:00  朝礼。シャチョツーが生成した「今週の安全注意」を読み上げ。
       「現在進行中の5現場のうち、現場Cで足場解体工事があります。
        過去のヒヤリハット34件を分析した結果、
        足場解体時の墜落リスクが最も高い工程です。
        以下の3点を特に注意してください…」
       → 【ナレッジ横断検索】全現場・全年度のヒヤリハットを横断分析。

08:30  協力会社の新規評価。
       シャチョツーに「○○工業の評価は？」と聞く。
       「過去5件の施工実績: 品質スコア4.2/5、納期遵守率92%、
        安全事故0件。同規模の協力会社平均と比較して上位20%です。
        ただし、最近の2件で是正指示が各1件ずつあります」
       → 【デジタルツイン】協力会社の実績が自動スコアリング。
         判断材料がデータで揃う。所要時間3分。

10:00  東京支店長からの入札判断。
       シャチョツーに「この案件に参加すべきか？」と聞く。
       「現在の手持ち工事を分析した結果:
        ・技術者: 1級建施管が2名空き予定（4月以降）
        ・利益率: 類似案件の平均粗利率18.3%。本件の概算粗利率16.1%
        ・リスク: 工期が重なる現場Aとの技術者兼務が必要
        → 参加推奨ですが、技術者配置の条件付きです」
       → 【What-if分析】手持ち工事・人員・利益率を総合判断。
         支店長に「参加OK、ただし△△の条件で」と即回答。

10:30  シャチョツーから能動提案のアラート（LINE通知）。
       「先月の原価率が前年同月比+2.3%上昇しています。
        主因は鋼材価格の上昇（+8.2%/3ヶ月）。
        今後3ヶ月の受注案件の見積りに、鋼材価格上昇分を
        織り込むことを推奨します。影響額は約1,200万円」
       → 【能動提案（Level 1）】社長が気づく前にリスクを検知。
         経営判断に必要な情報がLINE/メールでプッシュ通知。

12:00  銀行担当者との面談。
       「御社の粗利率は同業他社と比べてどうですか？」
       → シャチョツーのベンチマーク画面を見せる。
       「当社の粗利率18.3%は、同規模建設会社の75パーセンタイル。
        上位25%に位置しています。ただし、労務費率が業界平均より
        2.1%高く、ここに改善余地があります」
       → 【ネットワーク知性】匿名ベンチマークで業界内の立ち位置を可視化。
         銀行の評価向上 → 融資条件改善にも寄与。

13:30  現場巡回は週2回に削減（以前は毎日）。
       理由: 各現場の進捗・課題がシャチョツーで一元管理されている。
       → 本日は巡回なし。空いた時間で新規案件の戦略検討。
       ▶ 効果: 現場巡回 週5回 → 週2回。移動時間 月20h削減。

15:00  決裁。シャチョツーが定型決裁を自動分類（Level 2: 下書き作成）。
       「本日の決裁7件のうち、5件は過去の承認パターンに合致。
        部長決裁で問題ありません。
        残り2件は社長判断が必要です:
        ① 新規協力会社との年間契約（1,200万円）
        ② 重機リース契約の変更（+350万円/年）」
       → 【BPO Worker（Level 2→3）】定型決裁の自動振り分け。社長は本質的判断に集中。
       ▶ 効果: 決裁時間 月16.7h → 月5h（-70%）

17:00  工事会議。各所長がシャチョツーに入力済みの進捗を確認しながら議論。
       「3ヶ月前の現場Dの鉄骨の件は？」
       → シャチョツーが即答。「2025年11月の鉄骨建方で発生した
         ボルト締付不良について。原因は…対策は…結果は…」
       → 【ナレッジ検索】全会議・全現場の情報が検索可能。
         会議が「報告会」から「意思決定の場」に変わった。

18:00  退社。帰宅途中、千葉営業所からの近隣クレーム対応も
       シャチョツーが過去の対応パターンをLINEで提示。
       営業所長が自分で対応完了。社長への電話は不要に。
       ▶ 効果: 社長への緊急連絡 1日5-8件 → 1-2件（-75%）

18:30  帰宅。家族と夕食。仕事の電話は鳴らない。
```

### 1.4 導入ジャーニー

