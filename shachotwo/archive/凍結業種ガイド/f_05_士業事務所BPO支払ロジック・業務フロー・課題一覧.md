# 士業事務所BPO — 支払いロジック・業務フロー・課題一覧

> **対象**: 3〜30名規模の士業事務所（税理士・社労士・弁護士事務所が中心）
> **料金帯**: 月額5〜15万円（BPOコア¥250,000の内数 or 単体導入）
> **競合**: マネーフォワード クラウド（月¥3〜6万）、freee（月¥2〜5万）、達人シリーズ（年¥10〜30万）、事務所向けSaaS（MyKomon月¥3万〜、LEGAL FORCE月¥10万〜）
> **特殊性**: 士業事務所は「自らがBPOを提供する側」でもある。自事務所の事務を外部委託するパラドックスをどう解消するかが鍵

## 支払いロジックの大前提

### 士業事務所の特殊性：「BPOを売る側が、BPOを買う」パラドックス

士業事務所は顧問先企業の経理・労務・法務を代行する**プロのBPO事業者**である。にもかかわらず、自事務所の管理業務は「後回し」になり、所長が深夜に経理処理をしている。これは「医者の不養生」「紺屋の白袴」と同じ構造。**「自分の事務所こそ、最もBPOが必要」**と気づかせることが販売の鍵。

### 所長の時間単価 = 10,000〜50,000円/時間

| 士業 | 所長の平均年収 | 時間単価（年2,000h） | 顧問先への請求単価 |
|---|---|---|---|
| 税理士（所長） | 1,000〜2,000万円 | 5,000〜10,000円 | 15,000〜30,000円/時間 |
| 社労士（所長） | 600〜1,500万円 | 3,000〜7,500円 | 10,000〜20,000円/時間 |
| 弁護士（所長） | 1,500〜3,000万円 | 7,500〜15,000円 | 20,000〜50,000円/時間（着手金別） |

所長が事務作業に月20〜40時間を費やすと、**月20〜200万円の機会損失**。「月5〜15万円のBPO」は所長の事務時間のごく一部を解放するだけで元が取れる。

## 1. 期限管理・申告スケジュール

### 支払いロジック（月2〜5万円）

```
■ 直接効果①：期限ミスによる懲戒・賠償の回避
  税理士: 確定申告期限ミス→延滞税+加算税+顧問先からの賠償請求
  社労士: 届出期限ミス→クライアントの社会保険料過誤+賠償請求
  弁護士: 控訴期限ミス→依頼者の権利消滅→確実に賠償請求（数百〜数千万円）

  期限ミス1件の平均賠償額:
  ・税理士: 50〜200万円（延滞税+無申告加算税15%+顧問先賠償）
  ・社労士: 30〜100万円（過誤手続きの是正+顧問先賠償）
  ・弁護士: 100〜5,000万円（権利消滅の損害全額）

■ 直接効果②：所長のスケジュール管理工数削減
  顧問先50社の各種期限を手動管理 = 月10〜15時間
  × 所長時間単価10,000円 = 月10〜15万円の機会損失

■ 間接効果：精神的負荷の軽減
  「期限を忘れていないか」という常時不安 = 計測不能だが離職・体調不良の原因

■ 合計効果:
  賠償リスク回避（年50〜500万円）+ 所長時間削減（年120〜180万円）
  → BPO月3万円（年36万円）で ROI 3.5〜6.7倍（賠償回避除く）
  → 賠償回避を含めると ROI 数十倍
```

**なぜ払うか:** 期限ミスは士業にとって「致命傷」。税理士なら無申告加算税と延滞税で顧問先に実害が出る。弁護士なら控訴期限14日を1日でも過ぎれば依頼者の権利が消滅する。しかし顧問先が50社、100社と増えるほど「所長の頭の中」での管理は破綻する。Excel管理すら追いつかない。月3万円で「二度と期限ミスが起きない」保険が買えるなら、所長は即決する。なぜなら期限ミスの賠償額は、このBPO費用の数十倍〜数百倍だからだ。

### 業務フロー

```
【年間サイクル管理】
  年度初め: 全顧問先の年間申告・届出スケジュールを一覧化
  → 各顧問先の決算月・届出期日・特殊事情をカレンダー登録
  → 担当者への自動アサイン
                    ↓
【月次チェック】
  月初: 当月期限の案件を一覧表示
  → 各案件の進捗確認（未着手/作業中/レビュー待ち/提出済み）
  → 遅延案件のアラート発報（期限7日前→3日前→当日）
                    ↓
【日次管理（弁護士特有）】
  裁判期日カレンダーの毎日確認
  → 控訴期限・上告期限の自動計算（判決日+14日/+30日）
  → 時効の自動管理（債権: 起算日+5年、不法行為: +3年）
  → 消滅時効の完成猶予・更新事由の記録
                    ↓
【繁忙期対応（税理士特有）】
  1月: 年末調整の残務+法定調書+償却資産申告
  → 2月: 個人確定申告の本格着手（件数50〜200件）
  → 3月上旬: ラストスパート（全件の最終チェック+e-Tax送信）
  → 3月15日: 申告期限 ※ここで事務所全員が限界を迎える
                    ↓
【繁忙期対応（社労士特有）】
  6月: 算定基礎届の準備開始（4〜6月の報酬月額集計）
  → 顧問先から給与データ回収（催促が最大のボトルネック）
  → 7月10日: 算定基礎届+労働保険年度更新の同時提出
                    ↓
【期限超過時の対応】
  超過検知 → 即時アラート → 所長判断
  → 税務: 期限後申告の手続き+延滞税計算+顧問先への説明
  → 労務: 届出遅延の理由書作成+遡及手続き
  → 法務: 依頼者への説明+賠償保険への連絡
```

### 注意点

- 士業事務所は**保守的な業界**。「AI」という言葉への抵抗感がある事務所も多い
- 訴求は**「AI事務長」「所長の右腕」「事務所のバックオフィス代行」**が効果的
- **税理士向けから入るのが最善**。顧問先数が多く、定型業務が多く、IT親和性が比較的高い
- 社労士は税理士の次。弁護士は最後（個別性が高く、AI化の恩恵が相対的に小さい）
- **「自分たちがBPOのプロなのに、外部BPOを使うのか」**という心理的抵抗がある
  → 「靴屋の子供が裸足」「医者の不養生」という比喩で打破する
  → 「顧問先に薦められるほどの体験をまず自分で試す」というストーリーも有効

### 士業事務所がBPOツールに求めるセキュリティ要件

| 要件 | 内容 | シャチョツーの対応 |
|---|---|---|
| データの国内保管 | 顧問先データは日本国内のサーバーに保管 | Supabase東京リージョン（GCP asia-northeast1） |
| 通信の暗号化 | TLS 1.2以上 | ◎ 対応済み |

### 士業特有のコンプライアンス考慮事項

1. **依頼者の同意**: BPOツールに顧問先データを入力する際、顧問先の同意が必要か
   → 税理士法基本通達・弁護士職務基本規程に基づき、「業務委託先」としての位置づけで整理
   → 利用規約に「第三者提供には該当しない（委託先への提供）」旨を明記

2. **利益相反チェック**: 弁護士事務所では、同一のBPOツールを利用する他事務所の依頼者情報と利益相反が生じないことの保証
   → テナント分離（RLS）により物理的にデータ隔離。他テナントのデータは一切参照不可

3. **懲戒リスク**: BPOツールの不具合により期限ミスや情報漏洩が発生した場合の責任分界点
   → SLA（Service Level Agreement）で稼働率99.5%を保証
   → 障害時の損害賠償上限を契約で明定

4. **電子帳簿保存法対応**: 税理士が顧問先の電子データをBPOツールに保管する場合の要件
   → タイムスタンプ、検索機能、訂正削除の履歴管理に対応

5. **マイナンバー管理**: 税理士・社労士は顧問先従業員のマイナンバーを扱う
   → マイナンバーは**BPOツールには保管しない**設計（既存の安全管理措置を維持）
   → 将来的にはマイナンバー管理モジュール追加（特定個人情報ガイドライン準拠）

### シャチョツーでの対応

- 全テーブルに`company_id`ベースのRLS適用（テナント完全分離）
- 顧問先データは`knowledge_items`テーブルに暗号化保管
- 全アクセスを`audit_logs`に記録（いつ・誰が・何を操作したか）
- LLMモデルの学習にデータ不使用（商用API DPA）
- 追加オプション: **士業セキュリティパック ¥30,000/月**
  - マイナンバー管理連携
  - 利益相反自動チェック（弁護士向け）
  - 懲戒リスクアラート（期限管理との連動）
  - データ削除証明書の自動発行
  - SOC2 Type II準拠レポートの提供
