# 建設業BPO 8モジュール — 初学者向け完全ガイド

> **このドキュメントの目的**: 建設業を知らない人でも「なぜ建設会社がBPOに月30-50万円払うのか」を理解できるようにする。
> **対象読者**: 建設業未経験のメンバー、営業担当、投資家、パートナー企業
> **前提知識**: 不要

---

## 全体像: 建設会社の1ヶ月

まず、建設会社（従業員30名くらい）の1ヶ月を追ってみる。

```
月初:
  ・新しい現場が始まる → 安全書類を大量に作る（②）
  ・入札案件が来る → 図面を見て「いくらでできるか」計算する（①積算）

月中:
  ・現場が動いている → 写真を撮って整理する（⑥）
  ・材料費・外注費が発生 → 原価を記録する（④）
  ・新しい現場の準備 → 施工計画書を書く（⑤）
  ・下請業者とのやりとり（⑦）

月末:
  ・各現場の工事がどこまで進んだか計算 → 請求書を出す（③出来高・請求）
  ・経理が死にそうになりながらExcelと格闘

年次:
  ・建設業の許可更新（⑧）← これを忘れると会社が営業停止
```

**この全部を、今は「人の手」でやっている。** しかもベテラン社員の頭の中の知識に依存している。
シャチョツーのBPOは、これを丸ごとAIで代行する。

## ① 積算AI — 「工事の値段計算」をAIがやる

### 一言で言うと

**「この工事、うちがやったらいくらかかる？」を計算する作業を、AIが代わりにやる。**

### 具体的な場面

> 月曜の朝、建設会社の社長・田中さんのもとに市役所から図面が届く。
> 「この道路の舗装工事、入札しませんか？」
>
> 田中さんは入札するために「うちがやったらいくらかかるか」を計算しないといけない。
> これが**積算（せきさん）**。
>
> やることはシンプルだけど膨大:
> - 図面を1枚1枚読んで、コンクリートが何m3必要か、鉄筋が何トン必要か...と**数量を拾い出す**
> - それぞれに「1m3あたりいくら」という**単価**をつける
> - 作業員が何人×何日必要かの**歩掛（ぶがかり）**を計算する
>   （歩掛＝「この作業には作業員3人×2日かかる」という標準的な工数のこと）
> - 現場の管理費や会社の経費（**諸経費**）を上乗せする
> - 全部合計して**内訳書**（見積書のようなもの）を作る
>
> ベテランの山田さん（62歳）がExcelと電卓で3日かけてやっている。
> 山田さんが来年定年退職したら、**この会社は入札すらできなくなる**。

### 今のやり方（Before）→ AIのやり方（After）

| | Before（今） | After（AI） |
|---|---|---|
| 数量拾い出し | 図面を目で読んで手計算。1-2日 | 図面PDFをアップロード → AIが自動抽出。数時間 |
| 単価の決定 | ベテランの経験と勘。単価帳を手めくり | 過去の実績DBから候補を自動表示。信頼度つき |
| 歩掛の適用 | 国交省の歩掛表を手で引く | 工種を選ぶと自動適用 |
| 諸経費の計算 | 工事規模に応じた率を手計算 | 自動計算（公共/民間で切り替え） |
| 内訳書の作成 | Excelで手作業。書式を整えるだけで半日 | ワンクリックでExcel/PDF出力 |
| 所要時間 | **3-7日/件** | **半日/件** |

### なぜ月¥10-30万払うのか

```
■ 損失回避の観点
  積算ミス → 安すぎる金額で入札 → 赤字受注
  → 1件間違えると数百万〜数千万円の損失
  → 「絶対にミスできない」のに、できる人がいなくなっていく

■ コスト削減の観点
  現状の外注費: 1件5-20万円 × 月2-3件 = 月¥10-60万
  AI積算の月額: ¥10-30万
  → 外注費だけで元が取れる

■ 人材リスクの観点
  熟練積算士の平均年齢: 55歳以上
  10年後に半数が引退する見込み
  → 「人がいない」問題の解決に対する支払い
```

### 業務フロー

```
図面PDF/Excel/画像を受領
  → [Document AI + LLM] 数量拾い出し（工種/種別/細別/規格/数量/単位）
    → [自社実績DB + 公共労務単価 + LLM推定] 単価候補付与（信頼度付き）
      → [国交省基準 or 会社設定] 歩掛適用 + 諸経費計算
        → 内訳書自動生成（Excel/PDF）
          → ユーザー確認・修正 → 修正内容が単価DBにフィードバック（使うほど賢くなる）

Level 1: 拾い出し70%削減（3日→1日）、単価は手動入力
Level 2: 積算全体80%削減（3日→半日）、単価はAI推定+人間確認
Level 3: 積算士不要レベル（複数社の匿名データで市場単価推定）
```

### 用語解説

| 用語 | 意味 |
|---|---|
| 積算（せきさん） | 工事にかかる費用を、図面から計算すること。「見積もり」のもっと詳細版 |
| 数量拾い出し | 図面から「コンクリート何m3」「鉄筋何トン」などを読み取る作業 |
| 単価 | 「コンクリート1m3あたり○○円」のような単位あたりの価格 |
| 歩掛（ぶがかり） | 「この作業には作業員○人×○日かかる」という標準工数。国交省が基準値を公表 |
| 諸経費 | 直接の工事費以外にかかるお金の総称。現場事務所の費用、安全対策費、会社の利益など |
| 内訳書 | 工事費の内訳を項目ごとに記載した書類。入札や見積提出に必須 |
| 設計単価 | 発注者（国や自治体）が設定した標準的な単価 |
| 市場単価 | 実際の取引で使われている単価。地域や時期で変動する |
| 入札 | 公共工事の受注方法。「うちはこの金額でやれます」と提示して、条件の良い会社が選ばれる |

## ② 安全書類自動生成 — 「現場に人を入れるための書類」をAIが作る

### 一言で言うと

**建設現場で働くには大量の書類が必要。その書類作りを、AIがほぼ全自動でやる。**

### 具体的な場面

> 田中建設に新しい現場が決まった。来週から作業員10人を送り込む。
>
> その前に、元請（発注者から直接工事を請けた大手ゼネコン）に
> **「うちの会社のこの人たちが現場に入ります」**という書類を提出しないといけない。
>
> 事務の鈴木さんが作る書類は:
> - **作業員名簿**: 10人全員の氏名・生年月日・血液型・住所・保険番号・資格・健康診断結果...
> - **有資格者一覧**: 誰がどの資格を持っているか（クレーン免許、玉掛け技能講習 等）
> - **下請負業者編成表**: どの会社がどの部分を担当するかの組織図
> - **安全衛生計画書**: この現場でどんな危険があって、どう対策するか
>
> しかも元請によって**フォーマットが微妙に違う**。
> A社は「全建統一様式」（業界の標準フォーマット）、B社は独自様式、
> C社は**グリーンサイト**（安全書類をオンラインで管理するクラウドサービス）を使っている。
>
> 1現場分で20-40時間。田中建設は常時5現場を抱えているので、
> 鈴木さんは**毎月ほぼ安全書類だけで手一杯**。

### 今のやり方（Before）→ AIのやり方（After）

| | Before（今） | After（AI） |
|---|---|---|
| 作業員名簿 | Excelに1人ずつ手入力。現場ごとにコピペ+修正 | 作業員マスタから対象者を選ぶだけで自動生成 |
| 資格管理 | Excelの一覧表 or ノート。期限切れに気づかない | 有効期限90日前に自動アラート |
| 作業員の入退場 | 人が替わるたびに名簿を手修正 | 入退場を登録すると名簿が自動更新 |
| 元請への提出 | 紙 or Excel をメール送付 | グリーンサイトCSV出力 or PDF自動生成 |
| 所要時間 | **1現場 初回20-40h + 更新 月5-10h** | **初回1-2h + 更新はほぼ自動** |

### なぜ月¥3-5万払うのか

```
■ 人件費削減の観点
  全現場合計で月50-100時間 = 事務員1人分の工数
  事務員の人件費: 月¥15-25万
  AI月額: ¥3-5万
  → 人件費の1/5以下で同じことができる

■ リスク回避の観点
  ・資格の有効期限が切れた作業員を現場に入れてしまうと法令違反
  ・安全書類の不備で元請から現場入場を拒否される → 工事が止まる
  ・最悪の場合、労災事故時に「書類不備だった」と責任を問われる

■ 属人化解消の観点
  ・鈴木さんが辞めたら書類が作れる人がいない
  ・元請ごとのフォーマットの違いを把握しているのが鈴木さんだけ
```

### 業務フロー

```
新規現場開始（着工7日前に自動検知）
  → 作業員マスタから対象者選定（CCUS連携で自動取得）
    → 全建統一様式の書類を自動生成
      - 第5号: 作業員名簿（★最大の工数削減）
      - 第5号別紙: 社会保険加入状況
      - 第9号: 有資格者一覧表
      - 第1号-甲: 下請負業者編成表
      - 第6号: 工事安全衛生計画書（AI: 危険要因分析→安全対策提案）
    → 資格有効期限アラート（90日前通知）
    → 作業員の入退場 → 名簿自動更新
    → グリーンサイトCSV出力 or API連携（元請の要求に応じて）

完全自動: 名簿・社保・有資格者・編成表（マスタデータから100%自動）
半自動:   施工体制台帳・安全衛生計画書（テンプレ+AI補完→人間確認）
```

### 用語解説

| 用語 | 意味 |
|---|---|
| 元請（もとうけ） | 発注者から直接工事を請け負った会社。大手ゼネコンが多い |
| 下請（したうけ） | 元請から工事の一部を請け負った会社。中小建設会社が多い |
| 全建統一様式 | 全国建設業協会が定めた安全書類の標準フォーマット。第1号〜第13号まである |
| 作業員名簿（第5号） | 現場に入る全作業員の個人情報・資格・保険加入状況をまとめた書類。最も作成工数が大きい |
| グリーンサイト | MCデータプラス社が運営する安全書類のクラウド管理サービス。大手ゼネコンが導入しており、下請に利用を求めるケースが増えている |
| CCUS（建設キャリアアップシステム） | 作業員の資格・就業履歴を業界横断で蓄積するICカードシステム。国交省が推進 |
| 施工体制台帳 | 元請が作成する「どの下請がどの工事を担当するか」の一覧表 |
| 玉掛け（たまがけ） | クレーンで荷物を吊る際にワイヤーをかける技能。国家資格が必要 |

## ③ 出来高・請求書自動生成 — 「今月どこまで進んだか」を計算して請求する

### 一言で言うと

**工事は数ヶ月〜数年かかる。「今月はここまで進みました、なので○○万円ください」という計算と請求を自動化する。**

### 具体的な場面

> 田中建設は、A社から5,000万円の道路工事を請けている。工期は6ヶ月。
>
> 工事は6ヶ月で完成するが、完成するまで1円ももらえないわけではない。
> **毎月「今月はここまで進みました」と報告して、進んだ分だけ請求する。**
> これが**出来高（できだか）払い**。
>
> 例えば:
> - 契約内訳: 舗装工 2,000万円 / 排水工 1,500万円 / 土工 1,500万円
> - 今月の進捗: 舗装工 30% / 排水工 50% / 土工 20%
> - 今月の出来高: 600万 + 750万 + 300万 = 1,650万円
> - 先月までの累計: 1,200万円
> - **今月の請求額: 1,650万 - 1,200万 = 450万円（＋消費税）**
>
> この計算を、経理の佐藤さんが**全工事分（常時10-15件）をExcelで毎月末にやっている。**
> 間違えると元請からの信頼を失う。少なく請求すれば損をする。多く請求すれば信用問題になる。

### 今のやり方（Before）→ AIのやり方（After）

| | Before（今） | After（AI） |
|---|---|---|
| 出来高の計算 | 現場監督がExcelに進捗率を入力。経理が集計 | 進捗率を入力するだけで自動計算。前月との差分も自動 |
| 異常チェック | 目視で「先月と比べておかしくないか」確認 | 前月比で大幅変動があれば自動アラート |
| 請求書作成 | Excelテンプレートにコピペ → PDF化 | ワンクリックで請求書PDF生成（番号も自動採番） |
| 入金管理 | 通帳を見て手動で消込。見落としあり | 未入金一覧・支払期日超過を自動アラート |
| 所要時間 | **月末に全工事で20-30h** | **各工事10分、合計2-3h** |

### なぜ月¥3-5万払うのか

```
■ ミス防止の観点
  ・請求漏れ: 1件忘れるだけで数十万〜数百万円の入金が遅れる
  ・過請求: 元請からの信頼失墜 → 次の仕事がもらえなくなる
  ・計算ミス: 累計出来高と前月累計の差分間違い → 修正の手間と信用低下

■ 資金繰りの観点
  ・建設業は「先にお金を使って、後から入金される」ビジネス
  ・入金管理がずさんだと資金ショート（黒字倒産）のリスク
  ・支払期日超過の見逃し → 年間で数十万〜数百万の機会損失

■ 法令対応の観点
  ・電子帳簿保存法（2024年〜）: 電子取引データの保存が義務化
  ・インボイス制度: 適格請求書の要件を満たす必要がある
  → 手作業では対応漏れのリスクが高い
```

### 業務フロー

```

  → 月次出来高入力（各項目の進捗率を入力）
    → 累計出来高 − 前月累計 = 当月請求額を自動計算

            → 入金管理（未入金一覧・支払期日超過アラート・資金繰り予測）
```

### 用語解説

| 用語 | 意味 |

| 出来高（できだか） | 工事がどこまで進んだかを金額で表したもの。「進捗率 × 契約金額」で計算する |
| 出来高払い | 工事の進み具合に応じて、毎月分割で支払いを受ける方式。建設業の標準的な支払方法 |

## ④ 原価管理レポート — 「この工事、儲かってる？赤字？」をリアルタイムで見える化

> （※ 実行予算 ＝ 「お客さんに出す見積書」ではなく「自社内部のコスト計画」のこと）
>
> ところが工事が進むにつれて:
> - 材料費が予定より200万円高くなった（鉄鋼価格の高騰）
> - 外注の下請業者が予定より時間がかかり、追加費用が150万円
> - 雨で工期が延びて、仮設費用が100万円増加
>
> **合計450万円のコストオーバー。利益500万の見込みが、50万円しか残らない。**
>
> でも田中さんがこれに気づくのは、**工事が全部終わって経理が集計した後**。

### 今のやり方（Before）→ AIのやり方（After）

| | Before（今） | After（AI） |
|---|---|---|
| 予算管理 | Excelに実行予算を入力。あとは放置 | 実行予算を登録すると、実績を自動で突き合わせ |

```

  ・「竣工してから赤字発覚」→ 数百万円の損失が確定してからでは打つ手がない

```

```

          → 赤字転落リスクがある場合 → プロアクティブ提案と連動
            → 全社サマリー（工事別利益率ランキング・赤字工事一覧）
```

### 用語解説

| 用語 | 意味 |
|---|---|

```
  → 更新必要書類チェックリスト自動生成
    → 経審申請書の下書き自動生成（財務データから）
      → 技術者配置シミュレーション（資格要件の充足チェック）
        → 経管・専技の資格要件充足状況チェック
          → 経審スコア試算（改善ポイント提案つき）
```

### 用語解説

| 用語 | 意味 |
|---|---|
| 建設業許可 | 建設工事を請け負うために必要な免許。都道府県知事許可（1県のみ）と国土交通大臣許可（2県以上）がある。500万円以上の工事には必須 |
| 経審（経営事項審査） | 公共工事を受注するために必要な審査。会社の経営状況・技術力・社会性などを点数化する。略称「けいしん」 |
| P点（総合評定値） | 経審の最終スコア。この点数でランク付けされ、入札できる工事の規模が決まる |
| 経管（経営業務管理責任者） | 建設業許可の要件の一つ。建設業の経営経験が5年以上ある役員が必要 |
| 専技（専任技術者） | 建設業許可の要件の一つ。一定の資格（1級土木施工管理技士など）を持つ技術者を営業所ごとに配置する義務 |
| i-Construction | 国交省が推進する建設業のICT化政策。2040年までに建設現場の生産性を1.5倍にすることが目標 |

## 補足: 建設業の構造を理解するための基礎知識

### 建設業の商流（お金と仕事の流れ）

```
発注者（国・自治体・民間企業）
  │
  │ 工事を発注（入札 or 随意契約）
  │
  ▼
元請（ゼネコン・総合建設会社）  ← 発注者から直接受注する会社
  │
  │ 工事の一部を外注
  │
  ▼
1次下請（専門工事会社）  ← 元請から仕事をもらう会社。田中建設はここ
  │
  │ さらに一部を外注
  │
  ▼
2次下請（職人集団・一人親方）

※ シャチョツーのメインターゲットは「元請の中小」と「1次下請」
```

### なぜ建設業は書類が多いのか

```
理由1: 人の命に関わるから
  → 安全書類・施工計画書・資格確認が法律で義務付けられている

理由2: 税金（公共工事）が使われるから
  → 「ちゃんと工事しました」の証拠（写真・出来形・品質記録）が必要

理由3: 多重下請構造だから
  → 元請→1次→2次→...と関わる会社が多い
  → 全社の情報を書類にまとめる必要がある

理由4: 建設業法で定められているから
  → 許可制度、経審、施工体制台帳の作成義務 等
```
