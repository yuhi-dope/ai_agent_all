# 製造業BPO完全ガイド — 町工場の「困った」をAIで解決する

> **このドキュメントの目的**: 製造業を知らない人が「なぜ工場はこれにお金を払うのか」を理解し、8つのBPOモジュールの全体像を把握できるようにする
> **対象読者**: 製造業未経験の営業・PM・エンジニア・投資家
> **技術仕様**: `e_02_製造業BPO設計.md` を参照

## 1. まず知っておくべきこと：「町工場」ってどんな場所？

シャチョツーのターゲットは、従業員10〜300名の中小製造業。日本には約5.2万社ある。

具体的にはこんな会社：

```
・金属加工屋（鉄やアルミを削って部品を作る）
・プレス工場（金属板を型で押して形を作る）
・食品工場（パンやお菓子を作る製造ライン）
・樹脂成形屋（プラスチック部品を作る）
・電子部品メーカー（基板やセンサーを作る）
```

こういう会社には共通の特徴がある：

```
① 社長 or ベテランの頭の中に仕事のやり方が入っている
② 紙・Excel・ホワイトボードで仕事が回っている
③ 人を雇いたいけど、若い人が来ない（平均年齢50代〜）
④ ITに詳しい人がいない（"パソコンに強い人"が兼任SEみたいな状態）
⑤ でも仕事はちゃんと来る。むしろ忙しすぎて回らない
```

## 2. 町工場の社長の1日（Before）

これを読むと、8つのBPOモジュールが「なぜ必要か」がわかる。

```
AM 7:00  出社。工場長と今日の生産予定を確認（ホワイトボード）
AM 7:30  急ぎの見積依頼がメールで3件。図面をプリントして見積担当に渡す
         → 見積担当は今日も残業確定。1件に2時間はかかる【①見積AI】
AM 8:00  昨日の夜勤で設備が止まったらしい。修理業者を呼ぶ
         → いつも突然壊れる。交換時期がわからない【⑥設備保全】
AM 9:00  得意先から電話「例の部品、来週に前倒しできない？」
         → 工場長「うーん、MC（加工機）が来週いっぱいなんだよな…」
         → この判断が工場長の頭の中にしかない【②生産計画AI】
AM 10:00 品質クレームの報告。先月納品した部品の寸法がズレていた
         → 検査記録は紙。原因を遡るのに半日かかる【③品質管理】
PM 1:00  購買担当「すみません、材料の発注忘れてました。ラインが明日止まります」
         → 何がいつ足りなくなるか、誰も把握していない【④在庫・⑦仕入】
PM 2:00  新しく入ったベトナム人実習生に作業を教える
         → 手順書が5年前のまま。日本語しかない【⑤SOP管理】
PM 3:00  来月のISO審査の準備。書類が全然揃っていない
         → 品質管理責任者が1人で泣きながら準備中【⑧ISO文書管理】
PM 5:00  やっと見積が1件できた。あと2件は明日。
         → 顧客は3社に見積を出している。遅い会社は候補から落ちる【①見積AI】
PM 7:00  退社。今日も何も「経営」できなかった。
```

**この社長が求めているのは、「自分がいなくても会社が回る仕組み」。**
それがシャチョツーの製造業BPO。

## 4. 各モジュール詳細（What → Why Pay → How）

### ① 見積AI — 「図面を見せたら値段が出る」

#### What: これは何？

お客さんが「こんな部品を作ってほしい」と図面（設計図）を送ってくる。
工場は、その部品を作るのにいくらかかるかを計算して、値段を回答する。これが「見積」。

部品の値段を出すには、こういう計算が必要：

```
1. 図面を見て、どんな加工が必要かを判断する
   例：「まず鉄の棒を切って、旋盤で丸く削って、穴を開けて、表面を磨いて…」

2. 各工程にどれくらい時間がかかるかを見積もる
   例：「旋盤で削るのに1個15分。でも最初のセット（段取り）に30分かかる」

3. 設備の「時間単価」を掛け算する
   ※ 設備の時間単価＝その機械を1時間動かすのにかかるコスト
   （電気代+減価償却+人件費+場所代を、1時間あたりに割ったもの）
   例：「CNC旋盤は1時間6,000円。15分使うから1,500円」

4. 材料費・外注費・管理費を足す
   例：「鉄の棒が500円、メッキ外注が300円、管理費15%で…合計8,500円」
```

この計算ができる人は、**20年以上の経験がある熟練者**。
多くの工場で、この人が1人しかいない。そして60歳を超えている。

#### Why Pay: なぜ月¥10-30万を払うのか

```
■ 現状のコスト
  ・見積担当者の年収: 500〜700万円（月42〜58万円）
  ・この人が1日にできる見積: 3〜5件
  ・月に来る引合い: 100件以上
  → 全部に回答しきれない。回答できなかった分 = 逸した売上

■ AIを入れると
  ・見積担当の作業が50〜80%減る
  ・回答スピードが2時間→10分になる
  → 今まで断っていた引合いにも回答できる = 売上が増える

■ 社長の計算
  「月30万払って、受注が月1件増えたら？」
  → 1件の受注単価が50万〜500万の世界
  → 月30万は"元が取れる"どころか大幅な黒字

■ さらに怖いシナリオ
  ・見積できる人は60代のベテラン1人だけ
  ・この人が辞めたら？ → 会社が受注できなくなる
  → 「人を1人雇うより安い」が最強の支払い理由
```

#### How: 業務フロー（今 → AI）

```
STEP 1: 引合い受領
  ・顧客から図面（PDF/画像）+ 仕様がメールで届く
  ・仕様 = 材質（鉄？アルミ？ステンレス？）、数量、表面処理、納期
  └→ 今: 図面をプリントして見積担当の机に置く

STEP 2: 図面解析
  ・図面を見て、部品の形状・寸法・精度を把握する
  ・「ここに穴が3つ」「この面は精度が厳しい」など
  └→ AI: Document AI（Google）で図面をスキャン → 形状・寸法を自動抽出

STEP 3: 工程推定
  ・どんな順番で加工するかを決める
  ・例: 材料切断 → 旋盤（丸く削る）→ MC（穴・溝を加工）→ 研磨 → 表面処理 → 検査
  └→ AI: LLMが形状・材質・精度から最適な工程を推定

STEP 4: 工数算出
  ・各工程で「どれくらい時間がかかるか」を見積もる
  ・段取り時間（準備）+ サイクルタイム（1個あたりの加工時間）× 数量
  └→ AI: 過去の類似部品の実績データから推定。初期は精度60〜70%、学習で80%+に

STEP 5: コスト計算
  ・加工費 = 合計時間 ÷ 60分 × チャージレート（設備の時間単価）
  ・見積金額 = 材料費 + 加工費 + 表面処理費 + 外注費 + 管理費
  └→ AI: 自社のチャージレートマスタ × AI算出工数で自動計算

STEP 6: 見積書出力
  ・見積書をPDF/Excelで生成して顧客に送付
  └→ AI: ボタン1つで整形された見積書を出力

STEP 7: 受注率分析（AIならではの付加価値）
  ・「この価格帯なら受注確率70%」「5%下げれば85%」
  └→ 過去の受注/失注データから価格と受注率の関係を分析
```

#### 課題テーブル

| 項目 | 内容 |
|---|---|
| **何が起きているか** | 顧客から図面が来たら、ベテラン見積担当が1件30分〜数時間かけて手作業で見積書を作っている |
| **なぜ困るのか** | 月100件以上の引合いに対して、回答しきれない。顧客は通常3社に見積を依頼し、一番早く回答した会社に発注しがち |
| **誰が困っているか** | 社長（売上機会の逸失）、見積担当（残業続き）、営業（回答が遅くて顧客に催促される） |
| **起きている損失** | ①回答遅れ→失注（1件50万〜500万の受注を逃す）②見積ミス→赤字受注（1件で数百万の損失）③ベテラン退職→見積ができなくなる |
| **今どうやっているか** | Excelに計算式を組んで、図面を見ながら手入力。過去の見積書をコピーして修正することも多い |
| **AIで解決できる範囲** | 図面から工程推定→工数算出→見積書出力までを自動化。精度は初期60〜70%→学習で80%以上。人間の最終確認は必要 |

### ② 生産計画AI — 「工場長の頭の中を見える化する」

#### What: これは何？

工場にはたくさんの注文が同時に来る。
「どの注文を、どの機械で、いつ作るか」を決めるのが生産計画。

```
・機械Aは来週水曜まで別の注文で埋まっている
・機械Bは木曜にメンテナンスで使えない
・急ぎの注文が今日入った。でもどこに入れればいい？
・残業させる？外注に出す？納期を交渉する？
```

この判断を、多くの工場では**ベテラン工場長が頭の中だけで管理**している。
ホワイトボードに書いてあることもあるが、それは全体の一部。
工場長が風邪で休んだら、誰も判断できない。

#### Why Pay: なぜ月¥5-10万を払うのか

```
■ 現状のコスト
  ・納期遅延が起きると → 顧客からペナルティ or 取引停止
  ・1回の納期遅延の損失: 数十万〜数百万円
  ・年に数回は起きている会社が多い

■ AIを入れると
  ・「この注文を受けたら他の納期に影響するか？」が即座にわかる
  ・工場長が休んでも誰でも計画を確認できる

■ 社長の計算
  「月10万で、年2回の納期遅延を防げたら？」
  → 1回の損失が50万なら、年100万の損失回避
  → 月10万 × 12ヶ月 = 120万 vs 損失回避100万 → ほぼトントン
  → でも「信用を失わない」価値はお金に換算できない
  → 大口顧客を1社失うと年間売上の10〜30%が消える
```

#### How: 業務フロー（今 → AI）

```
STEP 1: 受注登録
  ・受注内容を登録（製品名・数量・納期・優先度）
  ・見積AIの工程データと連動
  └→ 今: Excelに手入力。見積書を見ながら工程を書き写す

STEP 2: 工程展開
  ・その製品を作るのに必要な工程と、使う機械を確定する
  └→ AI: 見積AIで推定済みの工程データを自動引用

STEP 3: 山積み（= 注文を納期順に機械に割り当てること）
  ・全ての受注を納期順に並べて、各機械に「いつ・何を作るか」を割り当てる
  └→ AI: 自動で最適な割当を計算

STEP 4: 山崩し（= 機械の能力を超えた分を調整すること）
  ・山積みした結果「機械Aが来週150%の稼働になる」等の過負荷を調整
  ・対策: 前倒しする / 外注に出す / 残業で対応 / 納期を交渉する
  └→ AI: 過負荷を自動検出し、対策案を提示

STEP 5: ガントチャート生成
  ・機械 × 日付 の表で、どの注文がどこに入っているかを可視化
  └→ 今: 工場長の頭の中 or ホワイトボード

STEP 6: シミュレーション
  ・「この新しい注文を受けたら、他の納期に影響するか？」を即座に回答
  └→ 今: 工場長に聞くしかない。工場長が不在なら判断できない
```

#### 課題テーブル

| 項目 | 内容 |
|---|---|
| **何が起きているか** | 「どの注文を、どの機械で、いつ作るか」がベテラン工場長の頭の中にしかない |
| **なぜ困るのか** | 急な注文が入った時、既存の納期に影響するかどうかの判断が工場長にしかできない。工場長が休むと工場の判断が止まる |
| **誰が困っているか** | 社長（受注判断ができない）、工場長（休めない）、営業（納期回答ができない） |
| **起きている損失** | ①納期遅延→ペナルティ or 取引停止（1回で数十万〜数百万）②設備の遊び時間が見えず投資判断が遅れる③工場長の退職=計画崩壊 |
| **今どうやっているか** | Excelとホワイトボード。工場長が毎朝30分かけて「今日はこれを作れ」と指示 |
| **AIで解決できる範囲** | 全注文の工程と機械の空き状況をガントチャートで可視化。新規注文のシミュレーション。設備稼働率の自動計算 |

### ③ 品質管理 — 「不良が出る前に気づく」

#### What: これは何？

作った部品がちゃんと図面通りの寸法になっているか、傷がないかを検査すること。

```
・検査データは紙に手書き → Excelに転記 → 誰も分析しない
・「最近ちょっと寸法がズレ気味だな」に気づけるのはベテランだけ
・不良が出てから原因を調べるが、記録がバラバラで追えない
・「機械のせい？材料のせい？作業者のせい？」の判断に時間がかかる
```

#### Why Pay: なぜ月¥3-5万を払うのか

```
■ 現状のコスト
  ・不良品が出ると → 材料代+加工時間が丸損 + 手直し費用
  ・クレームが出ると → 対応工数 + 信用低下 + 最悪は取引停止
  ・不良率1%改善 = 年間数百万のコスト削減
    （年間売上3億の工場で不良率2%→1% = 年300万の改善）

■ AIを入れると
  ・不良が出る「前に」異常を検知（予兆管理）
  ・月次レポートが自動で出る（今は手作業で3日かかっている）

■ 社長の計算
  ・効果は確実にあるが、「いつ効果が出るか」が読みにくい
  ・不良が減ったのがAIのおかげか判断しにくい
  → 月3〜5万なら「まあ試してみるか」のライン
```

#### How: 業務フロー（今 → AI）

```
STEP 1: 検査記録登録
  ・加工した部品の寸法・外観・各測定値を記録する
  └→ 今: 紙の検査シートに手書き → 後でExcelに転記

STEP 2: 統計分析（SPC = 検査データをグラフ化して異常を見つける手法）
  ・管理図: 「上限・下限」の線を引いて、データがその中に収まっているか見る
  ・工程能力指数: 「この工程はどれくらい安定して規格内に収まるか」を数値化
  └→ 今: できる人がいない。やっていない会社がほとんど

STEP 3: 不良分析
  ・不良が出た時「何が原因か」を分類する
  ・4つの分類: 機械のせい / 材料のせい / 作業方法のせい / 人のせい
  ・多い順に並べて「まずここを直すべき」を特定（パレート分析）
  └→ 今: ベテランの経験と勘で判断

STEP 4: 不良予測
  ・データのトレンドから「もうすぐ不良が出そう」を検知
  ・例: 寸法が少しずつ上限に近づいている → 明日には規格外になりそう
  └→ 今: 気づいた時にはもう不良品が出ている

STEP 5: 月次レポート生成
  ・不良率・主要原因・改善トレンドをまとめたレポートを自動生成
  └→ 今: 品質管理担当が月末に3日かけて手作業で作成
```

#### 課題テーブル

| 項目 | 内容 |
|---|---|
| **何が起きているか** | 検査データは取っているが、分析できていない。不良が出てから原因を調べるが、記録がバラバラで追跡に時間がかかる |
| **なぜ困るのか** | 不良品を顧客に納品→クレーム→対応工数→最悪は取引停止。不良の「予兆」に気づけないので、いつも後手に回る |
| **誰が困っているか** | 品質管理担当（データ整理に追われる）、工場長（原因究明を求められる）、社長（クレーム対応） |
| **起きている損失** | ①不良コスト=材料費+加工時間+手直し費（年間売上の1〜5%）②クレーム対応工数（1件あたり数日〜数週間）③最悪は取引停止 |
| **今どうやっているか** | 紙の検査シートに手書き→月末にExcelに転記→グラフを手作業で作成。統計分析はできる人がいない |
| **AIで解決できる範囲** | 検査データの自動集計、管理図の自動生成、異常の予兆検知、不良原因の自動分類、月次レポート自動生成 |

### ④ 在庫最適化 — 「足りない・多すぎるをなくす」

#### What: これは何？

部品を作るには材料がいる。材料の在庫が切れたらラインが止まる。
かといって大量に買い込むと、お金が材料に変わって手元に現金がなくなる。

```
中小製造業の在庫管理の実態：
・倉庫を見に行って「あ、そろそろ少ないな」で発注する
・発注してから届くまで2週間かかる材料を、残り3日分で気づく
・逆に「念のため」で多めに買って、1年使わない材料が倉庫を占拠
・棚卸（在庫の実数を数える作業）は年に1〜2回だけ
```

#### Why Pay: なぜ月¥3-5万を払うのか

```
■ 現状のコスト
  ・過剰在庫: お金が材料に変わって手元のキャッシュが減る
    中小製造業にとってキャッシュフローは命
    例：500万円分の材料が倉庫で1年寝ている = 500万の現金がない
  ・欠品: ラインが止まる → その日の生産が全部止まる

■ AIを入れると
  ・「3日後にこの材料が切れます」を自動通知

■ 社長の計算
  ・ライン停止を年2回防げたら → 損失回避で数十万

```

### ⑤ SOP管理 — 「作業手順書をAIで作って、翻訳もする」

#### What: これは何？

SOP = Standard Operating Procedure（標準作業手順書）。
「この作業はこの順番でこうやる」を文書にしたもの。料理のレシピのようなもの。

```
・手順書を作る時間がない（書くより作業した方が早い、と後回しにされる）
・作っても更新されない（5年前の手順書がそのまま）

#### Why Pay: なぜ月¥2-3万を払うのか

■ AIを入れると
  ・手順書のドラフトを自動生成（作成工数 1日→1時間）

■ 社長の計算
  ・「あったら便利だけど、なくても死なない」

  → パック（セット）の一部としてなら価値を感じる

#### How: 業務フロー（今 → AI）

```

  └→ 今: ファイル名に「_v2」「_最新」「_最新2」が増殖

STEP 5: 定期見直しリマインド

STEP 6: 多言語対応

### ⑥ 設備保全 — 「壊れる前に直す」

#### What: これは何？

工場の機械は使っていると摩耗して壊れる。
壊れてから直す（事後保全）より、壊れる前に部品を交換する（予防保全）方が安い。

```
・「前回いつベルトを交換したっけ？」の記録が紙でバラバラ
・突然壊れると、修理に3日+部品取り寄せに1週間 → ラインが止まる
・止まった間の損失 = 1日あたり数十万円
・保全にいくら使っているか、誰も正確に把握していない
```

#### Why Pay: なぜ月¥2-3万を払うのか

```
■ 現状のコスト
  ・突発故障1回の損失:
    修理費: 10〜100万円
    ライン停止: 1日数十万円 × 修理日数
    部品取り寄せ: 1〜2週間待ちもありうる
  ・年間の保全費: 売上の2〜5%が目安（3億売上なら600〜1,500万）

■ AIを入れると
  ・「この機械、前回の部品交換から○○時間経過。そろそろ交換時期」を通知
  ・保全コストを設備ごとに可視化（「この機械、修理代だけで年100万」）

■ 社長の計算
  ・突発故障を年1回防げれば元が取れる
  ・ただし予測精度が完璧ではないので、過信はできない
  → 月2〜3万なら「保険」として許容できるライン
```

#### How: 業務フロー（今 → AI）

```
STEP 1: 設備マスタ登録
  ・全設備の情報を登録（名前・種類・購入日・1日の稼働可能時間）
  └→ 今: 一覧表すらない会社も多い

STEP 2: 保全カレンダー生成
  ・日次点検 / 週次点検 / 月次点検 / 年次オーバーホール のスケジュールを自動作成
  └→ 今: 壁のカレンダーに手書き

STEP 3: 点検チェックリスト管理
  ・設備ごとの点検項目を管理
  └→ 今: 紙のチェックシートを印刷して使い回し

STEP 4: 保全記録登録
  ・交換した部品・停止時間・費用を記録
  └→ 今: ノートに手書き。後から振り返れない

STEP 5: 故障予兆検知
  ・過去の故障間隔のトレンドから「そろそろ危ない」を予測
  ・例:「この設備、前回故障から1,800時間。平均故障間隔は2,000時間。注意」
  └→ 今: 壊れてから「やっぱりか」と言う

STEP 6: 保全コストレポート
  ・設備ごとに年間の保全費用を集計
  ・「この機械、修理代だけで年100万。買い替えた方が安くない？」
  └→ 今: 感覚的に「金かかってるな」と思うだけ
```

#### 課題テーブル

| 項目 | 内容 |
|---|---|
| **何が起きているか** | 機械が突然壊れてラインが止まる。保全記録がバラバラで、交換時期が勘頼み |
| **なぜ困るのか** | 突発故障→修理業者を呼ぶ→部品取り寄せに1〜2週間→その間ラインが止まる |
| **誰が困っているか** | 工場長（ライン停止の責任）、保全担当（記録管理の負担）、社長（予算管理） |
| **起きている損失** | ①突発故障1回=修理費10〜100万+ライン停止損失（日数十万×日数）②保全費の総額を把握できず投資判断が遅れる |
| **今どうやっているか** | 紙の点検表。カレンダーに手書きの予定。保全記録はノートに手書き。「前回いつ交換したか」が分からないことも |
| **AIで解決できる範囲** | 保全カレンダー自動生成、故障間隔のトレンド分析による予兆検知、保全コストの設備別集計 |

### ⑦ 仕入管理 — 「何をいくつ買えばいいか自動で出す」

#### What: これは何？

部品を作るには材料や部品が必要。それを仕入先（サプライヤー）から買う。
「何を・いくつ・いつまでに・どこから買うか」を管理すること。

```
・1つの製品を作るのに必要な材料リスト（BOM = 部品表）がExcelに散在
・注文が入った → 必要な材料を手計算 → 在庫を目視確認 → 足りなければ発注
・この作業を購買担当が1人でやっている
・発注忘れ = ラインが止まる = 納期遅延 = 顧客からの信用失墜
```

#### Why Pay: なぜ月¥2-3万を払うのか

```
■ 現状のコスト
  ・発注忘れ → ラインが止まる（在庫の問題と連動）
  ・手計算ミス → 材料が多すぎ or 足りない
  ・仕入先の評価をしていない → もっと安い・早い仕入先があるかも

■ AIを入れると
  ・注文が入ったら必要な材料を自動で計算
  ・足りない分の発注書を自動生成

■ 社長の計算
  ・便利だけど、kintone等の既存ツールでカバーしている会社も多い
  → 単体では売りにくいが、在庫+仕入のセットなら月5〜8万の価値
```

#### How: 業務フロー（今 → AI）

```
STEP 1: BOM（部品表）登録
  ・「製品Aを1個作るには、材料Xが2kg、部品Yが3個、ネジZが10本必要」を登録
  └→ 今: Excelにバラバラに存在

STEP 2: BOM展開（= 受注数量から必要な材料の総量を自動計算すること）
  ・例: 製品Aを100個受注 → 材料Xが200kg、部品Yが300個、ネジZが1,000本
  └→ 今: 電卓で手計算

STEP 3: 在庫引当
  ・必要量から現在の在庫を差し引いて、不足分を算出
  └→ 今: 倉庫に見に行って確認

STEP 4: 発注書自動生成
  ・不足分の材料について、仕入先ごとの発注書を自動作成
  └→ 今: Excelで発注書を作ってFAXまたはメール

STEP 5: 検収管理
  ・材料が届いたら、注文通りかチェック（数量・品質）
  └→ 今: 納品書と現物を目視で照合

STEP 6: 仕入先評価
  ・仕入先ごとの品質（Quality）・コスト（Cost）・納期（Delivery）を評価
  └→ 今: 「あそこはたまに不良品混ざるけど安いんだよな」と感覚で判断
```

### ⑧ ISO文書管理 — 「審査準備を1ヶ月→1日にする」

#### What: これは何？

ISO = 国際標準化機構が定めた品質管理の認証制度。
ISO 9001（品質マネジメント）を取得している工場は「ちゃんと品質管理しています」という証明になる。
大手メーカーとの取引条件になっていることが多い。

```
・ISOを維持するには膨大な文書管理が必要
  （マニュアル → 手順書 → 作業指示書 → 記録、の4階層）
・文書の「版管理」が必須（誰が・いつ・何を変えたか）
・年1回の内部監査 + 外部審査の準備に毎年1〜2ヶ月かかる
・品質管理責任者が1人で全部やっている（しかも他の仕事と兼任）
```

#### Why Pay: なぜ月¥2-3万を払うのか

```
■ 現状のコスト
  ・品質管理責任者の工数: 年間200〜400時間（年収の15〜25%相当）
  ・ISO審査直前の残業: 1〜2ヶ月間
  ・認証取り消し → 大手との取引ができなくなる（最悪のケース）

■ AIを入れると
  ・監査チェックリストが自動生成
  ・文書の期限管理が自動化
  ・審査準備レポートがボタン1つ

■ 社長の計算
  ・年1回の審査のために毎月払うのは抵抗がある
  ・ただし「いつでも審査OK」の状態を維持できるなら安心料としてアリ
  → 月2〜3万が上限
```

#### How: 業務フロー（今 → AI）

```
STEP 1: 文書体系管理
  ・ISOでは文書を4階層で管理することが求められる
    品質マニュアル → 手順書 → 作業指示書 → 記録・帳票
  └→ 今: 共有フォルダに雑多に入っている。体系がわからない

STEP 2: 改訂管理
  ・文書を変更する時は「誰が・いつ・なぜ変えたか」を記録する
  ・承認フロー（作成→確認→承認）が必要
  └→ 今: ファイルを上書きして終わり。版管理が崩壊

STEP 3: 内部監査チェックリスト生成
  ・ISOの条項（ルール）に沿って「この部門はこの項目をチェックする」を生成
  └→ AI: ISO 9001の条項 + 自社の手順書からチェックリストを自動作成
  └→ 今: 品質管理責任者が毎年手作業で作成

STEP 4: 監査実施・是正処置管理
  ・監査で見つかった問題（不適合）の記録と、対策の進捗管理
  └→ 今: Excelで管理。対策が完了したかの追跡が甘い

STEP 5: マネジメントレビュー用レポート
  ・経営層に報告するための品質レポートを自動生成
  ・品質目標の達成状況、不適合の件数、顧客満足度等をまとめる
  └→ 今: 品質管理責任者が1週間かけて手作業で作成

STEP 6: 期限管理
  ・「この文書のレビュー期限は来月」「ISO審査は3ヶ月後」を自動リマインド
  └→ 今: 審査2週間前に「ヤバい」と気づいて慌てる
```

#### 課題テーブル

| 項目 | 内容 |
|---|---|
| **何が起きているか** | ISOの文書管理だけで品質管理責任者1人の年間工数の15〜25%を使っている。審査直前に慌てて準備する |
| **なぜ困るのか** | 文書の版管理が崩壊→審査で不適合→是正処置の追加工数。最悪の場合、認証取消→大手との取引ができなくなる |
| **誰が困っているか** | 品質管理責任者（1人で全部やっている。しかも他の仕事と兼任）、社長（認証維持のコスト負担） |
| **起きている損失** | ①品質管理責任者の工数（年200〜400時間）②審査前の残業（1〜2ヶ月間）③外部コンサル費用（年50〜200万円を払う会社も） |
| **今どうやっているか** | 紙+Excel+共有フォルダ。ファイルの版管理はファイル名。内部監査チェックリストは毎年手作業で作成。期限管理はカレンダーに手書き |
| **AIで解決できる範囲** | 監査チェックリスト自動生成、文書改訂管理の自動化、期限リマインド、マネジメントレビューレポート自動生成 |

## 7. 用語集（これだけ覚えれば会話できる）

| 用語 | 読み方 | 意味 |
|---|---|---|
| **チャージレート** | — | 機械を1時間使うコスト。電気代+減価償却+人件費+場所代を時間で割ったもの |
| **段取り** | だんどり | 加工を始める前の準備作業。治具のセット、プログラムの呼び出し等。1回の段取りに30分〜2時間かかることも |
| **サイクルタイム** | — | 部品1個を加工するのにかかる正味の時間 |
| **山積み** | やまづみ | 注文を納期順に並べて、各機械に「この日にこれを作る」と積み上げていくこと |
| **山崩し** | やまくずし | 山積みした結果、機械の能力を超えた部分を前倒し・外注・残業で調整すること |
| **SPC** | エスピーシー | Statistical Process Control。統計的工程管理。検査データをグラフにして異常を検知する手法 |
| **Cp/Cpk** | シーピー/シーピーケー | 工程能力指数。「この工程がどれくらい安定して規格内に収まっているか」を示す数値。1.33以上が目安 |
| **4M** | よんエム | 不良原因を Machine（機械）/ Material（材料）/ Method（方法）/ Man（人）の4つに分類する考え方 |
| **BOM** | ボム | Bill of Materials。部品表。1つの製品を作るのに必要な材料・部品のリスト |
| **MTBF** | エムティービーエフ | Mean Time Between Failures。平均故障間隔。「この機械は平均○○時間ごとに壊れる」 |
| **ISO 9001** | アイエスオー | 品質マネジメントシステムの国際規格。大手との取引に必要なことが多い |
| **MC** | エムシー | マシニングセンタ。穴開け・削り・溝掘りなどを1台でこなす万能加工機 |
| **旋盤** | せんばん | 材料を回転させて刃物を当てて削る機械。丸い部品を作るのに使う |
| **外注** | がいちゅう | 自社でやらず、別の工場に加工を依頼すること |
| **ガントチャート** | — | 横軸が日付、縦軸が機械や作業者の表。どの仕事がいつ入っているかが一目でわかる棒グラフ |
| **リードタイム** | — | 発注してから届くまでの日数。材料なら数日〜数週間 |
| **棚卸** | たなおろし | 倉庫にある在庫の実数を数えて、帳簿上の数と合っているか確認する作業 |
| **是正処置** | ぜせいそち | 問題が起きた原因を取り除くための対策。ISO審査で不適合が出たら必ず求められる |
