## 2. ステップガイド（StepGuide）

### 2.1 プラン別分岐（修正#1）

| プラン | ガイド表示 | 理由 |
|---|---|---|
| セルフ | 3ステップガイド表示 | 自力で始める必要がある |
| コンサル | 3ステップ + 「担当から連絡します」 | ガイド後にコンサルが補助 |
| フルサポート | **非表示** | 専任担当が直接セットアップする |

プラン情報は `GET /api/v1/onboarding/status` の `plan` フィールドで判定。
`plan === "full_support"` の場合、ガイドを一切表示しない。

### 2.2 ステップ構成

**Step 1: 業種を選ぶ**

```
ようこそ、シャチョツーへ！

まずは業種を選んでください。
業界の基本ルールが自動で登録されます。

[建設業]  [製造業]  [医療・福祉]
[不動産]  [物流・運送]  [卸売業]
[その他]
```

- API: `POST /api/v1/onboarding/apply-template { "industry": "construction" }`
- 完了条件: APIが200を返す
- DB更新: companies.industry + genome テンプレート適用 + **「使い方」ナレッジ10件投入**（修正#4）

**Step 2: 試しに質問する**（修正#2: 業種別質問）

```
✅ 業種を設定しました！

業界の基本ルールを登録しました。
試しに質問してみましょう。

下のボタンを押すだけでOKです：

[「{業種別の質問}」と質問する]
```

業種別の質問例:
| 業種 | 質問 |
|---|---|
| 建設業 | 見積もりの諸経費率はいくらですか？ |
| 製造業 | CNC旋盤のチャージレートはいくらですか？ |
| 医療・福祉 | レセプト点検の基準を教えてください |
| 不動産 | 家賃の督促タイミングはいつですか？ |
| 物流・運送 | 配車計画の基準を教えてください |
| 卸売業 | 受注から出荷までの標準リードタイムは？ |
| その他 | 経費精算の承認フローを教えてください |

- API: `POST /api/v1/knowledge/qa { "question": "{業種別質問}", "department": "全社" }`
- 完了条件: AIの回答が表示される

**Step 3: 自社のルールを1つ登録する**

```
✅ AIが回答しました！

これは業界の一般的なルールです。
御社のルールを登録すると、御社専用の回答ができるようになります。

例: 「うちの諸経費率は28%」

[ルールを登録する]  [あとで登録する]
```

- 「ルールを登録する」→ ナレッジ入力画面（`/knowledge/input`）に遷移。ガイドは完了扱い
- 「あとで登録する」→ ガイドは**完了扱い**にする（Step 3は任意。強制しない）
- API: 遷移のみ（ナレッジ登録自体は既存画面で行う）

**完了画面**（修正#5: 導線明確化）

```
🎉 設定完了！

シャチョツーを使う準備ができました。

【次のおすすめ】
・ナレッジをもっと登録する（目標: 10件）
・業務自動化を試す
・会社の状態を確認する

困ったときは右下の「💬 困った」ボタンから
いつでもAIに質問できます。

[ダッシュボードへ]
```

- 「ダッシュボードへ」→ ガイドを閉じる（既にダッシュボード上にいる）
- コンサルプランの場合、追加メッセージ: 「担当の○○から3営業日以内にご連絡いたします」

### 2.3 表示条件

| 条件 | 動作 |
|---|---|
| 初回ログイン（DB: onboarding_steps が空） | 自動表示 |
| 完了済み（DB: onboarding_steps に guide_completed あり） | 非表示 |
| 「あとで」で閉じた | 次回ログイン時に再表示（最大3回） |
| フルサポートプラン | 非表示 |
| ダッシュボード以外のページ | 非表示 |
| モバイル | 表示（レスポンシブ対応） |

### 2.4 進捗の保存（修正#3: DB連携）

**二重保存**: localStorage + DB

```
localStorage（即時性）:
  step_guide_completed: boolean
  step_guide_current: number (0-3)
  step_guide_defer_count: number
  step_guide_industry: string

DB（永続性）:
  companies.onboarding_steps: JSONB
  {
    "guide_completed": true,
    "guide_completed_at": "2026-03-30T...",
    "guide_current_step": 3,
    "guide_defer_count": 0,
    "industry_selected": "construction",
    "first_qa_done": true,
    "first_knowledge_done": false
  }
```

- Step完了時: `POST /api/v1/onboarding/complete-step { "step": "guide_step_1" }` → DB更新
- ページロード時: `GET /api/v1/onboarding/status` → DB値でlocalStorageを同期
- **ブラウザ変更してもDBから復元できる**

### 2.5 エラーハンドリング（修正#6）

| エラー | ユーザーに見せる表示 | 動作 |
|---|---|---|
| API通信エラー | 「通信エラーが発生しました。もう一度お試しください」 | リトライボタン表示 |
| テンプレート適用失敗 | 「設定中にエラーが発生しました。サポートにお問い合わせください」 | サポート連絡先表示 |
| Q&A回答失敗 | 「AIが回答を生成できませんでした。もう一度お試しください」 | リトライボタン表示 |
| ネットワーク切断 | 「インターネット接続を確認してください」 | 接続復帰後に自動リトライ |

全エラーメッセージはやさしい日本語。技術用語（401, timeout等）は表示しない。

## 3. コンテキストヘルプ（ContextHelp）

### 3.1 表示

- 位置: 全認証済みページの右下固定（`fixed bottom-6 right-6`）
- ボタン: 丸型 `w-14 h-14`、テキスト「💬」
- ラベル: ボタン横に「困った」テキスト（初回のみ表示、2回目以降はアイコンのみ）
- Z-index: `z-40`（ガイドの`z-50`より下）

### 3.2 チャットUI

```
┌─────────────────────────────────┐
│ 何かお困りですか？          [×] │
├─────────────────────────────────┤
│                                 │
│ よくある質問:                   │
│ [この画面の使い方]              │
│ [ナレッジの登録方法]            │
│ [業務自動化の始め方]            │
│                                 │
│ （チャット履歴）                │
│                                 │
├─────────────────────────────────┤
│ [質問を入力...]         [送信]  │
└─────────────────────────────────┘
```

- 幅: `w-80`（320px）、高さ: `h-[28rem]`（448px）
- クイックボタン3つ（ページに応じて変わる）:
  - ダッシュボード: 「この画面の使い方」「ナレッジの登録方法」「業務自動化の始め方」
  - ナレッジ管理: 「ナレッジの登録方法」「Q&Aの使い方」「ルールの書き方のコツ」
  - BPO: 「業務自動化の始め方」「実行結果の見方」「困ったときの対処法」
- 送信: `POST /api/v1/knowledge/qa` に質問を投げる
- 回答: マークダウンをHTMLにパースして表示

### 3.3 「使い方」ナレッジの事前投入（修正#4）

`POST /api/v1/onboarding/apply-template` 実行時に、業種ナレッジに加えて以下10件を自動投入:

| # | タイトル | カテゴリ | 内容 |
|---|---|---|---|
| 1 | ダッシュボードの見方 | usage | ナレッジ数・提案数・AIコストの意味と活用法 |
| 2 | ナレッジの登録方法 | usage | テキスト入力・PDF・Excel→AIが自動構造化 |
| 3 | Q&Aの使い方 | usage | 質問のコツ・部門フィルタ・出典の確認方法 |
| 4 | 業務自動化の始め方 | usage | パイプライン選択→入力→Dry-run→実行の流れ |
| 5 | AIの提案の確認方法 | usage | リスク・改善・機会の提案をaccept/rejectする方法 |
| 6 | 会社の状態の見方 | usage | 5次元スナップショットとWhat-if分析の使い方 |
| 7 | メンバーの追加方法 | usage | 招待メール→権限設定（admin/editor） |
| 8 | 困ったときの対処法 | usage | ヘルプチャット・サポート連絡先・よくある質問 |
| 9 | データの書き出し方法 | usage | CSV/PDF/JSONでのエクスポート手順 |
| 10 | サポートへの連絡方法 | usage | メール・Slack・電話の連絡先と対応時間 |

これらは `department: "全社"`, `category: "usage"`, `item_type: "tip"` で登録。
全業種共通（業種テンプレートとは別に常に投入）。

## 4. デザイン仕様

### 4.1 ステップガイド

- オーバーレイ: `bg-black/30 backdrop-blur-sm`
- カード: `bg-white rounded-2xl shadow-2xl max-w-lg mx-auto`
- ステップインジケーター: ドット3つ（完了=blue-600、現在=blue-600、未完了=gray-300）
- プライマリボタン: `bg-blue-600 hover:bg-blue-700 text-white rounded-lg px-6 py-3 text-base`
- セカンダリボタン: `border border-gray-300 text-gray-600 rounded-lg px-6 py-3 text-base`
- 業種選択ボタン: `border-2 border-gray-200 hover:border-blue-500 rounded-xl px-4 py-3`
- フォント: 本文16px（`text-base`）、見出し20px（`text-xl`）、補足14px（`text-sm`）
- アニメーション: `animate-in fade-in slide-in-from-bottom-4 duration-300`

### 4.2 コンテキストヘルプ

- ボタン: `w-14 h-14 rounded-full bg-blue-600 hover:bg-blue-700 text-white shadow-lg`
- チャットウィンドウ: `w-80 h-[28rem] bg-white rounded-2xl shadow-2xl`
- 入力欄: `border border-gray-200 rounded-lg px-3 py-2`
- AIの回答: `bg-gray-50 rounded-lg px-4 py-3 text-sm`
- ユーザーの質問: `bg-blue-50 rounded-lg px-4 py-3 text-sm ml-auto`

### 4.3 モバイル対応

- ステップガイド: `max-w-lg` → モバイルでは画面幅いっぱい（`mx-4`）
- 業種ボタン: 横3列 → モバイルでは横2列（`grid grid-cols-2 md:grid-cols-3`）
- ヘルプチャット: `w-80` → モバイルでは `w-[calc(100vw-2rem)]`
- ヘルプボタン: 位置は同じ（`bottom-6 right-6`）

## 5. API連携一覧

| タイミング | API | メソッド | 用途 |
|---|---|---|---|
| ページロード時 | `/api/v1/onboarding/status` | GET | プラン・進捗を取得 |

| Step 3遷移 | （ページ遷移のみ） | - | `/knowledge/input`へ |
| ガイド完了 | `/api/v1/onboarding/complete-step` | POST | guide_completed=true |
| ヘルプ質問 | `/api/v1/knowledge/qa` | POST | Q&A実行 |

## 6. テスト要件

| # | テスト | 確認内容 |
|---|---|---|
| 1 | セルフプランで初回ログイン | ガイドが自動表示される |
| 2 | フルサポートプランで初回ログイン | ガイドが表示されない |
| 3 | Step 1で業種選択 | テンプレート適用API成功 + Step 2に遷移 |
| 4 | Step 2でQ&A実行 | 業種に応じた質問が表示される + AIの回答が出る |
| 5 | Step 3「あとで」 | ガイドが完了扱いになる |
| 6 | ガイド完了後に再ログイン | ガイドが表示されない |
| 7 | 「あとで」3回 | 4回目以降はガイドが表示されない |
| 8 | ブラウザ変更 | DB値から復元してガイドが正しい状態で表示される |
| 9 | API通信エラー | エラーメッセージ + リトライボタン表示 |
| 10 | モバイル表示 | 業種ボタンが2列で崩れない |
| 11 | ヘルプチャットで質問 | AIが「使い方」ナレッジから回答 |
| 12 | ダッシュボード以外のページ | ガイド非表示、ヘルプボタンのみ表示 |

## 7. 多言語対応（修正#7）

Phase 1は日本語のみ。ただし将来の多言語対応を見据えて:

- 全テキストは定数オブジェクトに集約（ハードコードしない）
- `const GUIDE_TEXTS = { ja: { step1_title: "ようこそ...", ... } }` の形式
- Phase 2+で `en` キーを追加するだけで英語対応可能にする
