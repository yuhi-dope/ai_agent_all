## 設計思想

```
Layer 0: LLMデフォルト — 何も設定がなくても概算が出る（全24業種Day 1対応）
Layer 1: YAML設定     — 推定を事実に置き換えて精度を上げる（業種データ固定）
Layer 2: Pythonプラグイン — 計算の公式自体が違う業種に対応（例外のみ）

上のレイヤーがなければ下にフォールバック。重ねて使う。
```

### 解決順序（各データ項目ごと）

```
チャージレート: 顧客DB → YAML → LLM推定
材料単価:       顧客DB → YAML → LLM推定
工程リスト:     Plugin → YAMLルール → LLM推定
サイクルタイム: 顧客の過去実績 → YAML → LLM推定
```

### Confidenceスコア

| ソース | Confidence |
|--------|------------|
| LLMのみ（Layer 0） | 0.3-0.5 |
| YAML設定あり（Layer 1） | 0.6-0.8 |
| YAML + 顧客DB | 0.8-0.9 |
| Plugin（Layer 2） | 0.7-0.9 |
| 過去実績あり | 0.9-1.0 |

## エンジン設計（engine.py）

### クラス構造

```python
# DESIGN — 実装コードではない

class ManufacturingQuotingEngine:

    async def run(
        self,
        company_id: str,
        hearing: HearingInput,
    ) -> QuoteResult:
        sub_industry = self._resolve_sub_industry(hearing)
        plugin = get_plugin(sub_industry)        # Layer 2
        yaml_config = load_yaml_config(sub_industry)  # Layer 1
        customer = await self._load_customer_data(company_id)  # 顧客DB

        # 工程推定（解決順序: Plugin → YAML → LLM）
        if plugin and plugin.can_estimate_processes(hearing):
            processes = await plugin.estimate_processes(hearing, yaml_config, customer)
        elif yaml_config and yaml_config.get("process_routing_rules"):
            processes = self._estimate_from_yaml(hearing, yaml_config, customer)
        else:
            processes = await self._estimate_from_llm(hearing, sub_industry)

        # コスト計算（共通。チャージレート等は顧客DB→YAML→LLMの順で解決）
        costs = self._calculate_costs(processes, hearing, yaml_config, customer)

        # プラグイン固有の追加コスト（金型償却等）
        additional = []
        if plugin:
            additional = plugin.calculate_additional_costs(hearing, processes)

        quote_id = await self._save_quote(company_id, hearing, processes, costs, additional)
        return QuoteResult(quote_id=quote_id, processes=processes, costs=costs, additional_costs=additional)
```

### コスト計算（全レイヤー共通の公式）

```
材料費 = 材料単価 × 使用量 × ロス率 × 数量
加工費 = Σ（チャージレート × (段取り時間 + サイクルタイム × 数量) / 60）
外注費 = Σ（基本費 + 個あたり費 × 数量）
追加費 = Σ（プラグイン固有。金型償却、配合ロス等）
管理費 = (材料費 + 加工費 + 外注費) × 管理費率
利益   = (材料費 + 加工費 + 外注費 + 管理費) × 利益率
合計   = 材料費 + 加工費 + 外注費 + 追加費 + 管理費 + 利益
```

**この公式は全24業種で共通。** 各項目の「値」がレイヤーによって異なるだけ。

## Pythonプラグインインターフェース

### ABC（plugins/base.py）

```python
# DESIGN

class ManufacturingPlugin(ABC):
    """計算の公式自体が違う業種のみ実装する"""

    @property
    @abstractmethod
    def sub_industry_id(self) -> str: ...

    def can_estimate_processes(self, hearing: HearingInput) -> bool:
        return True

    @abstractmethod
    async def estimate_processes(
        self, hearing: HearingInput, yaml_config: dict | None, customer: CustomerOverrides,
    ) -> list[ProcessEstimate]: ...

    def calculate_additional_costs(
        self, hearing: HearingInput, processes: list[ProcessEstimate],
    ) -> list[AdditionalCostItem]:
        return []  # デフォルト: 追加費用なし
```

### Pluginが必要な理由（業種ごと）

| 業種 | 特殊な計算 | なぜYAMLだけでは無理か |
|------|-----------|---------------------|
| 樹脂成型 | 金型費 ÷ 生産予定数量 = 個あたり金型償却費 | 「償却」という概念がYAMLの工程ルーティングでは表現できない |
| 食品・化学 | レシピ配合比 × バッチサイズ → 各原料の必要量 | 「配合展開」は材料マスタの参照ではなく逆算ロジック |
| 電子部品 | BOMツリー展開 → 部品費 + 実装費の積上げ | 「親子関係のある部品表」はフラットなYAMLでは表現困難 |

## LLMデフォルトレイヤー（Layer 0）

### プロンプト構造

```
あなたは{業種名}の見積エキスパートです。
以下のヒアリング情報をもとに、製造工程・材料費・加工費を推定してください。

[ヒアリング回答6項目]

出力: JSON形式（processes, material_estimate, outsource_items, confidence）

ガイドライン:
- confidenceは正直に。根拠が薄ければ0.3
- 業界の一般的な相場を使い、notesに根拠を記載
- 工程は10以内
```

### YAMLがある場合のLLMの役割縮小

```
YAMLなし: LLMが全て推定（工程リスト、材料単価、チャージレート）
YAMLあり: LLMは「工程リストの推定」だけ担当。単価はYAMLから取得
顧客DBあり: LLMは「形状分類」だけ担当。工程も単価もDBから取得
```

## 顧客データによるオーバーライド

### データ構造

```python
# DESIGN

class CustomerOverrides:
    charge_rates: dict[str, int]          # 設備名 → チャージレート
    material_prices: dict[str, int]       # 材料コード → kg単価
    historical_averages: dict[str, dict]  # 設備名 → {avg_setup, avg_cycle, count}
    overhead_rate: float | None
    profit_rate: float | None
```

### 学習ループ

```
1. AIが見積生成（precision 0.4-0.7）
2. 顧客が修正（「CNC旋盤の段取りは25分」）
3. 修正データをDBに保存（user_modified = true）
4. 次回同じ設備×材質の組み合わせ → 過去実績の平均値を使用
5. 3件以上の実績 → confidence 0.9
```

### 匿名集約（Phase 2+）

```
全社のmfg_charge_ratesを匿名集約
  → 同じ equipment_type × sub_industry の中央値を計算
  → metalwork.yaml の charge_rate_yen_hour を更新候補として提示
  → 管理者レビュー後にYAMLにマージ
```

## DBスキーマ変更（015_mfg_quoting_engine.sql）

```sql
-- mfg_quotes に業種カラム追加
ALTER TABLE mfg_quotes
  ADD COLUMN IF NOT EXISTS sub_industry TEXT DEFAULT 'metalwork',
  ADD COLUMN IF NOT EXISTS layers_used JSONB DEFAULT '[]',
  ADD COLUMN IF NOT EXISTS overall_confidence DECIMAL(3,2),
  ADD COLUMN IF NOT EXISTS additional_costs JSONB DEFAULT '[]';

-- mfg_quote_items にレイヤーソース追加
ALTER TABLE mfg_quote_items
  ADD COLUMN IF NOT EXISTS layer_source TEXT DEFAULT 'yaml';

-- 学習ループ用の集約ビュー
CREATE OR REPLACE VIEW mfg_historical_averages AS
SELECT
  company_id, equipment_type,
  COUNT(*) as sample_count,
  AVG(setup_time_min) FILTER (WHERE user_modified = true) as avg_setup_min,
  AVG(cycle_time_min) FILTER (WHERE user_modified = true) as avg_cycle_min
FROM mfg_quote_items
WHERE user_modified = true AND created_at > now() - interval '6 months'
GROUP BY company_id, equipment_type
HAVING COUNT(*) >= 3;
```
