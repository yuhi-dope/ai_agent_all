# データ鮮度・学習ループ・横展開設計

> **位置づけ**: 製造業BPO 16モジュールを起点に、データ取得方法・入力負荷・学習精度向上・6業種横展開の共通アーキテクチャを定義する
> **上位文書**: c_02（プロダクト設計）、d_02（フィードバック学習ループ設計）、e_02b/e_02c（製造業BPO詳細/経営インテリジェンス設計）
> **対象**: 製造業BPO 16モジュール（①-⑧ 現場BPO + ⑨-⑬ 経営インテリジェンス + ⑭-⑯ バックオフィス）
> **実装場所**: `workers/bpo/manufacturing/`, `brain/genome/data/bpo/`, `shared/`

---

## 0. エグゼクティブサマリー

```
課題:
  ・16モジュールのデータ取得方法が設計書ごとに散在。顧客の入力負荷が不透明
  ・学習ループの「いつ・どれだけのデータで・どの精度まで」が未定義
  ・6業種展開時の共通基盤と業種固有部分の境界が曖昧

本設計で定義すること:
  A. 16モジュール×3カテゴリのデータ取得方法マトリクス + 初回セットアップウィザード
  B. 学習ループの統計的有意性・匿名化・反映タイミング・精度測定・フィードバックUI
  C. Base/Industry分離パターン + ゲノムJSON化による横展開インターフェース

設計原則:
  ・データが50%でも50%の価値を出す（グレースフルデグラデーション）
  ・初回セットアップ60分以内。月次更新は自動+CSV10分
  ・学習は3フェーズ（c_02 §3.1準拠）: ルールベース→企業適応→N社ネットワーク
```

### A.2 データ取得カテゴリ別UX設計

#### A.2.1 API自動（手間ゼロ）

| 連携先 | 対象モジュール | 認証方式 | データ取得頻度 | Phase |
|---|---|---|---|---|
| freee | ⑦⑨⑭ | OAuth 2.0 | 日次（PLは月次） | Phase 1 |
| 為替API（ECB/日銀） | ⑫ | APIキー | 日次 | Phase 1 |
| LME API（金属相場） | ⑫ | APIキー | 日次 | Phase 1 |
| 経産省 e-Stat | ⑪⑫ | 公開API | 月次 | Phase 1 |
| kintone | ②⑮ | APIトークン | リアルタイム | Phase 2 |
| LINE WORKS | ⑮ | OAuth 2.0 | 日次 | Phase 2 |

**認証フロー設計**:
```
freee OAuth:
  1. 「freee連携」ボタン → freee認証画面（リダイレクト）
  2. ユーザーが承認 → callback でアクセストークン取得
  3. リフレッシュトークンをSupabase暗号化カラムに保存
  4. 日次バッチでトークンリフレッシュ + データ取得
  → Phase 1: 手動OAuth。Phase 2: 自動リフレッシュ

為替/LME API:
  1. 初回設定で主力通貨・材質を選択（USD/EUR, SUS304/A5052等）
  2. 日次CRONで自動取得 → market_data テーブルに蓄積
  3. 閾値超過（前日比±3%）でSlack自動アラート
```

**データ取得失敗時のフォールバック**:
```
APIエラー → 3回リトライ（exponential backoff: 1s, 5s, 30s）
  → 失敗 → 前日のデータを「推定値」として表示（バッジ: ⚠ 推定値）
  → 24時間以上の取得失敗 → Slack通知 + ダッシュボードに警告バナー
  → CSV手動アップロードへのフォールバック導線を表示
```

#### A.2.2 CSV半自動（月1回10分）

**対応フォーマット**:
```

生産実績CSV:      日付, 設備名, 受注番号, 稼働時間, 良品数, 不良数

在庫棚卸CSV:      品目コード, 品名, 現在庫数, 単価, カテゴリ
銀行明細CSV:      取引日, 摘要, 入金, 出金, 残高
保全記録CSV:      日付, 設備名, 保全種別(定期/突発), 内容, 費用, 停止時間
勤怠CSV:          従業員名, 日付, 出勤時刻, 退勤時刻, 休憩時間
```

**CSV取込UX設計**:
```
1. ドラッグ&ドロップ or ファイル選択
2. プレビュー画面:
   ・先頭5行を表示
   ・カラム自動マッピング（ヘッダー名のファジーマッチング）
   ・マッピング修正UI（ドロップダウンで正しいカラムを選択）
3. バリデーション結果:
   ・✅ 正常: 128行
   ・⚠ 警告: 3行（数値フォーマット不正 → 自動変換済み）
   ・❌ エラー: 1行（必須項目欠落 → スキップ）
4. 「取込実行」ボタン → 処理完了通知

設計参考: freee のCSVインポート機能
  ・ヘッダー行の自動検出
  ・文字コード自動判定（Shift_JIS/UTF-8）
  ・重複データの検出（取引日+摘要+金額の一致）
```

### A.4 グレースフルデグラデーション設計

> **原則**: データが50%でも50%の価値を出す。欠損データのあるモジュールは「低精度モード」で稼働させ、利用不可にはしない。

#### モジュール別デグラデーション定義

| # | モジュール | 必須データ | あれば精度向上 | データ欠損時の挙動 |
|---|---|---|---|---|
| 1 | 見積AI | 加工種別(1つ以上) | 過去見積実績, 材料単価マスタ | テンプレート相場で見積。confidence=0.5表示 |
| 2 | 生産計画AI | 受注データ(1件以上), 設備(1台以上) | 稼働実績, 工程マスタ | デフォルトキャパ(8h/日)で計算。精度バッジ「参考値」 |
| 3 | 品質管理 | 検査記録(10件以上) | SPC用30件以上 | 10件未満: 個別値一覧のみ。管理図・Cpk算出不可 |
| 4 | 在庫最適化 | 品目リスト | 入出庫履歴3ヶ月 | ABC分析のみ。EOQ/安全在庫は「仮値」表示 |
| 5 | SOP管理 | 手順テキスト(1件以上) | 写真/動画 | テキストのみのSOP生成。多言語翻訳は可能 |
| 6 | 設備保全 | 設備登録(1台以上) | 保全履歴6ヶ月 | メーカー推奨周期でカレンダー生成。MTBF算出不可 |

#### B.1.3 ベンチマーク統計処理

```python
def calculate_benchmark(segment_data: list[dict], metric: str) -> BenchmarkResult:
    """
    セグメント別ベンチマーク算出
    ロバスト統計: 外れ値の影響を最小化
    """
    values = [d[metric] for d in segment_data]
    n = len(values)

    if n < 5:
        return BenchmarkResult(status="HIDDEN", reason="k-anonymity violation")

    # ロバスト統計（外れ値耐性）
    median = np.median(values)
    q1, q3 = np.percentile(values, [25, 75])
    iqr = q3 - q1

    # Tukey法で外れ値除外
    lower_fence = q1 - 1.5 * iqr
    upper_fence = q3 + 1.5 * iqr
    filtered = [v for v in values if lower_fence <= v <= upper_fence]

    # 信頼区間（N≥30の場合のみ）
    if n >= 30:
        mean = np.mean(filtered)
        se = np.std(filtered, ddof=1) / np.sqrt(len(filtered))
        ci_95 = (mean - 1.96 * se, mean + 1.96 * se)
    else:
        mean = np.mean(filtered)
        ci_95 = None  # サンプル不足

    return BenchmarkResult(
        n=n,
        mean=round(mean, 2),
        median=round(median, 2),
        q1=round(q1, 2),
        q3=round(q3, 2),
        ci_95=ci_95,
        status="BENCHMARK" if n >= 30 else "REFERENCE" if n >= 10 else "TREND",
    )
```

### B.2 匿名化ルール

> **法令参照**: 個人情報保護法第2条（個人情報の定義）、第43条（匿名加工情報の取扱い）

#### B.2.1 k-匿名性の実装

```
■ 基本ルール: k ≥ 5
  同一セグメント内に5社以上存在しない限り、ベンチマーク値を公開しない

■ セグメント判定
  セグメント = (加工種別, 従業員規模, 地域ブロック)
  例: (金属切削, 30-100名, 関東) に5社以上いればベンチマーク表示可

■ l-多様性の追加制約: l ≥ 3
  同一セグメント内の数値分布に3社以上の異なる値域が含まれること
  → 1社の突出した値で推定されることを防止
  例: 粗利率が全5社とも28%前後 → 特定社の粗利率が推測可能 → 公開しない

■ t-近似性（Phase 2+）
  セグメント内の値分布が全体分布からt以上乖離しないことを検証
```

#### B.2.2 匿名化処理パイプライン

```python
class AnonymizationPipeline:
    """匿名化処理パイプライン（d_02 §3 フィードバック学習ループと連動）"""

    def anonymize_for_benchmark(self, company_data: dict) -> dict | None:
        """個社データをベンチマーク集約用に匿名化"""

        # Step 1: 直接識別子の除去
        anonymized = {
            k: v for k, v in company_data.items()
            if k not in DIRECT_IDENTIFIERS
        }

        # Step 2: 準識別子の一般化
        anonymized["employee_count"] = generalize_range(
            company_data["employee_count"],
            ranges=[(10, 30), (31, 100), (101, 300)]
        )
        anonymized["region"] = generalize_to_block(
            company_data["prefecture"]  # 都道府県→8地域ブロック
        )
        anonymized["annual_revenue"] = generalize_range(
            company_data["annual_revenue"],
            ranges=[(0, 1e8), (1e8, 5e8), (5e8, 10e8), (10e8, 50e8)]
        )

        # Step 3: k-匿名性チェック
        segment = self.get_segment(anonymized)
        segment_count = self.count_segment(segment)
        if segment_count < K_THRESHOLD:  # K_THRESHOLD = 5
            return None  # ベンチマーク非公開

        # Step 4: l-多様性チェック
        diversity = self.check_l_diversity(segment, metric="gross_margin")
        if diversity < L_THRESHOLD:  # L_THRESHOLD = 3
            return None

        # Step 5: ノイズ付加（差分プライバシー）
        # 具体的な数値にラプラスノイズを加算
        for metric in NUMERIC_METRICS:
            epsilon = 0.1  # プライバシーバジェット
            sensitivity = METRIC_SENSITIVITY[metric]
            noise = np.random.laplace(0, sensitivity / epsilon)
            anonymized[metric] += noise

        return anonymized
```

```python
class AccuracyTracker:
    """精度測定・記録（d_02 フィードバック学習ループと連携）"""

    async def record_prediction_vs_actual(
        self,
        company_id: str,
        module: str,
        prediction: dict,
        actual: dict,
        metric: str,
    ) -> AccuracyRecord:
        """予測値と実績値の乖離を記録"""

        predicted_value = prediction[metric]
        actual_value = actual[metric]

        # MAPE計算
        if actual_value != 0:
            mape = abs(predicted_value - actual_value) / abs(actual_value)
        else:
            mape = None

        record = AccuracyRecord(
            company_id=company_id,
            module=module,
            metric=metric,
            predicted_value=predicted_value,
            actual_value=actual_value,
            mape=mape,
            recorded_at=datetime.utcnow(),
        )

        # DB保存（既存 accuracy_logs テーブル）
        await self.save(record)

        # 月次集計の自動トリガー
        if await self.should_recalculate(company_id, module):
            await self.recalculate_monthly_accuracy(company_id, module)

        return record

    async def get_module_accuracy_summary(
        self, company_id: str, module: str, months: int = 3
    ) -> AccuracySummary:
        """直近N月の精度サマリー"""
        records = await self.get_records(company_id, module, months)

        return AccuracySummary(
            module=module,
            n_predictions=len(records),
            mean_mape=np.mean([r.mape for r in records if r.mape]),
            median_mape=np.median([r.mape for r in records if r.mape]),
            trend="improving" if self.is_improving(records) else "stable",
            phase=self.determine_phase(records),  # Phase 1/2/3
        )
```

```python
# shared/base_pipeline.py

from abc import ABC, abstractmethod
from dataclasses import dataclass


@dataclass
class PipelineConfig:
    """ゲノムJSONから読み込まれるパイプライン設定"""
    industry: str                          # "manufacturing", "construction", ...
    cost_categories: list[str]             # 業種固有の原価区分
    allocation_method: str                 # 原価按分方法
    benchmarks: dict[str, tuple[float, float]]  # ベンチマーク値（min, max）
    thresholds: dict[str, float]           # 閾値
    templates: dict[str, str]              # レポートテンプレートID


class BasePLIntelligencePipeline(ABC):
    """PLインテリジェンスの共通基底クラス"""

    def __init__(self, config: PipelineConfig):
        self.config = config

    async def run(self, company_id: str, period: str) -> PLReport:
        """共通パイプラインフロー（Template Method）"""
        # Step 1: PLデータ取得（共通）
        raw_pl = await self.fetch_pl_data(company_id, period)

        # Step 2: コスト按分（業種固有）
        allocated = await self.allocate_costs(raw_pl)

        # Step 3: PL算出（共通エンジン）
        pl_results = self.calculate_pl(allocated)

        # Step 4: ベンチマーク比較（共通エンジン + 業種固有閾値）
        comparison = self.compare_benchmark(pl_results)

        # Step 5: レポート生成（共通エンジン + 業種固有テンプレート）
        report = await self.generate_report(pl_results, comparison)

        # Step 6: 異常検知（共通エンジン + 業種固有閾値）
        anomalies = self.detect_anomalies(pl_results)

        return PLReport(
            results=pl_results,
            comparison=comparison,
            anomalies=anomalies,
            report=report,
        )

    async def fetch_pl_data(self, company_id: str, period: str) -> dict:
        """PLデータ取得（共通: freee API / CSV）"""
        # 共通実装
        ...

    @abstractmethod
    async def allocate_costs(self, raw_pl: dict) -> dict:
        """コスト按分（業種固有: 製造業=設備稼働時間、建設業=工事別）"""
        ...

    def calculate_pl(self, allocated: dict) -> list[dict]:
        """PL算出（共通: 売上-原価=粗利の計算ロジック）"""
        results = []
        for category in self.config.cost_categories:
            revenue = allocated.get(f"{category}_revenue", 0)
            costs = sum(
                allocated.get(f"{category}_{cost_type}", 0)
                for cost_type in self.config.cost_categories
            )
            gross_profit = revenue - costs
            gross_margin = gross_profit / revenue * 100 if revenue else 0

            results.append({
                "category": category,
                "revenue": revenue,
                "total_cost": costs,
                "gross_profit": gross_profit,
                "gross_margin": round(gross_margin, 1),
            })
        return results

    def compare_benchmark(self, pl_results: list[dict]) -> list[dict]:
        """ベンチマーク比較（共通エンジン + ゲノムJSONの閾値）"""
        comparisons = []
        for result in pl_results:
            category = result["category"]
            benchmark = self.config.benchmarks.get(category, (0, 100))
            actual = result["gross_margin"]

            if actual < benchmark[0]:
                severity = "WARNING"
            elif actual > benchmark[1]:
                severity = "EXCELLENT"
            else:
                severity = "NORMAL"

            comparisons.append({
                "category": category,
                "actual": actual,
                "benchmark_low": benchmark[0],
                "benchmark_high": benchmark[1],
                "severity": severity,
            })
        return comparisons
```

```python
class GenomeLoader:
    """ゲノムJSONローダー（既存の loader.py を拡張）"""

    INTELLIGENCE_GENOME_DIR = "brain/genome/data/bpo/"

    async def load_intelligence_config(
        self, industry: str, module: str
    ) -> PipelineConfig:

        path = f"{self.INTELLIGENCE_GENOME_DIR}{industry}_intelligence.json"

        return PipelineConfig(
            industry=industry,
            cost_categories=module_config.get("cost_categories", []),
            allocation_method=module_config.get("allocation_method", "revenue_ratio"),
            benchmarks=module_config.get("benchmarks", {}),
            thresholds=module_config.get("anomaly_rules", {}),
            templates=module_config.get("report_template", ""),
        )

        """セットアップウィザード用テンプレートをロード"""
        path = f"{self.INTELLIGENCE_GENOME_DIR}{industry}_intelligence.json"
```
