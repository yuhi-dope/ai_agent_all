# i_05 リファクタリング実行手順書

## 目的
ファイル移動を安全に実行するための手順書。間違えた場合の戻し方も定義。

## 前提条件
- mainブランチから `refactor/3layer-architecture` ブランチを作成
- 全テストのベースライン記録済み
- i_01〜i_04の設計書レビュー済み

## 実行順序（依存関係順）

リファクタリングは依存関係の「根」から「葉」に向かって実行する。
先に移動すべきは「他から参照されるもの」。

### Step 1: Manager層 + Engine層の移動（最優先）
全パイプラインから参照されるため、最初に移動。

```
1.1 workers/bpo/manager/ → workers/manager/
1.2 workers/bpo/engine/ → workers/engine/
1.3 全ファイルの import を sed で一括置換
1.4 テスト実行（manager + engine テストのみ）
```

**この時点で壊れるファイル**: 全パイプライン（56本）+ ルーター + main.py
**修正方法**: import パスの置換のみ（ロジック変更なし）

**sedコマンド例**:
```bash
# manager
find shachotwo-app/ -name "*.py" -exec sed -i '' \
  's/from workers\.bpo\.manager/from workers.manager/g' {} +
find shachotwo-app/ -name "*.py" -exec sed -i '' \
  's/import workers\.bpo\.manager/import workers.manager/g' {} +

# engine
find shachotwo-app/ -name "*.py" -exec sed -i '' \
  's/from workers\.bpo\.engine/from workers.engine/g' {} +
find shachotwo-app/ -name "*.py" -exec sed -i '' \
  's/import workers\.bpo\.engine/import workers.engine/g' {} +
```

**検証コマンド**:
```bash
cd shachotwo-app
python -c "from workers.manager.task_router import route_and_execute"
python -c "from workers.engine.models import PipelineResult"
pytest tests/workers/manager/ tests/workers/bpo/manager/ -v --tb=short
```

**コミット**: `refactor: move manager/ and engine/ out of bpo/ (Step 1/8)`

### Step 2: micro層の業界中立化
compliance.pyから業界ルールを分離。

```
2.1 workers/industry/construction/rules/compliance_rules.py 新規作成（建設業法ルール）
2.2 workers/industry/manufacturing/rules/compliance_rules.py 新規作成（ISO規則）
2.3 micro/compliance.py から _CONSTRUCTION_RULES, _MANUFACTURING_RULES を削除
2.4 micro/compliance.py に汎用 check_rules(data, rules) インターフェースを定義
2.5 建設パイプラインが rules/compliance_rules.py を読んで micro/compliance に渡すように変更
2.6 テスト実行（micro + 建設 + 製造テスト）
```

**注意**: この Step は Step 3（業界プラグイン移動）より先に実行する。
理由: 業界固有コードを分離してからでないと、移動先の構造が確定しない。

**検証コマンド**:
```bash
cd shachotwo-app
python -c "from workers.micro.compliance import check_rules"
pytest tests/workers/micro/ -v --tb=short
pytest tests/workers/bpo/construction/ -v --tb=short
pytest tests/workers/bpo/manufacturing/ -v --tb=short
```

**コミット**: `refactor: extract industry rules from micro/compliance.py (Step 2/8)`

### Step 3: 業界プラグインの移動（Layer B）
他から参照されないため、安全に移動可能。

```
3.1 workers/bpo/construction/ → workers/industry/construction/
3.2 workers/bpo/manufacturing/ → workers/industry/manufacturing/
3.3 workers/bpo/clinic/ → workers/industry/clinic/
3.4 workers/bpo/nursing/ → workers/industry/nursing/
3.5 workers/bpo/realestate/ → workers/industry/realestate/
3.6 workers/bpo/logistics/ → workers/industry/logistics/
3.7 全ファイルのimportをsedで一括置換
3.8 PIPELINE_REGISTRY のモジュールパスを更新
3.9 テスト実行（全業界テスト）
```

**sedコマンド例**:
```bash
INDUSTRIES="construction manufacturing clinic nursing realestate logistics"
for ind in $INDUSTRIES; do
  find shachotwo-app/ -name "*.py" -exec sed -i '' \
    "s/from workers\.bpo\.${ind}/from workers.industry.${ind}/g" {} +
  find shachotwo-app/ -name "*.py" -exec sed -i '' \
    "s/import workers\.bpo\.${ind}/import workers.industry.${ind}/g" {} +
done
```

**凍結業種も同時に移動**:
```bash
FROZEN="dental hotel beauty auto_repair pharmacy ecommerce staffing architecture professional restaurant"
for ind in $FROZEN; do
  find shachotwo-app/ -name "*.py" -exec sed -i '' \
    "s/from workers\.bpo\.${ind}/from workers.industry.${ind}/g" {} +
done
```

**検証コマンド**:
```bash
cd shachotwo-app
python -c "from workers.industry.construction.pipelines import estimation_pipeline"
python -c "from workers.industry.manufacturing.pipelines import quoting_pipeline"
pytest tests/workers/bpo/construction/ tests/workers/bpo/manufacturing/ -v --tb=short
```

**コミット**: `refactor: move all industry plugins to workers/industry/ (Step 3/8)`

## ロールバック手順

各ステップ完了時にgit commitする。問題が発生した場合:

```bash
git stash           # 現在の変更を退避
git log --oneline   # 最後の安定commitを確認
git reset --hard <safe_commit>  # 安定状態に戻す
```

### Step別ロールバック

| Step | ロールバック難易度 | 方法 |
|---|---|---|
| 1 | 低 | git revert 1コミット |
| 2 | 中 | compliance.pyの復元 + rules/削除 |
| 3 | 低 | git revert 1コミット（import置換の逆） |
| 4 | 低 | git revert 1コミット |
| 5 | 中 | ファイル分割の逆統合が必要 |
| 6 | 低 | REGISTRY定義の復元のみ |
| 7 | 低 | git revert 1コミット |
| 8 | N/A | 削除済みディレクトリは git で復元可能 |

### 全体ロールバック（最悪ケース）

```bash
git checkout main
git branch -D refactor/3layer-architecture
```

mainブランチには一切影響しないため、最悪の場合はブランチごと破棄して最初からやり直せる。
