## 1. 不動産業の現実（業界人なら「分かる」話）

### 1.1 中小不動産管理会社の現場

管理戸数50-500戸の中小不動産管理会社、約1.5万社（国交省 賃貸住宅管理業登録簿）。
賃貸管理、売買仲介、賃貸仲介、オーナー対応。業態は違っても共通する現実がある。

**社長の1日（Before）**

```
AM 8:00  出社。昨晩の入居者クレーム3件がLINEに溜まっている
         「上階の足音がうるさい」「給湯器が壊れた」「隣の部屋がゴミ屋敷」
         → 1件ずつ状況確認して業者手配。この時点で30分消える
AM 8:30  月末の家賃入金チェック。300戸分の通帳記帳をExcelと突合
         → 入金名義が「カ）ヤマダ」「ヤマダタロウ」「山田」で3パターン。誰の分か判別に時間がかかる
         → 未入金15戸。督促電話リスト作成だけで1時間
AM 9:30  オーナーから電話「今月の収支報告書まだ？」
         → 毎月25日に送る約束だが、入金消込が終わらないと作れない
         → オーナー40名分の報告書を手作業でExcelからPDF化。丸1日仕事
AM 10:00 空室が3ヶ月続いている物件。オーナーから「家賃下げるべき？」
         → 相場を調べるためSUUMO/HOME'Sを30分見る。根拠のある提案ができない
AM 10:30 売買の査定依頼が入った。一括査定サイト経由で3社競合
         → レインズで事例を探す（30分）→ 査定書作成（1時間）→ 他社より遅い
AM 11:30 新規の賃貸申込。保証会社の審査書類を揃えて、重説の準備
         → 宅建士は社長1人。重説のたびに社長が対応
PM 1:00  内見のアテンド2件。物件まで車で移動、案内、戻り。3時間消える
PM 4:00  退去立会い。原状回復の範囲でもめる
         → ガイドラインの解釈で毎回揉める。写真を撮って見積もり依頼
PM 5:00  更新案件のチェック。来月更新の入居者10件に更新案内を送る
         → 2年ごとの更新日を手動管理。見落としで更新漏れが年に数件
PM 6:00  今日もREINSへの物件登録ができなかった。明日やる
PM 7:00  退社。帰宅後も入居者からの緊急電話が鳴る可能性がある。
         今日も「管理」しかしていない。「経営」は1秒もしていない。
```

**社長の1日（After — シャチョツー導入後）**

```
AM 8:00  出社。AIが昨晩のクレーム3件を自動分類済み
         「騒音」→テンプレ回答ドラフト済み。「給湯器故障」→緊急度高、業者に自動手配済み
         「ゴミ屋敷」→対応フロー提案+内容証明のドラフト付き。確認して送信、10分で完了
AM 8:30  入金消込AIが銀行API連携で300戸を自動消込（一致率97%）
         → 未入金12戸が自動抽出。督促SMSも下書き済み。確認してワンクリック送信
AM 9:00  オーナー報告書40名分が自動生成済み。収支+空室状況+修繕履歴+相場レポート付き
         → 確認してメール一括送信。15分で完了
AM 9:30  空室物件の賃料査定AIが周辺相場を分析「現在¥75,000 → 適正¥72,000。¥3,000下げで
         空室期間1.2ヶ月短縮、年間収支+¥4万の試算」
         → オーナーにデータ付きで提案。納得度が全然違う
AM 10:00 売買査定依頼。物件情報入力 → AIが3手法で査定書を10分で自動生成
         → 確認・微修正して30分で提出。他社より半日早い
AM 10:30 経営ダッシュボードで管理戸数・空室率・入金率・修繕予算消化率を確認
         → 「A物件の空室率が上昇傾向。リフォーム提案をオーナーに出す」
PM 1:00  内見アテンドに集中。AIがCRMで追客メールを自動送信
PM 4:00  退去立会い。AIが原状回復ガイドラインに基づく負担区分表を自動作成
PM 5:00  更新管理AIが来月の更新10件を自動通知済み。書類も下書き完了
PM 5:30  定時退社。夜間は24時間対応AIが一次受け。緊急時のみ電話転送。
         今日は「経営」ができた。来期の新規管理獲得戦略を考える時間もあった。
```

### 1.3 2025-2027年の業界変化

**賃貸住宅管理業法の影響（2021年施行→2025年本格運用）**

管理戸数200戸以上の事業者は国交省への登録が義務化。
管理受託契約の重要事項説明、財産の分別管理、定期報告が法的義務に。
→ コンプライアンス対応コストが増大。中小は「法令遵守しながら日常業務を回す」だけで精一杯。

**不動産IDの段階導入（2025年〜）**

全不動産に一意のIDが付与される。物件情報の名寄せ・履歴管理が劇的に楽になる一方、
対応システムの導入コストが発生。

```
→ データに基づく賃料設定・修繕提案・収支シミュレーションが差別化要因
```

### 2.2 管理戸数規模別の課題

| 規模 | 推定社数 | 従業員数 | 主な課題 | シャチョツーの価値 |
|---|---|---|---|---|
| **50-100戸** | ~5,000社 | 1-3名 | 社長が全業務兼務。入金管理が限界 | 入金消込+オーナー報告の自動化で月40時間削減 |
| **100-300戸** | ~6,000社 | 3-8名 | 事務員はいるが属人化。督促・更新漏れ | 督促自動化+更新管理で滞納率3pt改善 |
| **300-500戸** | ~2,500社 | 5-15名 | システムは入れたが使いこなせない。修繕管理が破綻 | 修繕管理+収支分析でオーナー満足度向上 |
| **500-1,000戸** | ~1,000社 | 10-30名 | 部門間連携（仲介→管理→修繕）が分断 | 全モジュール統合でワンストップ管理 |
| **1,000戸以上** | ~500社 | 30名超 | 既にシステム導入済み。カスタマイズ性が課題 | Phase 2+で対応（API連携型） |

**最優先ターゲット: 50-500戸の管理会社 約13,500社**

理由:
1. 社長が即決できる（稟議不要）
2. 既存システムへのロックインが弱い（Excel/紙が多い）
3. 管理業法対応の負担が重く、AIへのニーズが切実
4. 紹介ネットワークが強い（地域の管理会社コミュニティ）

## 3. 不動産業務の全貌（これが分かってないと話にならない）

### 3.1 賃貸管理（PM）の年間業務サイクル

```
■ 毎月の定型業務
  1日    家賃入金確認開始（銀行振込の突合）
  1-5日  入金消込（口座振替+振込+クレジット。名義照合が地獄）
  5-10日 未入金者への督促（電話→SMS→書面→保証会社→法的手続き）
  10日   オーナー送金（家賃-管理費-修繕費-借入返済を差し引いて送金）
  15日   オーナー収支報告書作成・送付
  随時   入退去対応（申込受付→審査→契約→鍵渡し / 解約受付→立会い→原状回復→募集）
  随時   修繕・クレーム対応（24時間365日）
  随時   更新管理（満了2ヶ月前に通知→条件交渉→更新契約→火災保険更新）

■ 年次業務
  1月    確定申告用資料作成（オーナー向け年間収支報告書）
  3月    繁忙期（入退去ピーク。管理戸数300戸で月20-30件の入退去）
  4月    繁忙期終了。空室対策の見直し
  7月    路線価発表。相続相談が増える
  10月   来年度の管理委託契約更新の準備
  12月   年末の退去ラッシュ対応

■ 不定期
  大規模修繕（築10-15年で外壁・防水。築20-30年で設備更新）
  相続対応（オーナー死亡時の相続人対応・名義変更）
  賃貸住宅管理業の定期報告（年1回、国交省へ）
```

### 3.2 入金消込の実態（最も工数がかかる業務）

```
■ なぜ入金消込が地獄か

  1. 入金手段がバラバラ
     口座振替:   60-70%（引き落とし日が統一できない）
     銀行振込:   20-30%（名義が一致しない。法人/個人/家族名義）
     現金持参:    3-5%（高齢者。受領書発行が必要）
     クレジット:  1-3%（導入している管理会社は少ない）

  2. 名義不一致のパターン
     ・契約者「山田太郎」→ 振込名義「ヤマダ タロウ」（カタカナ変換）
     ・契約者「山田太郎」→ 振込名義「カ）ヤマダショウジ」（法人名で振込）
     ・契約者「山田太郎」→ 振込名義「ヤマダ ハナコ」（配偶者が振込）
     ・金額不一致: 賃料¥75,000のところ¥74,000入金（駐車場代を引いてる？）
     ・複数月分まとめて振込: ¥150,000（2ヶ月分？ 1ヶ月+敷金？）

  3. 300戸の場合の工数
     口座振替: 200戸 → 自動消込可能。ただし引落不能が5-10件/月
     銀行振込: 80戸 → 1件3-5分の手動照合 × 80件 = 4-7時間
     未入金: 15-20戸 → 督促リスト作成1時間 + 電話/SMS 1件10分 × 15件 = 3.5時間
     合計: 月8-12時間（月末の1-5日に集中）

  4. シャチョツーの解決
     銀行API → 入金データ自動取得
     AIで名義マッチング（ファジーマッチ + 過去パターン学習）
     一致率97%以上。残り3%のみ人間確認
     月8-12時間 → 月1-2時間に削減
```

### 3.3 売買査定の実態

```
■ 売買査定のフロー
  Step 1: 物件情報の確認（所在地・面積・築年数・構造・間取り）
  Step 2: レインズで成約事例を検索（30分-1時間）
  Step 3: ポータルサイトで売出事例を確認（15-30分）
  Step 4: 取引事例比較法で概算（事例の補正作業が属人的）
  Step 5: 収益還元法（投資物件の場合）
  Step 6: 原価法で検算
  Step 7: 査定書作成（30分-1時間）
  → 合計1-2時間。ベテランの勘に±20%のブレ

■ 一括査定サイトの圧力
  HOME4U、イエウール、すまいValue等から査定依頼が流れてくる。
  1件の依頼に対して3-6社が競合。
  回答スピードが最大の差別化要因（「最初に電話をかけた会社に依頼」が40%）。
  → 査定書の作成が遅い会社は、永遠に媒介が取れない。

■ 査定ミスのリスク
  高すぎる査定: 売主は喜ぶが売れない → 長期滞留 → 価格改定 → 信用失墜
  低すぎる査定: 他社の高値査定に負けて媒介を逃す
  → 「根拠のある適正価格」を素早く出せるかが勝負

■ シャチョツーの解決
  物件情報入力 → 10分で3手法による査定書PDF自動生成
  公開データ（国交省API + 路線価 + 地価公示）ベースでLevel 1
  自社成約データ蓄積でLevel 2（±8%精度）
  → 一括査定サイト経由の依頼に最速で回答 → 媒介獲得率向上
```

### 3.4 重要事項説明（重説）の実態

```
■ 重説が必要な場面
  ・売買契約前（35条書面）: 宅建士による説明義務
  ・賃貸契約前（35条書面）: 同上
  ・2022年5月〜: IT重説（オンライン）解禁 → でも書面作成の工数は変わらない

■ 重説に記載する項目（売買の場合、主要なもの）
  1. 登記簿に記録された事項（権利関係・抵当権）
  2. 法令上の制限（都市計画法・建築基準法・条例）
  3. 道路・排水・電気・ガス等のインフラ
  4. 造成宅地防災区域・土砂災害警戒区域・津波災害警戒区域
  5. アスベスト・耐震診断の有無
  6. 住宅性能評価
  7. マンション管理規約・修繕積立金の状況
  8. 契約の解除に関する事項
  9. 損害賠償額の予定・違約金
  10. 手付金等の保全措置

■ 1件あたりの作成工数
  賃貸: 30分-1時間（テンプレ化しやすいが、物件固有の注意事項が毎回違う）
  売買: 1-3時間（登記簿・法令制限の確認に時間がかかる）
  → 宅建士が社長1人の会社では、重説がボトルネック

■ シャチョツーの解決
  物件情報 + 登記データ → AIが重説ドラフトを自動生成
  法令制限は用途地域・ハザードマップAPIから自動取得
  過去の重説パターンを学習 → 物件タイプ別の注意事項を自動挿入
  1件1-3時間 → 15-30分（宅建士の確認・署名のみ）
```

### 3.5 修繕・設備管理の実態

```
■ 修繕対応のフロー
  入居者から連絡（電話/LINE/メール）
  → 管理会社が状況確認（現地確認が必要な場合も）
  → 業者手配（水道/電気/建具/ガス等の専門業者）
  → 見積取得（1-3日）
  → オーナー承認（金額が大きい場合。¥5万以上が目安）
  → 工事実施（1日-2週間）
  → 完了確認・入居者報告
  → オーナーへ請求（収支報告書に反映）

  リードタイム: 2-3週間が平均。緊急（水漏れ・エアコン故障）は当日対応が必要

■ 修繕の分類
  緊急修繕:   水漏れ/ガス漏れ/鍵故障/エアコン故障（夏季）→ 当日対応必須
  一般修繕:   設備交換/クロス張替/建具調整 → 1-2週間
  大規模修繕: 外壁塗装/防水/屋上/配管更新 → 計画的（築10-15年周期）
  原状回復:   退去後の現状復帰 → 次の入居者募集前に完了必須

■ 修繕でもめるポイント
  ・オーナー負担 vs 入居者負担の線引き（原状回復ガイドライン）
  ・通常損耗（経年劣化）はオーナー負担 → 入居者に請求すると訴訟リスク
  ・経過年数による減価（クロス6年/設備15年/建具等は物件の耐用年数）
  ・見積が適正かどうか（相見積もりを取る工数がない）

■ シャチョツーの解決
  修繕依頼を自動分類（緊急度判定AI）
  緊急 → 提携業者に自動手配+入居者にSMS「本日中に対応します」
  一般 → 見積AI（過去の修繕実績から概算）→ オーナー承認フロー
  大規模 → 長期修繕計画AIが築年数・設備状況から予算計画を自動作成
  原状回復 → ガイドラインAIが写真+築年数から負担区分表を自動作成
```

| 電子署名サービス | 契約の電子化 | Phase 2+ |
| レインズ | 成約事例（閉鎖NW→CSV取込） | Phase 1はCSV |

### 8.3 学習ループ

```
1. AIが査定/消込/判定を実行（Confidence 0.4-0.7）
2. 担当者が確認・修正（「この入金は山田太郎のもの」「査定価格を¥200万下げる」）
3. 修正データをDBに保存
4. 次回同パターン → 過去実績を自動適用
5. 3ヶ月の運用 → Confidence 0.8-0.9+

入金消込の学習:
  「カ）ヤマダ = 山田太郎 = 102号室」を1回教えると、次月から自動消込

査定の学習:
  「実際の成約価格」をフィードバックすると、地域×物件タイプ別の精度が向上

修繕の学習:
  「この症状にはこの業者が最適」「この設備は10年で交換」のパターン蓄積
```

使えば使うほど、その会社専用の管理AIに育つ。

## 9. 導入フロー

### Phase 1（Month 1）: データ吸収

```
Week 1: アカウント開設・初期設定
  ・ブレイン基本情報入力（会社情報・管理物件一覧・オーナー情報）
  ・入金口座の設定（銀行API連携 or CSVインポート設定）
  ・入居者マスタの取り込み（既存管理ソフトからCSVエクスポート→インポート）

Week 2-3: 過去データの取り込み
  ・過去12ヶ月の入金データ取込（名義パターン学習用）
  ・過去の売買査定実績（成約データがあれば10-20件）
  ・修繕履歴（業者・費用・対応時間のパターン学習用）

Week 4: シャドーモード開始
  ・入金消込: AIと手動の並走。差異を確認して精度向上
  ・査定: AI査定ドラフトと手動査定を比較
  ・修繕: AI分類と手動判断を比較
```

### Phase 2（Month 2-3）: 並走運用

```
・入金消込AIが主、人間が確認 → 自動消込率90%→97%に向上
・査定書AIが下書き → 社長が確認・微修正 → 提出
・オーナー報告書が自動生成 → 確認して送信
・月次精度レポートで改善状況を可視化
```

### Phase 3（Month 4+）: 自走

```
・入金消込は99%自動。不明3%のみ人間確認
・定型査定（類似事例が豊富なエリア）はAI完結
・特殊案件（事業用・再開発地域・相続関連）のみ人間判断
・オーナー向け月次レポートに市場トレンド分析を追加
・四半期: ポートフォリオ分析 + 賃料改定提案
```
