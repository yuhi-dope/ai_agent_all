## 0. エグゼクティブサマリー

### なぜ経営インテリジェンスか

```
中小製造業の社長の現実（経営判断）:
  ・全社PLは見ているが「どの加工が儲かっているか」が分からない
  ・設備投資は経験と勘。「本当にペイするのか」の定量根拠がない
  ・工場移転/新設を検討しても、候補地の評価が感覚的
  ・国際情勢（為替・材料相場・需要変動）が自社にどう影響するか判断材料がない
  ・自社の加工技術が将来も通用するか見えない
  ・3ヶ月先の資金繰りが見えず、突然の資金ショートに怯えている
  ・熟練技能者の退職で「誰もできない工程」が発生するリスクを放置
  ・社長の平均年齢62歳。事業承継の準備が進んでいない

  → ①-⑧が現場の「作業」をAI化。⑨-⑯は社長の「経営判断」をAI化
  → データは①-⑧で蓄積済み。あとは分析するだけ
  → 8モジュール合計で年間1,400-2,400万円の意思決定価値
```

## 9. PLインテリジェンス（加工種別PL分析）

### 9.1 課題（金額換算の価値）

```
■ 直接効果①：赤字加工種別の特定
  「CNC旋盤は儲かっていると思っていたが、光熱費を按分したら粗利15%しかなかった」
  → 加工種別を絞る/チャージレートを見直すだけで年200-500万円改善

■ 直接効果②：価格交渉の根拠
  「この加工種別の粗利率は業界平均25%に対して18%です」
  → 根拠のある値上げ交渉が可能（年100-300万円の値上げ実現）

■ 直接効果③：社長の月次経営判断の速度向上
  月末に経理が3日かけてまとめていた資料 → 月初1時間で自動生成
  → 経営判断が3日早まる × 12ヶ月

■ 合計効果：年300-800万円
  → BPO月10万円（年120万円）で ROI 2.5-6.7倍
```

#### Step 3: calculator（加工種別PL算出）

| 項目 | 内容 |
|---|---|
| マイクロエージェント | `micro/cost_calculator`（calc_type=`process_pl`） |
| 入力 | Step 2の按分済みデータ |
| 処理 | 加工種別PLを算出 |
| 出力 | `process_pl_results[]`（加工種別ごとのPL） |

**計算式**

```python
# 加工種別PL計算
for process_type in process_types:
    # 製造原価
    material_cost = Σ(材料費_直課分)
    outsourcing_cost = Σ(外注費_直課分)
    direct_labor_cost = 直接工時間(h) × 時給(¥/h)
    depreciation = 設備別_帳簿価額 / 耐用年数 / 12  # 月額

    # 光熱費按分
    utility_cost = 設備別消費電力(kW) × 月間稼働時間(h) × 電力単価(¥/kWh)
    # 電力単価参考: ¥25-35/kWh（高圧受電の場合。低圧は¥30-40/kWh）

    # 間接工按分
    indirect_labor = 間接工人件費_総額 × (当該加工種別の直接工時間 / 全加工種別の直接工時間合計)

    # 管理部門按分
    admin_cost = 管理部門人件費_総額 × (当該加工種別の売上 / 全加工種別の売上合計)

    # 製造原価合計
    manufacturing_cost = (material_cost + outsourcing_cost + direct_labor_cost
                         + depreciation + utility_cost + indirect_labor + admin_cost)

    # 粗利
    gross_profit = revenue - manufacturing_cost
    gross_margin = gross_profit / revenue * 100  # 粗利率(%)

    # 労務費比率
    labor_ratio = (direct_labor_cost + indirect_labor) / revenue * 100

    # 光熱費比率
    utility_ratio = utility_cost / revenue * 100
```

```python
# 粗利率の判定

    severity = "WARNING"   # 業界平均未達
    suggestion = "チャージレート見直し or 稼働率改善を検討"
elif actual_margin > benchmark_high:
    severity = "EXCELLENT"  # 業界平均超え（強み）
```

#### Step 3: calculator（投資回収計算）

| 項目 | 内容 |
|---|---|
| マイクロエージェント | `micro/cost_calculator`（calc_type=`capex_simulation`） |
| 入力 | 設備仕様 + 稼働条件 + 財務前提 |
| 処理 | 回収期間・NPV・IRRの算出 |
| 出力 | `capex_result`（payback_years, npv, irr, annual_cf[]） |

**計算式**

```python
# 年間キャパシティ（売上ポテンシャル）
annual_capacity_hours = operating_days * daily_hours * utilization_rate
annual_revenue_potential = annual_capacity_hours * charge_rate  # ¥/h

# 年間増分コスト
depreciation = investment_amount / useful_life            # 定額法
annual_power_cost = power_kw * daily_hours * operating_days * power_unit_price  # ¥25-35/kWh
annual_maintenance = investment_amount * maintenance_rate  # 3-8%
annual_labor_increase = additional_operators * annual_salary  # 増員が必要な場合

annual_incremental_cost = depreciation + annual_power_cost + annual_maintenance + annual_labor_increase

# 年間増分利益
annual_incremental_profit = annual_revenue_potential * gross_margin_rate - annual_incremental_cost

# 回収期間（単純）
payback_years = investment_amount / annual_incremental_profit

# NPV（正味現在価値）
discount_rate = 0.05  # 5%想定（中小製造業のWACC相場）
npv = sum(annual_cf / (1 + discount_rate) ** t for t in range(1, useful_life + 1)) - investment_amount
# annual_cf = annual_incremental_profit + depreciation（減価償却は非現金）

# IRR（内部収益率）
# NPV = 0 となるdiscount_rateを二分探索で算出
irr = solve_irr(investment_amount, annual_cfs)

# 例: CNC旋盤新品 ¥3,000万
#   年間キャパ: 240日 × 8h × 80% = 1,536h
#   年間売上: 1,536h × ¥8,000/h = ¥12,288,000
#   年間コスト: 減価償却¥300万 + 電力¥86万 + メンテ¥120万 + 人件費¥400万 = ¥906万
#   年間利益: ¥1,229万 × 25% - ¥906万 ≈ ▲¥598万 ... 稼働率/受注が不十分
#   → 粗利込みの年間増分利益で計算:
#   年間粗利: ¥1,229万 × 25% = ¥307万... → 既存案件の生産性向上で計算し直し
```

**各スコアの計算式**

```python
# ① 労働力スコア（25点）
labor_score = (
    normalize(有効求人倍率, lower_is_better=True) * 10    # 低いほど人が集まりやすい
    + normalize(製造業従事者数, higher_is_better=True) * 8  # 多いほど採用しやすい
    + normalize(外国人技能実習生受入数, higher_is_better=True) * 4  # 受入実績
    + normalize(工業高校・高専数, higher_is_better=True) * 3  # 若手人材の供給源
)

# ② コストスコア（25点）
cost_score = (
    normalize(坪単価, lower_is_better=True) * 10
    + normalize(固定資産税率, lower_is_better=True) * 5
    + normalize(電力単価, lower_is_better=True) * 5
    + normalize(最低賃金, lower_is_better=True) * 5
)

# ③ サプライチェーンスコア（20点）
supply_chain_score = (
    normalize(同業種集積度, higher_is_better=True) * 8   # 協力企業が近い
    + normalize(材料調達拠点数, higher_is_better=True) * 6  # 鋼材商社等
    + normalize(表面処理・熱処理業者数, higher_is_better=True) * 6  # 外注先
)

# ④ 物流スコア（15点）
logistics_score = (
    normalize(高速IC距離, lower_is_better=True) * 6
    + normalize(港湾距離, lower_is_better=True) * 4      # 輸出がある場合
    + normalize(主要顧客への所要時間, lower_is_better=True) * 5
)

# ⑤ 補助金スコア（15点）
incentives_score = (
    normalize(税減免率, higher_is_better=True) * 5
    + normalize(設備補助率, higher_is_better=True) * 5
    + normalize(雇用助成金額, higher_is_better=True) * 5
)

total_score = labor_score + cost_score + supply_chain_score + logistics_score + incentives_score
```

**計算式**

```python
# 為替感応度

# 例: 輸入材料比率30% × 円安10% = +3%のコスト増

# 材料コスト影響
material_impact = material_cost_ratio * commodity_change_rate
# 例: 材料費比率25% × アルミ相場+15% = 3.75%のコスト増
# 年間売上5億 × 3.75% = ¥1,875万のインパクト

# 需要変動予測
demand_forecast = sum(
    industry_share * industry_growth_rate
    for industry, share in portfolio.items()
)
# 例: 自動車(ICE)60% × ▼25% + EV10% × ▲50% + 半導体20% × ▲20% + 産機10% × ▲5%
```


#### Step 4: calculator（技術ポジショニングスコア）

| 項目 | 内容 |
|---|---|
| マイクロエージェント | `micro/cost_calculator`（calc_type=`tech_positioning`） |
| 入力 | 技術評価結果 + 市場トレンド（⑫連携） |
| 処理 | 3軸スコアリング |
| 出力 | `positioning_scores[]`（技術別のスコア） |

**計算式**

```python
# ① 成長性スコア（0-100）
# 自社技術の市場需要トレンドに基づく
growth_score = weighted_avg(
    industry_growth_rate * industry_share  # ⑫から取得
    for industry in served_industries
) * 100

# ② 代替リスクスコア（0-100、高いほどリスク大）
substitution_risk = (
    tech_shift_risk_level * 40           # 技術シフトマップのリスクレベル
    + equipment_remaining_life_factor * 30  # 設備残存年数が短い=更新必要
    + competitor_adoption_rate * 30       # 競合の新技術採用率
)
# equipment_remaining_life_factor = 1 - (remaining_years / useful_life)

# ③ 補完機会スコア（0-100）
# 追加1設備で開拓できる市場の規模
complementary_score = (
    new_market_size * 40                 # 開拓可能な市場規模
    + synergy_with_existing * 30         # 既存設備との相乗効果
    + investment_accessibility * 30      # 投資のハードル（低いほど高得点）
)

# 総合ポジショニング
# 成長性が高く、代替リスクが低く、補完機会が大きい → 投資優先度高
investment_priority = growth_score * 0.4 + (100 - substitution_risk) * 0.3 + complementary_score * 0.3
```

#### Step 5: generator（技術ロードマップ提案書生成）

**ロードマップ構成**

```
1. 現在の技術ポートフォリオ（設備構成×加工実績×品質レベル）
2. 技術環境分析（各保有技術の将来性+代替リスク）
3. 短期提案（1年以内）
   例: 「ワイヤカット1台追加 → 金型部品の内製化（外注費¥500万/年削減）」
4. 中期提案（1-3年）
   例: 「5軸MC導入 → 航空機部品への参入（粗利率+15%）」
5. 長期提案（3-5年）
   例: 「金属3Dプリンタ → 試作の内製化（リードタイム1/3）」
6. 投資優先順位マトリクス
   横軸: 投資額 / 縦軸: 期待リターン
   → 右上象限=最優先、左下象限=見送り
7. ⑩設備投資シミュレーターとの連携結果（各提案の回収期間）
```

```

  ・金属3Dプリンタ（¥5,000万）
```

### 14.1 課題（金額換算の価値）

```

■ 直接効果④：経営判断の速度向上
  「この設備投資をすると3ヶ月後のキャッシュはどうなるか」を即座にシミュレーション
  → ⑩設備投資シミュレーターとの連動で投資判断の精度向上

■ 合計効果：年500-1,000万円（倒産回避の安心感を含む）
```

#### Step 3: calculator（3ヶ月先の資金繰り予測）

| 項目 | 内容 |
|---|---|
| マイクロエージェント | `micro/cost_calculator`（calc_type=`cashflow_forecast`） |
| 入力 | Step 2の構造化データ + 受注残（②生産計画AI連携）+ 設備投資計画（⑩連携） |
| 処理 | 3ヶ月先の月末キャッシュ残高を予測 |
| 出力 | `monthly_forecasts[3]`（3ヶ月分の予測CF） |

**計算式**

```python
for month in forecast_months:  # 3ヶ月分
    # === 営業CF ===

    # 売掛金回収予測
    receivable_collection = (
        receivables_balance
        × collection_rate_by_aging(aging_days)  # エイジング別回収率
        × seasonality_index[month]               # 季節指数
    )
    # エイジング別回収率（過去実績から算出）
    # 0-30日: 95%、31-60日: 85%、61-90日: 70%、91-120日: 50%、121日超: 20%
    COLLECTION_RATE_BY_AGING = {
        (0, 30):   0.95,
        (31, 60):  0.85,
        (61, 90):  0.70,
        (91, 120): 0.50,
        (121, 999): 0.20,  # 長期滞留は回収困難
    }

    # 新規売上入金予測
    new_revenue_forecast = (
        order_backlog × delivery_rate[month]     # 受注残からの出荷予測
        + recurring_revenue                       # リピート受注の予測（過去実績ベース）
    ) × seasonality_index[month]

    # 売上入金（回収サイト考慮）
    # 中小製造業の平均回収サイト: 月末締め翌月末払い（30-60日）
    revenue_cash_in = new_revenue_forecast × (1 - average_collection_days / 30)
                    + prev_month_revenue_forecast  # 前月売上の入金分

    # 買掛金支払予測
    payable_payment = (
        payables_balance                         # 既存買掛金
        + new_purchase_forecast                  # 新規発注予測（受注残から推定）
    )
    # 新規発注予測 = 受注残 × 材料費比率 × 前倒し発注率
    new_purchase_forecast = order_backlog × material_cost_ratio × 0.8  # 80%が前倒し発注

    # 人件費（固定）
    labor_cost = monthly_labor_total              # 月次人件費（ほぼ固定）

    # その他経費（固定+変動）
    other_opex = fixed_opex + (variable_opex_ratio × new_revenue_forecast)

    # 営業CF合計
    operating_cf = (
        receivable_collection + revenue_cash_in
        - payable_payment - labor_cost - other_opex
    )

    # === 投資CF ===
    # ⑩設備投資シミュレーターとの連動
    investing_cf = -planned_capex[month]          # 設備投資支出（⑩から取得）
                 + planned_asset_sales[month]     # 設備売却予定
                 + expected_subsidies[month]       # 補助金入金予定

    # === 財務CF ===
    financing_cf = (
        new_borrowing[month]                     # 新規借入予定
        - loan_repayment_schedule[month]          # 返済スケジュール（借入金DBから取得）
    )

    # === 月末キャッシュ残高 ===
    month_end_cash = prev_month_end_cash + operating_cf + investing_cf + financing_cf

    # === 安全余裕度 ===
    monthly_revenue_avg = annual_revenue / 12
    safety_months = month_end_cash / monthly_revenue_avg  # 月商の何ヶ月分か
```

**月商比率による安全水準**

| 安全余裕度（月商比） | 判定 | 意味 |
|---|---|---|
| 3ヶ月分以上 | SAFE | 十分な手元資金。成長投資が可能 |
| 2-3ヶ月分 | NORMAL | 標準的。突発的な支出にも対応可能 |
| 1-2ヶ月分 | WARNING | 要注意。資金繰り改善策の検討が必要 |
| 1ヶ月分未満 | CRITICAL | 危険。即座に銀行相談 or 回収前倒し交渉 |
| マイナス | EMERGENCY | 資金ショート。緊急融資 or リスケ交渉 |

**計算式**

```python
# === 個人別退職リスクスコア（0-100、高いほどリスク大） ===
def calc_retirement_risk(employee):
    age_factor = min((employee.age - 55) / 10, 1.0) * 35  # 55歳未満=0、65歳=35点
    # 55-65歳の10年間で線形にリスク上昇

    tenure_factor = min(employee.tenure_years / 30, 1.0) * 15  # 勤続30年で15点
    # 長期勤続者ほど定年退職が近い

    skill_criticality = max(
        SKILL_LEVEL_WEIGHT[skill_level] * (1 / max(substitute_count, 1))
        for process, skill_level in employee.skills.items()
    ) * 30
    # 高スキル × 代替者少数 = 高クリティカリティ
    # SKILL_LEVEL_WEIGHT = {0: 0, 1: 0.2, 2: 0.6, 3: 1.0}

    employment_type_factor = {
        "正社員": 0,
        "契約社員": 10,       # 契約満了リスク
        "技能実習生": 20,     # 在留期限リスク（最大5年）
    }.get(employee.employment_type, 5)

    retirement_risk = age_factor + tenure_factor + skill_criticality + employment_type_factor
    return min(retirement_risk, 100)

# === 工程別技能断絶リスク ===
def calc_skill_gap_risk(process, skill_matrix):
    workers_at_level = {
        level: count_workers_at_level(process, level, skill_matrix)
        for level in [0, 1, 2, 3]
    }

    # 指導可(3)が0人 → CRITICAL（伝承者不在）
    if workers_at_level[3] == 0:
        return "CRITICAL", "指導可能な人材がゼロ。外部から招聘 or 即座に育成開始"

    # 単独(2以上)が1人のみ → HIGH（1人が抜けたら止まる）
    if workers_at_level[2] + workers_at_level[3] <= 1:
        return "HIGH", "単独作業可能な人材が1名のみ。退職でラインが止まる"

    # 指導可(3)が1人、かつその人の退職リスクが50以上 → HIGH
    if workers_at_level[3] == 1 and get_retirement_risk(sole_instructor) >= 50:
        return "HIGH", f"{sole_instructor.name}({sole_instructor.age}歳)の退職で指導体制が崩壊"

    # それ以外
    return "NORMAL", "複数名が対応可能。育成計画は通常ペースで"

# === 採用必要数 ===
def calc_hiring_needs(employees, skill_matrix, forecast_years=3):
    # 3年以内の退職予測
    predicted_retirements = [
        emp for emp in employees
        if emp.age + forecast_years >= 60  # 定年60歳想定（再雇用は65歳まで）
        or (emp.employment_type == "技能実習生"
            and remaining_visa_months(emp) <= forecast_years * 12)
    ]

    # 育成中の人数（レベル1の人がレベル2になるまで平均2年）
    in_training = count_workers_at_level_1(skill_matrix)

    # 採用必要数 = 退職予測 - 育成中
    hiring_needs = max(len(predicted_retirements) - in_training, 0)

    # 工程別の内訳
    hiring_by_process = {}
    for emp in predicted_retirements:
        for process, level in emp.skills.items():
            if level >= 2:  # 単独以上の技能が失われる
                hiring_by_process[process] = hiring_by_process.get(process, 0) + 1

    return {
        "total_hiring_needs": hiring_needs,
        "predicted_retirements": len(predicted_retirements),
        "in_training": in_training,
        "hiring_by_process": hiring_by_process,
        "estimated_cost": hiring_needs * 1_200_000,  # 1人あたり採用コスト¥120万
    }
```

```
■ twin（デジタルツイン5次元 → ヒト軸+リスク軸）

    ・退職波（同時期に複数名退職）のリスク

■ proactive（能動提案）

    ・退職リスク50以上の従業員 → 育成計画の策定を自動提案
    ・在留期限6ヶ月以内 → 更新手続きリマインダー

  育成計画 → 実際の育成期間 → 予測精度の向上
```

## 16. 事業承継・企業価値評価

### 16.1 課題（金額換算の価値）

```
■ 直接効果①：事業承継の準備遅延防止
  中小製造業の約30%が後継者未定（中小企業庁 事業承継ガイドライン）
  社長の平均年齢62歳。承継準備には5-10年かかる
  → 「社長が65歳になる前に承継計画を策定すべき」を定量的に提示
  → 準備なしの廃業 = 従業員の雇用喪失 + 技術消滅 + 取引先への影響

■ 直接効果②：企業価値の適正評価
  M&Aの場面で「自社の値段が分からない」→ 安く買い叩かれる or 高すぎて成立しない
  → 3手法（EBITDA倍率/修正純資産/簡易DCF）で適正レンジを提示
  → M&A仲介手数料の交渉材料にも（仲介手数料: 通常5-10%。¥1億の企業で¥500-1,000万）

■ 直接効果③：税負担の最適化
  事業承継税制（特例措置）の適用可否 → 最大で相続税/贈与税の全額猶予
  → 株式評価額¥5,000万の場合、相続税率30% = ¥1,500万の税負担軽減
  → 知らずに期限を逃すケースが多い

■ 直接効果④：承継後の経営安定化
  後継者に「今の会社の実力」を定量的に引き継ぎ
  → 後継者の経営判断の精度向上 → 承継後3年間の経営安定化

■ 合計効果：年500-1,500万円（承継準備期間を通じた価値）
  → BPO月10万円（年120万円）で ROI 4.2-12.5倍
```

```python
# EBITDA正規化（一過性損益の除外）
def normalize_ebitda(financial_data):
    ebitda = financial_data["operating_profit"] + financial_data["depreciation"]

    # 一過性損益の調整（見つかった場合）
    adjustments = []
    # 役員報酬の市場水準との差額（中小企業は過大/過小なことが多い）
    # 中小製造業の適正役員報酬: ¥1,200-1,800万/年（売上5億規模）
    officer_comp_adjustment = actual_officer_comp - market_officer_comp
    if abs(officer_comp_adjustment) > 3_000_000:
        adjustments.append({"item": "役員報酬調整", "amount": officer_comp_adjustment})
        ebitda += officer_comp_adjustment

    # 不動産賃料の調整（社長個人所有の不動産を会社に貸しているケース）
    # 保険料の調整（経営者向け生命保険の過大計上）
    # 一過性の修繕費・訴訟費用等

    normalized_ebitda = ebitda + sum(adj["amount"] for adj in adjustments)
    return normalized_ebitda, adjustments

# FCF（フリーキャッシュフロー）
def calc_fcf(financial_data):
    fcf = (
        financial_data["net_profit"]
        + financial_data["depreciation"]
        - financial_data["capex"]
        - working_capital_change  # 運転資本の増減
```

```python
def modified_net_asset_valuation(financial_data, equipment_data):
    # 帳簿上の純資産
    book_net_assets = financial_data["net_assets"]

    # 設備の含み益/含み損
    equipment_adjustment = sum(
        eq["market_value"] - eq["book_value"]
        for eq in equipment_data
    )
    # 注: 償却済み設備でも市場価値がある場合がある
    # 例: 30年前のCNC旋盤（簿価0）→ 中古市場で¥200万の値がつく

    # 不動産の含み益（該当する場合）
    # 工場の土地: 取得時の簿価 vs 現在の路線価/公示地価
    real_estate_adjustment = current_land_value - book_land_value

    # 営業権（のれん）= 超過収益力の評価
    # 3年間の平均超過利益 × 3年分（簡易計算）
    industry_avg_roi = 0.05  # 製造業の平均ROA 5%
    excess_profit = ordinary_profit - (total_assets * industry_avg_roi)
    goodwill = max(excess_profit * 3, 0)  # マイナスの場合は0

    modified_net_assets = (
        book_net_assets
        + equipment_adjustment
        + real_estate_adjustment
        + goodwill
    )
    return modified_net_assets

# 手法3: DCF法（簡易版）
# ========================================

def simplified_dcf_valuation(fcf_history, growth_assumptions):
    # 過去3年のFCFから将来5年分を予測
    base_fcf = weighted_avg_fcf(fcf_history, weights=[0.2, 0.3, 0.5])  # 直近重視

    # 成長率の仮定
    # 中小製造業: 0-3%/年が現実的（高成長は期待しない）
    growth_rate = min(
        historical_revenue_cagr,  # 過去3年のCAGR
        0.03                       # 上限3%
    )

    # 割引率（WACC）
    # 中小製造業のWACC: 8-12%が一般的
    # （サイズプレミアム+非上場ディスカウントを含む）
    wacc = 0.10  # 10%をデフォルト

    # 5年分のFCF現在価値
    pv_fcfs = sum(
        base_fcf * (1 + growth_rate) ** t / (1 + wacc) ** t
        for t in range(1, 6)
    )

    # 残存価値（ターミナルバリュー）
    # 永久成長率: 0-1%
    terminal_growth = 0.005  # 0.5%
    terminal_value = (
        base_fcf * (1 + growth_rate) ** 5 * (1 + terminal_growth)
        / (wacc - terminal_growth)
    )
    pv_terminal = terminal_value / (1 + wacc) ** 5

    # 企業価値
    enterprise_value = pv_fcfs + pv_terminal

    # 株主価値 = 企業価値 - 有利子負債 + 現預金
    equity_value = enterprise_value - interest_bearing_debt + cash_and_deposits

    return equity_value

    # 目的に応じて重み付けを変更
    if purpose == "ma":
        # M&Aの場合: EBITDA倍率法を重視（買い手の視点）
        weights = {"ebitda": 0.50, "net_asset": 0.30, "dcf": 0.20}
    elif purpose == "succession_family":
        # 親族承継の場合: 修正純資産法を重視（税務評価に近い）
        weights = {"ebitda": 0.20, "net_asset": 0.50, "dcf": 0.30}
    else:
        # その他: 均等
        weights = {"ebitda": 0.35, "net_asset": 0.35, "dcf": 0.30}

    weighted_value = (
        ebitda_val * weights["ebitda"]
        + net_asset_val * weights["net_asset"]
        + dcf_val * weights["dcf"]
    )
    return weighted_value, weights
```

### 16.10 制約・前提条件

- Phase 1: 財務データは手動入力 or freee CSV。3期分の決算書が必要
- 株式評価は概算値。正式な税務評価には税理士の確認が必須（免責事項を明示）
- 事業承継税制の適用可否は概算チェックのみ。正式な認定申請は専門家に委託
- EBITDAマルチプルは初期値固定（製造業のサブセグメント別）。M&Aプラットフォーム連携はPhase 2+
- 不動産の時価評価は路線価ベースの簡易計算。不動産鑑定士の評価はPhase 2+
- 税額計算は概算。確定申告に使用不可（免責事項必須）
- 海外資産/海外取引先がある場合の国際課税はPhase 3+

## セキュリティ・コンプライアンス

### 保護対象（企業秘密）

```
経営インテリジェンス固有の保護対象:
  ・加工種別PL（粗利率/コスト構造）→ 競合に知られると価格交渉で圧倒的に不利
  ・設備投資計画 → 競合の先手を取られるリスク
  ・顧客ポートフォリオ（業界構成比）→ 営業秘密の中核
  ・技術ポートフォリオ（加工精度/Cpk）→ 技術力の定量指標
  ・立地戦略（候補地/予算） → 不動産交渉で不利
  ・キャッシュフロー予測（残高/資金繰り） → 信用情報の中核。漏洩で取引停止リスク
  ・従業員技能マトリクス（個人名+スキルレベル） → 個人情報+ヘッドハンティングリスク
  ・企業価値評価（算定額/承継計画） → 最高機密。M&A交渉で圧倒的に不利

保護措置:
  ① RLS: 全テーブル company_id ベース。テナント分離必須
  ② 匿名化: 横断分析時はcompany_id→ハッシュ化。個社特定不可
  ③ アクセスログ: PL/投資/技術データの閲覧/ダウンロードを全件記録
  ④ エクスポート制限: レポートのPDFダウンロードはadminロールのみ
  ⑤ 市場データ共有: mfg_market_data_cacheは企業固有情報を含まないため全テナント共有OK
     ただし、各社のインパクト分析結果は厳格にテナント分離
```

## 制約・前提条件（8モジュール共通）

### Phase 1の制約

- 外部データ（freee API/LME/為替等）の多くは手動入力 or CSV。API自動取得はPhase 2+
- ベンチマーク/市場データの自動更新はPhase 2+。初期値は手動設定
- 工業団地DB（⑪）は初期1,200件を手動整備
- 技術トレンドDB（⑬）は初期値固定
- 補助金DBは年2回の手動更新
- モンテカルロ・シミュレーション、機械学習モデルはPhase 2+
- 海外市場/海外立地の分析はPhase 3+
- キャッシュフロー予測（⑭）の売掛金回収率は過去実績ベース。ML個別予測はPhase 2+
- 技能マトリクス（⑮）は手動入力。IoT/作業ログベースの自動評価はPhase 2+
- 企業価値算定（⑯）は概算。正式な税務評価は税理士確認必須（免責事項明示）
- 事業承継税制の適用可否チェック（⑯）は概算のみ。正式認定は専門家委託
