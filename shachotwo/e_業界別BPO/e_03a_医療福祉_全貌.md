## 1. 医療・福祉の現実（業界人なら「分かる」話）

### 1.1 中小施設の現場

介護施設約14万事業所、クリニック約10万施設（厚労省 医療施設動態調査）。
通所介護、訪問介護、特養、老健、内科クリニック、歯科。施設種別は違っても共通する現実がある。

### 1.3 2025年問題と業界トレンド

**団塊世代75歳突入（2025年）→介護需要急増**

2025年に団塊世代（約800万人）が全員75歳以上に。要介護認定率は75歳を境に急上昇（65-74歳:4.3% → 75歳以上:32.2%）。介護サービスの需要は2025年から2040年にかけて1.5倍に膨張するが、供給側の人材は追いつかない。

**介護報酬改定のインパクト**

| 改定年 | 改定率 | 主な変更 |
|---|---|---|
| 2024年 | +1.59% | 処遇改善加算の一本化、科学的介護（LIFE）推進加算の拡充 |
| 2027年（予定） | 未定 | ICT活用による人員基準緩和の拡大が議論中 |

3年ごとの報酬改定で加算体系がリセットされる。改定のたびに算定ルールが変わり、事務負担が爆発する。**制度改正への追従こそがBPOの最大価値。**

**医療DXの追い風**

| 施策 | 時期 | 影響 |
|---|---|---|
| オンライン資格確認義務化 | 2023年4月〜 | マイナ保険証でレセプトオンライン化が加速 |
| 電子処方箋 | 2023年1月〜 | 処方関連業務のデジタル化 |
| 科学的介護（LIFE）推進 | 2021年〜 | ADLデータの入力・分析が加算要件に |
| 医師の働き方改革 | 2024年4月〜 | 医師のタスクシフト・書類業務削減が急務 |
| 介護現場のICT導入補助金 | 毎年 | 上限100万円（都道府県により異なる） |

**供給側の構造変化**

- 介護事業所の倒産件数: 2024年は過去最多の176件（東京商工リサーチ）
- 小規模事業所ほど倒産率が高い。人件費高騰+報酬改定の波を吸収できない
- M&Aが加速。2025年以降、施設の統廃合が本格化
- 生き残る施設 = **業務効率化+加算最大化で利益を確保できる施設**

結論: **報酬請求の最適化と人手不足の解消が死活問題。** 加算を取り漏れている施設は利益を失い、人手不足で現場が回らない施設は閉鎖に追い込まれる。

## 3. 報酬請求の全貌（これが分かってないと話にならない）

### 3.1 介護報酬請求の基本フロー

```
Step 1: サービス提供・実績記録（日々）
  │  利用者ごとに提供したサービス内容・時間を記録
  │
Step 2: 実績データの集計（月初1-5日）← 最も工数がかかる
  │  全利用者のサービス実績を介護ソフトに入力・確認
  │
Step 3: 算定チェック（月初3-8日）← 最も属人的
  │  サービス内容×要介護度の整合性
  │  加算の算定要件充足チェック（処遇改善/特定事業所/夜勤体制等）
  │  人員配置基準の充足確認
  │
Step 4: 請求データ作成・伝送（毎月10日まで）← 法定期限
  │  国保連（国民健康保険団体連合会）に電子請求
  │
Step 5: 審査・支払い（請求月の翌々月）
  │  国保連が審査→支払い。返戻があれば差し戻し
  │
Step 6: 返戻・過誤対応（翌月以降）
     返戻原因を調査→再請求 or 過誤申立
```

### 3.2 介護報酬の算定構造

介護報酬は**単位**で計算される。1単位の金額は地域・サービス種別ごとに異なる（10円〜11.40円）。

**サービス種類別の単位数（通所介護の例）**

| 要介護度 | 3-4時間 | 4-5時間 | 5-6時間 | 6-7時間 | 7-8時間 |
|---|---|---|---|---|---|
| 要介護1 | 368 | 386 | 567 | 581 | 655 |
| 要介護2 | 421 | 442 | 670 | 686 | 773 |
| 要介護3 | 477 | 500 | 773 | 792 | 896 |
| 要介護4 | 530 | 557 | 876 | 897 | 1,018 |
| 要介護5 | 585 | 614 | 979 | 1,003 | 1,142 |

**主な加算（通所介護の場合）**

```
■ 処遇改善加算（2024年改定後の一本化版）
  加算Ⅰ:  14.5%（旧・処遇改善Ⅰ+特定Ⅰ+ベースアップⅠ相当）
  加算Ⅱ:  13.0%
  加算Ⅲ:  10.5%
  加算Ⅳ:   8.0%
  → 算定要件: キャリアパス要件+職場環境等要件+賃金改善実績
  → 申請書類: 計画書（年度初め）+実績報告書（年度末）

■ サービス提供体制強化加算
  Ⅰ:  22単位/日（介護福祉士70%以上 or 勤続10年以上が25%以上）
  Ⅱ:  18単位/日（介護福祉士50%以上）
  Ⅲ:   6単位/日（介護福祉士40%以上 or 常勤75%以上）

■ ADL維持等加算
  Ⅰ:  30単位/月（評価対象者のADL利得が1以上）
  Ⅱ:  60単位/月（評価対象者のADL利得が2以上）
  → LIFEへのデータ提出が要件

■ 個別機能訓練加算
  Ⅰイ: 56単位/日（機能訓練指導員1名以上配置）
  Ⅰロ: 85単位/日（機能訓練指導員を専従で1名以上配置）
  Ⅱ:   20単位/月（LIFEへの情報提出+フィードバック活用）

■ 認知症加算:     60単位/日（認知症の利用者が一定割合以上）
■ 中重度ケア体制加算: 45単位/日（要介護3以上が30%以上+看護職員の配置）
■ 入浴介助加算Ⅰ:  40単位/日
■ 入浴介助加算Ⅱ:  55単位/日（個別入浴計画に基づく）
■ 栄養アセスメント加算: 50単位/月（管理栄養士の配置+LIFEデータ提出）
■ 口腔機能向上加算Ⅰ: 150単位/回（歯科衛生士等の配置）
■ 送迎加算:       片道47単位（送迎しない場合は減算）
```

**主な減算**

```
■ 人員基準欠如減算: 所定単位数の70%に減算
  → 配置基準を満たさない日が1日でもあると、その月全体が70%
  → 例: 通所介護で看護職員が不在の日がある = その月の全利用者が70%
  → 年間影響: 月間報酬300万 × 30% = 月90万の損失

■ 定員超過減算: 所定単位数の70%に減算
  → 利用定員の105%を超えた場合

■ サービス提供体制強化加算の返還
  → 算定要件（介護福祉士の比率等）を下回った場合、遡及返還

■ 送迎未実施減算: △47単位/片道
```

**返戻の主な原因（頻度順）**

| 原因 | 発生頻度 | 影響 | AI対策 |
|---|---|---|---|
| 被保険者資格の不一致 | 最多 | 要介護認定期間切れで請求不可 | 認定有効期限の自動監視+更新アラート |
| サービスコード誤り | 多い | 単位数の過大/過少請求 | コード×要介護度×時間の自動マッチング |
| 加算の算定要件不備 | 多い | 要件を満たしていない加算を請求 | 要件充足チェックの自動化 |
| ケアプランとの整合性 | やや多い | ケアプランにないサービスを請求 | プラン←→実績の自動照合 |
| 公費負担者番号の誤り | やや多い | 生活保護等の公費番号ミス | 番号の自動バリデーション |

### 3.3 診療報酬請求の基本フロー（クリニック向け）

```
Step 1: 外来診療（日々）
  │  診察→処方→検査→カルテ記載
  │
Step 2: 算定入力（日々 or 月末一括）← 医事課の主業務
  │  診療行為を点数に変換してレセコンに入力
  │  病名と診療行為の整合性チェック
  │
Step 3: レセプト点検（月初1-5日）← 最も属人的
  │  病名漏れ・算定漏れ・過剰算定のチェック

  │
Step 4: 請求（毎月10日まで）
  │  社会保険（支払基金）・国民健康保険（国保連）に電子請求
  │
Step 5: 審査・査定対応
     査定通知→再審査請求 or 算定方針の見直し
```

### ケース2: 内科クリニック（患者数1,200名/月・医事課3名・東京）

```
■ Before:
  月間診療報酬: 約1,500万円
  医事課3名でレセプト月1,200件
  返戻・査定率: 2.5%（月37.5万円の減収リスク）
  算定漏れ: 特定疾患管理料・生活習慣病管理料の漏れで月5-10万円
  主治医意見書・紹介状: 院長が月20通×30分 = 月10時間

■ After:
  返戻・査定率: 2.5% → 0.5%
  算定漏れ: AIが毎日チェック。月1-2万円に削減
  書類作成: AIドラフト生成で月10時間 → 3時間
  医事課の点検時間: 月60時間 → 月20時間

■ 効果試算:
  返戻・査定改善: 月30万円 × 12 = ¥360万/年
  算定漏れ改善: 月5万円 × 12 = ¥60万/年
  医事課の工数削減: 40時間/月 × ¥1,800/h = ¥86万/年
  院長の書類時間削減: 7時間/月 × ¥10,000/h = ¥84万/年
  ─────────────────────────
  年間効果合計: 約¥590万

■ 費用:
  ブレイン ¥3万 + BPOコア ¥25万 = ¥28万/月 = ¥336万/年

■ ROI: 1.8倍
```

### ケース3: 特養（利用者80名・職員50名・千葉）

```
■ Before:
  月間介護報酬: 約2,000万円
  事務員2名（請求+事務全般）
  返戻率: 3%（月60万円が2ヶ月遅延）
  加算の取り漏れ: 年間200万円の逸失（ADL維持等加算、栄養関連加算等）
  シフト管理: 夜勤+早番+遅番×50名。月40時間
  人員基準欠如: 年2-3回発生。1回で月100万円以上の減収

■ After:
  返戻率: 3% → 0.5%
  加算最適化: 年間200万円の取り漏れ → 年間30万円に削減
  シフト管理: 月40時間 → 月10時間
  人員基準欠如: AIシフトで発生ゼロに

■ 効果試算:
  返戻改善: 月50万円 × 12 = ¥600万/年
  加算最適化: ¥170万/年
  シフト管理工数: 30時間/月 × ¥1,500/h = ¥54万/年
  人員基準欠如の回避: 2.5回 × ¥100万 = ¥250万/年
  ─────────────────────────
  年間効果合計: 約¥1,074万

■ 費用:
  ブレイン ¥3万 + BPOコア ¥25万 + 追加2モジュール ¥20万 = ¥48万/月 = ¥576万/年

■ ROI: 1.9倍
```

**ROI試算の前提と注意:**
- 加算最適化の効果は施設の現状の加算取得状況による。既に加算を十分取得している施設では効果が小さい
- 返戻率の改善は3ヶ月の学習期間が必要。導入初月から最大効果にはならない
- 小規模施設（利用者20名以下）では月額費用に対してROIが出にくい。BPOスターターパックの検討が必要
- 「人員基準欠如の回避」は発生頻度×影響額の期待値。確率的な効果であることに留意
- 上記は定量化しやすい項目のみ。「施設長の経営時間確保」「利用者と向き合う時間の増加」「スタッフの離職率低下」は含んでいない

## 8. 技術アーキテクチャ（概要）

### 8.1 ゲノム駆動型アーキテクチャ

```
医療・福祉ゲノム（JSON）
  ├─ 業務フロー定義（介護報酬請求/レセプト点検/ケアプラン等）
  ├─ 業界用語辞書（介護保険用語/医療用語/加算名称）
  ├─ 算定ルール・検証ルール（加算要件/減算条件/返戻パターン）
  ├─ サービスコード・点数マスタ
  └─ 施設種別カスタマイズ（通所/訪問/施設/クリニック）

共通エンジン: 実績データ取込 → 算定チェック → 加算最適化 → 請求データ出力

新しい施設種別の追加:
  Layer 0: 即日（歯科エンジンのフォールバック）
  Layer 1: 数時間（ゲノムJSON追加。サービスコード+加算ルール）
  Layer 2: 数日（特殊な算定ロジック。障害福祉等）
```

### 8.2 歯科→介護→医科の横展開マップ

```
歯科（実装済み）
  │ レセプト点検エンジン
  │ 病名×処置の整合性チェック
  │ 加算要件チェック
  │
  ├─→ 介護（★Phase 1）
  │     サービス×要介護度の整合性チェック（≒歯科の病名×処置）
  │     加算要件チェック（構造は同じ。ルールDBを差替え）
  │     返戻パターンDB（歯科と同構造）
  │     差分: 国保連伝送フォーマット、区分支給限度額管理
  │
  └─→ 医科クリニック（Phase 2）
        病名×診療行為の整合性チェック（≒歯科と同構造）
        査定リスク判定（歯科の返戻リスクと同構造）
        施設基準管理（介護の人員配置基準管理と類似）
        差分: 診療報酬点数表、DPC対応（病院のみ）

共通コンポーネント:
  ├─ チェックエンジン（ルールベース+AI）
  ├─ 加算最適化エンジン
  ├─ 返戻/査定パターンDB
  ├─ レポート生成（施設長/院長向け）
  └─ 制度改正対応フレームワーク（ゲノム更新）
```

### 8.3 学習ループ

```
1. AIが請求データをチェック（返戻リスクスコア 0.0-1.0）
2. 事務員が確認・修正（「この加算は今月は算定できない。理由: ○○」）
3. 修正データをDBに保存
4. 次回同パターン → 自動的にリスクスコアを調整
5. 3ヶ月の蓄積 → 施設固有の返戻パターンが学習され、返戻率が半減

使えば使うほど、その施設専用の請求チェックAIに育つ。
```

## 9. 導入フロー

### Phase 1（Month 1）: データ接続

```
Week 1: アカウント開設・初期設定
  ・ブレイン基本情報入力（施設情報・職員・利用者数・加算取得状況）
  ・使用中の介護ソフト/レセコンとのデータ連携設定
  ・過去12ヶ月の請求データ取込（返戻パターン学習のため）

Week 2-3: 加算棚卸し
  ・現在取得している加算の一覧化
  ・未取得だが要件を満たしている加算の洗い出し（← ここで増収提案）
  ・返戻履歴の分析と原因分類

Week 4: シャドーモード開始
  ・新規請求に対してAIが算定チェックのドラフトを生成
  ・人間が従来通りの点検を実施
  ・両者を比較して精度を検証
```

### Phase 2（Month 2-3）: 並走運用

```
・AIが算定チェック → 事務員が確認・修正 → 請求
・修正内容がフィードバックされ、精度が継続的に向上
・目標: 返戻リスク検知率90%以上達成
・月次精度レポートで改善状況を可視化
・加算最適化の提案 → 施設長が判断 → 取得手続き
```

### Phase 3（Month 4+）: 自走

```
・定型チェック（要介護認定期限・サービスコード整合）はAI完結
・複雑な加算判定・新規ケースのみ人間が判断
・月次: 返戻率レポート + 加算取得状況 + 収支分析
・報酬改定時: ゲノム更新 → 全施設に自動反映
・横展開: シフト管理→ケアプラン→実地指導と追加モジュール導入
```

## 10. パイロット戦略

### 10.1 パイロット対象

**セグメント1: 通所介護（デイサービス）10-50名規模**

```
選定理由:
  ・施設数最多（44,000事業所）
  ・施設長=経営者で意思決定が速い
  ・報酬請求の痛みが直接的（事務員1名体制が多い）
  ・歯科ゲノムの横展開で最も開発効率が良い
  ・利用者数の変動が大きく、加算の取り漏れが多い

理想的なパイロット施設:
  ・利用者30-40名/日
  ・事務員1-2名
  ・カイポケ or ほのぼの使用中
  ・処遇改善加算Ⅱ以下（→Ⅰへのアップグレード提案が可能）
  ・返戻率3%以上
  ・施設長がITに前向き
```

**セグメント2: 内科クリニック 3-10名規模**

```
選定理由:
  ・院長即決。ROI重視
  ・レセプト件数が多く、点検の効率化効果が大きい
  ・歯科のレセプト点検エンジンとの親和性が最も高い

理想的なパイロット施設:
  ・外来患者600-1,500名/月
  ・医事課1-3名
  ・ORCA or メディコム使用中
  ・返戻・査定率2%以上
  ・施設基準加算を複数取得（算定チェックの価値が高い）
```

### 10.2 共創パートナーの選定基準

| 基準 | 必須/推奨 | 詳細 |
|---|---|---|
| 施設長のITリテラシー | 必須 | スマホ・PCを日常的に使っている |
| 介護ソフト/レセコン利用 | 必須 | データ連携の前提 |

  = 約8,400億円/年

  通所介護+グループホーム+内科クリニック（S1-S2規模）
  = 約7万施設 × ¥28万/月 × 12ヶ月

■ SOM（Serviceable Obtainable Market）— 3年後目標
  SAMの0.5% = 350施設 × ¥28万/月 × 12ヶ月
  = 約11.8億円/年

  → 子会社化基準（月商1,000万円） = 約36施設で達成
