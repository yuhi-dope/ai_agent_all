## 1. 介護報酬請求AI（★実装対象）

### 1.1 問題の本質

```
介護報酬請求の現状:
  1. サービス提供実績を記録（日々）
  2. 実績データを介護ソフトに入力
  3. 請求データを作成（月初1〜10日）
  4. 国保連（国民健康保険団体連合会）に電子請求
  5. 審査→支払い（請求月の翌々月）
  6. 返戻・過誤があれば再請求

  → 返戻率3-5%。1件の返戻=入金2ヶ月遅延
  → 加算の取り漏れが常態化（特に処遇改善加算・特定事業所加算）
  → 算定要件の充足チェックが手作業
  → 事務員1人が請求業務に月40-60時間
```

### 1.2 支払いロジック

```
■ 直接効果①：返戻・過誤の削減
  月間請求額500万円 × 返戻率2%改善 = 月10万円の入金安定化 = 年120万円

■ 直接効果②：加算の取り漏れ防止
  処遇改善加算Ⅰ（加算率8.3%）の計上漏れ月あり
  月間500万円 × 8.3% × 漏れ率5% = 月2万円 = 年24万円
  特定事業所加算、夜勤体制加算等を含めると年50-100万円

■ 直接効果③：事務員の請求業務時間削減
  月40-60時間 → 自動チェックで50%削減 = 月20-30時間削減
  時給1,500円 × 25時間 = 月3.75万円 = 年45万円

■ 合計効果：年189-265万円
  → BPO月5万円（年60万円）で ROI 3.2〜4.4倍
```

### 1.3 歯科レセプトとの構造比較

| 項目 | 歯科レセプト | 介護報酬請求 |
|---|---|---|
| 請求先 | 社保・国保 | 国保連 |
| 法定期限 | 毎月10日 | 毎月10日 |
| 主な審査 | 病名×処置の整合性 | サービス×要介護度の整合性 |
| 加算体系 | か強診加算等 | 処遇改善加算・特定事業所加算等 |
| 返戻対応 | 再請求+原因調査 | 再請求+過誤申立 |
| 共通パターン | ルールベースチェック→AI予測チェック→レポート生成 |

→ 歯科のレセプト点検エンジンをほぼそのまま転用可能。差分は算定ルールDB（ゲノム）のみ。

### 1.4 介護報酬の主な算定ルール（ゲノムに詰める内容）

**サービス種類別の単位数**
- 通所介護（デイサービス）: 要介護度×時間区分で単位が異なる
- 訪問介護: 身体介護/生活援助×時間で単位が異なる
- 施設サービス: 要介護度×施設種別で包括単位

**主な加算**
- 処遇改善加算Ⅰ〜Ⅳ（加算率3.3%〜8.3%）
- 特定処遇改善加算Ⅰ〜Ⅱ
- 特定事業所加算Ⅰ〜Ⅳ
- 夜勤体制加算
- ADL維持等加算
- 認知症加算
- 中重度ケア体制加算
- 送迎加算/入浴介助加算

**主な減算**
- 人員基準欠如減算（配置基準を満たしていない場合、70%に減算）
- 定員超過減算（定員の105%超で70%に減算）
- サービス提供体制強化加算の要件不備

**返戻の主な原因**
1. 被保険者資格の不一致（要介護認定期間切れ）
2. サービスコード誤り
3. 加算の算定要件不備
4. ケアプランとの整合性不一致
5. 公費負担者番号の誤り

### 1.5 業務フロー

```
【日次】
  サービス提供 → 実績記録（介護ソフトに入力）
  → 利用者の要介護認定有効期間の確認

【月初（1〜5日）】
  前月分の実績データ集計
  → サービス内容×要介護度の整合性チェック
  → 加算の算定要件充足チェック
    - 処遇改善: 計画書提出済み+賃金改善実績
    - 特定事業所: 資格者配置+会議開催+計画作成
    - 夜勤体制: 夜勤職員配置基準の充足
  → 請求データ作成

【毎月10日（法定期限）】
  国保連に電子請求（伝送）

【翌月〜翌々月】
  審査結果確認 → 返戻・過誤の対応
  → 再請求 or 過誤申立
```

### 1.6 課題テーブル

| 課題 | 具体的な問題 | 影響額（年間） | 発生頻度 |
|---|---|---|---|
| 加算の取り漏れ | 算定要件を満たしているのに加算を付けていない | 50-100万円の逸失 | 毎月 |
| 返戻率が高い | 要介護認定期間切れ、コード誤り | キャッシュフロー悪化 | 毎月 |
| 人員基準の管理不足 | 配置基準を一時的に下回り70%減算 | 月数十万円の減収 | 月数回 |
| 事務員の属人化 | 請求できるのは1人だけ。退職=業務崩壊 | 計測不能（事業リスク） | 常時 |
| 処遇改善加算の複雑さ | Ⅰ〜Ⅳの判定+計画書+実績報告 | 加算漏れ+行政指導リスク | 年次 |
| 制度改正への追従 | 3年ごとの報酬改定+随時の制度変更 | 誤請求リスク | 3年ごと |

### 1.7 セキュリティ・コンプライアンス

- 要配慮個人情報（利用者の介護度・病歴・家族構成）
- 個人情報保護法+介護保険法の規定に準拠
- 全アクセスをaudit_logsに記録
- テナント分離（company_idベースのRLS）

## 2〜8. 拡張ロードマップ

> 以下は#1モジュール（介護報酬請求AI）のPMF確認後に実装。

### ② ケアプラン作成支援（Tier1 / ¥3-5万）
- ケアマネが1人30件を抱えて限界
- アセスメント→課題分析→サービス計画→モニタリングの一連をAI支援
- LLMでプランドラフトを生成、ケアマネは修正・承認のみ
- 歯科の「治療計画→見積→同意書」パターンと類似

### ③ シフト・勤怠管理（Tier1 / ¥3-5万）
- 夜勤・早番・遅番の複雑なシフト
- 人員配置基準の自動チェック（違反=指定取消）
- 急な欠勤→代替スタッフの自動提案

### ④ 処遇改善加算管理（Tier2 / ¥2-3万）
- 加算Ⅰ〜Ⅳの判定+計画書+実績報告
- 賃金改善額の自動計算
- 年度末の実績報告書自動生成

### ⑤ 実地指導（監査）対応（Tier2 / ¥2-3万）
- 行政監査の書類準備で毎年数日消える
- チェックリスト自動生成+書類の充足状況可視化

### ⑥ 利用者管理・家族連絡（Tier2 / ¥1-2万）
- 利用者100名の状態変化・家族への報告が属人化
- 歯科の「リコール管理」パターンの横展開

### ⑦ 採用支援（Tier3 / ¥2-3万）
- 有効求人倍率3.6倍。2040年に69万人不足
- 歯科の「採用支援（衛生士）」パターンの横展開

### ⑧ ヒヤリハット・事故報告（Tier3 / ¥1-2万）
- 報告書の作成負荷。分析して再発防止に活かせていない
- 製造業の「品質管理」パターンの横展開
