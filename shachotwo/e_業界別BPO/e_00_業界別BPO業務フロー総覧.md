## 1. 建設業（e_01）

### ① 積算AI — ★キラーフィーチャー（月¥10-30万の価値）

**課題**: 熟練積算士が1案件3-7日。平均年齢55歳以上。ミスれば赤字受注で数百万〜数千万の損失

```
業務フロー:
  図面PDF/Excel受領
    → 数量拾い出し（工種/種別/細別/規格/数量/単位）
      → 単価選定（設計単価/市場単価/見積単価）
        → 歩掛適用（標準歩掛 × 補正係数）
          → 諸経費計算（共通仮設費/現場管理費/一般管理費等）
            → 内訳書作成（Excel/PDF出力）

AI化ポイント:
  Level 1: Document AI + LLM で数量構造化抽出（拾い出し70%削減）
  Level 2: 過去実績 + 公共労務単価 + LLM推定で単価候補付与（80%削減）
  Level 3: N社の匿名化データで市場単価推定（ネットワーク効果）

制約: 積算ソフト・単価DBアクセスなし → 「育つ単価DB」戦略で対応
```

### ② 安全書類自動生成（月¥3-5万の価値）

**課題**: 1現場あたり初回20-40時間、更新で月5-10時間。全現場合計で月50-100時間

```
業務フロー:
  新規現場開始
    → 作業員マスタから対象者選定
      → 全建統一様式の書類自動生成
        - 第5号: 作業員名簿（★最大の工数削減）
        - 第9号: 有資格者一覧表
        - 第1号-甲: 下請負業者編成表
        - 第6号: 工事安全衛生計画書（AI生成: 危険要因分析→安全対策提案）
      → 資格有効期限アラート（90日前通知）
      → グリーンサイトCSV出力（元請がグリーンサイト利用の場合）

AI化ポイント:
  マスタデータ → ワンクリックで全書類生成
  作業員入退場 → 名簿自動更新
  資格期限 → 自動リマインド
```

### ③ 出来高・請求書自動生成

**課題**: 月末の出来高計算と請求書作成がExcel地獄

```
業務フロー:
  工事台帳（契約内訳）登録
    → 月次出来高入力（各項目の進捗率）
      → 累計出来高 − 前月累計 = 当月請求額を自動計算
        → 請求書自動生成（PDF/Excel）
          → 入金管理（未入金一覧・支払期日超過アラート）

AI化ポイント:
  前月比大幅変動 → 異常検知アラート
  電子帳簿保存法対応（タイムスタンプ不要のクラウド保存）
  請求書番号の自動採番
```

### ④ 原価管理レポート

**課題**: 赤字工事の発見が竣工後になりがち

```
業務フロー:
  実行予算登録
    → 月次原価実績入力（材料/労務/外注/経費）
      → 予実差異分析（科目別消化率）
        → 完成予測原価AI推定

  消化ペースから完成時原価を推定

  赤字転落リスクをプロアクティブに通知
```

## 2. 製造業（e_02）

### ① 見積AI — ★キラーフィーチャー（月¥10-30万の価値）

**課題**: 熟練者が1件30分〜数時間。月100件以上の引合い。回答遅い=受注逸失

```
業務フロー:
  図面PDF/画像 + 仕様（材質、数量、表面処理）受領
    → 形状・寸法・加工要素を抽出
      → 加工工程を推定（切断→旋盤→MC→研磨→表面処理→検査）
        → 各工程の工数算出（段取り時間 + サイクルタイム × 数量）
          → チャージレート × 工数 = 加工費算出
            → 材料費 + 加工費 + 表面処理 + 外注費 + 管理費 = 見積金額
              → 見積書PDF/Excel出力

AI化ポイント:
  Level 1: Document AI + LLM で工程推定（50%削減）
  Level 2: 類似部品の過去実績で工数補正 + 受注率分析（80%削減）
  Level 3: N社の匿名化データでチャージレート市場単価推定

独自価値: 受注率分析「¥X万なら受注率70%、¥Y万なら50%」
```

```
  設備稼働率の可視化（投資判断の材料）
```

## 3. 歯科（e_03）

### ① レセプトAI — ★キラーフィーチャー（月¥5-15万の価値）

**課題**: 院長が月末2-3日をレセプトチェックに費やす。査定率0.5%改善で年間30-100万の収益改善

```
業務フロー:
  レセコン（Dentis/ORCA）からCSV出力
    → データ取り込み（患者×月のレセプトを構造化）
      → ルールベースチェック

        - 部位不整合（上顎の病名なのに下顎の処置）
      → AI予測チェック（過去の査定パターンからリスクスコア算出）
        → チェックレポート生成（エラー/警告/情報の3段階）
          → 査定結果フィードバック → 学習精度向上

AI化ポイント:
  Level 1: ルールベース（一般的エラー検出、精度60-70%）
  Level 2: 自院の過去査定パターン学習（精度80-90%）
  Level 3: N院の匿名化データで審査傾向学習（ネットワーク効果）

セキュリティ: 要配慮個人情報 → 3省2ガイドライン準拠必須
```

## 4. 不動産業（e_04）

### ① 物件査定AI — ★キラーフィーチャー（月¥8-20万の価値）

**課題**: 査定に1件30分〜1時間。経験と勘に依存。査定価格のブレが大きく、売主の信頼を失う or 安値で受託

```
業務フロー:
  物件情報入力（所在地、面積、築年数、構造、間取り）
    → 公開データ自動収集
      - 国交省: 不動産取引価格情報（土地総合情報システムAPI）
      - 国交省: 地価公示・都道府県地価調査
      - 国税庁: 路線価
      - ポータル: SUUMO/HOME'S等の売出価格（相場参考）
      ※ REINS: 公開APIなし・宅建業者専用・自動収集不可
        → ユーザーが手動でCSV/PDF貼付 or 目視転記で補完
    → 取引事例比較法による概算算出
      → 収益還元法（投資用の場合: 想定賃料 × 利回り）
        → 原価法（建物: 再調達原価 × 残存率）
          → 3手法の加重平均 + AI補正 → 査定書PDF出力

AI化ポイント:
  Level 1: 公開データ（国交省API + 路線価） + LLMで自動査定（精度±15%、40%工数削減）
  Level 2: 自社成約データ + ユーザー入力のREINS事例で精度向上（±8%、70%削減）
  Level 3: N社の匿名化成約データで地域・物件タイプ別精度向上（ネットワーク効果）

独自価値: 「この価格帯なら成約確率80%、¥X万上乗せで50%」の受注率分析
```

## 導入前必須対応チェックリスト

> パイロット企業への導入前に必ず完了させる7項目。

| # | 項目 | 優先度 | 実装先 | 影響範囲 |
|---|---|---|---|---|
| 1 | **エラーメッセージ安全化** | 高 | `security/error_handler.py` | `str(e)`で内部情報が漏洩する10箇所を汎用メッセージに置換 |
| 2 | **レート制限** | 最高 | `main.py` middleware | BPO: 10req/min/テナント、認証: 5req/min（暴走・DDoS防止） |
| 3 | **LLMコスト上限** | 最高 | `llm/client.py` CostTracker | テナント別¥50,000/月。超過→実行拒否+管理者アラート |
| 4 | **タイムアウト** | 高 | `llm/client.py` + `task_router.py` | LLM: 30秒、パイプライン全体: 5分 |
| 5 | **同時実行制御** | 高 | `task_router.py` Semaphore | テナント別同時3パイプライン。超過→429 |
| 6 | **テナント分離テスト** | 最高 | `tests/security/` | テナントAのデータがBから見えないことの自動検証 |
| 7 | **CORS制限** | 中 | `main.py` | 本番: 特定ドメインのみ許可。`allow_methods`限定 |

### なぜこの7項目か

```
#2 レート制限なし + #3 コスト上限なし = 1テナントの暴走で全テナント影響
  → 月間LLMコスト数百万円の請求が来るリスク
  → 最悪ケース: 悪意あるユーザーが大量リクエストでサービス停止

#6 テナント分離テストなし = 他社データ漏洩のリスク
  → 1件の漏洩で全顧客の信頼喪失 + 個人情報保護法違反（罰金1億円）

#1 エラーメッセージ漏洩 = ペネトレーションテストの攻撃情報を提供
  → スタックトレースからDB構造・内部APIパスが判明

#4 タイムアウトなし = LLM APIの応答遅延でスレッドが枯渇
  → 他のリクエストが全て503になる

#5 同時実行制御なし = 1テナントが全リソースを占有
  → 他テナントのパイプラインが実行できない
```
