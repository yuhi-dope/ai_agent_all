# 建設業 × シャチョツー — 全貌

> **このドキュメントの目的**: 建設業の業界人が読んで「完全に理解している」と感じるレベルで、
> シャチョツーが建設業に何をもたらすかを1つのドキュメントにまとめる。
> **対象読者**: 建設会社経営者、現場所長、積算担当、安全担当、コンサルタント、パートナー候補

## 1. 建設業の現実（業界人なら「分かる」話）

### 1.1 中小建設会社の現場

従業員10-300名の中小建設会社、約47万社（国交省 建設業許可業者数調査）。
総合建設、土木、電気工事、管工事、内装、塗装、足場。業種は違っても共通する現実がある。

### 2.3 元請/下請の構造とBPOニーズの違い

```
元請（発注者から直接受注）:
  ・入札→積算→施工計画→原価管理→竣工書類
  ・書類の種類: 多い（施工体制台帳、品質計画書、安全書類一式、竣工図書）
  ・ニーズ: 積算AI + 原価管理 + 施工計画書 + 写真整理
  ・月額: ¥35-55万（BPOコア + 追加モジュール2-3個）

下請（元請から受注）:
  ・見積提出→安全書類提出→作業→出来高報告
  ・書類の種類: 安全書類が中心（作業員名簿、有資格者一覧、施工体系図）
  ・ニーズ: 安全書類自動生成 + 出来高・請求書
  ・月額: ¥25-35万（BPOコア）

重層下請の現実:
  元請 → 1次下請 → 2次下請 → 3次下請
  ・同じ作業員の名簿を、元請用/1次用/2次用で3回作り直し
  ・資格証明書のコピーを何度も提出
  ・グリーンサイト/ビルディーに登録、かつ紙でも提出
  → シャチョツーはマスタデータから各元請フォーマットに自動変換
```

**シャチョツーの差別化:**

グリーンサイトは「箱」。データを入れるのは人間。
シャチョツーは「入力ロボット」。マスタデータからワンクリックで全書類生成→グリーンサイトCSV出力。

### 3.4 シャチョツーの独自ポジション

```
1. 競合は「1つの課題」を解くツール。シャチョツーは「8つの課題」をまとめて解くOS。
   積算AIで入って、安全書類→出来高→原価管理→写真整理と横展開する。

2. 「積算」と「バックオフィス」を一気通貫で解く唯一のプレイヤー。
   積算ソフトは計算だけ。施工管理SaaSは現場管理だけ。
   「社長が一番困っていること全部」を1つのサービスで解決する。

3. 使うほど精度が上がる。
   自社の実績単価・補正係数が蓄積され、3ヶ月で精度が±30%→±15%に収束する。

4. 中小の「ちょうどいい」価格帯。
   積算ソフト（年間70-350万円）+ 施工管理SaaS（年間36-240万円）= 年間100-600万円
   シャチョツー: 年間300-540万円で全部入り。かつ積算ソフトほどの習得コストがない。

正直な弱み:
  ・新参。導入実績がまだ少ない。パイロット企業での実証フェーズ
  ・積算ソフトほどの精度は出ない（特に建築積算は項目数が膨大）
  ・Level 0（LLMのみ）の精度は±30-50%。顧客データの蓄積が精度の鍵
  ・BIM/CIM対応はPhase 2+。現時点は2D設計書（PDF/Excel）のみ
```

## 4. 法令・規制環境

### 4.1 建設業法（最重要法令）

**許可制度**

```
建設業を営むには許可が必要（軽微な工事を除く）。

許可の区分:
  ・一般建設業: 下請に出す金額が1件4,500万円未満（建築一式は7,000万円未満）
  ・特定建設業: 上記を超える場合。元請として大規模工事を受注するのに必要

許可の要件（5要件）:
  ① 経営業務の管理責任者（経管）: 5年以上の経営経験
  ② 専任技術者（専技）: 1級施工管理技士 等
  ③ 財産的基礎: 一般500万円以上、特定4,000万円以上
  ④ 誠実性: 不正・不誠実な行為がないこと
  ⑤ 欠格要件に該当しないこと

許可は5年更新。更新を忘れると営業停止。
→ シャチョツーのモジュール⑧で6ヶ月前アラート + 必要書類チェックリスト自動生成
```

**経営事項審査（経審）**

```
公共工事を受注するには経審が必須。

評点:
  ・X1: 完成工事高（=受注規模）
  ・X2: 自己資本額/利益額（=財務基盤）
  ・Y: 経営状況分析（=財務健全性）
  ・Z: 技術職員数値/元請完成工事高（=技術力）
  ・W: 社会性等（=社会貢献）

総合評定値(P) = 0.25X1 + 0.15X2 + 0.20Y + 0.25Z + 0.15W

中小建設会社にとって:
  ・W点に加算される項目: 建退共加入、社会保険加入、CCUS登録 等
  ・技術者の資格がモロに反映される
  ・経審対策 = 入札参加資格の維持 = 会社の存続
  → シャチョツーで「経審シミュレーション」: 技術者配置の最適化提案
```

**技術者配置**

```
工事現場には「主任技術者」or「監理技術者」の専任配置が必要。

  ・主任技術者: 全ての工事現場に配置（一般建設業）
  ・監理技術者: 特定建設業の元請が配置（下請総額4,500万円以上）

実態:
  中小建設会社で1級施工管理技士は2-5名程度。
  複数現場を掛け持ちすると法令違反。
  → 技術者の「空き状況」を把握していないと受注できない
  → シャチョツーで技術者配置のシミュレーション
```

| 業種 | 土木一式 or 舗装 | 公共工事の標準積算が使える。AI化しやすい |
| 工事種別 | 公共工事が売上の50%以上 | 設計単価ベース。積算の標準化が進んでいる |
| 地域 | 地方（福岡・熊本・広島 等） | 社長決裁が早い。IT導入率が低く伸びしろが大きい |
| IT環境 | Excel + 紙が中心 | ANDPADなどを既に使いこなしている会社は後回し |
| 積算体制 | 社長 or 積算担当1-2名 | 属人的であるほどAI化の効果が大きい |
| 経営状態 | 年商3-30億、黒字 | 月額25-48万円の支払い能力がある |

### 9.2 共創パートナーの選定基準

```
必須条件:
  ① 社長が「書類に殺される」と実感している
  ② 月3件以上の積算がある（データ蓄積に必要）
  ③ 月額¥25万を「投資」と考えられる経営体力
  ④ フィードバックを出してくれる（修正→精度向上のサイクル）

望ましい条件:
  ⑤ 下請を5社以上使っている（安全書類モジュールの効果が大きい）
  ⑥ 経審を毎年受けている（モジュール⑧の検証ができる）
  ⑦ 地元の建設業協会で影響力がある（口コミ効果）
```

### 9.3 PMF指標

| 指標 | 目標値 | 測定方法 |
|---|---|---|
| NPS | ≥ 30 | 月次アンケート |
| WAU（週間アクティブ率） | ≥ 60% | ログインデータ |
| 「ないと困る」回答率 | ≥ 60% | 月次アンケート |
| 積算AI精度 | ±20%以内 | 修正率の計測 |
| 安全書類生成時間 | 従来の20%以下 | 作業時間計測 |
| 継続意向 | 3ヶ月後に80%以上が継続 | 解約率 |

## 10. 収益モデル

### 10.1 料金体系

```
ブレイン: ¥30,000/月（全業界共通。ナレッジ・Q&A・デジタルツイン）
BPOコア: ¥250,000/月（①積算AI + ②安全書類 + ③出来高・請求書 + ④原価管理）
追加モジュール: ¥100,000/個/月
  ⑤ 施工計画書AI
  ⑥ 工事写真整理
  ⑦ 下請管理
  ⑧ 許可更新・経審サポート

典型的な月額:
  小規模（15-30名）: ブレイン + BPOコア = ¥280,000/月
  中規模（30-100名）: ブレイン + BPOコア + 追加2個 = ¥480,000/月
  中堅（100-300名）: ブレイン + BPOコア + 追加4個 = ¥680,000/月
```

### 10.2 想定ARPU

```
パイロット期（Year 1）:
  平均ARPU: ¥350,000/月（ブレイン + BPOコア + 追加1個）

成長期（Year 2-3）:
  平均ARPU: ¥480,000/月（追加モジュールの利用拡大）

成熟期（Year 4+）:
  平均ARPU: ¥550,000/月（全モジュール利用 + バックオフィス追加）
```

### 10.3 TAM/SAM/SOM

```
TAM（潜在市場全体）:
  建設業47万社 × 平均ARPU¥35万/月 × 12ヶ月 = 約1.97兆円/年

SAM（アクセス可能市場）:
  対象セグメント（従業員15-300名の元請+1次下請）: 約5万社
  5万社 × ¥35万/月 × 12ヶ月 = 約2,100億円/年

SOM（獲得可能市場・3年目標）:
  初期シェア: SAMの0.1% = 50社
  50社 × ¥42万/月 × 12ヶ月 = 約2.5億円/年
```
