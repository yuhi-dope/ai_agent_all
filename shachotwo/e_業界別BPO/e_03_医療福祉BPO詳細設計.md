## 0. エグゼクティブサマリー

### なぜ医療・福祉BPOか

```
医療・福祉の中小事業所は：
  ・介護報酬請求の返戻率3-5%。月500万円請求で年120万円の入金遅延
  ・レセプト点検に院長が毎月5-10時間。時給換算で年120-240万円の機会損失
  ・ケアマネ1人が30件抱えて限界。ケアプラン作成が属人化
  ・夜勤シフト管理が地獄。人員配置基準違反=指定取消のリスク
  ・実地指導（行政監査）の書類準備で毎年数日消える
  ・介護記録・日誌が手書き or 二重入力
  ・服薬管理の抜け漏れが事故につながる
  ・未収金管理が属人化。公費適用判定に詳しい事務員が退職=崩壊

  → 全部AIで代行できる = 年間500-1,000万円の価値
  → 歯科レセプトエンジンの横展開で最も効率的に立ち上がる
```

## 1. 介護報酬請求AI（★キラーフィーチャー / 介護施設向け）

### 1.1 課題（金額換算の価値）

```
■ 直接効果①：返戻・過誤の削減
  月間請求額500万円 × 返戻率2%改善 = 月10万円 = 年120万円の入金安定化

■ 直接効果②：加算の取り漏れ防止
  処遇改善加算I（加算率13.7%）の計上漏れが月あり
  月間500万円 × 13.7% × 漏れ率5% = 月3.4万円 = 年41万円
  特定事業所加算、夜勤体制加算等を含めると年50-100万円

■ 直接効果③：事務員の請求業務時間削減
  月40-60時間 → 自動チェックで50%削減 = 月20-30時間削減
  時給1,500円 × 25時間 = 月3.75万円 = 年45万円

■ 合計効果：年206-265万円
  → BPO月5万円（年60万円）で ROI 3.4〜4.4倍
```

### 1.2 業務フロー

```
【日次】
  サービス提供
    → 実績記録（介護ソフトに入力 or 紙記録）
    → 利用者の要介護認定有効期間の自動確認
    → ★AI: 実績データのリアルタイムバリデーション

【月初（1〜5日）】
  前月分の実績データ集計
    → ★AI Step 1: record_reader（実績記録データ取得）
    → ★AI Step 2: service_code_mapper（サービスコード→報酬単位数マッピング）
    → ★AI Step 3: limit_checker（区分支給限度額チェック）
    → ★AI Step 4: addition_checker（加算チェック）
    → ★AI Step 5: deduction_checker（減算チェック）
    → ★AI Step 6: compliance_checker（コンプライアンスチェック）
    → ★AI Step 7: output_generator（国保連請求データ生成）
    → 事務員が最終確認→承認

【毎月10日（法定期限: 介護保険法施行規則第30条）】
  国保連に電子請求（伝送）

【翌月〜翌々月】
  審査結果確認 → 返戻・過誤の対応
  → ★AI: 返戻原因分析→再請求データ自動生成
```

### 1.3 AI化ポイント

```
Level 1: ルールベースチェック（実装済み）
  - サービスコード×要介護度の整合性チェック
  - 区分支給限度基準額の超過チェック
  - 加算算定要件の充足チェック（処遇改善加算I〜V、特定処遇改善加算I〜II）
  - サービス提供日と請求対象期間の一致確認

Level 2: AI予測チェック（Phase 2）
  - 過去の返戻パターンから返戻リスクを予測
  - 加算の取り漏れを過去実績から提案
  - 類似事業所との比較分析（匿名化データ）

Level 3: 完全自動請求（Phase 3）
  - 実績記録→請求データ生成→国保連伝送まで自動化
  - 返戻率0.5%以下を達成後に段階的に自律度を上げる
```

# 判定ロジック
def check_staff_shortage(facility_info: dict) -> float:
    """人員配置基準の充足チェック→減算率を返す"""
    service_type = facility_info["service_type"]
    if service_type == "通所介護":
        capacity = facility_info["capacity"]
        care_staff = facility_info["care_staff_count"]
        required = 1 if capacity <= 15 else 1 + ((capacity - 15) // 5 + 1)
        if care_staff < required:
            return 0.70  # 人員基準欠如減算
    return 1.00  # 減算なし
```

### 1.5 計算ロジック

**介護報酬の算出フロー**

```
① 基本報酬（単位数）の決定
   サービス種類 × 要介護度 × 提供時間 → 基本単位数
   例: 通所介護/要介護3/7時間以上8時間未満 = 896単位

② 加算の適用
   基本単位数 + 各種加算単位数 = 総単位数
   例: 896 + 入浴40 + 送迎47×2 + 個別機能訓練56 = 1,086単位

③ 減算の適用
   減算条件に該当する場合、所定単位数 × 減算率
   例: 人員基準欠如の場合 → 1,086 × 0.70 = 760単位

④ 処遇改善加算
   対象サービスの総単位数 × 加算率
   例: 760 × 0.094（通所介護 加算I）= 71単位

⑤ 金額換算
   (総単位数 + 処遇改善加算単位数) × 地域単価
   例: (760 + 71) × 10.84（4級地）= 9,004円

⑥ 利用者負担
   金額 × 負担割合（1割/2割/3割）
   例: 9,004 × 0.1 = 901円（利用者負担）
       9,004 - 901 = 8,103円（保険給付額 → 国保連請求）

⑦ 公費適用（生活保護等）
   生活保護受給者の場合、利用者負担分も公費から支給
   法令: 生活保護法第15条の2
```

### 1.8 制約・前提条件

- 介護報酬単位数表はゲノムJSONとして管理。3年ごとの報酬改定時にJSON更新

### 1.9 独自価値（競合にない差別化）

- **加算取り漏れの能動提案**: 既存ソフトは「入力した加算のチェック」のみ。シャチョツーは「取れるのに取っていない加算」を提案する

    ("特定疾患療養管理料", "生活習慣病管理料"),  # 同一月に併算定不可
    ("外来管理加算", "処置料"),                  # 処置実施日は算定不可
    ("初診料", "再診料"),                        # 同一日に併算定不可
]

# リスクスコア計算
def calculate_risk_score(receipt: dict) -> int:
    """査定リスクスコアの算出（点数）"""
    risk_points = 0
    for proc_name, max_count in FREQUENCY_LIMITS.items():
        actual = count_procedures(receipt, proc_name)
        if actual > max_count:
            excess = actual - max_count
            points_per = get_points(proc_name)
            risk_points += points_per * excess
    return risk_points  # 点数 → ×10で円換算

### 2.5 計算ロジック

**診療報酬の算出フロー**

```
① 基本診療料の算定
   初診料: 291点（2,910円）
   再診料: 75点（750円）
   外来管理加算: 52点（520円）← 処置なし再診時のみ

② 医学管理等
   特定疾患療養管理料: 225点 × 月2回まで
   生活習慣病管理料: 610点 × 月1回

③ 検査・画像診断
   血液検査（CBC）: 21点
   HbA1c: 49点
   X線撮影: 68点（単純1枚）

④ 投薬・注射
   処方料: 42点（3種以上の内服薬: 29点に減算）
   処方箋料: 68点（院外処方の場合）
   薬剤料: 薬価から算出

⑤ 処置・手術
   各処置の所定点数を算定

⑥ 合計点数 × 10 = 医療費（円）
   患者負担割合（1割〜3割）を乗じて患者負担額を算出

⑦ 推定返戻金額
   エラーfindingがある患者の合計点数 × 10 = 返戻リスク金額
```

# 状態変化検知ロジック
def detect_vital_change(current: dict, history: list[dict]) -> list[str]:
    """直近7日間のバイタルトレンドから異常を検知"""
    alerts = []
    for key, thresholds in VITAL_THRESHOLDS.items():
        value = current.get(key)
        if value is None:
            continue
        # 閾値チェック
        if "critical_high" in thresholds and value >= thresholds["critical_high"]:
            alerts.append(f"【緊急】{key}: {value}{thresholds['unit']}（基準値超過）→ 医師報告必須")
        elif "high" in thresholds and value >= thresholds["high"]:
            alerts.append(f"【注意】{key}: {value}{thresholds['unit']}（高値）")
        # トレンド分析（3日連続上昇/下降）
        recent = [h.get(key) for h in history[-3:] if h.get(key)]
        if len(recent) >= 3 and all(recent[i] < recent[i+1] for i in range(len(recent)-1)):
            alerts.append(f"【トレンド】{key}: 3日連続上昇傾向")
    return alerts

# 利用者負担割合の判定
COPAY_RATIOS = {
    "一般": 0.1,           # 1割負担（大多数の高齢者）
    "一定以上所得者": 0.2,  # 2割負担（年金収入280万円以上）
    "現役並み所得者": 0.3,  # 3割負担（年金収入340万円以上）
}

# 高額介護サービス費（月額上限）
HIGH_COST_LIMITS = {
    "生活保護受給者": 15_000,         # 月15,000円
    "市町村民税非課税（年金80万以下）": 15_000,
    "市町村民税非課税": 24_600,
    "一般": 44_400,
    "現役並み所得者I": 44_400,
    "現役並み所得者II": 93_000,
    "現役並み所得者III": 140_100,
}

# 公費負担制度
PUBLIC_EXPENSE_PROGRAMS = {
    "生活保護（介護扶助）": {
        "法令": "生活保護法第15条の2",
        "内容": "利用者負担額の全額を公費負担",
        "判定": "生活保護受給者証の番号確認",
        "負担者番号": "12xxxxxx",
    },
    "自立支援医療（精神通院）": {
        "法令": "障害者総合支援法第58条",
        "内容": "自己負担を1割に軽減+月額上限設定",
        "判定": "自立支援医療受給者証の番号確認",
    },
    "特定疾患医療（難病）": {
        "法令": "難病の患者に対する医療等に関する法律第5条",
        "内容": "自己負担を2割に軽減+月額上限設定",
        "判定": "特定医療費受給者証の番号確認",
    },
    "被爆者援護": {
        "法令": "原子爆弾被爆者に対する援護に関する法律第10条",
        "内容": "医療費の全額公費負担",
        "判定": "被爆者健康手帳の番号確認",
    },
}

# 自己負担金計算
def calculate_copay(
    total_amount: int,
    copay_ratio: float,
    income_class: str,
    public_expense: str | None,
) -> dict:
    """自己負担金の計算"""
    base_copay = int(total_amount * copay_ratio)

    # 公費適用
    if public_expense == "生活保護（介護扶助）":
        return {"copay": 0, "public_expense_amount": base_copay, "reason": "生活保護法第15条の2"}

    # 高額介護サービス費の適用
    monthly_limit = HIGH_COST_LIMITS.get(income_class, 44_400)
    if base_copay > monthly_limit:
        actual_copay = monthly_limit
        high_cost_benefit = base_copay - monthly_limit
    else:
        actual_copay = base_copay
        high_cost_benefit = 0

    return {
        "copay": actual_copay,
        "high_cost_benefit": high_cost_benefit,
        "public_expense_amount": 0,
    }
```
