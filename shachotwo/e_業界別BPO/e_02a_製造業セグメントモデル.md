# 製造業セグメントモデル — 5軸定義

> **作成**: 2026-03-27
> **目的**: 製造業22万社を5つの軸で分類し、攻めるセグメントの優先順位・モジュール構成・料金を決定する
> **依存**: `e_02_製造業BPO設計.md`（モジュール詳細）、`c_09_製造業GTM戦略.md`（戦略ストーリー）
> **設計の正**: この文書

---

## 0. 3層セグメントモデルの全体像

顧客理解の抽象度順（営業ヒアリングの順番）にLayerを定義する。

```
L1（最上位）  完成品産業 × Tier
              「あなたは誰か」
              → ゲノムテンプレート選択
              → BPOモジュール優先順位
              → 業界固有ルールの有無
                    │
                    ▼
L2（中位）    生産数 × サイズ × 品質要求
              「どんな仕事をしているか」
              → 月間見積件数（= AIのROI）
              → 見積精度の要求水準
              → 料金プラン
                    │
                    ▼
L3（最下位）  見積構造型
              「どうやって値段を出すか」
              → Type A/B/C/D のどのエンジンを使うか
              → 技術的な実装の問題（顧客には見せない）
```

**L1/L2は顧客に見える層**（営業資料・オンボーディングで使う）。
**L3は内部アーキテクチャ**（顧客にとっては「見積が速く正確に出る」ことが価値）。

## 1. L1: 完成品産業 × Tier

### 1.1 完成品産業の分類

| ID | 完成品産業 | 特徴 | 品質規格 | 発注特性 |
|---|---|---|---|---|
| **IND-A** | 自動車 | 最大市場。コスト圧力最強 | IATF16949 / PPAP / Cpk必須 | 年次単価改定。VA/VE要求 |
| **IND-B** | 半導体製造装置 | 高単価。景気循環大 | クリーン度。精密公差 | 単品〜小ロット。短納期 |
| **IND-C** | 産業機械 | 安定需要 | ISO9001レベル | リピート+新規半々 |
| **IND-D** | 医療機器 | 高単価。規制厳格 | 薬機法 / ISO13485。トレーサビリティ必須 | 小ロット。変更管理厳格 |
| **IND-E** | 航空宇宙 | 超高単価。参入障壁最大 | Nadcap / JISQ9100。特殊工程認定 | 極小ロット。材料証明必須 |
| **IND-F** | 建設機械 | 安定。中〜大ロット | ISO9001レベル | 大型部品。溶接+機械加工 |
| **IND-G** | 民生・雑多 | 特定産業に依存しない | 一般品質 | 多品種少量。件数最多 |
| **IND-H** | 電子・電気機器 | 筐体/放熱板/コネクタ | ISO9001〜業界規格 | 中ロット。アルミ板金多い |
| **IND-X** | 食品OEM/化学受託 | プロセス型 | HACCP / GMP | バッチ生産。見積構造が異なる |
| **IND-Z** | 見込み生産全般 | 素材/量産消費財 | 各種 | 見積AI不要。ブレインのみ |

### 1.2 Tier位置の分類

| ID | Tier | 規模感 | 特徴 | 元請けとの関係 |
|---|---|---|---|---|
| **T0** | 最終組立（OEM） | 大企業 | **対象外** | - |
| **T1** | Tier 1 | 100-300名 | 設計能力あり。品質保証体制あり。見積チームがいる | 直接取引。監査あり |
| **T2** | Tier 2 | 30-100名 | 加工特化。見積できる人が2-3人 | Tier1経由。指示図面で加工 |
| **T3** | Tier 3 | 5-30名 | 単工程特化。社長が全部やる | 複数のT1-2から受注 |
| **TI** | 独立型 | 10-100名 | 特定の親会社を持たない。30-100社の得意先 | エンドユーザーと直接 |

## 2. L2: 生産数 × サイズ × 品質要求

### 2.1 生産数

| ID | 区分 | ロットサイズ | 月間見積件数（目安） | 1件の見積時間 |
|---|---|---|---|---|
| **V1** | 単品・試作 | 1-5個 | 100-200件 | 10-30分 |
| **V2** | 小ロット | 5-50個 | 30-80件 | 30分-1時間 |
| **V3** | 中ロット | 50-500個 | 10-30件 | 1-3時間 |
| **V4** | 量産 | 500個超 | 3-10件 | 半日-数日 |

### 2.2 サイズ

| ID | 区分 | 寸法目安 | 設備 | 見積への影響 |
|---|---|---|---|---|
| **SZ-S** | 小型 | ~100mm | CNC旋盤/小型MC | 工程シンプル。AI推定しやすい |
| **SZ-M** | 中型 | 100-500mm | MC/5軸MC | 標準。エンジンの想定ど真ん中 |
| **SZ-L** | 大型 | 500mm-1m | 大型MC/門型MC | 段取り・搬送コスト追加。クレーン要 |
| **SZ-XL** | 超大型 | 1m超 | 5面加工機/特殊設備 | 別世界。追加コスト計算が必要 |

### 2.3 品質要求レベル

| ID | レベル | 公差目安 | 要求事項 | ゲノムへの追加要素 | 見積への追加工程 |
|---|---|---|---|---|---|
| **Q1** | 一般 | ±0.1mm | 外観検査+寸法抜取 | 最小限 | 検査1工程 |
| **Q2** | 精密 | ±0.02-0.05mm | 全数検査+測定記録 | 検査基準・測定器リスト | 検査+測定工程 |
| **Q3** | 高精密 | ±0.005-0.01mm | 3次元測定+SPC+Cpk | 工程能力管理基準、測定手順 | 研磨+測定+管理工程 |
| **Q4** | 超精密/規制品 | ±0.001mm or 規制品 | 全数3次元+トレーサビリティ+認定 | 規制要件全体 | 認証・書類作成（見積の15-30%） |

### 2.4 L2が決めるもの

#### ROIスコア（見積AI導入価値）

```
ROI = 月間見積件数 × 1件あたり削減時間 × 時間単価
```

| | SZ-S（小型） | SZ-M（中型） | SZ-L（大型） | SZ-XL（超大型） |
|---|---|---|---|---|
| **V1（単品・試作）** | **ROI最高** | **ROI高** | ROI中 | ROI中 |
| **V2（小ロット）** | **ROI高** | **ROI高** | ROI中 | ROI低 |
| **V3（中ロット）** | ROI中 | ROI中 | ROI低 | ROI低 |
| **V4（量産）** | ROI低 | ROI低 | ROI最低 | ROI最低 |

#### 見積精度の要求水準

| 品質レベル | AI精度目標（Level 1時点） | 理由 |
|---|---|---|
| **Q1** | ±20-25% | 単価が小さく誤差の金額インパクトが低い |
| **Q2** | ±15-20% | 標準的。研磨・測定工程の追加判断が重要 |
| **Q3** | ±10-15% | 1件の金額が大きい。研磨・特殊工程の積算精度が必要 |
| **Q4** | ±10% + 書類コスト正確 | 認証・書類コストが見積の15-30%。ここの精度が肝 |

#### 料金プランの対応

| L2特性 | 推奨プラン | 理由 |
|---|---|---|
| V1-V2 × SZ-S/M × Q1-Q2 | **スターター ¥15万** | 件数多い×精度要求低い。コスパ最高のゾーン |
| V1-V2 × SZ-M/L × Q2-Q3 | **スタンダード ¥25万** | 件数多い×精度要求もある。2ndモジュール必要 |
| V2-V3 × SZ-L/XL × Q3-Q4 | **プレミアム ¥40万** | 1件の金額大×高精度要求。フルサポート必要 |
| V3-V4（中ロット〜量産） | **スタンダード ¥25万**（原価管理中心） | 見積件数少ないが原価管理の価値が高い |

## 3. L3: 見積構造型（エンジン選定）

### 3.1 4つの見積構造型

| ID | 型 | 原価の決まり方 | 対応エンジン | 実装状況 |
|---|---|---|---|---|
| **Type A** | チャージレート型 | 設備 × 時間 = 加工費 | metalwork.yaml / plastics / rubber | **✅ 実装済み** |
| **Type B** | 面積/回数/長さ型 | 単位 × 単価 = 加工費 | 新規開発 | ❌ 未実装 |
| **Type C** | BOM組立型 | 部品表展開 + 組立工数 | 新規開発 | ❌ 未実装 |
| **Type D** | バッチ配合型 | 原料配合 × バッチ数 | 新規開発 | ❌ 未実装 |

### 3.2 L1セグメント → L3エンジンのマッピング

| セグメント | 主要エンジン | 補助エンジン | 備考 |
|---|---|---|---|
| **S1** 民生×独立 | **Type A**（切削/成型） | Type B（板金兼業の場合） | 多くはType A単体で対応 |
| **S2** 半導体装置 | **Type A** | - | 精密切削。Type Aの精度要件が高い |
| **S3** 産業機械 | **Type A**（部品）/ **Type C**（完成品） | - | Tier位置で分岐 |
| **S4** 自動車 | **Type A** | - | 量産品のピース単価計算 |
| **S5** 医療機器 | **Type A**（部品）/ **Type C**（完成品） | - | 認証コスト加算が必要 |
| **S6** 航空宇宙 | **Type A** | - | 難削材の工具消耗費加算が必要 |
| **S7** 建設機械 | **Type A** | Type B（板金・溶接構造物） | 大型部品の搬送コスト加算 |
| **S8** 電子・電気 | **Type B**（板金主体） | Type A（切削部品） | 板金+アルマイトが中心 |
| **S9** 食品OEM/化学 | **Type D** | - | 全く別のエンジン |
| **S10** 見込み生産 | - | - | 見積エンジン不要 |

### 3.3 Type A エンジンの業種別カスタマイズ

同じType Aでも業種で微調整が必要な部分:

| カスタマイズ項目 | S1(民生) | S2(半導体) | S4(自動車) | S6(航空) |
|---|---|---|---|---|
| 材料マスタ | 標準14種 | +PEEK/セラミック | 標準 | +チタン/インコネル |
| 設備マスタ | 標準14種 | +クリーンルーム加工 | 標準 | +特殊工程設備 |
| 追加コスト | 表面処理/熱処理 | クリーン洗浄 | PPAP書類作成費 | 材料証明/Nadcap認証費 |
| 工具消耗費 | 標準(含む) | 標準 | 標準 | **要加算（難削材で工具費が2-5倍）** |
| 検査工程 | 抜取検査 | 全数+3次元 | 全数+Cpk管理 | 全数+3次元+書類 |

→ これらは**YAML設定ファイルの差し替え**で対応可能。エンジン本体の改修は不要。

## 4. 5軸統合: セグメントカードの定義

1社を完全に定義するセグメントカードのフォーマット:

```
┌──────────────────────────────────────────────────────┐
│  セグメントカード                                     │
│                                                      │
│  L1: 完成品産業  [IND-A〜Z]                           │
│      Tier位置    [T1/T2/T3/TI]                       │
│      → セグメントコード: [S1〜S10]                    │
│      → ゲノムテンプレ: [選択]                         │
│      → 1stモジュール: [選択]                          │
│                                                      │
│  L2: 生産数     [V1/V2/V3/V4]                        │
│      サイズ     [SZ-S/M/L/XL]                        │
│      品質要求   [Q1/Q2/Q3/Q4]                        │
│      → 月間見積件数: [推定]                           │
│      → ROIスコア: [最高/高/中/低]                     │
│      → 推奨プラン: [スターター/スタンダード/プレミアム]│
│      → 精度目標: [±X%]                               │
│                                                      │
│  L3: 見積構造   [Type A/B/C/D]                       │
│      → 使用エンジン: [選択]                           │
│      → YAML設定: [業種別カスタマイズ]                 │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 具体例

```
例1: 東大阪の金属切削町工場（20名、売上¥8億）
  L1: IND-G(民生) × TI(独立型) → S1
      ゲノム: 汎用金属加工テンプレ
      1st: 見積AI
  L2: V1(単品〜試作) × SZ-M(中型) × Q2(精密)
      月見積: 80件  ROI: 最高
      プラン: スターター ¥15万
      精度: ±15-20%
  L3: Type A → metalwork.yaml（標準）

例2: 浜松の自動車部品加工（80名、売上¥25億）
  L1: IND-A(自動車) × T2 → S4
      ゲノム: 自動車サプライヤーテンプレ（PPAP/VA・VE込み）
      1st: 原価管理
  L2: V3(中ロット) × SZ-S(小型) × Q3(高精密)
      月見積: 15件  ROI: 中（ただし原価管理の価値大）
      プラン: スタンダード ¥25万
      精度: ±10-15%
  L3: Type A → metalwork.yaml + 自動車カスタマイズYAML

例3: 諏訪の半導体装置向け精密加工（35名、売上¥12億）
  L1: IND-B(半導体装置) × T2 → S2
      ゲノム: 精密加工テンプレ + 半導体装置特化
      1st: 見積AI
  L2: V1(単品) × SZ-M(中型) × Q3(高精密)
      月見積: 60件  ROI: 高
      プラン: スタンダード ¥25万
      精度: ±10-15%
  L3: Type A → metalwork.yaml + 精密加工カスタマイズYAML

例4: 大田区の板金加工（15名、売上¥6億）
  L1: IND-G(民生) × TI(独立型) → S1
      ゲノム: 汎用板金加工テンプレ
      1st: 見積AI
  L2: V1(単品〜試作) × SZ-M(中型) × Q1(一般)
      月見積: 100件  ROI: 最高
      プラン: スターター ¥15万
      精度: ±20-25%
  L3: Type B → 板金エンジン（Phase 2で開発）
```

## 5. 攻める優先順位（5軸統合版）

### 5.1 優先度マトリクス

| 優先度 | セグメント | L2典型 | L3 | 社数 | 1stモジュール | 参入理由 |
|---|---|---|---|---|---|---|
| **★1** | S1 民生×独立/T3 | V1-V2 × SZ-S/M × Q1-Q2 | Type A | ~12,000 | 見積AI | 件数最多。エンジン既存。品質要求低=AI許容幅大。社長即決 |
| **★2** | S2 半導体装置×T2-3 | V1 × SZ-M × Q2-Q3 | Type A | ~5,000 | 見積AI | 高単価=失注コスト大。ITリテラシー高。精密ノウハウが溜まる |
| **★3** | S3 産業機械×T2-3 | V2 × SZ-M/L × Q2 | Type A | ~6,000 | 見積AI+生産計画 | 安定市場。リピートで実績蓄積しやすい |
| **★4** | S1 民生×独立（板金） | V1-V2 × SZ-M × Q1-Q2 | Type B | ~8,000 | 見積AI | ★1と同じ工業団地。紹介で横展開。Type Bエンジン開発要 |
| **★5** | S4 自動車×T2-3 | V3 × SZ-S/M × Q3 | Type A | ~8,000 | 原価管理 | 「原価低減AI」としてポジショニング。IATF対応が参入障壁 |
| **★6** | S7 建設機械×T2-3 | V2-V3 × SZ-L/XL × Q1-Q2 | Type A+B | ~4,000 | 見積AI+生産計画 | 大型部品の搬送コスト計算追加が必要 |
| **★7** | S8 電子・電気×T2-3 | V2 × SZ-S/M × Q2 | Type B | ~3,000 | 見積AI | 板金主体。★4のType Bエンジン流用 |

## 7. 繊維/木材/家具/窯業など小規模受注型の扱い

### 方針: 最低限のベース + 個社ゲノムで対応

これらの業種は個別にエンジンを作るほどの市場規模はないが、受注生産型であれば見積AIのニーズはある。

| 業種 | 事業所数 | 見積構造 | 対応方針 |
|---|---|---|---|
| 繊維OEM（E-11の一部） | ~3,000 | Type B（裁断面積×単価+縫製工数） | β型エンジン+繊維YAMLで対応 |
| 木材・木製品（E-12） | ~2,000 | Type A（木工機械のチャージレート） | α型エンジン+木工YAMLで対応 |
| 家具・造作（E-13） | ~3,000 | Type C（BOM+組立。だが簡易） | 簡易BOMモード（Type Cの軽量版） |
| 窯業・土石（E-21の一部） | ~2,000 | Type D（焼成バッチ+仕上げ） | ε型。Phase 3以降 |

**共通方針**: ゲノムの「製造業共通レイヤー」は全業種で使える。業種固有部分はYAML設定ファイルの追加で対応し、エンジン本体の改修は最小限にする。個社ごとの独自ブレインが入ることで、テンプレートが薄くても運用で育つ設計。

### A.2 生産方式の解説

| 方式 | 正式名 | 順序 | 特徴 | 見積AIとの関係 |
|---|---|---|---|---|
| **ETO** | Engineer to Order（個別受注設計） | 受注→設計→生産 | 毎回仕様が違う。都度見積 | **◎ 見積AIの最大ターゲット** |
| **MTO** | Make to Order（受注生産） | 設計済→受注→生産 | 設計は終わっている。リピートあり | ○ 新規分は見積AI、リピートは前回踏襲 |
| **MTS** | Make to Stock（見込み生産） | 設計→生産→在庫→販売 | 定番品を在庫して売る | ✕ 見積作業が発生しない |
| **ATO** | Assemble to Order | 標準モジュール→受注→組合せ組立 | 家電/PC等 | △ 構成選択型の見積（製造業BPOスコープ外） |

### A.3 見積構造型の解説

| 型 | 原価計算の基本式 | 典型的な業種 | 見積1件の工程数 |
|---|---|---|---|
| **Type A: チャージレート型** | 加工費 = (段取り時間 + サイクルタイム × 数量) ÷ 60 × チャージレート(¥/h) | 金属切削、プラスチック射出成型、ゴム成型、木工 | 3-8工程 |
| **Type B: 面積/回数/長さ型** | 加工費 = 単位量(面積/回数/長さ) × 単価 | 板金（レーザー切断長×単価+曲げ回数×単価）、印刷（版数×部数×紙代+加工費）、縫製 | 2-6項目 |
| **Type C: BOM組立型** | 原価 = Σ(部品費) + 組立工数 × レート + 設計費 + 試運転費 | はん用機械、生産用機械（完成品）、業務用機械、家具 | 部品10-100点+組立工程 |
| **Type D: バッチ配合型** | 原価 = Σ(原料 × 配合比 × kg単価) × バッチ数 + 工程費 | 食品OEM、化学受託、化粧品OEM、窯業 | 配合+工程2-5段 |
