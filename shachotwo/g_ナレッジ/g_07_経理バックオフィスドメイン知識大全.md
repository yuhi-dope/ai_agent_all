# 経理・バックオフィス ドメイン知識大全

> 中小企業（従業員10-300名）の経理・バックオフィス実務を網羅的に理解するためのドキュメント。
> 製品紹介ではなく、業務そのものの深い理解を目的とする。
> **対象読者**: 中小企業の経理に提案・支援を行う全ての人。営業・PM・エンジニア・経営者。
> **関連**: `b_07_バックオフィスBPO詳細設計.md`（BPOモジュールの実装仕様に特化）

## 1. 中小企業の経理とは何か

### 1.1 業務の全体像

中小企業の経理は「社長の右腕」的存在。帳簿をつけるだけでなく、資金繰り、税務対応、社会保険、給与計算まで1-3名で全てをカバーする。

```
経理担当者の守備範囲（中小企業の場合）:

  ① 記帳代行: 仕訳入力・領収書整理・通帳突合
  ② 請求管理: 請求書発行・入金消込・督促
  ③ 支払管理: 買掛管理・振込データ作成
  ④ 月次決算: 試算表作成・部門別PL・経費分析
  ⑤ 給与計算: 勤怠集計・社保計算・明細発行
  ⑥ 届出・法定調書: 年末調整・償却資産・法定調書
  ⑦ 資金繰り管理: 入出金予測・手形期日管理
  ⑧ 税理士との窓口: 月次データ提出・税務相談の窓口
```

### 1.2 経理担当者の人数と現実

| 会社規模 | 経理人数 | 担当範囲 | 課題 |
|---|---|---|---|
| 10名以下 | 社長の妻 or 0名 | 記帳は税理士丸投げ | 月次数字が出るのが翌々月 |
| 10-30名 | 1名（兼任） | 記帳+請求+支払+給与 | 1人休むと全業務が止まる |
| 30-100名 | 2-3名 | 分業が始まる | 属人化。引継ぎが困難 |
| 100-300名 | 3-5名 | 経理部として組織化 | システム投資の余力が出てくる |

### 1.3 顧問税理士との関係

```
中小企業の99%が顧問税理士を持つ。

典型的な顧問契約:
  月額顧問料: ¥3-10万（売上規模による）
  決算申告料: ¥15-40万（年1回）
  年末調整: ¥3-10万（従業員数による）
  合計年間: ¥60-200万

税理士がやること:
  ・月次の仕訳チェック・修正
  ・決算整理仕訳
  ・法人税・消費税の申告書作成
  ・税務調査の対応
  ・経営相談（するかどうかは税理士による）

税理士がやらないこと:
  ・日常の仕訳入力（記帳代行は別料金）
  ・請求書発行・入金消込
  ・給与計算・社保手続き
  ・資金繰り管理

★ 「税理士に全部やってもらっている」は幻想。
  実際は会社側で入力したデータを税理士がチェックしているだけ。
  入力が遅れれば月次も決算も遅れる。
```

### 1.4 経理の年間サイクル

```
【月次サイクル（毎月の繰り返し）】

月初 ──────────────────────────────── 月末
  │                                      │
  │ 1日-5日: 前月の証憑回収締切・督促     │
  │ 5日-10日: 前月分の記帳完了・通帳突合   │
  │ 10日: 源泉所得税の納付期限（前月分）   │
  │ 10日-15日: 請求書発行（当月分）        │
  │ 15日-20日: 月次決算処理・試算表作成    │
  │ 20日-25日: 給与計算・明細作成          │
  │ 25日: 給与振込                        │
  │ 末日: 社保料の納付・住民税の納付       │
  │       支払手形の期日管理               │
  │       月次経営報告                    │
  └────────────────────────────────────┘

【年次の主要イベント（3月決算の場合）】

4月: 新年度予算策定。固定資産税第1期。新入社員の社保手続き
5月: ★決算申告（法人税・消費税）。税理士と連携して申告書作成
6月: 労働保険の年度更新（6/1-7/10）。住民税の切替。夏季賞与
7月: ★算定基礎届（7/1-7/10）。源泉所得税の納期の特例（1-6月分）
8-9月: 中間申告（法人税・消費税）
10月: 社保料の新等級適用開始。最低賃金改定
11月: ★年末調整の準備開始。従業員から書類回収
12月: ★年末調整の計算・実行。冬季賞与
1月: ★法定調書の提出（1/31期限）。償却資産申告（1/31期限）
2月: 固定資産税第4期。確定申告準備（個人事業主がいる場合）
3月: ★決算整理仕訳。棚卸。減価償却。引当金
```

## 2. 記帳の実務

### 2.1 証憑の回収

```
証憑（しょうひょう）= 取引の証拠となる書類

回収すべきもの:
  ・領収書・レシート
  ・請求書（受領分）
  ・通帳コピー or ネットバンクCSV
  ・クレジットカード明細
  ・出金伝票・経費精算書

★ 最大のボトルネック: 証憑が集まらない
  営業マンのポケットに領収書が溜まっている
  建設業は現場が散在し、そもそも事務所に来ない
  「月末に出して」が守られない
```

### 2.2 仕訳の基本

```
仕訳 = 取引を「借方」「貸方」に分解して記録すること

例: 事務用品を現金¥5,000で購入
  借方: 消耗品費 ¥5,000 / 貸方: 現金 ¥5,000

実務での判断ポイント:
  ・勘定科目の選択（消耗品費? 事務用品費? 雑費?）
  ・消費税区分（課税? 非課税? 不課税? 免税?）
  ・部門・プロジェクトへの紐付け
  ・インボイス番号の確認（適格/非適格の判定）

★ 取引先×勘定科目のパターンは80%がリピート
  → パターン学習で自動仕訳の精度が上がる
```

### 2.3 通帳突合

```
ネットバンクCSV or 通帳コピーと仕訳を突き合わせる作業

  ・全入金に対応する仕訳があるか
  ・全出金に対応する仕訳があるか
  ・残高が一致するか

不一致の原因:
  ・未記帳（仕訳を入れ忘れている）
  ・二重計上（同じ取引を2回入力）
  ・計上漏れ（振込手数料、利息等の小額取引）
  ・日付のずれ（月末に振り込んだが銀行処理は翌月）
```

## 3. 請求と入金消込

### 3.1 請求書発行

```
請求パターン:
  A) 月末締め翌月末払い（最も一般的）
  B) 出来高払い（建設業。当月の作業分を翌月請求）
  C) 完成払い（納品完了後に一括請求）
  D) 前受金+出来高（着手金→以降月次払い）

インボイス要件（2023年10月〜）:
  ・適格請求書発行事業者の登録番号
  ・税率区分（10%/8%/対象外）の明記
  ・税額の端数処理ルール（1請求書につき1回の端数処理）
```

### 3.2 入金消込

```
消込 = 請求と入金を突き合わせて、誰が何を払ったかを確定する作業

難しいケース:
  ・振込名義が請求先と違う（家族名義、略称、会社名の揺れ）
  ・振込手数料が差し引かれている（差額処理が必要）
  ・部分入金（請求額の一部だけ振り込まれる）
  ・複数請求の合算振込（3件分をまとめて1回で振り込まれる）
  ・相殺（元請から返品や手直し費用を差し引かれる）

建設業特有:
  ・手形受領 → 満期日に口座入金（90-120日後）
  ・電子記録債権 → でんさいネット経由で入金
  ・ファクタリング → 売掛金を早期に現金化
  ・出来高払い+相殺の複合パターン
```

## 4. 支払管理

### 4.1 支払の流れ

```
① 請求書受領・登録
  ・下請・外注先・資材業者からの請求書を受領
  ・金額・支払期日・振込先を確認
  ・インボイス登録番号の確認
  ・工事別/プロジェクト別に原価を紐付け

② 支払承認
  ・支払一覧表を作成 → 社長 or 経理責任者の承認
  ・予算対比チェック
  ・資金繰り表と照合（支払可能か）
  ・支払優先度: 税金・社保 → 手形決済 → 外注費 → 材料費 → 経費

③ 振込実行
  ・全銀フォーマットでFBデータ作成
  ・ネットバンクにアップロード → 振込実行
  ・振込手数料の管理（先方負担/当方負担）
```

### 4.2 源泉徴収

```
支払先によって源泉徴収の要否が変わる:

  法人への支払い → 源泉徴収不要
  個人の請負代金（一人親方等）→ 源泉徴収不要
  個人の設計士・士業・コンサル → 源泉徴収必要（10.21%）

★ 一人親方が「請負」か「雇用」かの判断を間違えると
  源泉徴収漏れ+社保未加入のダブルリスク（偽装請負問題）
```

## 5. 月次決算

### 5.1 月次決算の流れ

```
① 月次締め処理（翌月5-10営業日が目標）
  ・売上計上の確定
  ・原価の確定
  ・経過勘定の計上（前払費用・未払費用・前受金・未収入金）
  ・減価償却費の月割計上
  ・仮勘定の精算

② 試算表・管理帳票作成
  ・合計残高試算表（BS/PL）
  ・部門別損益計算書
  ・資金繰り表
  ・前期比・予算比の差異分析

③ 経営報告
  ・社長への月次報告資料作成
  ・異常値・注意点のサマリー
  ・キャッシュフロー予測
```

### 5.2 月次決算で見るべき指標

```
最低限チェックする項目:
  ・売上高の前月比・前年同月比
  ・粗利率の推移（原価率が上がっていないか）
  ・販管費の異常値（前月比で大きく変動した科目）
  ・現預金残高と来月の支払予定の比較
  ・売掛金の滞留（回収遅延が発生していないか）
  ・借入金の返済スケジュール
```

## 6. 給与計算

### 6.1 給与計算の構成

```
総支給額 = 基本給 + 各種手当（残業・通勤・資格・役職等）

控除額 = 社会保険料 + 所得税 + 住民税

差引支給額 = 総支給額 - 控除額
```

### 6.2 社会保険料

```
健康保険: 標準報酬月額 × 保険料率（約10%。会社と折半）
厚生年金: 標準報酬月額 × 18.3%（会社と折半）
雇用保険: 賃金 × 料率（建設業は他業種より高い。事業主負担あり）
労災保険: 賃金 × 料率（全額事業主負担。業種で料率が異なる）

★ 算定基礎届（7月提出）で標準報酬月額が改定される
  → 4-6月の報酬で等級が決まるため、この期間の残業が影響
```

### 6.3 年末調整

```
目的: 毎月の源泉徴収額と年間の正確な税額を精算する

従業員から回収する書類:
  ・扶養控除等申告書
  ・配偶者控除等申告書
  ・保険料控除申告書 + 控除証明書
  ・住宅ローン控除申告書 + 残高証明書

★ 建設業は現場作業員が多く、書類回収が最大の手間
  → 事務所に来ない人から書類を回収する方法の工夫が必要
```

## 7. 会計ソフトの実務比較

### 7.1 主要3社の比較

| 項目 | freee | マネーフォワード | 弥生 |
|---|---|---|---|
| 設計思想 | 簿記知識不要。自動化重視 | 簿記知識前提。正統派 | 従来の経理実務を踏襲 |
| 仕訳入力 | 独自UI（取引→自動仕訳） | 標準的な仕訳入力 | 従来型の仕訳入力 |
| 銀行連携 | 強い（API連携多数） | 強い | やや弱い |
| クラウド | ネイティブクラウド | ネイティブクラウド | クラウド版あり（やよいオンライン） |
| API | REST API公開 | REST API公開 | 限定的 |
| 料金 | ¥2,380-39,800/月 | ¥2,980-5,980/月 | ¥8,800-30,000/年 |
| 得意な規模 | スタートアップ〜中小 | 中小〜中堅 | 零細〜中小 |
| 税理士の評判 | 好き嫌いが分かれる | 概ね好評 | 最も馴染みがある |

### 7.2 実務上の違い

```
freee:
  ・銀行明細から「自動で経理」。仕訳候補が自動生成される
  ・簿記を知らなくても使える設計だが、逆に簿記を知っている人には違和感
  ・勘定奉行やPCAからの乗り換えで「仕訳が探しにくい」という声
  ・API連携が豊富。外部ツールとの連携に向いている

マネーフォワード:
  ・標準的な仕訳入力画面。経理経験者には直感的
  ・グループ会社向けの連結機能がある
  ・給与計算（MFクラウド給与）との連携がスムーズ
  ・税理士事務所の採用率が高い

弥生:
  ・最も歴史が長い。中小企業の経理に最も浸透している
  ・デスクトップ版（弥生会計）とクラウド版（やよいオンライン）がある
  ・デスクトップ版のほうが機能が豊富
  ・「うちは弥生で30年やってます」という会社が多い
```

### 7.3 税理士との相性

```
税理士事務所の会計ソフト利用状況:
  弥生: 約50%（圧倒的シェア）
  マネーフォワード: 約20%
  freee: 約15%
  勘定奉行・PCA等: 約15%

★ 顧問税理士が使っているソフトに合わせるのが最も摩擦が少ない
  → 「うちの税理士が弥生なので弥生にしてください」は日常
  → 会計ソフトの変更は税理士変更とセットになることが多い
```

## 8. 消費税の実務

### 8.1 インボイス制度

```
2023年10月から施行。仕入税額控除にはインボイス（適格請求書）が必要。

影響が大きい場面:
  ・一人親方（免税事業者）への外注: 控除できない消費税分をどうするか
  ・経過措置: 2026/9まで80%控除 → 2029/9まで50% → 以降0%
  ・国税庁のインボイス登録番号照会が必要

実務対応:
  ① 受領する請求書のインボイス登録番号を確認
  ② 未登録の取引先リストを管理
  ③ 未登録先への対応方針を決定（値下げ交渉 or 自社負担）
```

### 8.2 消費税の申告

```
消費税の中間申告回数（前期の消費税額による）:
  48万円以下: 不要
  48万円超〜400万円以下: 年1回（半期）
  400万円超〜4,800万円以下: 年3回（四半期）
  4,800万円超: 年11回（毎月）

★ 建設業は売上が大きく消費税額も大きいため四半期申告が多い
```

## 9. 資金繰り

### 9.1 中小企業の資金繰りの現実

```
「黒字倒産」は中小企業で実際に起きる。

原因:
  ・売上はあるが入金が遅い（売掛金の回収サイクル60-120日）
  ・支払いは先行する（材料費・外注費は即払い or 翌月払い）
  ・手形の不渡りは2回で銀行取引停止 → 実質倒産

資金繰り表の作り方:
  入金予定: 工事別の出来高予定 × 得意先ごとの支払サイト
  出金予定: 外注費+材料費+給与+社保+税金+手形決済
  過不足判定: 不足月 → 手形割引 or ファクタリング or 短期借入

★ 「来月300万足りません」を早期に把握できるかが経理の価値
```

## 11. 税理士法の境界線

経理業務を支援する際、税理士法（税理士の独占業務）との境界を理解しておく必要がある。

```
無資格でOK:
  ・記帳代行（仕訳入力）
  ・請求書発行・入金消込
  ・給与計算
  ・社保の届出代行（社労士法にも注意）
  ・「過去の仕訳パターンに基づく候補提示」

税理士の独占業務（違反は2年以下の懲役 or 100万円以下の罰金）:
  ・税務相談（「この経費は損金になりますか？」への回答）
  ・税務申告書の作成・提出
  ・税務調査の代理

グレーゾーン:
  ・「この仕訳はこうすべき」→ 税務判断に近い
  ・対策: 判断が必要なものは顧問税理士に確認するフローを明確にする
```
