# ADR-011: ANDPAD連携戦略（競合ではなくチャネルとして使う）

## ステータス

**承認** (2026-03-21)

## 日付

2026-03-21

## コンテキスト

建設業vs製造業BPO比較（`建設業vs製造業_BPO比較.md`）で以下が判明:

```
ANDPAD: 23万社 / 68万ユーザー
  → 施工管理SaaS（写真・工程・書類・検査）
  → API公開あり
  → シャチョツーとターゲット・機能が補完関係（競合ではない）
```

ANDPADが持つもの: 施工現場の「実行記録」（写真・工程進捗・検査記録）
シャチョツーが持つもの: 会社の「頭脳」（見積・ルール・判断基準・自動化）

→ 連携することで「ANDPAD上の現場記録」をシャチョツーのナレッジとして自動取り込みできる。

## 検討した選択肢

### 選択肢A: 無視（連携しない）
- メリット: 実装コストゼロ
- デメリット: 23万社のユーザーベースを活用できない。導入企業でデータ二重入力が発生

### 選択肢B: ANDPAD経由でパートナー展開（採用）
- メリット: 23万社へのリーチ。ANDPAD認定パートナーとして信頼性向上。導入障壁低下
- デメリット: API連携の実装コスト + パートナー契約交渉コスト

### 選択肢C: 直接競合する（施工管理機能を自社実装）
- デメリット: ANDPADの本丸に正面衝突。勝ち目なし。避けるべき

## 決定

**選択肢B**を採用する。「ANDPADと連携してシャチョツーの価値を高め、ANDPADをチャネルとして使う」。

### 具体的アクション

#### Phase 1（今すぐ: Week 7〜）

**ANDPAD APIの調査と連携設計**

```
ANDPAD公開API（調査済み・参考）:
  - 案件情報API: プロジェクト・工事情報の取得
  - 工程API: 工程表データの取得
  - 写真API: 現場写真の取得
  - ドキュメントAPI: 書類一覧の取得

シャチョツーが利用したい情報:
  1. 工事原価実績 → 見積精度向上のナレッジとして蓄積
  2. 工程遅延パターン → リスク検知AIの学習データ
  3. 安全書類ステータス → BPO安全書類チェックの自動化
```

**パートナー申請**
- ANDPADパートナープログラムに申請（パイロット投入前）
- ANDPAD公式マーケットプレイス掲載を目標とする（Phase B）

#### Phase 2（パイロット後: Week 10〜）

**双方向連携の実装**
```python
# workers/connector/andpad.py（実装予定）

class ANDPADConnector:
    """ANDPAD APIコネクタ。読み取り専用（Phase 1）→ 書き込みも（Phase 2+）"""

    async def fetch_project_costs(self, project_id: str) -> dict:
        """工事原価実績をANDPADから取得してナレッジに変換"""
        # ANDPAD API → シャチョツーknowledge_itemsに自動登録
        ...

    async def fetch_safety_documents(self, project_id: str) -> list[dict]:
        """安全書類のステータスを取得してBPOチェックに活用"""
        ...

    async def push_estimation_result(self, project_id: str, estimation: dict):
        """シャチョツーの見積結果をANDPADの案件に紐付け（Phase 2+）"""
        ...
```

#### Phase 3（本格展開）

**ANDPAD公式パートナー + マーケットプレイス掲載**
- ANDPAD上のアプリとして「シャチョツー AI見積・業務自動化」を掲載
- ANDPADの23万社ユーザーへのダイレクトリーチ
- 「ANDPADを使っている建設会社はシャチョツーも使う」標準セットになる

### 連携の優先順位

```
優先1: 工事原価実績の取り込み（見積精度向上・最重要）
優先2: 安全書類ステータス（BPO安全書類チェックの自動化）
優先3: 工程データ（リスク検知・原価変動予測）
後回し: 写真連携（Phase 3: 施工品質AI検査）
```

## 影響

- **workers/connector/andpad.py**: 新規実装が必要（Phase 2）
- **パイロット提案**: 「ANDPADを使っているなら連携でき、入力が減る」を営業トークに使える
- **競合リスク低減**: ANDPAD側に「シャチョツーと競合しない」と認識させることで、ANDPADが独自のAI見積機能を開発するリスクを下げる

## 営業での使い方

```
営業トーク（ANDPAD使用中の建設会社向け）:
「今ANDPADで管理している原価データや安全書類を、シャチョツーが自動で読み込みます。
 見積書を作るときにANDPADの実績データを参照して、より精度の高い見積が出せます。
 ANDPAD側での入力は変わらず、シャチョツーが裏側で賢くなる仕組みです。」
```
