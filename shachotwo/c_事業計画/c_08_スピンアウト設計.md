## 2. 事業スキームの全体像

### 「2つの帽子」モデル

```
【帽子1】杉本個人 = コンサル会社の正社員（経営企画）
  ・経営企画業務を担当。給与をもらう
  ・この業務の成果物はコンサル会社に帰属

【帽子2】湊合同会社 = コンサル会社の業務委託先
  ・AI BPOプロダクト（シャチョツー）の開発・提供
  ・業務委託報酬をもらう
  ・この業務の成果物（プロダクトIP）は湊合同会社に帰属
```

### 構造図

```
湊合同会社
  │
  │  業務委託契約（法人対法人）
  │  ・AI BPOプロダクトの開発・提供
  │  ・IPは湊合同会社に帰属
  │
  ▼
コンサル会社 ←── 杉本が正社員（経営企画）として在籍
  │
  │  コンサル + AIツール付きで提案
  │
  ▼
コンサル会社の顧客（製造業・建設業・不動産・医療等）
```

### フェーズ別移行プラン

```
Phase 1（今〜6ヶ月）: 正社員メイン + 業務委託少
  ・正社員: 週5日（経営企画）
  ・業務委託: 業務時間外 + 週末にプロダクト開発
  ・報酬: 給与 + 業務委託¥0-10万/月
  ・先方に副業承認を書面で取得

Phase 2（6ヶ月〜1年）: 正社員縮小 + 業務委託拡大
  ・正社員: 週3-4日に短縮（時短 or パート切替）
  ・業務委託: プロダクト導入支援に時間が必要になる
  ・報酬: 給与減 + 業務委託¥20-30万/月 + 成果報酬

Phase 3（1年〜）: 業務委託のみ
  ・正社員: 退職（円満退社）
  ・業務委託: 湊合同会社として月額固定 + 成果報酬
  ・コンサル会社は「最大顧客 + 株主」として継続
```

## 3. IP（知的財産）の帰属

### 「2つの帽子」でIPを明確に分離

| 帽子 | 業務内容 | IP帰属先 | 根拠 |
|---|---|---|---|
| 正社員（経営企画） | 戦略立案・顧客対応・社内業務 | コンサル会社 | 著作権法15条（職務著作） |
| 湊合同会社（業務委託） | AI BPOプロダクト開発・運用 | **湊合同会社** | 業務委託契約。職務著作の適用外 |

### 契約書に明記する事項

```
① 杉本の正社員業務 = 経営企画（戦略立案、顧客対応等）
② 湊合同会社の委託業務 = AI BPOプロダクトの開発・運用
③ ②の成果物のIPは湊合同会社に帰属
④ コンサル会社は杉本の副業（湊合同会社の運営）を承認する
⑤ ①と②の業務範囲が重複しないことを双方確認
```

## 4. エクイティ設計

### 資本構成（株式会社化後）

```
杉本（創業者）:       普通株式 90-95%
コンサル会社:          新株予約権 5%（提案）→ 最大10%（交渉上限）
将来VC・従業員SO枠:    残り
```

### 新株予約権の設計

| 項目 | 内容 |
|---|---|
| 対象 | コンサル会社に対して発行 |
| 比率 | 完全希薄化ベースで **5%（提案）→ 最大10%（交渉上限）** |
| 行使価格 | 株式会社化時の簿価（低額設定可能） |
| 行使条件 | MRR 300万達成 / シリーズA調達完了 / 提携開始から2年経過のいずれか |
| 行使期間 | 発行から10年間 |
| 議決権 | 行使するまでなし |
| 発行タイミング | 合同会社→株式会社に組織変更したタイミング |

### 5%提案の根拠

| 先方の貢献レベル | 妥当な範囲 |
|---|---|
| R&D費を数千万投資 | 15-20% |
| 人員・設備を提供 | 10-15% |
| 顧客アクセス+協業関係のみ（今回） | **5-10%** |

経産省「起業家主導型カーブアウト実践のガイダンス」（令和6年4月）に準拠。

## 5. 業務委託報酬の設計

### フェーズ別報酬テーブル

| フェーズ | 固定報酬 | 成果報酬 | 合計イメージ |
|---|---|---|---|
| Phase 1（正社員メイン） | ¥0-10万/月 | なし | 給与で生活 |
| Phase 2（移行期） | ¥20-30万/月 | 導入1社あたり¥10-20万 | ¥30-50万 |
| Phase 3（独立後） | ¥30-50万/月 | 導入1社あたり¥20-30万 + MRRの10-15% | ¥50万〜 |

### 成果報酬の設計

- **導入ボーナス**: コンサル会社経由で新規顧客が1社入るたびに、初年度MRRの1-2ヶ月分
- **継続リベート**: コンサル会社経由の顧客のMRRの10-15%を毎月コンサル会社に還元
- **効果**: 先方は「売れば売るほど儲かる」構造になり、積極的に顧客を紹介する

## 6. 顧客データの利用範囲

| データ種別 | 利用可否 | 条件 |
|---|---|---|
| 顧客の業務データ（図面・見積等） | **利用可** | AI学習に使用。匿名化必須。個社特定不可な形で |
| 顧客の個人情報（社名・担当者名等） | **利用不可** | 湊合同会社には渡さない。コンサル会社が管理 |
| 集計・統計データ（業界平均値等） | **利用可** | 匿名化済みの統計データとしてプロダクトに組み込み |
| コンサル会社の業務ノウハウ | **利用可** | ゲノムJSON/YAMLの設計に反映。コンサル会社の承認制 |

### 契約書に入れるデータ条項

```
① 委託業務の遂行に必要な範囲で、甲（コンサル会社）の顧客データを
   乙（湊合同会社）に提供する
② 乙は当該データを匿名化した上でAIモデルの学習に利用できる
③ 個社を特定可能な形でのデータ利用・第三者提供は禁止
④ 契約終了後、乙は甲から提供を受けた生データを削除する
   （ただし匿名化済みの学習済みモデルは乙に帰属）
```

**最重要ライン**: 「生データは返す。学習済みモデルはうちのもの」

## 7. 独占 / 非独占

### 基本方針: 非独占。ただし先方に最恵待遇

| 条件 | 内容 |
|---|---|
| 基本 | 湊合同会社は他社にも自由にプロダクトを販売できる（**非独占**） |
| 優遇① | コンサル会社は市場価格の**50%割引**で永続利用 |
| 優遇② | 新機能の**先行アクセス権**（一般公開の3ヶ月前に利用可能） |
| 優遇③ | コンサル会社経由の顧客のMRRの**10-15%を還元** |
| 任意制限 | コンサル会社の直接競合（建設コンサル同業）への販売は**1年間控える** |

### 先方が独占を求めてきた場合の落とし所

- 「建設業界に限り、6ヶ月間の独占利用権」（期間限定・業界限定）
- プロダクト全体の独占は絶対に渡さない

## 8. コンサル会社での立場

### 正社員 + 副業承認

| 項目 | 内容 |
|---|---|
| 肩書き | 経営企画（現状維持） |
| 副業承認 | 湊合同会社の運営を書面で承認してもらう |
| 業務範囲 | 経営企画業務とAI BPO開発業務を明確に分離 |
| 移行条件 | Phase 2以降、時短勤務 or 業務委託に切替 |

### 副業許可申請書に書くこと

```
① 副業の内容: AI BPOプラットフォームの開発・運営（湊合同会社）
② 本業への影響: 経営企画業務に支障を来さない
③ 競合関係: 競合しない（ツール提供者と顧客の関係）
④ 会社の利益: コンサル会社の顧客にAIツールを提供することで
   コンサル事業の付加価値が向上する
```

## 9. 就業規則リスク分析（ブリッジワン就業規則 R8.1.1改訂）

### 重要条文と対応

| 条文 | 内容 | リスク | 対応 |
|---|---|---|---|
| **第25条（兼業禁止）** | 会社の許可なく社外業務・自営を禁止 | **許可制**。許可を取れば合法 | 兼業許可申請書を提出 |
| **第26条（機密情報）** | 機密情報の社外持出・個人利用を禁止。退職後も適用 | AI開発に機密情報が混入するリスク | 業務データの分離を明記 |
| **第80条（職務発明）** | 会社の業務範囲に属する発明は会社が原始取得可能 | AI開発がブリッジワンの業務範囲と重なるリスク | 兼業許可書で業務範囲の分離を確認 |
| **第83条(21)（懲戒）** | 許可なく在籍のまま他に雇用されると懲戒対象 | 無許可兼業の場合のリスク | 必ず書面許可を取得 |

### 兼業許可申請書に盛り込む内容

```
■ 基本情報
  兼業の内容: 湊合同会社によるAI BPOプラットフォームの開発・運営
  根拠: 就業規則第25条に基づく兼業許可の申請

■ 本業への影響なし
  ① 兼業の時間帯: 業務時間外（夜間・休日）を基本
  ② 経営企画業務に支障を来さない
  ③ 競合しない（ツール提供者と顧客の関係）

■ 機密情報の分離（第26条対応）
  ④ ブリッジワンの機密情報をAI開発に流用しない
  ⑤ 使用データは湊合同会社が独自取得 or 業務委託契約で適正提供されたもののみ
  ⑥ ブリッジワンの業務用端末・システムをAI開発に使用しない

■ 職務発明の分離（第80条対応）
  ⑦ AI BPOプロダクトは、ブリッジワンの業務範囲（コンサルティング）とは
     異なる独立した事業（ソフトウェア開発）である
  ⑧ 本兼業により生じる発明・著作物は、ブリッジワンの職務上の成果物ではなく、
     湊合同会社の事業として創出されたものであることを双方確認する
     ★この一文に会社の承認印をもらうことが最重要★
```

### 業務委託管理規程（ビーバーズ PMS-10）への対応

ブリッジワン/ビーバーズが湊合同会社に業務委託し顧客の個人情報を扱う場合:
- 「委託先リスト」への湊合同会社の登録が必要
- 「個人情報委託先評価シート」での評価が必要
- 個人情報に関する条項を含む契約を締結（Section 6のデータ条項で対応）

## 10. 業務提携契約の条項（全12項目）

| # | 条項 | 内容 |
|---|---|---|
| 1 | **IP帰属** | プロダクトIPは湊合同会社に帰属。契約書に明記 |
| 2 | **業務範囲の分離** | 正社員業務（経営企画）と委託業務（AI開発）を明確に分離 |
| 3 | **副業承認** | コンサル会社は杉本の湊合同会社運営を書面で承認 |
| 4 | **業務委託報酬** | Phase別の固定報酬+成果報酬（Section 5参照） |
| 5 | **エクイティ** | 株式会社化後、新株予約権5%を発行（最大10%） |
| 6 | **経営不干渉** | コンサル会社は湊合同会社の経営に関与しない |
| 7 | **先買権（ROFR）** | コンサル会社が株式売却時、杉本が同条件で優先買取可能 |
| 8 | **優先ライセンス** | コンサル会社は永続的に50%割引+先行アクセス権（非独占） |
| 9 | **顧客データ** | 匿名化の上でAI学習に利用可。生データは契約終了後削除。学習済みモデルは湊合同会社に帰属 |
| 10 | **秘密保持（NDA）** | 双方の機密情報を保護 |
| 11 | **競業避止** | なし（ツール提供者と顧客の関係） |
| 12 | **契約期間・解除** | 1年自動更新。解除は3ヶ月前通知。解除後もIPは湊合同会社に帰属 |

## 11. 先方への説得ロジック

### 先方のメリット

1. **開発コストゼロ** — 湊合同会社が開発する
2. **全業界で使えるAIツールを50%割引で永続利用** — 多業界展開の武器
3. **新株予約権5-10%** — 成功すればリターン。失敗しても損失ゼロ
4. **MRR還元10-15%** — 顧客を紹介するほど御社にも収益が入る
5. **経産省ガイダンス準拠** — 国の推奨スキーム

### 先方へのトーク

> 「私は自分の法人でAIプロダクトを開発し、御社の業務パートナーとして提供します。
>
> 御社にとってはリスクゼロです：
> - 開発コストは私の法人が持つ
> - 御社は業務委託費だけ
> - AIツールを御社の全クライアントに優先展開できる
> - 御社経由の顧客からMRRの10-15%を毎月還元
> - プロダクトが成功したら、貢献対価として新株予約権5-10%を差し上げる
>
> お互いがお互いのIPOを目指す。御社はコンサルで1000億、私はプロダクトで1000億。
> その過程で、お互いの武器になる関係です。」

### 想定される懸念と回答

| 懸念 | 回答 |
|---|---|
| 「顧客を持っていかれるのでは」 | ツール提供者と顧客の関係。御社の顧客は御社のもの。永続ライセンスで保証 |
| 「5-10%は少ない」 | 御社の実質コスト負担はゼロ。経産省ガイダンスの推奨レンジ内 |
| 「失敗したら」 | 御社の損失ゼロ。業務委託期間中のAIツール利用という実益だけが残る |
| 「うちの中で作った方がよくない？」 | 社内開発だとVC資金調達不可、エンジニア採用困難、投資の意思決定遅延 |
| 「他社に売るのは困る」 | 非独占だからこそプロダクトが早く成長し、御社が使うツールの品質が上がる。最恵待遇3点で保証 |

## 12. 税務上の注意点

| 場面 | 課税関係 | 対策 |
|---|---|---|
| 業務委託報酬 | 湊合同会社の売上として法人税 | 経費（開発費・クラウド代等）と相殺 |
| 給与+法人報酬の二重取得 | 給与所得+法人利益は別管理 | 税理士に二重課税を回避する構成を相談 |
| 組織変更（合同会社→株式会社） | 原則非課税 | そのまま変更するだけ |
| 新株予約権発行（コンサル会社へ） | 有償発行なら発行時は非課税 | 行使価格を適正に設定 |
