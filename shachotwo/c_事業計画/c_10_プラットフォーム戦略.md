## 4. フェーズ計画

### Phase 1: 1社で全部やる（〜PMF達成）
- 建設・製造でPMF検証
- 共通エンジン + ゲノム設計を固める
- 業界人アドバイザーは巻き込む（株主ではなく）
- ゲノムJSON追加で新業種テスト可能な状態にする

### Phase 2: 成功業界から分社化（PMF後〜1年）
- 月商1000万超の業界 → 業界BPO子会社を設立
- 業界パートナーに株を渡して経営を任せる
- 本体はプラットフォーム利用料（SaaS的）で回収
- ゲノムオーサリングUI提供（業界人がブラウザでBPOフロー定義）

### Phase 3: 横展開（1年〜）
- 成功モデルを見た他業界パートナーが自発的に参加
- 「うちの業界でもやりたい」→ ゲノム定義して子会社化
- プラットフォームとしてのネットワーク効果が発動

## 5. 収益構造

### 本体（プラットフォーム）
```
収入:
  - プラットフォーム利用料: 子会社売上の30%
  - ブレイン基本料: ¥30,000/月/テナント（全業界共通）
  - AI進化ライセンス: ゲノム精度向上・新機能の提供

コスト:
  - インフラ（GCP Cloud Run + Supabase）
  - LLM API（Gemini/Claude）
  - コアエンジン開発・保守
```

### 子会社（業界BPO）
```
収入:
  - BPO月額: ¥15-45万/テナント（業界による）
  - 導入支援: 初期セットアップ費用
  - カスタマイズ: 業界固有のゲノム調整

コスト:
  - プラットフォーム利用料（売上の30%）
  - 営業・カスタマーサクセス人件費
  - 業界ドメイン専門家
```

### シミュレーション（3年目）
```
コア6業界 × 平均250社 × 平均¥25万/月 = 月商3.75億円
  本体取り分（30%）: 月1.13億円 → 年13.5億円
  + ブレイン基本料（1500社 × ¥3万/月）: 年5.4億円
  本体合計: 年約19億円
```

## 8. 自社Agent OS（ドッグフーディング）

> **2026-03-28 追加**
> シャチョツー社自身がAgent OSを使って運営する。自社が最初の顧客。

### 8エージェント体制

| エージェント | 役割 | 使うパイプライン |
|---|---|---|
| CEO Agent | KPIダッシュボード・What-if・競合監視・意思決定記録 | brain/visualization |
| Sales Agent | リード発見→アウトリーチ→商談→提案書→契約 | sales/* |
| CS Agent | ヘルススコア→プロアクティブ介入→問い合わせ→解約防止 | cs/* |
| Product Agent | 全顧客精度監視→改善コード生成→A/Bテスト→ゲノム進化 | evaluation engine |
| Dev Agent | バグトリアージ→自動修正→テスト→セキュリティ→設計書同期 | Claude Code連携 |
| Finance Agent | 請求→入金→PL/BS→キャッシュフロー→税務準備 | common/expense |

### メリット

- 「自社で使っている」は最強の営業トーク
- 共有基盤は1回作れば顧客向け・自社向けの両方で使える
