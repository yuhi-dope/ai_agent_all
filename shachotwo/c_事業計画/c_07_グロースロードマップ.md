## Phase A: 建設業で「狂信者」を作る（〜Week 12）

> **目標**: 建設会社3社が「これなしで見積できない」と言う状態

### KPI

| 指標 | 目標 | 現状 |
|---|---|---|
| NPS | ≥ 40（業界平均15） | 未計測 |
| DAU/MAU | ≥ 60% | 未計測 |
| 「ないと困る」率 | ≥ 70% | 未計測 |
| 見積業務時間削減 | ≥ 50% | 未計測 |
| Q&A精度（keyword hit rate） | ≥ 75% | Eval未整備 |

### 施策

1. **見積精度を死ぬほど上げる**
   - 建設業ゴールデンデータセット100問作成 → 週次Eval実行
   - RAG enhanced_search（クエリ拡張+再ランキング）本番導入
   - 実際の見積書PDFからの学習品質を改善

2. **HitL（承認フロー）でゼロ事故を保証**
   - 見積書・請求書は必ず人間が確認してから送付
   - 承認UIでの「修正して承認」機能を充実させる

3. **ネットワーク効果の種を蒔く**
   - 見積単価データを匿名集計できるスキーマを今から設計
   - 「業界平均単価ベンチマーク（β）」をパイロット企業に先行提供

### 判断ゲート（Phase B移行条件）

```
建設業NPS ≥ 40
AND 建設業MRR ≥ ¥500万（20社 × ¥25万）
AND Q&A精度 keyword_hit_rate ≥ 75%
AND HitL承認フロー: 事故ゼロ（3ヶ月連続）
```

## Phase E: シャチョツー基盤を活用した事業拡張（Year 3〜5）

> **前提**: Phase A〜Dでシャチョツーがローンチ・安定稼働し、導入企業のデジタルツインデータが十分に蓄積された状態。
> **考え方**: シャチョツーのデータ資産・AI基盤・顧客基盤を「次の事業」に転用する。

---

### E-1. 見積もりマーケットプレイス（最優先）

> **一言**: 「発注したい会社」と「受注したい会社」をAIが自動マッチング

#### 現状の課題

```
建設会社が下請けに見積もりを取りたい
→ 知り合いの3社に電話 → 1週間待つ → 比較 → 決定
→ 価格の妥当性がわからない、時間もかかる
```

#### シャチョツーで作れる世界

```
案件をプラットフォームに投稿
→ AIが過去データから適正価格レンジを算出
→ 登録済みの受注候補に自動通知
→ 各社がAI見積もりで即回答（数時間以内）
→ 発注者は価格・納期・評価で比較して決定
```

#### なぜシャチョツーだからできるか

- 各社のデジタルツインに**原価構造・得意分野・稼働状況**が入っている
- AIが「この案件ならA社が最適」と**根拠付きで推薦**できる
- 業界ベンチマークデータ（Phase B〜Dで蓄積済み）が価格の妥当性を担保

#### 収益モデル

```
成約手数料 3-5%
月10件成約 × 平均500万円/案件 × 3% = 月150万円/エリア
全国10エリア展開 → 月1,500万円の追加MRR
```

#### 対象業界（横展開順）

1. 建設業（元請→下請けの見積もり）
2. 製造業（部品加工の外注先マッチング）
3. 人材派遣（人材とニーズのマッチング）

---

### E-2. 成果保証型BPO（料金モデルの進化）

> **一言**: 「効果が出なければほぼ無料」で中小企業の導入障壁を消す

#### 現状の課題

```
BPOコア ¥250,000/月 → 効果が出ても出なくても同じ料金
中小企業「本当に効果出るかわからないのに月25万は…」
```

#### 成果保証モデル

```
基本料 ¥50,000/月（最低ライン）
+ 削減できた人件費の20%を成果報酬

例: 営業3人分の業務を自動化 → 月60万円の人件費削減
→ シャチョツーの報酬: ¥50,000 + ¥120,000 = ¥170,000/月
→ クライアントは差額43万円を純粋に得する
```

#### なぜこれが強いか

- 「効果がわからない」→ 導入失敗パターン（head_db BRAIN-012）を根本解決
- 「IT投資に踏み切れない」中小企業（head_db BRAIN-026）が導入しやすい
- 実績データ（年間3,600万円削減等）がそのまま営業資料になる
- **開発不要**。料金体系の変更だけで即実施可能

#### 実施条件

- デジタルツインで効果測定が定量化できる状態（導入前後のBefore/After計測）
- 最初は建設業の積算BPO・製造業の見積もりBPOに限定して検証

---

### E-3. 中小企業の経営データバンク

> **一言**: デジタルツインの匿名データを「業界の共有資産」にする

#### シャチョツーに蓄積されるデータ

```
導入企業が増える
→ 16業界のコスト・プロセス・リスクデータが蓄積
→ 匿名化して集計
→ 「建設業の原価率は平均68%、上位25%は61%以下」
→ 「製造業で見積もり精度が高い企業の共通点はXYZ」
```

#### 売り先と収益

| 顧客 | 提供価値 | 料金目安 |
|---|---|---|
| 銀行・信用金庫 | 中小企業融資の審査データ（決算書だけでは見えない実態） | ¥500万〜/年 |
| M&A仲介 | 事業承継時のデューデリジェンスデータ | ¥10万/件 |
| 保険会社 | 業種別リスクデータ（経営リスク保険の設計） | ¥300万〜/年 |
| 導入企業自身 | 「業界平均と比べてうちはどうか」ベンチマークレポート | BPO契約に含む |

#### 前提条件

- **導入100社以上**でデータに統計的意味が出る
- 匿名化・利用同意の仕組み（Phase Dのデータ事業設計と接続）

---

### 拡張事業の優先順位

| 順序 | 施策 | 実現しやすさ | 収益インパクト | 着手条件 |
|---|---|---|---|---|
| **1st** | 成果保証型BPO | ★★★★★ | ★★★ | 即実施可能（料金変更のみ） |
| **2nd** | 見積もりマーケットプレイス | ★★★★ | ★★★★★ | Phase B以降（積算AI安定+20社） |
| **3rd** | 経営データバンク | ★★★ | ★★★★ | Phase C以降（導入100社超） |

> **判断基準**: まず「成果保証」でパイロット企業の導入障壁を下げ、企業数を増やす。
> 企業数が増えたら「マーケットプレイス」と「データバンク」が自然に立ち上がる。
> 3つは独立ではなく、**1→2→3の順で基盤が積み上がる構造**。
