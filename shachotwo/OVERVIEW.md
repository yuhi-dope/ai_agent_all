### OS Level 0→3 成長パス（6ヶ月タイムライン）

| 期間 | OSレベル | 精度目標 | マイルストーン |
|---|---|---|---|
| Week 1-2 | Level 0構築 | -- | OS基盤デプロイ |
| Week 3-4 | データ投入 | -- | 初回遡及学習完了（500件） |

| Week 24 | Level 3昇格 | 成功率90%超+ロールバック5%以下 | 完全自律到達 |

### キルスイッチ + ホットスワップ

- **異常検知**: ガードレール3回連続超過 / 修正率20%以上上昇 / クレーム報告 / LLMタイムアウト3連続
- **発動時**: 前回正常スナップショットに即座切替（5分以内復帰）
- **復帰**: 5段品質ゲートを全通過してから

### 品質改善ループ（5つのサブシステム）

1. **フィードバック収集**: 人間の修正を全件構造化記録
2. **品質改善エージェント**: 週次で精度分析→改善案生成→サンドボックステスト→承認→デプロイ
3. **攻撃テストエージェント**: 異常入力を自動生成→正しく拒否できるか検証
4. **自己評価ループ**: 生成(Flash)→評価(Sonnet)→品質スコア自動付与
5. **カナリアリリース**: 1社→10社→全社の段階展開

### 品質ゲート（5段自動ゲート）

| Gate | 内容 | 判定 |
|---|---|---|
| Gate 1 | シミュレーション精度 >= 現在精度 | ゴールドセット100件で新旧比較 |
| Gate 2 | サンドボックスtest全件pass | ユニット+統合+エッジケース |
| Gate 3 | カナリア24時間メトリクス正常 | 1社展開・エラー率/精度/コスト監視 |
| Gate 4 | 全体展開 | 1社→10社→全社（各段階1時間確認） |
| Gate 5 | 72時間後精度再測定 | 精度低下→自動ロールバック |

### 3層チェック体制

| Layer | 手法 | 役割 |
|---|---|---|
| Layer 1 | 自動メトリクス（ルールベース） | 数値異常の検知（100%） |
| Layer 2 | AIエージェント評価 | 判断品質の評価（90%目標） |
| Layer 3 | 人間サンプル監査（月50件） | Layer 1-2の校正 |

## 6. ゲノム駆動アーキテクチャ

### 設計思想

**従来**: 1業種=1ディレクトリ（16業種x個別パイプライン=80+ファイル）
**新方針**: 共通エンジン + ゲノムJSON で全業種対応。**新業種追加 = JSON 1ファイル追加のみ**。

### 共通エンジン5本

| エンジン | 機能 | 対応業務例 |
|---|---|---|
| ocr_extract | OCR→構造化抽出 | 図面読取、FAX受注、レセプト |
| document_check | 書類照合・チェック | 安全書類、品質管理、契約照合 |
| estimation | 見積・計算 | 積算、見積、原価計算 |
| report_generate | レポート生成 | 月次報告、出来高、在庫レポート |
| approval_flow | 承認ワークフロー | 発注承認、請求承認、休暇申請 |

### 3層ガードレール

1. **Layer 1: 法規制ガードレール**（絶対不可侵）-- 法定保存期間・資格要件・申告期限
2. **Layer 2: ゲノムガードレール**（業界常識の範囲）-- validation_rulesのmin/max/deviation
3. **Layer 3: 会社別ガードレール**（顧客が設定）-- 例: 発注100万以上は必ず人間承認
