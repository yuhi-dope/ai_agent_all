# AI自律都市 実装計画

## ビジョン
人口がなくなった農村部で、AIだけで経済圏・一次産業が全て回る街を作る。
エネルギーは「その街に必要だから」という形で自然に参入する。

---

## 各層の実現可能性（2026年時点）

### エネルギー層

| 技術 | 状態 | 備考 |
|---|---|---|
| ペロブスカイト太陽電池 | ✓ 商用化開始 | 積水化学2025年出荷。初期はシリコン併用が現実的 |
| 蓄電池 | ✓ 即導入可能 | テスラPowerwall/Megapack、中国製LFP電池 |
| VPP制御 | ✓ 技術確立済み | 電力小売ライセンス取得に半年〜1年。初期はFIT/FIP売電 |
| 高温岩体発電 | △ 国内商用実績なし | Phase 1ではスキップ。数億〜数十億円の掘削コスト |

**結論: ◎ 今すぐできる。最も技術的に確実な層**

### 一次産業層

| 技術 | 状態 | 備考 |
|---|---|---|
| 自動運転トラクター | ✓ 市販中 | クボタ「アグリロボトラクタ」2022年〜。1人が複数台遠隔監視 |
| ドローン農薬散布 | ✓ 認可済み | DJI Agras。1台で1日10-20haカバー。最も実現しやすい |
| AI作物管理 | ✓ 構築可能 | 土壌・気象センサー安価。衛星画像モニタリング（Planet Labs等） |
| 自動選果・出荷 | △ 大規模のみ | 1台数千万円。小規模はコスト回収困難。規模が出てから |

**結論: ○ 7割は今すぐ。完全無人化は3-5年先**

### 経済層

| 技術 | 状態 | 備考 |
|---|---|---|
| シャチョツー拡張版 | ○ 自社開発 | MVPが動けば拡張可能。技術的障壁なし |
| 売電収入 | ✓ 制度あり | FIT（固定価格買取）/ FIP（市場連動、2022年〜） |
| 農産物AI販路 | ✓ 既存活用 | 食べチョク・ポケマル等。AI需要予測+最適価格は自社開発 |

**結論: ◎ シャチョツーがあれば大半は解決**

### インフラ層

| 技術 | 状態 | 備考 |
|---|---|---|
| 自動運転バス | △ 限定運行中 | BOLDLY境町2020年〜。時速20km、決まったルートのみ。過疎地は交通量少なく最適 |
| ドローン配送 | △ 実証中 | Level 4解禁済み（2022年）。重量5-10kg制限。軽い物から開始 |
| 遠隔医療 | ✓ 恒久化済み | オンライン初診OK（2022年〜）。AI問診（Ubie等）商用化済み |
| 衛星通信 | ✓ 即契約可能 | Starlink月額約7,000円、下り100-200Mbps。山間部OK |

**結論: △ 半分は今すぐ。自動運転・ドローンは限定的に可能**

## 最短最速安全の実行プラン

### Phase 0: 場所を決める（Month 0-6）

- 候補地3-5カ所を視察
- 自治体首長と面談（最重要）
- 利用可能な補助金・特区制度を調査
- 1カ所に決定

**最速の方法**:
- 「デジタル田園都市国家構想交付金」に応募中の自治体を探す（内閣官房リスト公開）
- または「福島イノベーション・コースト構想」に問い合わせ（復興庁窓口）

### Phase 1: 最小構成で実証（Month 6-18）

**最小構成（これだけで始める）**:
- 太陽光パネル（シリコン）+ 蓄電池 → 1棟分
- ドローン農薬散布 → 1圃場
- 土壌・気象センサー → データ収集開始
- Starlink → 通信確保
- シャチョツー → 街の管理ダッシュボード
- 遠隔医療 → 既存サービス導入

**コスト概算**:
| 項目 | コスト |
|---|---|
| 太陽光+蓄電池（50kW規模） | 1,500-2,000万円 |
| ドローン（DJI Agras） | 200-300万円 |
| センサー群 | 100-200万円 |
| Starlink | 初期5万円 + 月7,000円 |
| シャチョツー | 自社開発（追加コストなし） |
| **合計** | **約2,000-3,000万円** |

**利用可能な補助金**:
| 補助金 | 金額 |
|---|---|
| 環境省「脱炭素先行地域」 | 最大50億円 |
| 農水省「スマート農業実証」 | 数千万〜数億円 |
| デジタル田園都市交付金 | 数千万〜数億円 |
| 経産省「需要家主導太陽光」 | 設備費の1/2〜2/3 |

→ **自己負担: 実質数百万円で始められる可能性がある**

### Phase 2: 拡張（Month 18-36）

- 自動運転トラクター導入（クボタ/ヤンマー）
- ペロブスカイトを建物壁面に追加
- VPP制御システム稼働（余剰電力の売電開始）
- 自動運転低速バス（決まったルート）
- ドローン配送（軽量物から）
- AI作物管理の精度向上（1年分のデータで学習）
- 2-3棟目の建物にエネルギー設備拡張

### Phase 3: モデル完成（Month 36-60）

- 街全体のエネルギー自給率80%+
- 農業の人手を50%削減
- 経済的に自立（売電+農産物+観光で黒字）
- 「AIタウンパッケージ」として商品化
- 2号・3号の街に展開開始

## 安全にやるための鉄則

1. **小さく始めて大きく育てる** — 1棟・1圃場から。動くことを証明してから拡大
2. **既存技術を組み合わせる** — クボタ、DJI、テスラ等「買えるもの」を使う。新技術開発から始めない
3. **住民を敵にしない** — 「仕事を奪う」ではなく「体がきつい作業を代わりにやる」
4. **補助金を最大活用する** — 国・自治体は過疎地再生に本気で金を出す
5. **データを最初から取る** — エネルギー・作物・気象・土壌。2号・3号展開時の最大の武器

## 都市AI構想（蓄電池制御 → 街全体の自律制御）

### 方針
蓄電池の制御AIはテスラ（Autobidder）に依存せず、自社で作る。
シャチョツーの拡張ではなく、**「都市AI」として独立したプロダクト**に育てる。

### テスラの蓄電池ソフトウェアとの比較

| | テスラ Autobidder | 都市AI（自社） |
|---|---|---|
| 蓄電池制御 | ✓ | ✓ |
| 電力市場の自動売買 | ✓ | ✓ |
| VPP（複数拠点の束ね） | ✓ | ✓ |
| 農業・一次産業の制御 | ✗ | ✓ |
| 物流・配送の最適化 | ✗ | ✓ |
| 住民サービス（医療・移動） | ✗ | ✓ |
| 街全体の経済最適化 | ✗ | ✓ |
| 他の街への展開（パッケージ） | ✗ | ✓ |

### 都市AIがカバーする領域

```
都市AI
├── エネルギー制御
│   ├── 発電量予測（天候×パネル状態）
│   ├── 蓄電池の充放電スケジューリング
│   ├── 電力市場（JEPX）の自動売買
│   ├── 街内の電力配分最適化
│   └── 異常検知・メンテナンス予測
│
├── 一次産業制御
│   ├── 作物生育予測・収穫タイミング判断
│   ├── 自動運転トラクター/ドローンの作業スケジュール
│   ├── 灌漑・施肥の自動制御
│   └── 出荷量予測・販路マッチング
│
├── インフラ制御
│   ├── 自動運転バスの配車最適化
│   ├── ドローン配送のルーティング
│   ├── 水道・排水の監視
│   └── 設備の劣化予測・保全計画
│
└── 経済制御
    ├── 街全体のキャッシュフロー管理
    ├── 売電/農産物/観光の収益最適化
    ├── 補助金・助成金の申請管理
    └── 住民への収益分配
```

### ハードウェア戦略
- 蓄電池: **中国製（CATL/BYD）が最もコスパが良い**。テスラより安い
- 制御AI: **自社開発（都市AI）**。テスラのAutobidderに依存しない
- 理由: ハードは汎用品、ソフトで差別化。テスラと同じ戦略を「街」スケールでやる

### シャチョツーとの関係
- シャチョツー = 企業の経営OS（1社単位）
- 都市AI = 街全体の経営OS（複数企業+インフラ+住民）
- シャチョツーで培ったAI基盤・データパイプライン・LLM統合を都市AIに転用
- 別プロダクトだが、技術基盤は共有
