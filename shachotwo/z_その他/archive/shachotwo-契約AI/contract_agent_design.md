# 契約エージェント設計書

> 商談で「契約します」となった瞬間から、契約締結・決済・アカウント作成・オンボーディング開始まで、
> 人間がやることは「電子署名をクリックする」だけ。

---

## 0. なぜ契約エージェントが必要か

```
現状（人間が全部やる場合）:
  商談で「契約します」
    → 見積書をExcelで作る（30分）
    → メールで送る
    → 契約書をWordで作る（1時間）
    → 印刷・製本・押印・郵送（2-3日）
    → 相手が押印して返送（3-7日）
    → 届いたら入金確認を待つ（月末締め翌月末払い → 最大60日）
    → 入金確認したらアカウント作成（手動）
    → オンボーディング案内メール（手動）
    ━━━━━━━━━━━━━━━━━━
    合計: 2-4週間。途中で先方が忘れて流れるリスク大

契約エージェント導入後:
  商談で「契約します」
    → エージェントが5分で全部やる
    → 相手は届いたメールの「署名する」を1クリック
    → クレカ入力 or 請求書払い選択
    → アカウント即日発行
    ━━━━━━━━━━━━━━━━━━
    合計: 最短30分。商談の熱が冷めない
```

**「商談の熱が冷めない」が最大の価値。** 中小企業の社長は忙しい。「後で契約書送ります」→ 忘れられる。その場で完結させる。

### 2.3 ステップ③: 電子契約書 自動生成 & 送信

#### なぜ電子契約か

| | 紙の契約 | 電子契約 |
|---|---|---|
| **時間** | 郵送で3-7日 | 最短5分 |
| **印紙税** | 必要（SaaS契約は通常不要だが念のため） | **不要**（電子契約は印紙税法の対象外） |
| **保管** | 物理的に保管 | クラウドに自動保管 |
| **検索** | 紙をめくる | 全文検索 |
| **法的効力** | ◎ | ◎（電子署名法で同等） |

#### 電子署名サービス選定

| サービス | 月額 | API | 日本語対応 | 推奨度 |
|---------|------|-----|----------|--------|
| **CloudSign** | 無料〜¥10,000/月 | ✅ REST API | ◎ 日本No.1 | ★★★ |
| **GMOサイン** | 無料〜¥8,800/月 | ✅ REST API | ◎ | ★★ |
| **DocuSign** | $25/月〜 | ✅ REST API | ○ | ★ |

**CloudSign を採用。** 理由:
- 日本の中小企業に最も認知されている（「CloudSignで署名をお願いします」で通じる）
- 相手側は無料（送信者のみ課金）
- 弁護士ドットコムが運営 → 法的信頼性が高い
- API で完全自動化可能
- 電子帳簿保存法に対応

### 2.4 ステップ④: 決済セットアップ

#### 決済サービス: Stripe

| 比較 | Stripe | PAY.JP | GMO-PG |
|------|--------|--------|--------|
| **手数料** | 3.6% | 3.6% | 3.2〜3.5% |
| **API品質** | ◎ 世界最高 | ○ | △ |
| **サブスク** | ✅ Billing | ✅ | ✅ |
| **請求書払い** | ✅ Invoicing | ❌ | ✅ |
| **開発速度** | ◎ ドキュメント充実 | ○ | △ |
| **日本語対応** | ◎ | ◎ | ◎ |

**Stripe を採用。** API品質とサブスクリプション管理が圧倒的。

#### 2つの決済フロー

```
署名完了
    ↓
決済方法選択画面を表示
    ├→ A: クレジットカード払い（推奨）
    │     → Stripe Checkout Session を生成
    │     → 相手がカード情報入力
    │     → サブスクリプション即日開始
    │     → 初月無料（トライアル期間設定）
    │
    └→ B: 請求書払い（銀行振込）
          → Stripe Invoice を自動生成
          → PDFをメール送信
          → 振込確認後にアカウント作成
          → ※ 中小企業は「請求書じゃないとダメ」が多い
```

**Stripe サブスクリプション設定:**
```
Stripe::Subscription.create({
  customer: customer_id,
  items: [{ price: plan_price_id }],
  trial_period_days: 30,          # 初月無料
  payment_behavior: 'default_incomplete',
  metadata: {
    company_id: company_uuid,
    plan: 'starter',
    deal_id: deal_uuid
  }
})
```

## 3. 相手が迷っている場合 — ナーチャリング分岐

商談後、全員がすぐ「契約する」とは限らない。温度に応じた自動フォローを設計。

```
商談完了後のシグナル分岐:

  ① 即契約（HOT 🔥）
     → 契約フロー直行（上記の流れ）

  ② 「社内で検討します」（WARM 🟡）
     → 3日後: 事例メール送信
        「同業の○○工業様では月40時間の削減を実現しました」
     → 7日後: 限定オファーメール
        「今月中のご契約で初月+2ヶ月目も無料（合計2ヶ月無料）」
     → 14日後: 最終フォロー
        「ご検討状況はいかがでしょうか？ご不明点があればお気軽に」
     → 21日後: 送信停止。3ヶ月後にリトライ

  ③ 「もう少し考えたい」（COOL 🔵）
     → 資料送付のみ（ホワイトペーパー + 事例集）
     → 1ヶ月後にフォロー1回
     → 反応なし → 休眠リスト
     → 6ヶ月後にリトライ

  ④ 「見送ります」（LOST ❌）
     → 失注理由ヒアリングメール（1通のみ）
        「差し支えなければ、見送りの理由をお聞かせください」
        選択肢: [予算] [タイミング] [機能不足] [他社採用] [その他]
     → 失注理由をCRMに記録
     → 6ヶ月後にリトライ（機能追加・価格改定があった場合のみ）
```

## 7. 法令チェックリスト（契約関連）

| 法令 | 対応 |
|------|------|
| **電子署名法** | ✅ CloudSign は電子署名法2条・3条に対応。書面契約と同等の法的効力 |
| **電子帳簿保存法** | ✅ CloudSign が電子保存要件（真実性・可視性）を満たす |
| **印紙税法** | ✅ 電子契約は課税文書に該当しない → 印紙税不要 |
| **特定商取引法** | ✅ SaaS（役務提供）は通信販売に該当。特商法表記をLP/サイトに掲載 |
| **消費者契約法** | ✅ B2B契約のため適用外だが、不当条項は避ける |
| **下請法** | ⚠️ シャチョツー→顧客は「上→下」ではないため通常適用外。念のため確認 |
| **個人情報保護法** | ✅ 契約時に取得する個人情報の利用目的を明示。プライバシーポリシー掲載 |
| **適格請求書等保存方式（インボイス制度）** | ✅ 請求書に登録番号・税率を記載 |
| **反社チェック** | ✅ 契約書に反社排除条項を含む |
