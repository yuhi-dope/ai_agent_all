## 2. STEP 1: 初回ヒアリング

弊社の担当者が以下をヒアリングいたします。お客さまに技術的な知識は一切不要です。

- **現在の業務フロー**: どんな業務を、誰が、どのツールで行っているか
- **お困りごと**: 時間がかかっている作業、属人化している業務、ミスが多い作業
- **優先する業務領域**: まずどの業務から AI社員に任せたいか
- **ご利用中のツール**: Notion / Slack / Google Drive / Chatwork のどれを使っているか

### ヒアリング後にお伝えすること

- 御社に最適な入力チャネルの推奨
- 最初に取り組む業務領域（10ジャンルのうちどれか）
- 最初の1ヶ月で実現できる具体的な成果イメージ

## 3. STEP 2: 入力チャネルの接続設定

御社が普段お使いのツールと AI社員を接続します。**複数チャネルの同時利用も可能**です。
以下から、ご利用になるチャネルの手順をご確認ください。

---

### 3-A. Notion で接続する場合

#### 御社側でご準備いただくこと

**1. Notion データベースの作成**

Notion 上に以下のプロパティを持つデータベースを1つ作成してください。

| プロパティ名 | 種類 | 説明 |
|------------|------|------|
| **名前** | Title | 要件の短いタイトル（例:「売上管理画面」） |
| **ステータス** | Select | 以下の4つの選択肢を作成してください |
| | | `実装前` / `実装希望` / `実装中` / `完了済` |
| **要件** | Rich text | 詳しい要件（任意。未記入の場合はページ本文を使います） |
| **ジャンル** | Select | 任意。AI が自動判定するため、通常は空欄で OK |
| **run_id** | Rich text | AI社員が自動で書き込みます。作成だけしてください |

**2. Notion インテグレーションの連携**

弊社からお送りする **Notion インテグレーション招待リンク** をクリックし、上記データベースへのアクセスを許可してください。

```
手順:
1. 弊社から招待リンクが届きます
2. リンクをクリック → Notion にログイン
3. 「アクセスを許可するページ」で作成したデータベースを選択
4. 「許可する」をクリック
→ 完了
```

**3. Webhook の自動設定（弊社側で実施）**

データベースの接続確認後、弊社側で Webhook を設定します。これにより、ステータスを「実装希望」に変更するだけで AI社員が自動的に動き出します。

### 3-B. Slack で接続する場合

#### 御社側でご準備いただくこと

**1. AI社員用チャンネルの作成**

Slack で AI社員専用のチャンネルを1つ作成してください。

```
推奨チャンネル名: #ai-dev
```

**2. AI社員 Bot の追加**

弊社からお送りする **Slack アプリインストールリンク** をクリックし、御社の Slack ワークスペースに Bot を追加してください。

```
手順:
1. 弊社からインストールリンクが届きます
2. リンクをクリック → Slack にログイン
3. 「許可する」をクリック（AI社員 Bot がワークスペースに追加されます）
4. 作成した #ai-dev チャンネルに Bot を招待
   → チャンネル内で「/invite @AI社員」と入力
→ 完了
```

**3. 使い方**

`#ai-dev` チャンネルにメッセージを投稿するだけで、AI社員が自動的に処理を開始します。進捗はスレッドで報告されます。

### 3-C. Google Drive で接続する場合

#### 御社側でご準備いただくこと

**1. AI社員用フォルダの作成**

Google Drive 上に AI社員専用のフォルダを1つ作成してください。

```
推奨フォルダ名: AI社員 - 要件
```

**2. Google アカウントの連携**

弊社からお送りする **Google OAuth 認証リンク** をクリックし、上記フォルダへの読み取りアクセスを許可してください。

```
手順:
1. 弊社から認証リンクが届きます
2. リンクをクリック → Google アカウントにログイン
3. 「AI社員がドキュメントを読み取ることを許可しますか？」→「許可」
4. 作成したフォルダを弊社の担当者に共有
→ 完了
```

**3. 使い方**

指定フォルダ内に **Google Docs で要件ドキュメントを作成**するだけです。AI社員が自動的に内容を読み取り、処理を開始します。

```
ドキュメントの書き方（例）:
────────────────────
タイトル: 売上管理ダッシュボード

【やりたいこと】
月別の売上推移をグラフで見たい。
営業担当ごとの成績一覧も欲しい。

【注意事項】
個人情報は暗号化してほしい。
────────────────────
```

## 5. 使い方: AI社員への指示の出し方

### 指示のポイント

AI社員は日本語の指示を理解します。以下のポイントを意識すると、より精度の高い成果物が得られます。

#### 良い指示の例

```
✅ 「月別の売上推移をグラフで表示するダッシュボードを作って。
     営業担当ごとのフィルターも欲しい。
     金額は3桁ごとにカンマ区切りにして。」

✅ 「顧客からの問い合わせを管理する画面が欲しい。
     ステータスは「未対応・対応中・完了」の3つ。
     担当者を割り当てられるようにして。
     対応期限を過ぎたら赤くハイライトして。」

✅ 「毎月の請求書を自動生成したい。
     取引先名、品目、単価、数量を入力すると PDF が出力される。
     消費税は10%で自動計算。」
```

#### 避けたほうがよい指示の例

```
❌ 「いい感じのシステムを作って」
   → 何を作るか具体的にお伝えください

❌ 「Salesforce みたいなものを全部作って」
   → 大きすぎます。まず1つの機能（例:「商談管理」）から始めましょう

❌ 「データベースのスキーマを○○にして、API は REST で〜」
   → 技術的な指定は不要です。業務の要望をお伝えください
```

#### 指示に含めると良い情報

| 項目 | 例 |
|------|-----|
| **やりたいこと** | 「売上の月次推移を見たい」 |
| **誰が使うか** | 「営業マネージャーが使う」 |
| **必要な機能** | 「フィルター、ソート、CSV出力」 |
| **表示したいデータ** | 「取引先名、金額、担当者、日付」 |
| **特別なルール** | 「100万円以上は部長承認が必要」 |

### チャネル別の指示方法

| チャネル | 指示方法 |
|---------|---------|
| **Notion** | データベースに行を追加 → 要件を記入 → ステータスを「実装希望」に変更 |
| **Slack** | `#ai-dev` チャンネルにメッセージを投稿 |
| **Google Drive** | 指定フォルダに Google Docs を作成 |
| **Chatwork** | グループチャットで AI社員にメンション付きメッセージ |

## 6. 使い方: 要件定義書の確認と承認

**確認モード（推奨）** では、AI社員が要件定義書を作成した段階で一旦停止します。

### 確認から承認までの流れ

```
1. AI社員が指示を受けて要件定義書を作成
   → ダッシュボードの Run List にオレンジ色の「Spec Review」バッジで表示

2. Run をクリックして Detail を表示
   → 「Spec Document」に要件定義書が表示される

3. 内容を確認
   → 日本語で書かれた設計書です。以下をチェック:
     ・やりたいことが正しく反映されているか
     ・不要な機能が含まれていないか
     ・特別なルール（承認フロー等）が反映されているか

4. 問題なければ「Start Implementation」ボタンをクリック
   → AI社員が実装・テスト・デプロイまで自動で実行

5. 完了するとバッジが緑色の「Published」に変わる
   → 御社のダッシュボードに成果物が反映される
```

### 要件定義書に問題がある場合

現時点では、ダッシュボード上での修正コメント機能は開発中です。以下の方法で修正をご依頼ください:

- **同じチャネルで修正指示を再送信**
  - 例（Slack）: 「さっきの売上管理だけど、グラフは棒グラフじゃなくて折れ線グラフにして」
- **弊社サポートに連絡**
  - 要件定義書の内容について、具体的な修正箇所をお伝えください

## 7. 使い方: 成果物の確認とデプロイ

### 自動デプロイの仕組み

AI社員が生成したコードは、全てのテスト（Lint → 単体テスト → E2E テスト）をパスした後に自動でデプロイされます。

```
AI社員が実装
    ↓
Docker コンテナ内で安全にテスト
    ↓
全テスト合格
    ↓
GitHub リポジトリに push
    ↓
Vercel / Cloud Run が自動デプロイ
    ↓
御社のダッシュボードに即反映
```

### 成果物の確認方法

ダッシュボードで Run を選択すると、以下が確認できます:

| 項目 | 内容 |
|------|------|
| **Summary** | 生成された機能の概要 |
| **Generated Files** | 生成・変更されたファイルの一覧 |
| **Status** | Published（成功） / Failed（失敗） |

### 成果物に問題がある場合

- AI社員に修正指示を出してください（通常の指示と同じ方法）
- 例:「売上管理画面の日付フォーマットを YYYY/MM/DD に変えて」
- AI社員が既存のコードを読み取り、差分のみを修正します

## 8. ダッシュボード機能一覧

| 機能 | 説明 |
|------|------|
| **Settings: Auto Execute トグル** | ON: 指示から完成まで全自動 / OFF: 要件定義書で一旦停止して確認 |
| **Next System Proposal** | AI が過去の実装履歴を分析し、次に作るべき機能を提案 |
| **Run List** | 全ての実行履歴。ステータスでフィルター可能 |
| **Status Filter** | All / Spec Review / In Progress / Published / Failed |
| **Detail: Spec Document** | 要件定義書のプレビュー |
| **Detail: Start Implementation** | 確認モード時の実装開始ボタン |
| **Detail: Generated Files** | 生成されたファイル一覧 |
| **自動リフレッシュ** | 30秒ごとに画面が自動更新されます |

## 9. 運用モードの切り替え

### 確認モード（推奨: 導入初期）

```
設定: Auto Execute → OFF

動作:
  指示 → ジャンル判定 → 要件定義書作成 → 【ここで停止】
                                          ↓ 人間が確認・承認
                                      実装 → テスト → デプロイ
```

- 導入初期はこちらを推奨します
- AI社員の仕事ぶりを確認しながら進められます
- 要件の齟齬による手戻りを防げます

### 全自動モード（慣れてきたら）

```
設定: Auto Execute → ON

動作:
  指示 → ジャンル判定 → 要件定義 → 実装 → テスト → デプロイ（全自動）
```

- AI社員の品質に信頼が持てたら切り替えてください
- ダッシュボードの Settings でワンクリックで切替可能
- 全自動でも、テスト不合格のコードはデプロイされません

## 10. セキュリティについて

AI社員は以下のセキュリティ対策を全ての実行で自動適用しています。

### 御社のデータを守る仕組み

| 対策 | 内容 |
|------|------|
| **コード実行の隔離** | AI が生成したコードは使い捨ての Docker コンテナ内で実行。御社のサーバーや本番環境には一切触れません |
| **秘密鍵の漏洩防止** | API キー等のパターンを自動検知。検出された場合はデプロイをブロックし、再生成します |
| **危険操作のブロック** | ファイル削除、権限変更等の危険なコマンドは物理的にブロックされます |
| **段階的テスト** | Lint → 単体テスト → E2E テストの順に実行。全てパスしたコードのみデプロイ |
| **変更量の制限** | 1回のタスクで変更できるのは最大200行。大きな変更は小さなタスクに分割 |
| **監査ログ** | AI社員の全操作（ファイル作成・コマンド実行・テスト結果）をタイムスタンプ付きで記録 |

### 監査ログについて

AI社員が実行した全ての操作は記録されています。監査対応が必要な場合は、弊社にお問い合わせいただければ、ご指定期間のログを提出いたします。

記録される情報:
- **いつ**: 操作日時（タイムスタンプ）
- **何を**: 実行したコマンド・作成/変更したファイル
- **結果**: 成功/失敗・テスト結果・エラー内容
- **承認者**: ダッシュボードで承認した担当者（確認モード時）

## 11. よくあるご質問（FAQ）

### 基本的な使い方

**Q: AI社員に指示を出すのに、プログラミングの知識は必要ですか？**
A: 不要です。「売上管理画面を作って」「請求書を自動生成して」など、やりたいことを日本語で伝えるだけです。

**Q: 1回の指示でどのくらいのものが作れますか？**
A: 画面1〜2ページ程度の機能が目安です。大きな機能は小さなタスクに分けて指示していただくと、品質が上がります。

**Q: 複数の指示を同時に出せますか？**
A: はい。複数の要件を同時に投入でき、AI社員が並列で処理します。

**Q: 指示を出してからどのくらいで完成しますか？**
A: タスクの複雑さによりますが、通常は数分〜十数分程度です。確認モードの場合は、御社の承認待ち時間が加わります。

### チャネルについて

**Q: 複数のチャネルを同時に使えますか？**
A: はい。例えば「普段は Slack で指示、まとまった要件は Notion で管理」といった使い分けが可能です。

**Q: 途中でチャネルを変更できますか？**
A: はい。いつでも追加・変更できます。弊社サポートにご連絡ください。

### セキュリティ・データ

**Q: AI に社内のデータを見られることはありますか？**
A: AI社員は御社が入力した要件テキストと、生成されたコードのみを処理します。御社の社内データベースや他のシステムにアクセスすることはありません。

**Q: 生成されたコードのセキュリティは大丈夫ですか？**
A: 5重のセキュリティチェック（秘密鍵スキャン・隔離実行・危険操作ブロック・段階テスト・監査ログ）を自動適用しています。全テストをパスしたコードのみデプロイされます。

**Q: AI が生成したコードが暴走することはありますか？**
A: ありません。コードは使い捨ての隔離コンテナ内で実行されます（メモリ上限 512MB・実行時間上限あり）。コンテナ外には影響しません。

### 料金・契約

**Q: 追加料金が発生するケースはありますか？**
A: 契約プランの実行回数を超えた場合は、事前にご連絡のうえ追加オプションをご案内します。知らないうちに課金されることはありません。

**Q: 解約はいつでもできますか？**
A: はい。月単位の契約のため、翌月から解約可能です。

### トラブル

**Q: AI社員の結果が期待と違った場合はどうすればいいですか？**
A: 修正指示を追加で出してください。例:「さっきの画面に検索機能を追加して」。AI社員が既存のコードを読み取り、差分のみを修正します。

**Q: ステータスが「Failed」になった場合は？**
A: AI社員が自動修正を3回試みても解決できなかった場合に Failed になります。弊社のサポートチームが原因を確認し、対応いたします。ダッシュボードから弊社にご連絡ください。

**Q: ダッシュボードにログインできません。**
A: マジックリンクの有効期限が切れている可能性があります。再度メールアドレスを入力し、新しいリンクを取得してください。それでもログインできない場合は、弊社サポートまでご連絡ください。
