現在のブランチの差分をレビューしてください。

対象: $ARGUMENTS （ブランチ名またはコミットハッシュ。未指定の場合はmainとの差分）

手順:
1. `git diff $ARGUMENTS...HEAD` で差分を取得（$ARGUMENTS未指定時は `git diff main...HEAD`）
2. 変更ファイルごとに以下を確認:
   - セキュリティリスク（.env漏洩、SQLインジェクション、XSS、PII露出）
   - RLSテナント分離（company_idフィルタの有無）
   - 型安全性（型ヒントの欠如）
   - テストの有無（実装変更に対するテスト追加）
3. 問題を重要度（高/中/低）で分類して報告
4. 問題がなければ「LGTM」と報告
