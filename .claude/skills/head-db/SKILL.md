# 頭のDB管理スキル

## ファイル情報

- **TSVファイル**: `/Users/sugimotoyuuhi/code/ai_agent/shachotwo-SNSAI/shared/head_db_entries.tsv`
- **フォーマット**: TSV（タブ区切り）、ヘッダー行あり
- **カラム**: `id`, `カテゴリ`, `テキスト`, `タグ`, `重要度`

## 追加時の手順

### Step 1: 現在のDBを読み込み

TSVファイルを読み込み、最後のIDを確認する（例: BRAIN-044 → 次は BRAIN-045）。

### Step 2: ユーザー入力を構造化

ユーザーの自由記述を以下のルールで構造化する:

**カテゴリ（いずれか1つ）**:
- `実績`: 具体的な数字付きの成果・Before/After
- `思想`: AI・DX・経営に対する持論・信念
- `ツール知見`: GAS/Gemini/Apify等の実践ノウハウ・Tips
- `業界知識`: 各業界の現場のリアルな課題・実態
- `失敗談`: うまくいかなかった経験と学び
- `顧客の声`: クライアントの反応・言葉・エピソード
- `フレームワーク`: 判断基準・思考モデル・分析手法
- `名言・引用`: 影響を受けた言葉・本

**テキスト**:
- 2-5文。SNS投稿に引用しやすい粒度にする
- 数字・固有名詞・ツール名は必ず保持
- 口語のニュアンスは残しつつ文章として整える
- 1つの入力から複数エントリを抽出してよい（1入力→1-3エントリ）

**タグ**:
- カンマ区切り、3-5個
- 業種名があれば必ず含める（建設業, 製造業, 歯科, 介護, 士業 等）
- ツール名があれば含める（GAS, Gemini, Apify, Supabase, kintone 等）
- テーマタグ（属人化, 自動化, コスト削減, 教育 等）

**重要度**:
- 5: SNS投稿の「フック」として単独で使える。数字ありインパクト大
- 4: 複数投稿で繰り返し引用できる核心的な知見
- 3: 特定テーマの投稿で使える
- 2: 補足情報
- 1: 背景情報

### Step 3: ユーザーに確認

構造化した内容を表形式で表示し、確認を求める:

```
以下の内容で追加します:

| id | カテゴリ | テキスト | タグ | 重要度 |
|---|---|---|---|---|
| BRAIN-045 | 実績 | ... | ... | 5 |

問題なければこのまま追加します。修正があれば教えてください。
```

### Step 4: TSVに追記

確認OKならTSVファイルの末尾に追記する。Editツールでファイル末尾に行を追加。

**重要**: タブ区切りを正確に維持すること。テキスト内にタブ文字が含まれないよう注意。

### Step 5: 追加件数を報告

```
頭のDBに{N}件追加しました（BRAIN-XXX〜BRAIN-XXX）。合計{M}件。
```

## 検索時の手順

ユーザーが「頭のDBで○○に関するエントリを探して」等と言った場合:
1. TSVファイルをGrepでタグ/テキスト検索
2. 該当エントリを表形式で表示

## 更新・削除時の手順

ユーザーが「BRAIN-XXXを修正して」「BRAIN-XXXを削除して」等と言った場合:
1. 該当行をEditツールで修正/削除
2. 変更内容を報告
