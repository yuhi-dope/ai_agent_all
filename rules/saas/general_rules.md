# SaaS 操作 共通ルール

## 基本原則

- 操作は最小限にとどめ、不要な API 呼び出しを行わない
- データの取得（READ）操作を先に行い、更新（WRITE）操作は後に実行する
- 一括操作が可能な場合は、個別操作よりも一括操作を優先する

## 安全性

- 削除操作は計画に含めない（手動対応を推奨）
- 大量データの更新（10件以上）は警告を出し、分割実行を検討する
- 認証エラーが発生した場合は即座に停止し、トークンリフレッシュを試みる

## エラーハンドリング

- API レート制限に達した場合は適切な間隔を空けて再試行する
- バリデーションエラーは引数を見直して再計画する
- タイムアウトは操作を分割して再試行する
