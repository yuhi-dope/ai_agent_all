# M&A 専門ルール

## ドメイン概要

M&Aエージェントは、買収候補企業の探索・デューデリジェンス（DD）・企業価値評価・案件管理を支援するシステムである。ロングリストからショートリスト、DD、クロージングまでの M&A プロセスを一元管理する。

---

## 必須エンティティ

| エンティティ | 説明 | 必須フィールド例 |
|---|---|---|
| **Target（ターゲット企業）** | 買収候補 | name, industry, revenue, employees, location, status, source |
| **Valuation（企業価値評価）** | バリュエーション | target_id, method(DCF/マルチプル/純資産), estimated_value, assumptions(JSONB) |
| **DDChecklist（DDチェックリスト）** | デューデリジェンス項目 | target_id, category(財務/法務/税務/人事/IT), item, status, findings |
| **Deal（案件）** | M&A案件 | target_id, deal_type(買収/合併/資本提携), stage, offer_amount, advisor |
| **CompetitorAnalysis（競合分析）** | 競合比較 | target_id, competitor_name, metric, target_value, competitor_value |

---

## ビジネスルール（Spec Agent / Coder Agent 共通）

### M&A プロセス

1. **パイプライン**: ロングリスト → ショートリスト → 初期接触 → NDA締結 → DD実施 → 条件交渉 → 最終合意 → クロージング
2. **ロングリスト**: 業種・売上規模・地域等の条件でフィルタリング
3. **ショートリスト**: ロングリストから有望候補を選定（選定理由を記録）
4. **DD 完了条件**: 全カテゴリ（財務/法務/税務/人事/IT）のチェックリストが「完了」または「該当なし」

### 企業価値評価

- **DCF法**: 将来キャッシュフロー予測と割引率から算出。assumptions に前提条件を記録
- **マルチプル法**: 類似企業の EV/EBITDA、PER 等の倍率で算出
- **純資産法**: 時価純資産をベースに算出
- 1ターゲットに対し複数の評価手法を併記し、レンジで表示

### DDチェックリスト

- カテゴリ別（財務/法務/税務/人事/IT）にチェック項目を管理
- 各項目のステータス: 未着手 / 調査中 / 完了 / 問題あり / 該当なし
- 「問題あり」の項目には findings（発見事項）と risk_level（高/中/低）を記録

### リスクスコアリング

- DD の findings から自動的にリスクスコアを算出（問題あり×高リスク = 3点、中 = 2点、低 = 1点）
- トータルリスクスコアをターゲット企業のサマリーに表示

---

## 受入条件テンプレート

- [ ] ターゲット企業を登録し、ロングリストに追加できる
- [ ] ロングリストからショートリストへの移行（選定理由付き）ができる
- [ ] 企業価値評価を DCF / マルチプル / 純資産で登録し、比較表示できる
- [ ] DD チェックリストをカテゴリ別に管理し、進捗率が表示される
- [ ] 「問題あり」項目にリスクレベルと発見事項を記録できる
- [ ] M&A パイプライン全体をステージ別に俯瞰できるダッシュボードがある

---

## Coder 向け実装指針

- 金額は `BIGINT`（円単位）。億単位の表示フォーマットはフロント側で処理
- バリュエーションの assumptions は JSONB で柔軟に格納（手法ごとに前提が異なるため）
- DD 進捗率 = 完了 + 該当なし / 全項目数 × 100
- パイプラインはカンバン表示（ステージ別の横カラム）
- ターゲット一覧は業種・売上規模・リスクスコアでソート・フィルタ可能
