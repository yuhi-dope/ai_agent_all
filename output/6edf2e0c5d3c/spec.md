# 概要
プロジェクトの進捗状況、今後の計画、および協議事項を関係者間で円滑に共有するための、定型的なMarkdownレポートを生成するツールの設計を行う。ユーザーが所定のフォームに内容を入力することで、一貫したフォーマットのレポートを迅速に作成し、コミュニケーションの効率化と報告の標準化を図る。

# 機能要件
- **報告対象者の指定**
  - レポートの宛先となる人物名（例: 大我）を入力するためのテキストフィールドを設ける。
  - 入力された名前は、レポート内の「ディスカッションしたい内容」セクションの見出しに反映される。
  - このフィールドは任意入力とし、空欄の場合は汎用的な見出し（例: 「ディスカッションしたい内容」）を使用する。

- **レポート内容の入力**
  - 以下の3つの項目について、内容を自由に入力できるテキストエリアを設ける。
    1.  `進捗報告`: 完了したタスクや現在の進捗状況を記述する。
    2.  `今後の動き`: 次に取り組む予定のタスクや計画を記述する。
    3.  `ディスカッションしたい内容`: 相談・確認したい事項を記述する。
  - 各テキストエリアでは、Markdownの箇条書き (`- `) 記法が利用できることを想定する。

- **Markdownレポートの生成**
  - 「生成」ボタンをクリックすると、入力された内容を元に、以下のフォーマットで整形されたMarkdownテキストを生成し、画面上の出力エリアに表示する。
  ```markdown
  ### 進捗報告
  - （入力内容1）
  - （入力内容2）

  ### 今後の動き
  - （入力内容1）
  - （入力内容2）

  ### （報告対象者名）さんとディスカッションしたい内容
  - （入力内容1）
  - （入力内容2）
  ```

- **クリップボードへのコピー**
  - 生成されたMarkdownレポートの横に「コピー」ボタンを配置する。
  - ボタンをクリックすると、生成された全文がユーザーのクリップボードにコピーされる。
  - コピー完了後、ユーザーに成功したことをフィードバックする（例: ボタンの文言が「コピーしました！」に一時的に変わる）。

- **入力内容の永続化**
  - ユーザーが入力した内容は、ブラウザのローカルストレージに自動で保存される。
  - ユーザーがページをリロードしたり、ブラウザを閉じても、次回アクセス時には前回の入力内容が各フォームに復元されている。

# 非機能要件
- **実行環境**: モダンブラウザ（Chrome, Firefox, Safari, Edge）上で動作するクライアントサイドアプリケーションとして実装する。サーバーサイドの処理は不要。
- **パフォーマンス**: 各操作（生成、コピー）は即座に完了し、ユーザーを待たせないこと。
- **UI/UX**: レスポンシブデザインに対応し、PCおよびスマートフォンの両方で快適に操作できること。

# データ・API の概要
- **使用API**:
  - `Clipboard API` (`navigator.clipboard.writeText`): 生成されたMarkdownテキストをクリップボードにコピーするために使用する。
  - `Web Storage API` (`localStorage`): フォームの入力内容をブラウザに永続化するために使用する。
- **データ構造**: `localStorage` に保存するデータは、以下のキーを持つJSONオブジェクトとする。
  ```json
  {
    "targetPerson": "大我",
    "progress": "- タスクAを完了\n- タスクBの設計を80%完了",
    "nextActions": "- タスクCに着手",
    "discussionPoints": "- XXXの仕様について相談したい"
  }
  ```

# 画面・フロー概要
## 画面構成
単一のページで構成される。
1.  **ヘッダー**:
    - ツール名（例: 「進捗報告ジェネレーター」）
2.  **入力エリア**:
    - `報告対象者`：テキスト入力フィールド
    - `進捗報告`：複数行テキストエリア
    - `今後の動き`：複数行テキストエリア
    - `ディスカッションしたい内容`：複数行テキストエリア
3.  **アクションエリア**:
    - `生成` ボタン
4.  **出力エリア**:
    - 生成されたMarkdownを表示する読み取り専用のテキストエリア
    - `コピー` ボタン

## ユーザフロー
1.  ユーザーがツールページにアクセスする。
2.  `localStorage` に保存されたデータがあれば、各入力フォームに内容が自動的にセットされる。
3.  ユーザーは各フォーム（報告対象者、進捗報告など）に内容を入力・編集する。入力内容は随時 `localStorage` に保存される。
4.  ユーザーが「生成」ボタンをクリックする。
5.  入力内容に基づき整形されたMarkdownテキストが、出力エリアに表示される。
6.  ユーザーが「コピー」ボタンをクリックする。
7.  出力エリアのテキストがクリップボードにコピーされる。
8.  ユーザーはコピーしたテキストを、チャットツールやWikiなどに貼り付けて報告を行う。
