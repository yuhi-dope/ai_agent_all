# 頭のDB 初期構築インタビュー質問リスト

> **目的**: 杉本さんの知見・思想・経験を構造化し、LLMに「この人だから書ける文章」を生成させるためのナレッジベースを構築する。
> **所要時間**: 約60分（音声回答推奨 → Whisper文字起こし → Gemini構造化）
> **目標件数**: 30-50エントリ
> **使い方**: 各質問に1-3分で回答。箇条書きでも口語でもOK。後でGeminiが構造化する。

---

## やり方

### Option A: 音声インタビュー（推奨・最速）

```
1. このリストを見ながら、スマホの録音アプリで自分の声を録音
2. 1質問につき1-3分で回答（考えすぎず、思い出すままに）
3. 音声ファイルを Whisper API で文字起こし
4. 文字起こしテキストを Gemini に渡して構造化（shared/prompts/structure_head_db.md のプロンプト使用）
5. 構造化結果をスプレッドシート「頭のDB」シートに貼り付け
```

### Option B: テキスト直接入力

```
1. このリストの質問に対して、スプレッドシートやメモアプリに直接書く
2. 1質問につき3-5行で十分
3. 全部書き終えたら Gemini で構造化
```

---

## カテゴリ定義（拡張版）

| カテゴリ | 説明 | 頭のDBでの用途 |
|---|---|---|
| `実績` | 具体的な数字付きの成果 | 信頼構築。P1/L1の根幹 |
| `思想` | AI・DX・経営に対する持論 | 差別化。「この人の考え方」を出す |
| `ツール知見` | GAS/Gemini/Apify等の実践ノウハウ | How-to系コンテンツの素材 |
| `業界知識` | 各業界の現場で見た・聞いたリアル | 業種別コンテンツの素材 |
| `失敗談` | うまくいかなかった経験と学び | 共感・信頼。人間味を出す |
| `顧客の声` | クライアントの反応・言葉 | 社会的証明。説得力を上げる |
| `フレームワーク` | 自分の中の判断基準・思考モデル | 図解コンテンツの素材 |
| `名言・引用` | 影響を受けた言葉・本 | コラム・パーソナルストーリーの素材 |

## 質問リスト

### Section 1: 実績（10問）— 最重要。ここを厚く。

> 全て「具体的な数字」で答えてください。

1. **これまでで最もインパクトがあった業務改善は？**
   - 何を → 何にしたか、Before/Afterの数字は？

2. **最も短時間で構築できた自動化は？**
   - 何時間で作って、どのくらいの効果が出た？

3. **最もコスト削減効果が大きかった案件は？**
   - 月額/年額でいくら削減？

4. **最も「属人化を解消した」と実感した案件は？**
   - 誰の何の知識を、どう移植した？

5. **「社長の時間」を最も取り戻した案件は？**
   - 社長が何に時間を使っていて、何時間/日 減った？

6. **新人教育・引継ぎを改善した案件はある？**
   - 教育期間がどう変わった？定着率は？

7. **OCR・ドキュメント処理系の実績は？**
   - 何の書類を、どのツールで、精度は何%？

8. **複数のツールを連携させた「仕組み」の実績は？**
   - 例: GAS→Gemini→スプシ→Slack みたいな連携

9. **クライアント以外の自分自身の業務で自動化している事は？**
   - 日常のルーティンで自動化しているものがあれば

10. **「これはまだ成果が出ていないが、取り組んでいる」ものは？**
    - 現在進行中のプロジェクト・実験

## 回答後の処理フロー

```
[音声録音 or テキスト回答]
  ↓
[Whisper API 文字起こし]（音声の場合）
  ↓
[Gemini API: structure_head_db.md プロンプト]
  │ 入力: 質問番号 + 回答テキスト
  │ 出力: 頭のDBフォーマット（id, カテゴリ, テキスト, タグ, 重要度）
  ↓
[人間: ざっと確認]
  │ 明らかに間違い・不適切なものだけ修正
  ↓
[スプレッドシート「頭のDB」シートに貼り付け]
  ↓
完了（30-50エントリ）
```
