# shachotwo-SNSAI

SNS運用の半自動化システム。3プラットフォーム横断で「AIによる業務改善」を発信し、経営者層からのリード獲得を狙う。

## プラットフォーム構成

| フォルダ | プラットフォーム | 目的 | ステータス |
|---|---|---|---|
| `x/` | X (旧Twitter) | リーチ拡大・DM獲得 | 要件確定済 |
| `note/` | note | 深掘り記事・SEO/AIO・アフィリエイト | 要件確定済 |
| `linkedin/` | LinkedIn | BtoBリード・経営者ネットワーク | 要件確定済 |
| `gas/` | GAS（clasp管理） | 全自動化のバックエンドコード | 未実装 |
| `prompts/` | 共通プロンプト | Gemini API用プロンプトテンプレート | 一部作成済 |

## 共通戦略

- **ペルソナ**: 従業員10〜300名の中小企業 社長・経営者層
- **テーマ**: AIエージェント × 業務改善 × 中小企業DX
- **トーン**: 専門家だが偉そうではない。「やってみた」ベースで語る
- **原則**: 1ネタ3媒体展開（各媒体に最適化。コピペ厳禁）

## 開発・保守方針

**GASコードはclasp経由でgit管理し、Claude Codeで保守可能にする。**

```
shachotwo-SNSAI/
├── gas/                    ← clasp管理。GASコードの実体
│   ├── .clasp.json         ← claspプロジェクト設定
│   ├── appsscript.json     ← GASマニフェスト
│   ├── main.js             ← エントリポイント（トリガー登録）
│   ├── generate.js         ← 3媒体コンテンツ一括生成
│   ├── gemini.js           ← Gemini API呼び出しラッパー
│   ├── geminiImage.js      ← Gemini 画像生成APIラッパー
│   ├── dashboard.js        ← ダッシュボードシート更新
│   ├── apify.js            ← Apify Webhook受信・パース
│   ├── sheets.js           ← スプレッドシート操作ユーティリティ
│   └── config.js           ← 設定値（APIキーはGASプロパティで管理）
├── x/                      ← X要件定義

```

### clasp セットアップ手順

```bash
# 1. clasp インストール
npm install -g @google/clasp

# 2. Google アカウントでログイン
clasp login

# 3. 既存のGASプロジェクトにクローン or 新規作成
cd shachotwo-SNSAI/gas/
clasp create --type standalone --title "shachotwo-SNSAI"

# 4. コードをGASにプッシュ
clasp push

# 5. GASエディタで確認
clasp open
```

### 開発フロー

```
Claude Code で gas/*.js を編集
  ↓
clasp push（ローカル→GASにデプロイ）
  ↓
GASエディタ or スプシで動作確認
  ↓
git commit（バージョン管理）
```

### 1ネタの3媒体展開フロー（具体例）

```
ネタ: 「請求書処理をfreee + GAS + Geminiで月40時間→2時間に自動化した」

  ① GASが「業務改善ネタシート」から未処理ネタを取得

  ② Gemini APIに3媒体分を一括生成依頼:

     ┌─ X短文（280字）
     │  「請求書処理を自動化したら月40時間→2時間になった話。
     │   freee×GAS×Geminiで〜（略）〜 DMで相談乗ります」
     │
     ├─ X記事ダイジェスト（2,000字）
     │  note記事の前半50% + 末尾に「全文はnoteで → {URL}」
     │
     ├─ note記事フル版（5,000字）
     │  完全ガイド + 手順 + 比較表 + アフィリリンク(freee/MF)
     │  + AIO最適化（定義文・FAQ・構造化）
     │
     └─ LinkedIn投稿（800字）
        プロフェッショナルトーン + 1コメ目にnoteリンク

  ③ 画像生成:
     ├─ Gemini API → アイキャッチ（抽象ビジュアル）
     ├─ Mermaid → フロー図（GAS→freee→Geminiの構成図）
     └─ Google Slides → Before/After比較画像

  ④ スプレッドシートに4件書き込み（X短文/X記事/note/LinkedIn）

  ⑤ ダッシュボードシートに「未確認」ステータスで表示

  ⑥ 人間がダッシュボードを開いて確認（30分以内）:
     - 4件の下書きを確認・微修正
     - noteにアフィリリンクが正しく配置されているか確認
     - PR表記があるか確認

  ⑦ 投稿（時間差で）:
     Day 1 朝: X短文投稿
     Day 1 夜: note記事公開 → X記事ダイジェスト投稿（noteリンク付き）
     Day 2 朝: LinkedIn投稿（1コメ目にnoteリンク）
```

### GAS関数一覧（3媒体統合）

| 関数名 | トリガー | 処理内容 |
|---|---|---|
| `doPost(e)` | Webhook受信 | Apifyデータパース → シート書き込み |
| `scheduledGenerateAll()` | 毎日 6:00 | ネタシートから1件取得 → 3媒体分の下書き一括生成 |
| `generateXShortDraft(row)` | 内部 | X短文+引用RT下書き生成 |
| `generateXArticleDraft(noteContent)` | 内部 | note記事からX記事ダイジェスト版を生成 |
| `generateNoteDraft(row)` | 内部 | note長文記事+アフィリ候補提案 |
| `generateLinkedInDraft(row)` | 内部 | LinkedIn投稿（プロトーン変換） |
| `generateImages(row)` | 内部 | Gemini画像+Mermaid+Slides画像生成 |
| `callGeminiAPI(prompt, systemInstruction)` | 内部 | Gemini API ラッパー（リトライ・レート制限対応） |
| `callGeminiImageAPI(prompt)` | 内部 | Gemini 画像生成API ラッパー |
| `updateDashboard()` | 生成完了後 | ダッシュボードシートに未確認件数・下書きサマリーを更新 |
| `logPerformance()` | 毎日 22:00 | 当日投稿のパフォーマンス記録行を準備 |
| `weeklyReport()` | 毎週月曜 9:00 | 3媒体横断の週間KPIサマリーをダッシュボードに集計 |
| `monthlyAffiliateReport()` | 毎月1日 9:00 | アフィリエイト月次収益をダッシュボードに集計 |

### 投稿タイミング（時間差展開）

| 時間 | 媒体 | 内容 |
|---|---|---|
| **Day 1 朝 7:00** | X | 短文投稿（バズ狙い・反応テスト） |
| **Day 1 昼 12:00** | X | 引用RT（別ネタ。昼の露出） |
| **Day 1 夜 or Day 2 朝** | note | フル記事公開 |
| **Day 2 朝 8:00** | X | 記事ダイジェスト投稿（「全文はnoteで」） |
| **Day 2 昼 12:00** | LinkedIn | 投稿（1コメ目にnoteリンク） |

> **同じネタを2日間かけて3媒体に展開**。同日一斉投稿はしない（各媒体のアルゴリズムを最大限活用）。
