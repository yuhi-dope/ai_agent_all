## 2. 頭のDB（知見ナレッジベース）シート

全媒体（X / note / LinkedIn）共通のRAGソース。自分のコアメッセージ・知見・経験を蓄積。
投稿生成時にタグマッチで関連エントリを5件選出し、Geminiプロンプトに差し込む。

> **初期構築**: `shared/head_db_interview.md` のインタビューで30-50件投入
> **自動蓄積**: `shared/prompts/auto_enrich_head_db.md` で週3-5件追加
> **フィードバック強化**: 高パフォーマンス投稿に使われたエントリの重要度を自動+1

| カラム | 型 | 説明 | 例 |
|---|---|---|---|
| A: id | TEXT | エントリID | `BRAIN-001` |
| B: カテゴリ | TEXT | `実績` / `思想` / `ツール知見` / `業界知識` / `失敗談` / `顧客の声` / `フレームワーク` / `名言・引用` | `実績` |
| C: テキスト | TEXT | 内容（2〜5文。SNS引用しやすい粒度） | `建設業の積算をAI化し、3日→4時間に短縮。精度99%。` |
| D: タグ | TEXT | カンマ区切り3-5個（業種名・ツール名・テーマ） | `建設業,積算,自動化,コスト削減` |
| E: 重要度 | NUMBER | 1-5（5が最重要。フィードバック強化で自動加算） | `5` |
| F: 作成日 | DATE | | `2026-03-15` |
| G: 最終使用日 | DATE | GASが投稿生成時に更新 | `2026-03-20` |
| H: 使用回数 | NUMBER | 累計使用回数（同じ内容の多用を防ぐ） | `3` |
| I: ソース | TEXT | `初期投入` / `棚卸し` / `自動抽出` | `初期投入` |

## データフロー

```
[業務改善ネタ] ──→ GAS scheduledGenerate ──→ [投稿管理(下書き)]
                                                     │
[頭のDB] ──→ GAS generateQuoteDraft ──┘              │
     ↑                                                │
[バズポスト収集] ←── Apify Actor 1                    │  人間が確認・投稿
                                                     ↓
[ニュース収集] ←── Apify Actor 2              [パフォーマンス]
                                                     │
                                              DM発生時 ↓
                                              [リード管理]
```

## バリデーションルール（GAS側で実装）

- `ステータス` カラム: ドロップダウンで選択肢を制限
- `重要度`: 1-5の整数のみ
- `いいね数` / `imp数` 等の数値: 0以上の整数
- `url`: `https://` で始まる文字列
- `日付`: YYYY-MM-DD 形式
