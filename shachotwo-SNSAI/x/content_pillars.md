# コンテンツ4本柱 × テンプレート集

> 全ポストはこの4カテゴリに分類される。
> Geminiへのプロンプトにもこの構造を渡す。

---

## P1: 実績公開（Build in Public）— 35%

**目的**: 「この人は実際にやっている人だ」という信頼を獲得する。

### テンプレート A: Before → After 型

```
【フック】
{業務名}を自動化したら、月{X}時間が{Y}時間になった話。

【Before】
・{課題の具体的な描写}
・{誰が・どのくらいの時間をかけていたか}

【After】
・{ツール名}で{何をしたか}
・結果: {数字で表現}

【学び】
・{再現性のあるポイント1つ}

自社の{類似業務}も自動化できないか気になる方、DMで相談乗ります。
```

### テンプレート B: 設計図公開型

```
【フック】
今日作った自動化の設計図を公開します。

【概要】
{何を自動化するか、1行で}

【アーキテクチャ】
{ツール構成: 例「Apify → GAS → スプシ → Gemini → Slack通知」}

【ポイント】
・コスト: 月{X}円
・構築時間: {Y}時間
・削減効果: {Z}

【図解】※画像添付
{Mermaidで生成した構成図}

同じ仕組み、あなたの会社でも作れます。興味あればDMへ。
```

### テンプレート C: 失敗→改善型

```
【フック】
{自動化名}で大失敗した。原因と修正を共有する。

【やったこと】
{試みた内容}

【失敗した理由】
{具体的な原因}

【修正後】
{どう直したか、結果どうなったか}

失敗も含めて全部公開します。
```

## P2: 業種別あるある課題 × 解決策 — 30%

**目的**: ターゲット業種の経営者に「自分のことだ」と思わせる。

### テンプレート D: あるある → 解決型

```
【フック】
{業種}の社長が毎日やってる「{あるある業務}」、AIに任せたら{結果}だった。

【あるある（共感パート）】
・{課題1: 社長あるある}
・{課題2: 業種特有の課題}

【解決策】
{ツール名}を使って{手順を3ステップで}:
  ① {ステップ1}
  ② {ステップ2}
  ③ {ステップ3}

【効果】
{Before → After を数字で}

{業種}で同じ課題を抱えている方、DMで具体的な方法をお伝えします。
```

### 業種別ネタリスト（初期ストック）

**建設業**:
- 積算の属人化（ベテランしか単価を知らない）
- 協力会社の評価が頭の中にしかない
- 日報が紙 → 集計に毎月丸1日
- 安全書類の作成が毎回ゼロから
- 現場写真の整理・報告書作成

**製造業**:
- 品質判断が「あの人に聞かないとわからない」
- 新人教育マニュアルが10年前のまま
- 設備メンテナンスの勘と経験
- 在庫管理がExcel手打ち
- 顧客ごとの特殊仕様が口頭伝達

**歯科クリニック**:
- 症例の判断基準がドクターの頭の中
- スタッフ教育に院長が時間を取られる
- 患者説明の品質がスタッフでバラバラ
- レセプト業務の手作業
- 材料発注のタイミング判断

**介護福祉**:
- ケアプランの書き方が人によって違う
- 夜勤の引き継ぎが口頭のみ
- 介護報酬改定への対応が遅れる
- ヒヤリハット報告が形骸化
- 家族への報告書作成に時間がかかる

**士業事務所**:
- 所長の暗黙知 = 事務所の最大資産
- 顧問先ごとの対応方針が属人化
- 法改正のキャッチアップが追いつかない
- 繁忙期の業務ピーク管理
- 若手の育成に時間を割けない

## P3: 図解・フレームワーク — 20%

**目的**: 保存・シェアされやすいコンテンツで拡散力を得る。

### テンプレート E: フレームワーク図解型

```
【フック】
中小企業がAIを導入する時に必ず通る5ステップを図解した。

【図解】※Mermaid画像添付

【解説（各ステップ1行）】
① {ステップ1の要点}
② {ステップ2の要点}
③ {ステップ3の要点}
④ {ステップ4の要点}
⑤ {ステップ5の要点}

保存しておくと、いつか使う時が来ます。
```

### 図解ネタリスト

- 中小企業AI導入の5ステップ
- 属人化を解消する3つのレイヤー
- 「社長にしかできない」を仕分けるマトリクス
- DXの失敗パターン Top 5
- GAS + AI で自動化できる業務マップ
- ナレッジが消える5つのタイミング
- 経営者の時間の使い方 Before/After

## P4: 引用RT・時事コメント — 15%

**目的**: バズポストの波に乗ってリーチを拡大する。

### テンプレート F: 専門家コメント型

```
{元ポストの要点を1行で要約}

これ、実際にやってみた立場から補足すると:

・{具体的な経験ベースの知見1}
・{経営者目線での示唆1}

{業種名}だと{具体的な適用例}。
```

### テンプレート G: 反論・補足型（慎重に使用）

```
{元ポストの主張}、半分同意で半分違う視点がある。

同意: {同意点}
補足: {見落とされている観点}

実際に{N}社の{業種}で試した結果、{事実}。

{元ポストの主張}を活かすなら、{具体的な提案}がおすすめ。
```

### 引用RT対象の選定基準

| 基準 | 条件 |
|---|---|
| いいね数 | 50以上（リーチが見込める） |
| トピック | AI / DX / 業務改善 / 中小企業 / 経営 |
| 鮮度 | 24時間以内 |
| コメント可能性 | 自分の経験・知見で「新しい視点」を追加できる |
| リスク | 炎上・政治・批判系でない |
